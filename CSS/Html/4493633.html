CaseBuddy is a tool that helps users of Service Desk Case, Knowledge, and support data transfer tool (DTM).

Production install

MS code-signed build that is update approximately once a month
• https://casebuddy.blob.core.windows.net/install/x64/production/CaseBuddy64.application 

Insider install

MS code-signed build that is updated approximately every two weeks; more stable than Alpha but you get new features and fixes more often
• https://casebuddy.blob.core.windows.net/install/x64/insider/CaseBuddy64.application 

Alpha install

This build is not MS code-signed so you will get a blue Windows Defender popup at each update.  The build is updated frequently – sometimes daily; and though you will be the first to get new features and fixes; it may have reliability/stability issues
• https://casebuddy.blob.core.windows.net/install/x64/alpha/CaseBuddy64.application

FAQ
  • What are the meanings of the highlight colors?
    • Case number is highlighted in orange = case's internal title contains the word "CoronaVirus"
    • Entire line is highlighted in light red = case is tagged as "Life And Safety"
    • Entire line is highlighted in yellow = data in DTM was scanned and found to match a known issue
    • Entire line is highlighted in light magenta = Incomplete Support Area Path (SAP)  e.g. no SAP, SAP to product only, or SAP to L2 where an L3 exists
  • What is the meaning of the "Life and Safety" tag?
    • New case category to help with CoronaVirus cases
    • There is an optional column you can add to the view to show which category (FirstResponder, Govt, Critical Infrastructure, Retail, Finance).
  • What is the red dot on the DTM icon mean?
    • It is intended to show if files available. Sometimes the API being used does not always accurately reflect that files are available, so sometimes the dot will not be present, but files will be available.
