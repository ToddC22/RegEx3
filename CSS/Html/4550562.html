1. Tool features

Features have been added into the tool to allow for a more customized experience.

  1. Full width/Half width – The button next to the X to close KIT will extend the window to full width of the browser windows or half width.
  1. Expand all/Collapse all – Use this to expand/collapse all subsections after searching.
  1. Simple view/Full view toggle – Simple View KIT will not show fields that do not contain data returned. Full view will show all fields even if they contain no data.
  1. Expand/Collapse individual sections – Expand or collapse each section in KIT with the arrow on the right.

Use the X in the upper right corner to exit the tool.

2. Tool section - Search

During troubleshooting the customer’s issue related to installation/set-up/activation failure, you can use KIT to look up a customer’s product key, review their activation summary and history, and access general product key information.

Search using a 5x5 Product Key or a Product ID. Alternatively, Microsoft internal could also use GCS to look up 5x5 details.

3.1 Activation blocked, validation blocked

If the Product Key is blocked for any of these reasons, this will be highlighted in the results. Additional fields will also be displayed providing further details as to why it was blocked, by who, the location for which it’s blocked, and other helpful information.

Important: Blocked Product Key details should never be shared with a customer.

Activation blocked

This information is primarily used by PACTRUN, APS, and CELA. Advocates should be primarily concerned with the fact that the Product Key was blocked, and to whom they should escalate if the customer claims they are the rightful owner.

Regardless of the reason, the product key the customer is attempting to use to the complete activation process is blocked, it cannot or can no longer be used to activate a product. However, the information provided in the Activation Blocked Reason may clarify the type of product key the customer is attempting to use to activate their software. For example, if the reason is identified as Default, then this is a product key that cannot be activated. In general, most other reasons suggest an action was taken to block the product key from enabling additional activations, based on historical activity of the product key. Examples include the product key had become pirated/abused, or the product was returned by the customer.

For specifics on Tennant ID mapping see KB 3147392 Procedure: MPA: Handling Escalations for Blocked, or to Block, Product Keys.

Note: It is not uncommon for KIT to return a Blocked Reason of Unknown. In these cases, it cannot be determined which process was used to block the product key.

4. Tool section – Configuration Summary

Product keys are configured to help automatically manage the level of activity for each key. This page provides information on what Limits have been set, and the level of activity against these limits. When limits are met, and depending on which ones, this will trigger different types of activation failures. You’re also able to see the first time the product key was used.

Important: Never share this information with customers. This is for Microsoft internal use only! See KB 4042868 TUG: KIT: Usage and product key checking requirements for further details.

The main items for you to look for are:

  • Product Name
  • NLS Date
  • Total Number of Activations
  • ROT Override Count
  • Activation Override Count

Description

Name of the product, including version, to which it was activated for the first time.

NLS Date

Total Activations

The total actual activations that were completed against the key

  • For OEM/Retail: Total Activations = 1 (for NLS) + RIT Count + ROT Count + Activation Override Count
  • For DMAK: Total Activations = DMAK Count + DMAK Extension Count + Activation Override Count

Important: This calculation is applicable to Vista and later products only.

RIT Limit

(Reissue in Tolerance) Activations allowed on same machine.

RIT Count

Actual number of activations on the same machine.

NLS MPC

The 5-digit Microsoft Product Code with which it was activated for the first time.

RAT (D : H : M)

(Reissue after Tolerance) Time interval in days/months in which additional ROT allowed.

ROT Limit

(Reissue out-of-Tolerance) Activations allowed on different machines.

ROT Count

The number of different machines the key is activated on.

ROT Override Limit

Number of offline override activations happened on a Retail or OEM.

ROT Override Count

Actual number of offline override activations happened on a Retail or OEM.

DMAK Activation Limit

Number of online activations allowed on a DMAK.

DMAK Activation Count

Activation Override Count

Actual number of offline override activations happened on a DMAK.

DMAK Override Limit

Number of offline override activations allowed for a DMAK.

DMAK Activation Extension Count

Actual number of offline override activations happened on a DMAK after the DMAK Activation Limit is reached.

DMAK Activation Extension Limit

Number of offline override activations allowed for a DMAK once the DMAK Activation Limit is reached.

5. Tool section – Activation History

When an activation attempt is made, it’s logged in the history. You’ll be able to identify the mode used, the date of the attempt, including the related hardware.

Important: Never share this information with customers. This is for Microsoft internal use only! See KB 4042868 TUG: KIT: Usage and product key checking requirements for further details .

Certain PKID's have been purged (Activation history information) and retained in the Datawarehouse housing the product activation data. Contact PACTRUN team for further assistance.
