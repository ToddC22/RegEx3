Overview

Request is received from any of the frontline support regions and it is determined that the request needs to be escalated to PayOps.

  • Escalation is billing related.
  • Escalation needs to be triaged by PayOps for Commerce Platform Team (CP).

Follow the below guidelines for SR creation and management:

  • Cases can be re-opened within 14 days as long as it’s the same issue.
  • Strikes happen as follows (72 weekday hours):
    • Strike 1 - After 24 weekday hours
    • Strike 2 - After another 24 weekday hours
    • Strike 3 - After another 24 weekday hours
  • Keep attention to STRIKE PROCESS. If 3 struck, then don't re-open if it’s after 14 days, do the following instead:
    • Make a copy of old SR that was 3 struck, then add details in discussion as needed.
    • Add the PayOps owner’s name of the old SR. This will help with KPIs.
  • If same customer asks different question that is not related to the original SR, then create a new SR.
    • SR can be re-opened if the issue is not resolved, as long as the ask is the same.

Description of information

After case has been scoped and determined to be a PayOps escalation, Tier 3 Engineering needs to create an escalation using one of the templates below:

General Escalations

Locate Account & Unknown Charge

Split Payment

Remove PI

Tenant Deletion

Please refer to the following Help links:

Submitting a Service Request
Managing a Service Request to Closure

The templates have some areas already prefilled however the information currently listed may need to be modified.

For example: In the General Escalations Template, update the Customer Type field when it’s relevant to do so.

Do not remove information in brackets from the Titles on above templates.

  1. Complete the following fields for processing:
    1. Title should include a description of the ask.

      [Asset 4025247]
    2. After adding the Title, click on the PAYOPS tab:

      [Asset 4025248]
    3. Complete the fields below Submit and Triage:

      1. Business Ops Team – select PayOps if not prepopulated
      2. Priority: 2
      3. Created by Team – CPC (select region)
      4. Customer Urgency – select one of the options below:
          * S400 (M365 only)
          * Premier/Pro-direct
          * N/A
      5. SR Reason – select applicable option from drop down
       
    4. Description: provide brief summary of ask or complete prepopulated template based on ask.
    5. Customer Details:

      1. Customer Type: Select from drop down

      2. Customer MSA/OrgID: Email alias of admin troubleshooting

      3. Billing Account ID: Billing Account ID found in CMAT

      4. Tenant GUID – found in CMAT

      5. Customer Country: found in CMAT

      6. Purchasing Storefront: O365 store

      7. Post-Sales Customer Portal Usage: O365/CRM

      8. Company Name: found in CMAT

    6. Repro Steps: provide any repro steps if applicable.
    7. Analytics:

      1. Product/Service: O365 Commercial (if not prepopulated)
      2. Prod Environment: Prod - GLOBAL (if not prepopulated)
      3. SR #:
      4. Tier 1 Case Created Date: Date the case was created by the customer
  2. Once all fields are completed, click Save. The escalation will be created and a VSO ticket number will be issued.
