1. How does Azure support work for CSP partners in Partner Center?

Read through each scenario in the table below in column 1. Each scenario maps to the next best steps that would resolve the CSP partner's issue.

A good general description of ASMS support and scope boundaries can be found here.

Redirect to Azure
After transferring an Azure subscription from EA/web-direct to CSP or from one partner to another, the subscription does not show any consumption in Partner Center or via API and subscription charges are not visible in the recon file immediately after the subscription transfer Inform the partner that it might take up to a month for the usage to be visible in PC or via API. If after a month, the estimated usage is still not available, partner should open a ticket with PCTS. For the recon file, inform the partner that – due to the same delay – the charges should be visible in the next recon file.

Billing cycle for a specific Azure subscription is not in sync with the partner’s billing cycle (charge start date is different from the partner’s AD).

In Azure whenever subscription is cancelled and re-activated it changes its billing cycle for the subscription by the number of days for which the subscription was cancelled.

Ex.:

Partner’s AD is 1st of every month.

Subscription created on 8/27/2018, but later cancelled on the same day and re-activated on 09/03/2018, which is 7 days later.

The billing cycle for the subscription is moved forward by 7 days and starts from the 8th day of the month.

RSC and ASMS to collaborate

*RSC to check with ASMS if the subscription billing cycle was impacted by any suspension/reactivation activity.

Note: For phone only, guide the partner on how to raise Support Requests from the Azure portal (Ibiza). For Web Submissions/Email, use the escalation process.

The CSP partner can raise a Service Request from the Azure portal.

1. After they log into the Azure portal, the partner should click the question mark button from the top right side of the page, and then select Help + Support.

[Asset 4293636]

2. On the Help + support page, the partner can view other available resources (forums and public documentation), review their active support requests, and also create a new one by selecting +New support request.

[Asset 4293637]

3. The agent should guide the partner through completing the SR form steps, depending on the issue type.
