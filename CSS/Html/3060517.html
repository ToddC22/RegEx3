External senders receive a '550.5.4.1 [user@contoso.com]: Recipient address rejected: Access denied' NDR when they send mail to an Office 365 user

When external senders send email messages to an Office 365 user, they receive a nondelivery report (NDR) that contains the following error message:
'550 5.4.1 [<nosuchuser>@<recipient_domain>]: Recipient address rejected: Access denied'.
The message that was sent to user@contoso.com was rejected by Directory Based Edge Blocking (DBEB) because it considers the recipient address to be an invalid address. The message was rejected because there's no valid object in Office 365 that matches the SMTP address of the recipient in Exchange Online.

Note DBEB is enabled when the domain is set to Authoritative. If the domain is set to Internal Relay, messages to invalid users are rejected and senders receive a '550 5.1.1 RESOLVER.ADR.RecipNotFound; Not found' error instead.
To resolve this issue, follow these steps:
  1. Make sure that the user exists in the cloud.
  2. Create and run the Provisioning - Recipient Object Full Sync Request.
  3. After the SOP is completed, run the Provisioning - Company Object full sync.

Note Dynamic Distribution groups (both on-premises and EXO) are not supported by DBEB.

Also, you need to run Directory Based Edge Blocking Recovery Action SOP for the affected user. Mentioned Action is available in Viewpoint.
