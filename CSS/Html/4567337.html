Procedure: Windows / Azure Support Collaboration

[Asset 4462485]

Who should use this article?

The Windows Commercial and Azure support engineer teams.

What is this article about?

This article explains best practices for when and how support engineers should collaborate and engage when working with on-premise Windows customers that might also have an Azure Support plan.

There is a business need to ensure that Windows and Azure teams carefully handle and manage customer cases that require the two team’s collaboration.

Scenario: The customer has an active support plan and submits a technical case for Azure support however there is a need to collaborate with Windows support.

Action Plan:

  • The Azure support engineer scopes the case with the customer and works this through to resolution with the customer.
  • If the engineer determines this issue is not an Azure issue and is presented with any of the scenarios below, please read the scenario and any follow-up actions / activities the engineer(s) should apply.
If the scenario is: Then the engineer should follow-up as documented:
A. Pure On-Prem Windows issue only and no support needed from Azure team
  1. Azure Engineer informs customer this issue is isolated to Windows and they will need support from windows team.
  2. Azure Engineer checks customer's entitlement
    • If the customer has Premier support, transfer the customer through to relevant Windows Premier queue.
    • If the customer has no Premier Support agreement, they are transferred through to the Broad Commercial support team for support.
B. Azure is present but has already been ruled out; the remaining is On-Prem Windows issue. Azure team validates the need to transfer case with Azure PTA (Partner Technical Advisor).
  1. Azure engineer transfers the case through to the Windows support team who will own the case through to resolution,
    • Need to lock and document plan between the two (2) groups. 
    • Need to include PTA process.
C. The issue is unclear where the problem might reside within the Azure infrastructure or the workload running into an Azure VM or Windows.
  1. Azure PTA to collaborate with Windows PTA to isolate the issue and agreement on case ownership moving forward.
D. Technical gap between Azure DP (Delivery Partner) and Windows DP and difficult to identify.
  1. Azure PTA to collaborate with Windows PTA to define and isolate issue.
E. Azure team needs immediate assistance on Severity A and no PTAs are available to validate scenario.
  1. Azure engineer(s) transfers the case to the Windows team on immediate assistance on Severity A when no PTAs are available to validate the scenario.
F. Azure customer requesting on RCA which is related to Azure Windows VM.
  1. Leverage current agreed process.
  2. Azure team creates a Collaborate task to the Windows Premier FTE team.
  3. Windows Premier FTE team will provide an RCA and Azure team will own the customer experience and delivery to the customer.
  4. Refer to the RCA Procedure KB 4456527

 

Scenario: The customer raises a Windows On-Premise case where Azure knowledge is needed.

Action Plan: The Windows support engineer scopes the case with the customer and determines the issue is not related to Windows and needs to be handled by Azure support team.

If the Scenario is: Then the Engineer should follow-up as documented:
A. Windows engineer should validate with Windows PTA before transferring the case.
  1. The Windows engineer transfers the case through to the Azure support team who will own the case through to resolution.
  2. If the Windows engineer needs to collaborate with Azure team, the engineer creates a collaboration request for help.
B. Windows Team needs immediate assistance and no PTAs are available to validate scenario.
  1. Windows engineer creates a collaborate task and works with Azure team on immediate customer issue and PTAs can validate once available.
C. The issue is unclear if it’s with Azure infrastructure or Windows.
  1. Windows PTA collaborates with Azure PTA to identify the issue and agreement on case ownership & action plan moving forward.
