Introduction

This article lists the Office 365 technical support boundaries articles and describes the terminology that the articles use.

Each of the support boundaries articles contains guidelines for the kinds of issues for which Office 365 support engineers provide technical support. Support issues that are outside these guidelines should be redirected to the appropriate support documentation. 

Legend

Heading Description
Scenario Brief description of the scenario
Scope Whether the scenario is supported, not supported, or allowed:
  • Supported: Scenario is supported by engineering and Broad Commercial support can assist.
  • Not Supported: Scenario is not supported by engineering. Therefore, Broad Commercial support will not assist.
  • Allowed: Scenario is supported by engineering. However, Broad Commercial support will not assist.
More Information Additional information that you should know about the scenario
Content Links to content to assist you in providing help for the scenario
Level 100 - All technology queues should understand this information. It should be part of your fundamental knowledge.

200 - All technology queues should understand this information but may require additional content to support the scenario

300 - Scenarios focused for a technology queue. If you are not the team who is responsible for the scenario, the scenario should be redirect to the appropriate Frontline team.

400 - Advanced scenario that may require escalation by using the Escalation Path identified. Make sure that due diligence, common troubleshooting, and all steps are performed based on the content that's available.
Team Frontline team that is responsible for the support boundary scenario
Escalation Path The next path for escalation if due diligence has been performed.
