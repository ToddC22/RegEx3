Launching a virtual application fails with error code 0x41E04304-00000028

When a user attempts to launch an application virtualized with Microsoft Application Virtualization (App-V), the application fails with the following error:

The application failed to launch.
This may be due to a network failure.
Error code: 0x41E04304-00000028
This can occur if the user attempts to launch a virtual application executable from an unpublished package version. This is by design.

To prevent this issue from occurring, always launch virtual applications from extension points (e.g., shortcuts and File Type Associations or FTAs).
When a package version is unpublished, the extension points are removed and the executables in the package version cannot be launched. When a new package version is published, extension points are updated to launch from the most recently published package version. Accordingly, virtual applications should always be launched from extension points, never directly from an executable within a particular package version.
