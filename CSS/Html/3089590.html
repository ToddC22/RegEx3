1. General troubleshooting - First steps

Some customers are experiencing what are called Blue Screens or Stop Errors either during the setup process or when running Windows 10. Both hardware and software problems can cause these errors. Resolving this problem starts with understanding what changed recently on the customer’s PC, and what error they are encountering.

1.1 Confirm that it is a Stop Error

Can the customer still see other Windows elements, such as the taskbar, Start, or icons? If so, this is not a blue screen error. For instance, note the title bar, minimize/maximize and close buttons, and other windows features in the following picture. Even though the error window has a blue background, this is not a Stop Error:

[Asset 4567472]

Capture as much detailed information as possible from the error shown, like the full numeric code, and notes on when you saw the error. Search for solutions in VKB and online resources. The rest of this article applies to Stop Errors only.

Stop Errors will not have any Windows features, they are just blue backgrounds with text and nothing to be selected:

[Asset 4567473]

After you’ve verified that it is a Stop Error:

  1. Ask the customer if they changed anything on their computer recently, including installing new hardware or installing updates. Find out when the error happened. Common situations: While upgrading to Windows 10, after installing updates, after installing hardware, or just during normal use of Windows.
  2. Review public content for troubleshooting Stop Errors: https://support.microsoft.com/help/14238/windows-10-troubleshoot-blue-screen-errors.
  3. If the customer can boot Windows, direct them to Check for Updates. From Start, search for check for updates, select Check for Updates from the search results, and then follow the on-screen instructions.
  4. Near the bottom of every Stop Error is an error code; a series of words all in capital letters connected by an underscore. MEMORY_MANAGEMENT or INTERNAL_POWER_ERROR are two examples. This will help you search for a solution in the KB and on Microsoft.com.

Many resolutions require recovery options. For more information about recovery: https://support.microsoft.com/help/12415/windows-10-recovery-options.

1.2 Stop Error while upgrading to Windows 10

During the upgrade process to Windows 10, a small set of customers may encounter a Stop Error that interrupts setup.

  1. Setup may make several attempts to complete and then initiate a rollback to the previous OS. When this happens, the customer might receive a rollback error code at end of the process. Make a note of this code as it can be used like a Stop Error to find specific information.
  2. Stop Error during setup that neither continues setup, nor rolls back, when this happens, it’s best to manually enter recovery mode and either attempt to boot into Safe Mode or initiate a rollback to the previous OS.    

Note: By design, if Windows 10 fails to boot three times, it will often go into Automatic Repair. When this happens, choose Advanced Options to enter the Windows Recovery Environment (WinRE). From there the customer can select Troubleshooter, Advanced Options, then select Go Back to uninstall Windows 10.

1.3 Stop Error after installing a Windows Update

The customer may have received a Windows Update or a device driver update. If after installing this update, the computer produces a series of Stop Errors, consider removing the recent update.

  1. If the customer can boot into normal mode, search for View Installed Updates and select the control panel result. Identify the recent update or updates; they may have one and must uninstall one at a time to see if it improves performance. Start with any device driver updates. Select the update and then choose Uninstall. Reboot if directed. If uninstalling an update prevents the Stop Error, consider temporarily blocking this update from re-installing by using this troubleshooter.
  2. If after several reboots the customer is given the option for Automatic Repair, choose Advanced Options, Troubleshooting, Startup Settings, and then select Safe Mode with Networking. Once there, follow directions from Step 1 to identify a problem update, remove it and temporarily block it from re-installing.
  3. If the customer is unable to boot their device, they can also create and boot from recovery media. This will allow them to use System Restore to restore the device to a previous state or use the Reset function for more serious issues.

1.4 Stop Error that just happened during normal computer use

Sometimes it’s not obvious what has changed that may have caused a Stop Error. On older PCs, it may even by that hardware is malfunctioning. When this happens, start with the assumption that a software change has taken place that needs to be corrected. The steps are the same as Stop Error after installing a Windows Update. It may even be that an update was installed but the customer is unaware.

2. Some known errors

  1. 0x0000007B  INACCESSIBLE_BOOT_DEVICE
    1. Most common early in upgrade.
    2. Change storage access method setting in Firmware:
      • Change BIOS/firmware setting for storage from IDE to AHCI. Note: If the customer is in a RAID configuration it can be risky to make changes.
      • Then restart and see if the upgrade continues. 
    3. If this does not resolve the issue, try going to the Recovery menu and Go Back.
      • If you can: Update drivers, uninstall security software and upgrade again.
      • If you cannot, escalate. We may need to help the move data files to a safe location while we clean install Windows.
  2. 0x00000119 VIDEO_SCHEDULER_INTERNAL_ERROR or VIDEO_TDR_FAILURE
    1. Reported for PCs already running Windows 10.
    2. Likely to be issue with display driver.
    3. If intermittent:
      • Check for latest drivers in Windows Update and the PC manufacturer. Install updated drivers, if newer than the current drivers.
      • If already on latest, recommend watching for updated drivers and installing when available.
  3. 0x0000007E SYSTEM_THREAD_EXCEPTION_NOT_HANDLED
    1. Likely to be driver or need BIOS/firmware update.
    2. Restart if in the middle of upgrade, and check for the option to Go Back.
    3. Try to get to safe mode with networking.
      • Troubleshoot as driver issue there.
      • Check for updated firmware for the PC.
  4. 0x0000000A IRQL_NOT_LESS_OR_EQUAL
    or
    0x000000C2 BAD_POOL_CALLER
    or
    0x000000D1 DRIVER_IRQL_NOT_LESS_OR_EQUAL
    1. Likely driver issue. Check the error screen carefully for a named driver file.
    2. Restart if in the middle of upgrade, and check for the option to Go Back.
    3. Check for updated drivers.
  5. 0x00000133  DPC_WATCHDOG_VIOLATION
    or
    0x000000EA THREAD_STUCK_IN_DEVICE_DRIVER
    1. Miscellaneous driver issues.
    2. Restart if in the middle of upgrade, and check for the option to Go Back.
    3. Remove nonessential hardware.
    4. Check for updated drivers.
  6. 0x00000139 KERNEL_SECURITY_CHECK_FAILURE
    1. Remove nonessential hardware. Update drivers.
    2. Memory diagnostics, if possible.
    3. Chkdsk.
  7. 0x000000CE
    1. Could be driver issue: DRIVER_UNLOADED_WITHOUT_CANCELLING_PENDING_OPERATIONS
    2. Some issues reported with DVD playback software.
    3. Update drivers
    4. Update software that includes drivers (for example, DVD Playback, AV, Security, Utilities, and Backup).

3. Collecting log files

If it is necessary to escalate a Stop Error case, try collecting log files.

3.1 Stop Error during upgrade causes rollback

We should collect the setup logs using MSDT. If there was a memory dump created it will be captured by the data collector.

Use the Windows Setup and Rollback Data Collector manifest, which is in the Download < Install < Activate category on the SDC site.

3.2 Stop Error in Windows 10

If we are seeing a Stop Error in Windows 10 and we can sign in to Windows 10, we can try to collect any of the following files that show a modified data/time around the time of the Stop Error:

%windir%\minidump\*.dmp

%windir%\memory.dmp

You can collect these using MSDT. The Custom Data Collector will enable you to upload multiple files.

If we can’t find any files in these locations that were created in the right timeframe, check Problem Repots:
  1. Open Control Panel and search for all problem.
  2. Select View all problem reports in the results.
  3. Sort the results by date and check for problem reports from the timeframe when the Blue Screen was found.
  4. If you find a related system failure or another related event, open it.
  5. Check for an option at the bottom to view temporary files.
    1. If there is an option, capture a copy of these files using the MSDT Custom Data Collector.
    2. If there is not an option, use the Copy to Clipboard option at the bottom to capture the text error details and paste into the SR.
