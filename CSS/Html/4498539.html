1. Overview

Rave is an application that support advocates use to see real time customer issues, take customer support requests, and manage their support cases.

For help with when to use some of these features see KB 4514518 Procedure: Case management in Rave.

Return to top

2. How to access

Rave is a web-based tool. Open Edge and go to https://rave.office.net/.

Return to top

3. Important phone interaction note

Rave is integrated with OneCloud that allows for more seamless managing of the phone experience. You will not use Workspace Web Edition (WWE) while you are using Rave.

5.2 Account authentication

Account authentication lasts only if you are on the tab. If you leave the tab and come back, you’ll need to re-authenticate the customer. Do not leave the tab until you are done creating a case or reviewing the customer details.

Verification codes can only be sent to the primary Microsoft account email right now. Delivery of the codes should be very fast, within seconds in most cases.  

This is the initial account authentication prompt. Select Send to send a verification code.

Note: If you leave this screen after sending a code and come back, you’ll need to start from the beginning. Another code must be sent to the customer.

 [Asset 4498578]

The customer receives an email from Microsoft Support with the subject line Microsoft Support verification code.

The codes are five digits long.

Enter the code and select verify. Successful verification brings you to the profile screen.

Note: Resending the code does not make the prompt look different.

6. Finding my cases

There are three case status types in Rave.

  • Active: Open cases.
  • Pending: These are waiting for final documentation and to be closed.
  • Closed: Closed cases.

To review cases of each type, go to My cases in the top bar and select My active cases, My pending cases, or My closed cases.

[Asset 4498579]

You are brought to a list of your cases. You can filter your cases even further using the available options.

The case views refresh on a timer. If you are moving quickly between actions like looking for a case you just created, you may not see it right away. Select refresh in the upper right-hand corner of Rave to force it to sync immediately.

Return to top

7. Customer account view

The customer view contains known data about the customer’s account.

To access the customer account:

  • Lookup the customer's account and authenticate.
  • Select the customer's account on the case in the upper left, next to the case number.

The customer's account is broken into four sections:

  • Profile: Microsoft account data and previous support cases.
  • Subscriptions: Subscription information associated to the Microsoft account, this is a read only view
  • Orders: Orders that have been made from the Microsoft account, this is a read only view
  • Tools: ORST functionality is located here.

Return to top

A quick tour of what a case looks like.

[Asset 4498566]

Left section: Yellow box, Number 1: Customer Information.

9.1 Case Summary

The Case Summary is the first box and describes various details about the case.

[Asset 4498570]

  • Agent: Who it is assigned to.
  • Status: Active or closed.
  • Modality: Phone or chat.
  • Request ID: Copies an internal tracking number. Only used for troubleshooting case issues.

Return to top

9.2 Actions (brief)

Case status (internal)

Do not worry about manually setting this right now. It defaults to Troubleshoot and will change based on a few actions you take in the tool.

[Asset 4498548]

Case actions

All actions work in roughly the same way. You’ll select the action you want to perform, select OK, and either a control box will fly out from the right side of the screen or a dialog box appears.

We will give a brief description of the actions here, but as some of these are complex, we’ll describe them in more detail in their own sections.

[Asset 4498563]

  • Get feedback: Ends the current call or chat and sends a survey to the customer. This puts the case in a pending state. This option can only be used if a customer account is associated to the case.
  • Close case: Closes a case.
  • Transfer: Transfers the case to another skill.
  • Reassign: Assigns the case to another advocate. Use this to assign a case to yourself, directly to a supervisor, or to another advocate at a supervisor’s direction.
  • Request assistance: Do not use this right now, this is a feature to request assistance on a case. More to come, but this isn’t quite ready yet.
  • Schedule callback: This allows you to schedule a callback with the customer.
  • Reopen: Only available on closed cases. This reopens the case.
  • Email transcript: Used to send a copy of the chat transcript to the customer.

Return to top

Setting the resolution steps is not enough, you need to use this action to close the case.

13. Reopen a case

Reopened cases in Rave behave very differently than ASD. When you reopen a case in Rave, it creates a copy of the case, with the same case number. This is so the customer doesn’t have to remember or reference multiple case numbers. Inside of Rave the reopened case is treated as a new case that is linked to the original. The below will shows you how to reopen a case and navigate between the original and new cases.

Go to My cases -> My closed cases to find the closed case.

[Asset 4498579]

Select the case number of the case you want to reopen. Review the case summary and notes to confirm it should be reopened.

After confirming it is the correct case, select Reopen from the case actions and select OK.

[Asset 4498601]

Select the reopen reason. You should treat the option that starts Customer emailed… as the correct option for any customer contact method (for example, phone or chat).

[Asset 4498603]

Select Reopen and select Ok on the confirmation screen.

 

You are returned to the case you reopened. This is not the new version of the case. You are still looking at the original case. Close this case tab.

 

Go to My cases -> My active cases and refresh the view. Select the new version of the case you just reopened.

  

The Customer Interaction and Notes section is very different than the original version of the case. You will also see two new system (purple) dialog boxes.

[Asset 4498602]

These boxes represent either the original case or the new case. The one labeled (this case) is the one you are currently looking at. Remember that the Active case is the new case. The original case is still in a closed state.

[Asset 4498604]

Selecting these boxes opens the other version of the case. This is useful as the case notes, description, and resolution steps do not carry forward into the new case. Both cases have the same case number so it’s hard to determine which you are looking at on the menu bar.

  

You need to work the new case to completion as you would any brand-new case. The summary tab needs to be filled out again. Make sure you note in the customer description, what caused the case to be reopened.

15. Receive a call

There are two types of calls you can receive within Rave:

  • Outbound calls: These are customer callbacks requested from the web. This type of call has a customer account associated and does not place you directly into a phone call.
  • IVR calls: These are calls that came through the IVR. This type of call does not have a customer account associated to the case and does place you directly into a phone call. Phone transfers will also appear this way.

Return to top

15.1 Outbound calls 

Remember you can tell it’s an outbound call because you’ll receive a case that has a customer account associated with it, and there is no active phone call.

Go into available in the upper right-hand corner of the tool.

 

When a call is available you will receive this notice.

[Asset 4340451]

Select Accept and a case will load.

  

Select the Call button in the customer profile section. You must initiate the call to the customer.

[Asset 4498691]

An outbound call is placed to the customer. If they do not answer should follow your normal callback retry process.

The integrated OneCloud client is used to manage the call.

Refer to KB 4340443 Tool User Guide: Rave 2.0: How to use call features for more details on OneCloud integrated call controls.

Note: When the article references both Genesys OneCloud Softphone and Skype directions, you want to follow the Softphone directions.

 

Return to top

15.2 IVR calls

Remember IVR calls arrive with no customer account associated, and there will be an active phone connection.

You will be prompted to accept the call.

Note: The prompt will likely look a little different from this. There will not be an accept button. 

[Asset 4340451]

The customer account will be listed as unknown.

To associate a customer account, you need to select Associate user to this case.

 [Asset 4508068]

Search for a customer’s account with their Microsoft account email.

 [Asset 4508070]

The search results will populate. Select the customer’s account and select Save at the bottom.

Note: You cannot assume that this customer has passed authentication. Do not share information as if the customer has authenticated at this point.

 [Asset 4508071]

The customer will be associated to the case at this point. Work the case as normal.

Refer to KB 4340443 Tool User Guide: Rave 2.0: How to use call features for more details on OneCloud integrated call controls.

Note: When the article references both Genesys OneCloud Softphone and Skype directions, follow the Softphone directions.

16. File attachments and DTM

DTM is a tool used to receive file attachments from the customer. You can use this with both chat and email.

To start a DTM instance for the customer go to the bottom of the Communications tab. Select the Generate file sharing link

[Asset 4526439]

Copy the file sharing link by selecting Copy file sharing link, then highlight the full URL and select Copy.

[Asset 4526441]

[Asset 4526442]

Send the link to the customer via either email or chat.

 

When the customer opens the link, they may be prompted to login. They must login with the Microsoft account associated to their case. They can select an +Add button and browse for the files they need to send us.

This is what the customer sees on the DTM page.

[Asset 4526443]

 

 

When a customer has added a file, you will not see it right away.

Select sync attachments in the initial customer problem in the Communications tab. This causes the file attachments to sync. This must be done each time the customer adds one or more files.

Just select the file to view the attachment.

After syncing:
