What is this article about?

This document provides the process that CSS will use to cover Motherboard Replacement (referred to as MBR in this document) for Windows 10 devices.  

PC’s running Windows 10, only Motherboard replacements are eligible for replacement product key. 

MBR scenarios are broken down into three main categories.

For Surface advocates, see section 5.

2. The Windows 10 device had to undergo motherboard replacement (MBR) in-warranty and the original the operating system has not been reloaded 

When the customer boots up the device, it will have lost the old Windows 10 license and will need to reactivate.

Return to top

2.1 OEM has provided the product key, or the product key is embedded in motherboard 

  • No key replacement required for this scenario. The OEM will have provided the customer a 5x5 product key printed on a card or the product key will be embedded into the new motherboard, using this the customer can complete online activation. Device must be connected to the internet to complete activation. Offline activation is not allowed.

Note: The MBR 5x5 key provided by the OEM can also read Win Pro 8 in the description of the key when looked up in KIT. Do not refer the customer back to the OEM, the customer will be able to activate Windows 10 using such a key. 

Example: PK6 Win Pro 8 01801 OA3 OEM:DM.

Return to top

2.2 The device was upgraded from Windows 7 or 8.1 to Windows 10 

  • No key replacement required for this scenario.  

  • The OEM will have provided the customer a product key printed on a card or the product key will be embedded into the new board, using this the customer can complete online activation. Device must be connected to the internet to complete activation. Offline activation is not allowed. 

To retrieve the embedded product key use the below command. 

  • wmic path SoftwareLicensingService get OA3xOriginalProductKey.
  • The customer should see a product key returned in the format XXXXX-XXXXX-XXXXX-XXXXX-XXXXX.

To activate Windows 10 using Windows 7/8.1/10 key: Select Start > Settings > Update & security > select the Activation tab, then select Change product key 

Return to top

3. The Windows 10 device had to undergo MBR in-warranty or Surface device replacement, and the original operating system has been reloaded or the replacement was downgraded

When the customer boots up the device, they will need to reactivate their previous OS first, then upgrade back to Windows 10.

Return to top

3.1 Original system has been reloaded or the replacement was downgraded

For Surface devices that are replaced with a Windows 8.1 device, use the exception process outlined in these KBs: 

  • PA advocates: KB 3100924 Procedure: MPA: Windows 10 upgrade exceptions
  • All other advocates: KB 3099777 Procedure: Windows: Windows 10 upgrade exception

Return to top

3.2 OEM devices that have the product key embedded in motherboard

If the device came with Windows 10 the customer will be able to use the product key provided by OEM and complete activation of their Operating System. Device must be connected to the internet to complete activation. Offline activation is not allowed. No key replacement required. 

Note: The MBR 5x5 key provided by the OEM can also read Win Pro 8 in the description of the key when looked up in KIT. Do not refer the customer back to the OEM, the customer will be able to activate Windows 10 using such a key.

Example: PK6 Win Pro 8 01801 OA3 OEM:DM.

Return to top

3.3 OEM devices where the product key is not embedded into the motherboard

  • If the device came with Windows 10, the customer will be able to use the product key provided by OEM and complete activation of their operating system.
  • If the device was upgraded to Windows 10 from down-level Windows like 7/8/8.1, and the OEM has reloaded the original operating system, use the exception process outlined in these VKBs and ensure the customer has activated their Windows before using the below links to upgrade back to Windows 10.
  • PA advocates: KB 3100924 Procedure: MPA: Windows 10 upgrade exceptions.
  • All other advocates: KB 3099777 Procedure: Windows: Windows 10 upgrade exception.

Return to top

4. Windows 10 device had to undergo MBR out-of-warranty and the original operating system has not been or has been reloaded

When the customer boots up the device, reactivation is required to acquire a valid activation status. Any device that is downgraded to the original operating system will need to be re-upgraded to Windows 10.

Return to top

4.1 OEM devices that undergo MBR outside of the manufacturer or reseller warranty 

  • In this scenario, the manufacturer or retailer will not usually provide a product key to the customer, therefore they are eligible for a replacement product key at no cost through CSS if they are able to provide a proof of purchase.  

  • If a customer is unable to provide a valid Proof of Purchase, they will need to buy the replacement product key at $40 USD or equivalent from CSS. For details on the product replacement process, please refer to KB 2967104 Procedure: Office: Product key replacement.

Return to top

5. The customer previously upgraded their edition to Windows 10 Pro via the Store App but after MBR or device exchange, they want to return to Windows 10 Pro (Allies to Surface)

  • In this scenario, the key provided by the manufacturer or retailer will be for the original operating system, therefore they are eligible for a replacement product key at no cost through CSS. 

For advocates with access to MST: Ask what Microsoft account was used to purchase Edition upgrade to Windows 10 Pro. Validate the purchase via MST. 

For advocates without access to MST or if the Microsoft account does not show purchase info, ask customer to provide proof of purchase. 

Note: If the customer upgraded to Pro using a product key, guide them through the edition upgrade process using the change product key button in the Activation page.

Return to top

6. Key replacement process 

6.1 A valid proof of purchase 

The customer is not required to send a copy or image of the proof of purchase. The customer is only required to read out details for any one of the following: Advocates have to document the Proof of Purchase information the customer provided in PID Plus as part of key generation process. 

  • Product key or COA/GML (Genuine Microsoft Label) sticker information: A product key of Windows 10/8.1/8 or 7 can be accepted as part of this process.
  • To locate the product key, use the following command:
  • wmic path SoftwareLicensingService get OA3xOriginalProductKey
  • The customer should see a product key returned in the format XXXXX-XXXXX-XXXXX-XXXXX-XXXXX
  • To locate COA/GML: It is often attached to the body of a computer for some versions of Windows or Windows Server. You can typically find the COA sticker on the body of the computer or, for some newer laptops, inside the battery compartment.
  • Microsoft account used for purchase of Windows 10/8.1/8 or 7.
  • Invoice Number, of the OEM device purchased with Windows 10/8.1/8/7 pre-installed.
  • Electronic purchase confirmation number of physical or digital copy of Windows 10/8.1/8/7 purchased from an authorized reseller/retailer. 

Return to top

Notes:

  • If a customer is unable to provide a valid proof of purchase, they will need to buy the replacement product key at $40 USD or equivalent from CSS. For details on the product replacement process, please refer to KB 2967104 Procedure: Office: Product key replacement. Devices that originally shipped with Single Language (SL) product key can be replaced with a regular product key from the above list. CSS is unable to provide SL product keys.
  • Windows 10 Editions that are in S mode do not require a different replacement product key.  Replacement Product keys depend on Edition only, and whether they are in S mode or not doesn’t matter.
  • Windows 10 S PCs should be updated to the latest version of Windows (1803 or later) before entering a replacement product key.
  • For Windows 10 version 1803 or later, the customer will have to use a Power Shell command to enter the replacement product key:
    • Important: If the replacement product key is being issued to change the edition of Windows 10, the customer will first have to enter it using the Change product key button in Activation settings, and then follow the below Power Shell steps. If the customer is not changing editions, they should just follow the below steps.
    • From Start, search for power shell. Select Power Shell from the results.
    • At the command prompt in Power Shell, type slmgr /ipk and then the new product key. Press Enter.
    • The customer should receive a notice that the new key was installed. Reboot the device and verify that Activation succeeded.
