1. What’s new

In addition to the already existing information on how to deal with ATA not being available from EMS when purchased through CSP or Direct Channel, this update also provides information about System Center Configuration Manager (SCCM) and Microsoft Identity Manager (MIM), encountering the same problems with availability.

2. Description

Enterprise Mobility + Security (EMS)/M365 offer includes an Advanced Threat Analytics (ATA) license. However, when EMS/M365 is purchased directly via O365 portal or through the Cloud Solution Partner licensing models, ATA is not getting entitled.

In Microsoft articles such as this - https://docs.microsoft.com/en-us/enterprise-mobility-security/solutions/learn-about-ems, explains in detail what is included in EMS/M365 license. EMS/M365 is also marketed to include ATA - https://partner.microsoft.com/en-US/marketing/campaigns#/(us)-enterprise-mobility-plus-security.

4. How can customers who purchase EMS via direct and CSP licensing model obtain the key for Advanced Threat Analytics (ATA)

Customers need to download ATA bits from the Eval Center. Verify that the customer has valid EMS/M365 license before providing them the following key to convert the eval build to a production build - GFFNK-T3G9P-6DCMM-6M27F-JHWGT.

Note: Inform the customer that the key should not be used/shared illegally.

1. The ATA is on-prem, so the ATA Admin needs to login to the on-prem console, and under configuration, they have an option to provide a license key.

[Asset 4036136]

2. As soon the key was provided in the ATA Configuration, the customer does not need to do anything as the product switched to “licensed mode” and works normally.

Also, when the product is in “Eval” mode – the user will get notification on the ATA console as follows:

[Asset 4036137]

Note: If the customer presses this notification, they will be directed to the license configuration above.

5. How can customers who purchase EMS/M365 via direct and CSP licensing model obtain the key for System Center Configuration Manager (SCCM) and Microsoft Identity Manager (MIM)?

Note: Inform the customer that the key should not be used/shared illegally below SCCM and SQL static key.

In order to ensure the delivery of promised functionality to customers for all impacted services, in this case System Center Configuration Manager (SCCM) and Microsoft Identity Manager (MIM), use the following:  

  • System Center Configuration Manager:

Link to Eval Center: https://www.microsoft.com/en-us/evalcenter/evaluate-system-center-configuration-manager-and-endpoint-protection

Static Key: D4BKT-XJ3K9-T86DD-YM4TC-74FD7

As per Product Terms SCCM customers have rights to use SQL Server technology too. To fulfill this right, customers should install the desired version of SQL from the Microsoft Evaluation Center and activate with the correct Static key.

  • SQL Server 2019

Link to Eval Center: https://www.microsoft.com/en-us/evalcenter/evaluate-sql-server-2019

SQL 2019 Static key (key is valid for both Standard and Enterprise): FTYHH-G4BBN-HVCCR-JK8GP-27MR2

  • SQL Server 2017

Link to Eval Center: https://www.microsoft.com/en-us/evalcenter/evaluate-sql-server-2017-rtm

SQL 2017 Static key (key is valid for both Standard and Enterprise): 8BXJ8-BQ2TQ-JTVNF-8PTR8-4VW6H

For previous SQL Server versions please escalate to MPN Global support via CAP Child SR.

  • Microsoft Identity Manager:

Link to Eval Center: https://www.microsoft.com/en-us/evalcenter/evaluate-microsoft-identity-manager-2016

Static Key: No key needed

6. Q&A

Question

Answer

When will be the expiry date of the product key (the one being provided)?

The provided product key you received is permanent. ATA does not have limited-time product keys. It is either Eval (90 days) or Licensed (permanent).

What is the official avenue to activate ATA license via EMS/M365 subscription?

We are working to update the Licensing Datasheet in the “How to buy ATA” page with information how to activate the ATA license via EMS/M365 subscriptions, but it may take few days until the document will be published.

Since the key generated and shared is exceptional, where is the location of the portal that the partner should get the ATA license key?

Here is the location customers who purchase EMS/M365 through CSP or direct can access the ATA download bits and keys.

Microsoft Online Software Download Center - https://www.microsoft.com/online/downloads/HomeRealmDiscovery.aspx
