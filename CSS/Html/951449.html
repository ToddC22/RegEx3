BUG #: 88808 (Content Maintenance)
When you try to install a Web Services for Devices (WSD) printer on a Windows Server 2008-based computer, the operation fails. You do not receive an error message.
This problem occurs if the Print Services role has not been added on the computer. By default, the Print Services role is not added in Windows Server 2008. Additionally, the WSD print proxy is not installed. When you add the Print Services role, the WSD print proxy is also installed.
To successfully install a WSD printer, add the Print Services role, and then configure an exception in Windows Firewall to allow for network discovery. To do this, follow these steps:
  1. Add the Print Services role. To do this, follow these steps:
    1. Click Start, point to Administrative Tools, and then click Server Manager.
    2. In Server Manager, click Add Roles in the Roles Summary area.

      Note If the Before You Begin page is displayed, click Next.
    3. Click to select the Print Services check box, and then follow the instructions in the Add Roles Wizard to add the Print Services role.
  2. Configure an exception in Windows Firewall to allow for network discovery. To do this, follow these steps:
    1. Click Start, click Run, type firewall.cpl, and then click OK.
    2. In the Windows Firewall dialog box, click Allow a program through Windows Firewall.
    3. In the Windows Firewall Settings dialog box, click the Exceptions tab, and then click to select the Network Discovery check box.
    4. Click OK.
