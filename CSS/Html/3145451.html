Azure Support: Azure Kubernetes Services

 

 

Service Description:

  • Azure Container Service (ACS) is a container management service that manages creation of all the compute resources, that will be needed to host and run containers in Azure. ACS goal is to eliminate the need for you to install, operate, and scale your own cluster management infrastructure. ACS will create VMs, VM Scale Sets, storage accounts, availability sets, network security groups, virtual networks, network interfaces and other resources on your behalf. You can use ACS to manage docker containers on Linux VMs in Azure using open source container management tools like Mesos, Swarm & DockerMachine.

 

 

How Does Azure Container Service Work?

  • Azure provides a choice of either Docker Swarm or Apache Mesos (with DCOS Marathon and DCOS Chronos frameworks) for orchestration.

  • More info at: https://azure.microsoft.com/en-in/documentation/articles/container-service-deployment/

 

 

What is the difference between running containers on regular V2 VM vs running Azure Container service based container cluster?

  • ACS is more convenient and will run well (E.g.: Marathon with Cassandra etc. will take 2 weeks, but with the DCoS package on ACS it will take 5min).

 

How can customer provision the service?

  • Customers can use one of the templates provided to deploy a Mesos or Docker Swarm cluster. More information at: https://azure.microsoft.com/en-in/documentation/articles/container-service-deployment/.

What are the limitation of using ACS (Quota, size, Orchestrator type, OS of VMs etc)?

 

  • Quota: None specific to ACS – Same as with they run into today.
  • Size: All sizes supported by CRP.
  • Orchestrator Type: Mesos, SwarmPreview, OpenDcosPreview (Only these are available during Public preview).
  • OS for the VMs: Ubuntu only for now; 14.04 LTS (Mesos and Docker) and OpenDcos will be on 16.04.
  • All clusters only support max 40 Agents and min 1 agent min 1 master; Max masters 5 (get choices of 1,3 and 5 to do quorum)

 

 

What services does Azure Container service support?

  • Support everything that CRP VMs support

 

 

Pricing/Billing:

  • ACS is currently available for Standard A-series Linux Virtual Machines. You are only charged for the compute instances you choose, as well as the other underlying infrastructure resources consumed such as storage and networking.
  • There are no incremental charges for the ACS itself.
  • For details on the virtual machine prices, visit the Virtual Machine pricing page.
  • Ref: https://azure.microsoft.com/en-in/pricing/details/container-service/

 

 

Default and hard limits:

  • None specific to ACS

 

 

Partner Engagement:

Azure Container Instance (ACI) and Azure Container Services (AKS/ACS) are two different services. Select the appropriate engagement path based on customer request Partner Engagement Path;

  1. For Azure Container Instance (ACI) Quota - Create ICM using the template link. (please notice that only the amount of container groups can be increased. The limit of 4 cores and 16 Gb RAM per single Container Group cannot be exceeded)
  2. AKS currently doesn’t have quota limit. The quota limit is on the Compute. So if customers asked to increase the quota, its quota for their VM Cores. Do NOT open ICMs for AKS quota limit; you should instead assist the customer with increasing the VM Cores for the customer subscriptions. The CSS Wiki for AKS can be found here

SLA:
