ASMS – Support and Scope Boundaries

Submit a ModTrack request to update this TSG.

Overview

This TSG outlines the general scope and support boundaries of the Azure Subscription Management Support (ASMS) team.

Policy

  • A customer’s case should be owned by the team that is equipped to provide the solution to the customer’s issue.
  • The ASMS team should always be ready and willing to assist any team on any issue.
    • To leverage the ASMS team, the team requesting ASMS help should create a [Asset 4498193] and assign it to ASMS for consultation.
  • The ASMS team should not own the main [Asset 4497459] case for a customer issue that the ASMS team does not support. This will diminish the level of support the customer receives pertaining to their issue.
     

Supported Scenarios

ASMS supports the following categories of issues:

  • Azure Accounts.
    • Creation, provisioning, and management.
  • Azure Billing.
    • Pricing, payment, tax, credit, and refund questions
  • Azure compliance questions. (SOC, PCI, FedRAMP, etc.)
  • Azure Marketplace.
    • Cancellation issues.
    • Credits, refunds, and billing issues.
  • Azure Portal login issues.
  • Azure Quota Increases.
  • Azure Subscriptions.
    • Creation, provisioning, and management.
  • Billing API scenarios not related to coding/scripting using the Billing APIs.
  • Enterprise Agreement (EA) subscriptions.
    • Portal access and management.
    • Pricing and credits.
  • Ibiza blades pertaining to Pricing or Billing.
    • To check the supportability of Ibiza blades, select the blade, then press Ctrl-Alt-D to bring up the platform blade ownership information. If the blade name does not include Microsoft_Azure_Billing, it is not supported by ASMS.
  • Management of the following roles in the Ibiza portal and azure.microsoft.com:
    • Account Administrators.
    • Co-Administrators.
    • Service Administrators.
  • Migration of RDFE contents of one Azure Subscription to another Azure Subscription.
    • ARM or Ibiza service migrations are supported by the team that owns the service.
  • Role-Based Access Control (RBAC) billing role issues.
  • Support case submission issues.
    • Includes Technical Support Case submission.
  • Visual Studio Online (VSO), as it relates specifically to an Azure account/subscription.


In general, ASMS should not own the main [Asset 4497459] support case if the customer's issue does not fall into one of the above categories, or if there is no ASMS support topic or root cause classification pertaining to the customer issue. However, ASMS should always be ready to own a [Asset 4498193] to assist with any collaboration questions.

Note: The full list of ASMS Support Topics and Root Cause Classifications can be found in [Asset 4497459].

Common Misroutes

The following is a list of common misroutes which arrive at the ASMS queue, but are not supported by ASMS. See the Online References section for information on the MSaaS Support Portal, which can help you redirect the case to the proper team.

  • Access to Azure Active Directory (AAD).
    • ASMS can ensure that the AAD subscription is in an Active state.
    • Most Azure Active Directory issues are covered by the Cloud Identity and Office 365 (O365) teams, excluding Enterprise Mobility Suite (EMS) subscription issues.
  • Adding, deleting, or managing any Azure services, features, or deployments.
  • Adding, deleting, or managing users, tenants, or directories in AAD.
  • Any technical issues with any Azure services, features, or deployments.
    • This includes customers needing assistance getting their services back up and running after encountering a Subscription Disabled scenario.
  • Azure Portal or Ibiza issues not related to logging in, support case submissions, making purchases, or managing subscriptions.
  • Azure RBAC questions relating to anything other than Billing Roles.
  • Customer questions on Planned Maintenance downtime in Azure.
  • Deletion or management of storage accounts.
  • Office 365 Commerce issues.
  • PowerShell management of Azure accounts or subscriptions.
    • This is supported by the IaaS Platform queue.
  • Removing a domain from one tenant and associating it to a new tenant.
    • This is supported by the Cloud Identity team.
       

Engagement and Escalation Path

If you have additional questions, please reach out to your Technical Advisor (TA).

Online References

  • MSaaS Support Portal – An internal web site which lists all support teams and queues. Use this tool when searching for the correct queue to send a customer's case to. 
    • If you are unable to find the correct queue using the MSaas Support Portal, please follow the Misroutes TSG to engage the Duty Managers (DMs) for assistance.
