How to resolve crash and stability issues in Office programs by using the OffCAT tool

This article has been retired. To get more information about this update, please refer to Content Idea Request 69935. Submit a Content Idea via http://contentidea if any questions.

Microsoft Office programs may sometimes crash or stop responding. In this situation, you may see a dialog box that contains the following error message:

<Application> has encountered a problem and needs to close.

Sometimes, the program exits without warning or error. If the program stops responding, you may have to force the program to exit.
This kind of instability problem may occur for various reasons. That's why it can be frustrating and time-consuming to identify the source of these problems.
To help you identify and resolve known causes of instability in Office programs, a diagnostic tool, the Office Configuration Analyzer Tool (OffCAT), was developed for customers by a team of escalation engineers from Microsoft Support. The list of known issues that can be detected by OffCAT grows each month. Therefore, if no issues of yours were detected in any month, it is a good idea to use OffCAT the following month to check whether any new issues were detected.

To use the OffCAT tool as your first step in troubleshooting these issues in an Office program, follow these steps:
  • Step 1: Download and install OffCAT (Office Configuration Analyzer Tool).

    To do this, go to Microsoft Office Configuration Analyzer Tool.
  • Step 2: Start OffCAT, and then scan the Office program that's crashing.
    1. Start OffCAT by using the shortcut that was created during the installation process.
      • Windows 7 and earlier versions

        [Asset 4052461]
         
      • Windows 8

        [Asset 4052462]
    2. When OffCAT is starting, it automatically checks for the list of latest known issues. Make sure that you let this process finish so that the scan of your Office program uses the latest issues list.
    3. On the NEW SCAN page, click the Office program that's crashing.

      [Asset 4052464]


      Then, click Scan.

      Note If you cannot start Outlook because it crashes soon after you try to start it, right-click the Outlook tile and then select Offline.

      [Asset 4052465]
       
    4. After the scan finishes, exit any Office programs that are running.
  • Step 3: Review the known crashing problems that have been identified by OffCAT.

    Note If your Office program exhibits instability other than crashes, go to
    step 4.
    1. In the OffCAT scan results, click the Filter icon.

      [Asset 4052466]

      Note If you do not see an All Issues tab in the scan results, there were no known issues of the Warning or Critical type that were detected by OffCAT. Therefore, the Filter icon is not displayed.
       
    2. Clear the Select All box, select the Crash symptom in the list, and then click OK.

      [Asset 4052467]

      Note If you do not see Crash in the list of symptoms, OffCAT did not detect any known crashing problems. In this case, go to step 4.
       
    3. The first issue with a Crash symptom is displayed and expanded to show you more information about the issue. Click the Click here to see possible solutions to this issue option to open the article that contains information about a resolution.

      The following screen shot shows an example of one of the known crashing issues that might be detected by OffCAT:

      [Asset 4052468]
       
    4. Follow the resolution steps that are provided in the article.
    5. Repeat steps 3c and 3d for all other issues that are listed in the report under the Crash symptom.
    6. Test your Office program to see whether it continues to crash.
    7. If your Office program continues to crash after you follow the steps in this section, go to step 4.
  • Step 4: Check for Office product updates
    1. In your OffCAT results, click the Filter control one more time.
    2. In the Filter dialog box, clear the check boxes for any selected symptoms, select the Product not updated check box, and then click OK.

      [Asset 4052469]

      Note If you do not see the Product not updated check box in the list of symptoms, your Office installation has the latest updates installed. Go to step 5.
    3. Under Product not updated, expand the first issue that was found, as shown in the following screen shot. Then, click the Click here to see possible solutions to this issue option to open the article that describes the download for the latest update.

      [Asset 4052470]
    4. Follow the steps that are provided in the article to install the latest update.
    5. If there are two or more items under Product not updated, repeat steps 4c and 4d for each item that's listed.
    6. After you install all the latest updates that were suggested by your OffCAT results, restart the computer if you received a notification that a restart is required to complete the installation.
    7. Test the Office program to see whether it continues to crash.
    8. If the Office program continues to crash after you installed all the latest available updates, restart OffCAT.
    9. On the MANAGE SCANS page, click your previous scan to see the list of options for the scan.

      [Asset 4052471]
       
    10. Click View report to reopen your previous scan results.
    11. Go to step 5.
  • Step 5: Review additional instability-related symptoms.

    Program instability may become noticeable in many ways. Therefore, we recommend that you also filter and review your scan results for the following symptoms:
    • Add-ins
    • Configuration (unsupported or problematic)
    • Delay
    • Error message
    • Exchange connection
    • Freeze
    • Not responding
    • Slowness
    • Startup

    As a reminder, most of the issues that are detected by OffCAT have more than one symptom associated with them. Therefore, you'll probably see the same issue under one or more of the symptoms in the list. Regardless of where you find an issue (when it's listed under multiple symptoms), the resolution will be the same, and you have to act only on the issue under one of the symptoms.

    To filter your results for these additional symptoms, click the Filter control on the All Issues tab, and then select each symptom that appears in the list. The following screen shot shows the Filter control after two symptoms are selected.

    [Asset 4052472]

    Then, follow the recommendations that are provided for any issues that fall under these symptoms and that are related to program instability.
