1. What is the Microsoft Demos Portal?

The Microsoft Demos Portal (https://cdx.transform.microsoft.com/) is the location where users can get the latest demo resources from across Microsoft. In the past, this portal offered only Microsoft Service Account (MSA) credentials as a login option. The Portal has now transitioned into Work Account (AAD based) credentials as a login option.

2. Login Guidance for Partners

The https://transform.microsoft.com site uses the Microsoft Partner Center for authentication. Partners are required to be enrolled with the Microsoft Partner Center https://partner.microsoft.com to access the site. If your organization is not enrolled with Microsoft Partner Center, you will be unable to access these new resources. If you previously had access to these tools, but cannot access them now, it is likely that your partner organization is not enrolled with the Microsoft Partner Center. Please enroll your organization at https://partner.microsoft.com for continued access to these tools. If you need further assistance, please review the information on the Partner Center https://partner.microsoft.com/en-us/support/partner-center-help

IMPORTANT: If partners are encountering login issues (i.e. the "Not a Partner" error/screen), it is likely that their organization may not have completed all the necessary steps to connect the organization's Azure Active Directory to the Partner Center (this is a key step in the migration process). If users encounter this error, they should contact their organization's Global Admin to ensure all necessary migration steps have been performed.

This issue may be caused by an Azure Active Directory app consent issue. The AAD Admin should ensure that the tenant has given app consent to ID 'cdb1c100-cbe5-4a62-acb0-0a2ad77f698e' named 'Modern Workplace Tools (Service)' and that Enterprise Applications are enabled.

Please ask the partners to login follow these steps below to ensure they have the needed access for the Demos page:

A user with AAD Admin privileges can go to https://portal.azure.com

-Click on Azure Active Directory

-Click on Enterprise Applications

-Search for Modern Workplace Tools (Service)

-Click Permissions

-Click Grant admin consent for

Note: The option will approve this app for EVERYONE in their AD org.

Also, the users that are trying to access the Microsoft Demos Portal should enable the MFA, following the steps described in the article below:

https://docs.microsoft.com/en-us/azure/active-directory/authentication/howto-mfa-userstates
