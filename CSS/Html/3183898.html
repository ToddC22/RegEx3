Process: Microsoft Advertising: Using Google Import

[Asset 4509089]

Who should use this article?

  • Microsoft Advertising

What is this article about?

This article is about how advertisers can import campaigns from Google Ads directly through the web user interface using their Google Ads credentials. They can also import using a Google Ads import file, using the process outlined in KB 3183709 Process: Microsoft Advertising: Bulk Upload (Import from File).

1. How to use the Google Import tool

Advertisers can import campaigns from Google Ads directly through the web user interface using their Google Ads credentials. An Import Preview will show a summary of the changes that will be applied by the import. Advertisers will also have the option to opt out of certain settings or features like Ad Distribution and determine what aspects of their Google Ads campaigns to import.

The preview will display the additions, updates, and total for the main entities campaign, ad groups, ads, and keywords. All other entities will be grouped into others.

Advertisers can use the Google Import Tool to import both individual and multiple accounts into Microsoft Advertising.

Before import, be aware of the following:

  • We currently do not support mixed campaigns. If a customer wants to import Search and DSA, they will need to do two separate imports.

Importing process:

Advertisers can use the process outlined in Import Campaigns Directly from Google Ads to import both single and multiple Google Ads campaigns.

For imports, note the following:

  • Google Ads sign-in is required, and advertisers will need to agree to grant permissions to Microsoft Advertising’s UI to import all data successfully.
  • During import, advertisers can use What to Import to select several additional import options such as tracking, lists, or new/existing items.
    • Selecting Show Advanced Options will further increase their customization for these options.
  • Advertisers can set up a recurring Sync Schedule. See Schedule Imports from Google Ads and Review Import Error Logs for more information.
  • By default, we will raise bids/budgets to meet our system’s minimum requirements, unless an advertiser opts out of this during the import process.
  • Budget/Bid note: A common error when importing from Google Ads is to overlook the bid/budget options within the What to Import > Updates to Existing Items section, as these are hidden under the Show Advanced Options drop-down.

These operate separately from the bids and budgets values for new imports. Ask advertisers to review both sets of bid or budget options, especially if they plan to set up recurring Google Imports, to avoid unexpected changes being made in their Microsoft Advertising account.

  • Multiple account import notes:
    • Account level URL options – We currently support only a single check box to opt in/out of the multiple account level URL options for all the accounts being imported. Customers can select the Show Tracking Template link, to open a pop-up box which will contain the tracking templates for all the accounts.
    • Microsoft Merchant Center – We currently do not support multiple MMC store mappings for different accounts or for different campaigns for an account.

2. How to create an Import file

You can import data from other online advertising programs into Microsoft Advertising using a file.

Simply fill out the import template which you can download from Microsoft Advertising by selecting Import Campaigns, then Import from file. Select Download the import template, fill it out with your campaign data, and save it as a .csv file. You can then import the file into your Microsoft Advertising account.

For guidance on using the import file, see the Import data using a file help page.

3. What gets imported from Google Ads

Not all data is included in an import from Google Ads (for example, radius targeting or zip code targeting information). Be aware that you may not see a warning about the data that is not imported.

See What gets imported for more information on what does and doesn’t get brought in from Google Ads.

After the import, be sure to review your campaigns and ensure that targeting, bids, and all other data is configured correctly. Then make any needed changes within Microsoft Advertising.

Things to be aware of:

  • Audience Campaigns (Display Ads, in Google): Only images with supported aspect rations will be imported.
  • Location Targeting/Bid Adjustments: Be aware that if a location target from Google Ads doesn’t exist in Microsoft Advertising, campaigns targeting those locations will be mapped to the parent location if matched. Unmatched locations aren’t imported, but the corresponding campaigns may be active. We flag Location targeting errors with a warning message during import.
  • Campaign Start/End Date: If the campaign end-date is already in the past, the campaign will be set to paused. Otherwise, the campaign start and end dates will be applied to all Ad Groups within the campaign.
  • Certain Automated Bid Strategies in Google Ads are either unavailable in Microsoft Advertising, or require a minimum number of conversions to be available (see KB 3183081 Informational: Microsoft Advertising: Bidding and price estimation for more information). In these scenarios, the campaigns will be imported as eCPC and use a bid value taken from the Bid Management Platform.
  • Currency note: In Google Ads and Microsoft Advertising, currency is set at the account level when the account is created, and cannot be updated later. When importing from a Google account using one currency, into a Microsoft Ads account with another currency, the system will provide two options:
  1. Import using a currency converter, which will apply a monthly conversion rate to the bids/budget values. (Default option.)
  2. Import bids/budget values as-is, with no conversion for currency. (Advertisers will then need to manually adjust bids/budget.)

4. Best practices for Google Import

  • Your web browser must be set to allow pop-up windows.
  • Make sure your import file doesn’t exceed the Microsoft Advertising UI limits for campaigns or ad groups, as noted in the account setup and size limitations section of KB 3183363 Tool User Guide: Microsoft Advertising UI.
  • Each keyword or phrase is limited to 100 characters.
  • If you use keywords as dynamic text, make sure that they do not exceed 70 characters.
  • Adding multiple negative match keywords at the keyword level is limited to 1,022 characters.
  • The total file size must be 2 megabytes (MB) or smaller. Note: In instances where the file size exceeds 2 MB and it is not practical, divide the file into smaller multiple files. You may use Editor for campaign import.
  • Look for duplicate keywords in the file. For example, "Apple" and "apple" will appear as duplicates.
  • Look for keywords that may not be permitted. For example, "gambling" and "sexually explicit."
  • When importing from Google Ads, select not importing for the following columns to avoid creating errors due to unsupported functionality:
    • Tracking Template
    • Destination URL
    • Customer Parameters
    • Mobile final URL

5. Audience Campaign Imports (Google Display Network - GDN)

Advertisers can import Audience Campaigns (known in Google as GDN) into their accounts, including images, associated feed lists and Audiences and Targets.

Things to note:

  • Feed Lists: If Google feeds haven’t yet been imported into Microsoft Merchant Center, advertisers should import product offers to Microsoft Merchant Center first. Once the Microsoft Merchant Center store is set up, select this store under Import Options in the Google Import tool to link it to feed-based Audience Campaigns.
  • Campaign Placement Exclusions: To import these, select Website Exclusions > Advanced Import Options.
  • Ad Extensions: Imported into an account’s shared library, but Microsoft Advertising currently doesn’t support association of Ad Extensions to Audience Campaigns.

Audiences and Targets:

The following Audiences and Targets are available for import:

Audience Ads in Search Campaigns

Dedicated Audience Campaigns

In-Market Audiences.

In-Market Audiences.

Rule-based remarketing lists.

Rule-based remarketing lists.

Custom combination lists.

Custom combination lists.

Campaign-level and ad group-level associations for the audiences.

Ad group-level associations for the audiences (campaign-level not supported for dedicated Audience campaigns).

Importing Audience Campaigns

Advertisers can use the majority of the process outlined in Import Campaigns Directly from Google Ads to import both single and multiple Google Ads campaigns, with the following extra callouts:

  • At the What To Import stage, choose Show Advanced Options.
  • Select the tick-box for Audience Lists and targets.

Note: Customers can currently only import Audiences and Targets into single accounts. Shared lists will not be imported.

  • New Remarketing List, Audience Goal, or Conversion Goal Note: If these haven’t previously been imported the advertiser must do the one-time manual step of associating their UET tag.
  • Microsoft Advertising matches Google Ads In-Market Audience Associations to the corresponding In-Market Audience Segment in Microsoft Advertising, importing all associated attributes including status and bid boost.
  • If an In-Market Audience in Google Ads does not exist in Microsoft Advertising, ad groups associated with those segments in Google Ads will be mapped to the parent audience in Microsoft Advertising instead.
  • If there are multiple ad group associations for the same in-market audience, the exact matching segment with the audience will be imported (for example, Apparel & Accessories > Active wear). If no exact matching segment is available, then active targeting segments with the highest bid boost will be imported. If an association with parent segment was imported because the child segment wasn’t available at the time, when the child segment later become available, we will delete the original parent association and add the child association.

6. Technical issues preventing Google Import

If the advertiser is having trouble importing campaigns, here are a few tips that can help:

  • Set the web browser to allow pop-up windows. For more information, see the help section in the web browser.
  • Make sure the data you import meets the entity limits, features that are supported in Microsoft Advertising, and entities that do not get imported because they are not supported in Microsoft Advertising yet:
    • Refer to What gets imported for more information on what to check after importing from Google Ads. Review all the areas under Items you should check after import - Some specific situations require special attention during import. If you are importing any of the items below, make sure to review the following information. The import of these attributes may cause customer DSAT due to feature disparity. These are the areas where the customer may request for refunds and credits.
      • Multiple Campaign Languages: Go to What gets imported, to understand how languages are handled during import.
        • Problem: Due to how Microsoft Advertising handles languages, your ad group language may get set to English during the import and can cause low impressions if the customer wants to target different language customers.
      • CPA Bidding: Microsoft Advertising doesn't have CPA (cost-per-acquisition) bidding. When a CPA bid is imported from Google Ads, Microsoft Advertising changes the CPA bid to a CPC (cost-per-click) bid. If you use CPA bidding in Google Ads, we recommend that you review your ad group bids after importing.
        • Problem: This will result in running out of budget very quickly since CPA bids are usually way higher than CPC bids.
      • A detailed list of entities and attributes that do and do not get imported can be found at What gets imported (Note: Some information can be re-created after import using Microsoft Advertising).
  • Have the customer try a different browser.

For detailed troubleshooting on Google Imports, see KB 4294834 Campaign Management: Errors signing into Google Ads (Owned by Global Support).

7. Viewing Import History Logs

  1. Select Import Campaigns, then Import schedule and history. The table in the Logs section contains details on imports that ran in the last 90 days. Note: To review an error file, in the Error file column, select Download error file.
  2. Advertisers can check their past imports using the Import Settings column. This will provide a summary of the settings used during previous imports, with a selectable ellipsis control.
  3. Selecting the ellipsis will open a pop-up window that will display the import settings.

8. Exporting from Microsoft Advertising

Advertisers can export ads to a comma-separated values (.csv) file in Microsoft Office Excel, which they can then edit and import back into Microsoft Advertising or into another online advertising program.

9. Customer cannot log into Google Ads

If Google two-step verification is turned on, advertisers will need to create an application-specific password for Microsoft Advertising. Refer advertisers to support.google.com/adwords for guidance on establishing a unique password for Microsoft Advertising.

A unique password is required for Microsoft Advertising within Google Ads because Microsoft uses the advertiser sign in information to request campaign data from Google. Microsoft Advertising doesn’t retain sign in information for any purpose other than the import process.
