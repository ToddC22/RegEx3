2.1  Technical Support & Subscription Management  

IMPORTANT

All Microsoft Azure customers are entitled to support - EVEN IF THE CUSTOMER HAS NO SUPPORT CONTRACT.

  1. “Do you happen to have a Premier Access ID?”
    1. If they do, you will use it to create the customer’s SR
    2. If they do not, you will create a case using Professional Unpaid as the entitlement (see more details on case creation in section 1. Summary above)
  2. “Which Azure service do you need assistance with?”
  3. “May I have a brief description of the issue with Azure you are calling about today?”
    1. Paraphrase the problem description you have captured back to the customer to ensure it is correct.
    2. When creating the Service Desk case, you will choose Azure as the Product Family, then perform keyword searches to locate the most applicable Product Version and Support Topic, based upon the service and problem description provided by the customer
  4. Did the customer indicate that they need support with either:
    • Azure Stack HUB
    • Azure Stack HCI
    • Azure Stack Edge

Azure Stack Hub, Azure Stack HCI & Azure Stack Edge

Azure Stack Hub, Azure Stack HCI & Azure Stack Edge are products with very similar names but completely different call handling.

  • Azure Stack Hub & Azure Stack Edge  are handled via the Azure support information detailed in this article. 
  • Azure Stack HCI is a name used for a hardware product and a software product.  Customers may say Azure Stack HCI when using hardware or software.
    • Prior to July 2020, the Azure Stack HCI name was used for a server hardware solution running Windows Server 2019 Datacenter software. Today, support is handled as a standard Broad Commercial/Premier issue with Windows Server 2019 Datacenter as the problem product.
    • In July 2020, Microsoft announced a new operating system that re-purposes the Azure Stack HCI name for operating system software.  Support for the Azure Stack HCI operating system software is handled via the Azure support information detailed in this article with Azure Stack HCI-Preview as the problem product. 

If the customer simply says they need with Azure Stack, expand the section below and use the steps included to verify if it is Azure Stack Hub, Azure Stack HCI, or Azure Stack Edge; then apply the appropriate call handling.

Azure Stack / Azure Stack Hub / Azure Stack HCI / Azure Stack Edge - Detailed Call Handling

 

Presales Questions

If the customer's question is Pre-Sales related, you do not need to spend time verifying which product the customer has.  Refer the customer to https://azure.microsoft.com/overview/azure-stack/ and at the top of the screen the applicable phone number for the customer's location will appear.

  • Azure Stack HCI Call Handling:  If the customer states specifically that they need assistance with Azure Stack HCI, ask the customer:
    • "Is the operating system Windows Server 2019 Datacenter or Azure Stack HCI?"

  • If the customer mentions Azure Stack as the service they are calling about, ask the customer if the administration portal says Microsoft Azure Stack, Microsoft Azure Stack Hub, or Azure Stack Edge in the upper/left corner of the screen.
    • Screen caps are included in the collapsible section below.
  • [Asset 4580187]
  • If it is not yet apparent whether the customer has the Azure Stack Hub, Azure Stack HCI, or Windows Server 2019 Datacenter operating system, ask the customer:
    • "Does the server with the issue have a desktop or command prompt from which you can run 'msinfo32' ?"
    • YES
      • This can be either Azure Stack HCI or Windows Server 2019 Datacenter
      • If msinfo32 shows the OS Name as Microsoft Windows Server 2019 Datacenter (see the screenshots above):
        • Support is handled as a standard Broad Commercial/Premier issue. 
        • Discontinue the Azure call flow outlined in this article.
        • Instead follow standard call handling to create a Service Desk case choosing Windows Server 2019 Datacenter (as applicable) as the product.
      • If msinfo32 shows the OS Name as Microsoft Azure Stack HCI (see the screenshots above):
        • Support is handled via the Azure support information detailed in this article. 
        • Continue the Azure call flow via the Azure support information detailed in this article.
        • Choose Product Family = Azure and choose Product = Azure Stack HCI-Preview.
    • NO
      • This is most likely Azure Stack Hub
      • Continue with Azure call handling outlined in this article
      • Choose Product Family = Azure and choose Product = Azure Stack Hub.
  • If you are still unsure whether the customer has Azure Stack Hub or Azure Stack HCI (e.g. they are not able to take the steps or provide the information requested above currently):
    • Continue the Azure call flow via the Azure support information detailed in this article.
    • Choose Product Family = Azure and choose Product = Azure Stack HCI-Preview

    1. Record the address
  2. “Thank you.  And do you also happen to have the Azure Subscription ID?”
    1. The Azure Subscription ID is a key piece of information for support to begin investigation of the issue and needs to be recorded if at all possible.
    2. Customers can find their Subscription ID from a few places, for example:
      • Azure Portal
      • Accounts Portal
      • Aux Portal (classic portal)
      • Invoice emails from Azure
      • MSDN if connected benefits

    2. If for any other reason the customer cannot or will not provide their Subscription ID then enter "CUSTOMER_DECLINED_TO_PROVIDE_SUBSCRIPTION_ID" in place of the Subscription ID and proceed with the support request.
    3. Inform the customer that the engineer who will be contacting them will be unable to begin troubleshooting until the Subscription ID is provided. 
    4. IMPORTANT:
      “The support engineer who will be calling you cannot begin troubleshooting your issue without your Subscription ID.  If you don’t have that available now you should obtain it while awaiting their call.”
  4. Determine the issue severity based upon standard guidelines
    1. For additional information, see:  330504   Questions for determining severity levels and callback times - Policy and process

5. How to

5.1. What is SubscriptionID (subid) and how to I find it? 

SubscriptionID is how Microsoft Azure deployments are identified internally. One customer can have multiple SubscriptonIDs.

To find the SubscriptionID, ask the customer to follow these steps. If customer cannot provide their Subscription ID, ask for their OrgID or LiveID/Microsoft Account.

  1. Open your internet browser and navigate to http://manage.windowsazure.com
  2. On the left-hand side menu (blue vertical bar), find Settings (the very last option in the bottom)
  3. You should be able to see all your subscriptions with LiveID and SubID associated with it)

Return to top

 

5.2. What is Microsoft Account (MSA) – previously known as LiveID?

Microsoft Account - MSA (previously .NET Passport, and most recently Windows Live ID) is a single sign-on web service developed and provided by Microsoft that allows users to log into many websites using one account.

Microsoft Account (MSA) offers a user two different methods for creating an account:

  1. Use an existing e-mail address: Users are able to use their own valid e-mail address to sign up for a Microsoft account. The service turns the requesting user's e-mail address into a Microsoft account (i.e. @gmail.com, @yahoo.com or even @company.com)
  2. Sign up for a Microsoft e-mail address: Users can also sign up for an e-mail account with Microsoft's webmail services designated domains (i.e. @hotmail.com, @live.com, @msn.com, @passport.com and @outlook.com or any variant for a specific country) that can be used as a Microsoft account to sign into other Microsoft account-enabled websites.

5.3. What is a Federated account or OrgID?

Certain companies can establish a special connection to Azure and Microsoft, called Federation or ADFS. That allows those customers to use their OrgID (corporate email address) to access Microsoft Azure.
