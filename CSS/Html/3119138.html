Azure Support - VS Marketplace Subscription

 

Visual Studio Marketplace Subscription
This program brings together a set of exclusive benefits into one experience, making the developers job easier. The Visual Studio Marketplace is a new destination, the exclusive place for purchasing Visual Studio subscriptions and Visual Studio Team Services extensions.

Visual Studio Subscriptions are new flexible ways of purchasing Visual Studio Professional and Visual Studio Enterprise to provide great options for customers who are embracing the cloud for software development.

Visual Studio Professional and Visual Studio Enterprise will each be offered as monthly or annual subscriptions, sold through the new Visual Studio Marketplace. Visual Studio subscriptions are a new way to acquire the Visual Studio IDE along with cloud services. Visual Studio is launching new subscription options that allow subscribers more flexibible options when it comes to leveraging products and services.

  

Note: From the ASMS support perspective, a Visual Studio Marketplace subscription is similar to a VSTS license and the usage will appear as a separate line item in the usage report. There is no separate Offer type for Visual Studio Marketplace subscription as such. Example: the user will have a PAYG subscription, and the VS Marketplace subscriptions will be a line item in the CSV file.  No separate subscription will show up in CMAT like an Azure Marketplace subscription does.

Support Boundaries 
ASMS team will support scenarios involving purchase experience of an Azure subscription and other refund-related scenarios. For any technical issues, customer should be instructed to create a support ticket directly from the Visual Studio Marketplace portal with the technical team.

Note: ASE Team does not support issues related to the URL: https://marketplace.visualstudio.com/subscriptions or issues related to managing VS users.

If you have a case requesting to have the API logs or user management issues, please engage VSTS Developer Team Queue: Developer VSTS.

Only OMS accounts are eligible for purchasing a Visual Studio Marketplace subscription.

MINT accounts need to be backfilled to OMS before the customer can purchase a Visual Studio Marketplace subscription.

Can I buy the new Visual Studio subscriptions from my software reseller?

No, these new Visual Studio subscriptions are only sold via the Visual Studio Marketplace which uses Azure for billing. Resellers can sell Azure (prepaid amounts of Azure services) through the Enterprise Agreement, MPSA, and Open programs, but while prepaid Azure funds on the Enterprise Agreement and MPSA can be used toward most Azure services, they cannot be used to pay for Visual Studio subscriptions. Prepaid funds in Open can be used to pay for Visual Studio subscriptions.

What types of Azure subscriptions can be used to purchase Visual Studio subscriptions?

Visual Studio subscriptions can be purchased through almost any type of Azure subscription: Enterprise Agreement, Gov offers, EA Dev/Test, Open, Pay-As-You-Go, Visual Studio subscriber monthly Azure credits, etc. However, the $200 free trial cannot be used to pay for these Visual Studio subscriptions.

Can monthly Azure credits be used to purchase Visual Studio subscriptions?

No, monthly Azure credits (such as the $150/month credit for Visual Studio Enterprise with MSDN subscribers or the $100/month credit for Microsoft Partner Network organizations) cannot be used to purchase Visual Studio subscriptions.

Can Visual Studio Team Services extensions be purchased with prepaid funds on the Enterprise Agreement or MPSA?

Yes.

Are Visual Studio subscriptions the only services that cannot be purchased with prepaid funds?

No, there are other Azure services which cannot be purchased with prepaid funds, notably Azure Support Plans, 3rd party offerings in the Azure Marketplace, and Oracle VMs.

Why can I not use my subscriber monthly Azure credit to purchase Visual Studio subscriptions?

The monthly Azure credit you get as a Subscriber Benefit is for trying out Azure services risk-free, for prototyping and proofs-of-concept, and for light development and testing workloads. This monthly credit enables Visual Studio subscribers to build skills in using Azure as part of their software development. New Azure services are being released regularly, so having this credit is key to staying on top of the rapid changes underway as Azure evolves.

The Visual Studio IDE is already included in Visual Studio standard subscriptions (e.g., Visual Studio Enterprise 2015 is included for Visual Studio Enterprise with MSDN subscribers) and Visual Studio Team Services access is included as well, so the monthly Azure credit only applies to other Azure services.

Are there supplemental Enterprise Agreement discounts available for Visual Studio subscriptions?

No, the only discounts available are based on volume of purchases.

Are there renewal discounts?

No, prices for Visual Studio subscriptions are flat—the same price is offered for new and continuing customers.

What happens if an Azure subscription linked with VS cancelled and customer requests to reinstate the subscription?

ASMS will assist the customer with re-enabling a cancelled Azure subscription as long as there are no outstanding charges. For guidance on re-enabling a subscription refer to TSG 2651536. 

Can a customer cancel an Azure subscription with a connected VS subscription? If yes, what are the implications?

Customers can cancel their Azure subscription at any time. However, when the Azure subscription is cancelled all the linked VS licenses get cancelled, even if they have already paid for the rest of the term. Even after the Azure subscription is re-enabled, the VS licenses don't get re-enabled. The customer will need to repurchase the licenses and assign them to users.

How do I cancel/downgrade VS subscription?

1. From manage.visualstudio.com, click Change Quantity.
 

2. Change the Users field to to 0.

3. Click Update, and then click Confirm.

 

Note: Annual plan – If an annual subscription is cancelled, customer still has access to select benefits. We’ll have a way to opt-out of auto-renewal in early 2016, and we’ll send a reminder email well in advance of renewal.

Account owner/Service admin for this subscription is no longer accessible and there are no co-admins. Our subscribers are blocked. How can we take ownership of this subscription?

A best practice would be to ensure that there are more than 1 admin on a given account. This will ensure we avoid situations where the admin and the team is locked out.

If the customer does not have more than one admin on the account, ASMS to proceed with Subscription Ownership Transfer as per the available process.

Note: Subscription Ownership Transfer will not cause any loss/downtime on the VS side. The admins on the VS also change post transfer.

Follow TSG: AZURE SMS Account&Portal/Subscription/Subscription Ownership Transfer (OMS & Mint) to ensure the customer scenario is supported using the proper ownership transfer process. 

My credit card failed. I am going to put a new credit card in. How long before my users get reinstated?  Do I need to go through and reassign them (rehydrate scenario)?

In the current implementation, the customer would essentially be removed from the My Team site, and have to be re-entered. This means that the subscriber has to start over. This may change in the future.

Where and how can I see my usage/bill?

The usage will show up as separate line item on your Azure subscription invoice. For the annual plan, it will show up once. For the monthly plan, the usage/line item will show up monthly.

There won't be a recurring monthly email(s) sent.

How to engage VS Ops?

The engagement process is similar to the current VSO/VSTS engagement.

  1. Use //Myaccess - search for DevDiv Work Items
  2. Goto: http://aka.ms/WASMS-VSO-DTS
  1. Fill out the work item:
  • Always fill in the [Asset 4497459]case number. 
  • Include a clear description of the issue, with as much detail as possible.
  • Always include the Azure Subscription ID.
  1. When you are ready to send the TFS ticket to the VS Commerce team (Ops) do the following:
    • Save the work item once. 
    • Change the [STATE] field to “CSS Triage” and save again.
      • This will trigger an email to that team and send it to their triage.

Important: No action will be taken on the request until this is done. The work item must be moved to “CSS Triage” or it will remain invisible to the VSTS PG. 

  • VS Commerce Ops will comment the work item in the History section and\or email you. They will then assign the work item back to you with questions or a resolution. You should check on the work item periodically to see if it has been updated (copy a link to the work item for future reference after submitting it).
  • If the work item is sent back with questions, answer them in the HISTORY section then assign the work item back to the person who assigned it to you (you might send them a courtesy email as well telling them you’ve done this). You can see who assigned you the work item on HISTORY. Set these fields when sending it back:
  • State: 5 - PG Engaged
  • Who's Got the Ball? - Product Group
  • Assigned To: <The person who assigned it to you>
  • Once the issue is resolved VS Commerce Ops will assign the work item back to the WASMS engineer who opened it.
