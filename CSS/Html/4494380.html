Summary


Who should follow this procedure?
  • Modern Premier support resources (Premier General) supporting Microsoft 365 Premier cases, such as Cloud Knows Premier (CKP POD) engineers, Project Beacon engineers, Fairfax Frontline Engineers.
What is this procedure about?
  • How to handle Root Cause Analysis (RCA) cases.
  • Root Cause Analysis (RCA) is the process of identifying the cause of a technical issue after the issue has been remediated. It should only be performed at customer request and with clear boundaries set in advance to the extent both parties will invest in identifying cause. Where cause is required to be found before an issue can be resolved, that is considered troubleshooting and is out of scope for this procedure. Technical knowledge limitation from support on an issue during trouble shoot should go through the proper support escalation process, and it’s not in scope for RCA.
When should this procedure be followed?
  • When there is a legitimate RCA request landing in M365 Premier (Premier Support) team and needing a warm hand-over to internal Premier team (Premier Escalation).

Procedure

Review Procedure

Review the details to determine if the customer issue is eligible for RCA. If customer meets one of the scenarios below, they are eligible for RCA:

  • Scenario 1: Information on Post Incident Reviews (PIR) - RCA for Service Incident can be found in article 4456527 Procedure: Root Cause Analysis.
  • Scenario 2: FL should review the customer name (Profile tab) in RAVE Quick facts section to determine if [Asset 4577209] badge is available.  If it is, that means the customer has O365 Engineering Direct and the customer issue is eligible for RCA.
  • Scenario 3: Does the customer has a contract type showing in Quick facts including: Premier – Mission Critical, Mission Critical – Unified and Advanced Support, Performance support? If yes, the customer is eligible for RCA. However, the Critical SMC cases automatically receive RCA support, see 4456542 | Procedure: SMC CritSit for additional details.
  • Scenario 4: Hybrid issue

If the issue is on the Cloud side:

FL should check if the customer has Premier customer badge in the Quick facts section on the “Profile” tab in the company info page. If yes, this means the customer’s hybrid issue has a Premier or Unified support contract and is eligible for RCA.

If the issue is on the on-premise side:

FL should verify in the Case history section to see if the customer has an Access ID and Contract ID. If yes, this means the customer has a Premier or Unified support contract and is eligible for RCA.

  • Scenario 5: Premier support for partners (PSfP) tenant needs to have one of the scenarios mentioned above from 1-4 in order to qualify for RCA support.  Just PSfP status alone will not qualify the partner or the end customer for RCA support.

Additional information:

  • SPO has specific eligibility criteria for Root Cause Analysis requests outside of a PIR. Please follow this blog posting for additional guidance. If you have any questions regarding RCA entitlement, engage with your TA contact.
  • There should not be a significant amount of cases being transferred or reviewed as all items for the first 3 bullets above (under RCA Overview are routing to Premier escalation today).

 

New case enters queue with an RCA request
  1. After customer passes standard entitlement, FL will perform the Review Procedure to confirm that the case is a valid RCA case. Engage TL/PTL/TA for additional confirmation as needed.
  2. Add an internal note on the case with the following details after confirmation is completed:
    • Customer Ask:
    • Next steps: #RCA - PremierEscalation support needed
  3. Vertically move the case to the Premier Escalation team and select: Routing-Non-DAO and Service Offer-PremierEscalation, no matter what the language or Support Area is. On “Reason for transfer”, add “#RCA – Premier Escalation resource needed”. For more information, see Transfer between Premier General and Premier Escalation under Case transfers section in 4339885.

 

Case turns into RCA (break-fix issue is resolved)

The existing break-fix issue must be resolved completely before RCA engagement can start.

If a case is not escalated to TA/SEE:

  1. FL should perform Review Procedure to confirm that the case is a valid RCA case. Engage TL/PTL/TA for additional confirmation as needed.
  2. If confirmed, engage TL/TM/QM to create a new Rave case and keep ownership: 
    • New case is required for on-prem support or issue may be caused by an on-prem setting in a hybrid support
    • New case is not required for cloud only support.
  3. Add an internal note on the case with the following details:
    • Customer Ask:
    • Next steps: #RCA - PremierEscalation support needed
  4. Vertically move the case to the Premier Escalation team, and select: Routing-Non-DAO and Service Offer-PremierEscalation, no matter what the language or Support Area is. On “Reason for transfer”, add “#RCA – Premier Escalation resource needed”. For more information, see Transfer between Premier General and Premier Escalation under Case transfers section in 4339885.

If a case is already escalated to SEE:

  1. (EMEA) SEE engages on Teams Channel OOB/Product design/RCA/Advisory.
  2. SEE collaborates with Premier team to check entitlement, and follows Review Procedure.
  3. If confirmed, engage TL/TM/QM to create a new Rave case and keep ownership:
    • New case is required for on-prem support or issue may be caused by an on-prem setting in a hybrid support
    • New case is not required for cloud only support.
  4. Add an internal note on the case with the following details:
    • Customer Ask:
    • Next steps: #RCA - PremierEscalation support needed
  5. Vertically move the case to the Premier Escalation team and select: Routing-Non-DAO and Service Offer-PremierEscalation, no matter what the language or Support Area is. On “Reason for transfer”, add “#RCA – Premier Escalation resource needed”. For more information, see Transfer between Premier General and Premier Escalation under Case transfers section in 4339885.
