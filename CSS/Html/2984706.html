Procedure: Microsoft account: Handling "Sign-in is blocked" contacts

[Asset 4049349]

Who should use this article?

  • All Consumer

What is this article about?

What to do when a customer is prevented from signing into their Microsoft account because of one of two messages saying that their sign-in is blocked.

“Sign-in with <account> is blocked for one of the following reasons:

Someone entered the wrong password too many times.”

Or

“You've tried to sign in too many times with an incorrect email address or password.”

1.1 Why is this happening?

This happens when too many incorrect passwords were entered consecutively while attempting to sign-in, either by the account owner or someone trying to sign-in to the wrong account. However, this can also be caused by a computer or mobile app that is still using an old password or that needs to use an “app password”.

Return to top

1.2 How do I fix this?

The account will automatically unblock after a short period of time, which can range from one minute or more depending on how frequently sign-in attempts were being made. If this was due to repeated sign-in attempts with bad passwords, use of the correct password or a password reset will resolve this. However, if this was due to an app or device automatically attempting to sign-in with the wrong password, the device or app in question will need to be updated with the right password or an app password before this can be resolved.

2. Things to try

2.1 Was the password recently changed?

Ask if the password was recently changed and double-check the ASD MSAS tool to ensure the right account name is being used and if the password was recently changed.

If the customer’s info does not match what is shown in ASD MSAS, the customer may be trying to sign in to the wrong account. See KB 2979990 Microsoft account: Process: Error "That Microsoft account doesn't exist".

If the password wasn’t recently changed, skip to section 2.2.


If the password was recently changed and...

Scenario

Advocate action

If customer is sure they know the password-

Advise that customer should check for apps or devices, especially email on smartphones, that sign-in to their account and make sure the sign-in info for those apps is updated.

If customer is sure they didn't change their password recently but ASD MSAS shows there was a recent password change-

Ask again if customer is sure they, or someone who they gave their password to, didn’t change their password recently.

If the customer is certain they have not changed the password, this may be a compromised account. Have the customer reset their password per section 2.3 and then refer to section 2.2 of KB 2980022 Process: Microsoft account: How to handle error "It looks like someone else might be using your account" for further instructions.

If customer is unsure if they can remember their password and has the "You've tried too many times..." message-

Advise that customer can keep trying but will have to enter the characters shown each time they guess incorrectly. However, if they want, they can just reset their password to get back in. (See section 2.3 for password reset.)

If the customer is unsure if they can remember their password and has the "Sign-in is blocked" message-

Advise the customer that they can keep trying but they will have to wait for the temporary block to clear. They can try again in 30-minute increments. However, if they want, they can just reset their password to get back in. (See section 2.3 for password reset.)

If customer cannot remember their password-

Skip to section 2.3.

2.2 Was Two-step verification recently enabled?

Check ASD to see if Two-Step Verification is enabled for the account. If it is, ask if Two-Step Verification was recently enabled and if there were any problems signing into to any apps or devices afterward.

If yes, explain that some aps and devices, like Xbox 360 and older versions of Windows Phone, are not fully compatible with Two-Step Verification and require an “App Password” to work. While the sign-in screen will usually suggest that you need an app password, this is easy to miss so it’s easy for an account to get locked out this way.

To resolve this, refer the customer to the App Passwords and Two Step Verification FAQ at http://windows.microsoft.com/en-us/windows/app-passwords-two-step-verification so they can create an app password for their device or app.

2.3 Walk customer through password reset

1. Depending on the message shown, select Reset your password or Can't access your account?

[Asset 4493710]

2. Select the most appropriate option, then select Next.

[Asset 4493729]

3. Enter the name (email address or phone number) of the Microsoft account that needs reset and then type the characters shown in the image on the page, then select Next.

Note: Customer can do one of the following if they cannot read the characters on screen:

  • Select New to generate a new image.
    • Select Audio to have the characters read to you.
 [Asset 4493717]

4. Select where to send a security code:

  • Via email
  • Via SMS text message
  • Via phone call
  • Via the Authenticator App

The list of options is based on the security info a customer has associated with their account, so some of these options may not appear.

Note: If none of the security info options listed are accessible, selecting I don't have any of these will start the account recovery process. This should only be done as a last resort. (See KB 3123259 Process: The Recover your Microsoft account form if customer has to use this option.)

[Asset 4493718]

5. If customer elects to receive a code via email, customer will need to supply the first part of the email account address referenced by the masked address above.

(For example, if the “Email” shown refers to 1account@outlook.com, the customer would enter “1account” in the field shown to match the 1a***** shown above).

Note: Email masking always uses 5 asterisks; they do not equal the number of masked characters.

If customer selects to receive a code via phone text message or phone call, customer will need to enter the last 4 digits of the phone number referenced above.

Once customer confirms the masked info, select Send code to continue.

[Asset 4493719]

6. A window prompting for a security code will appear while the code is sent to the security info contact the customer selected.

Important: Tell customer to leave this window open while checking email or phone for that security code message.

[Asset 4493720]

7.a. If sent to email, this message from “Microsoft account team” account-security-noreply@account.microsoft.com should arrive in the customer’s Inbox shortly.

If the email does not arrive within a couple minutes, check the “Junk” folder or refresh.

[Asset 4493721]

7.b. If sent to phone via text (SMS), the security code should appear shortly.

Make sure you copy the security code in the message, not the number the message was sent from.

If sent via voice call, have customer write down the code.

[Asset 4493722]

8. Copy the code and paste it into the Enter your security code window, then select Next.

[Asset 4493723]

9. Choose a new password, enter it again, then select Next to finish resetting the password.

[Asset 4493724]

10. If customer has followed all steps correctly, the following “Your account has been recovered” message should appear. Select Next to close and have customer try to sign in with the new password.

[Asset 4493725]

11. The customer may receive the following message after performing a password reset. This is a safety mechanism to alert the customer in case of an unauthorized change and can be ignored if customer initiated the change.

[Asset 4493726]

12. If the customer had Two-Step Verification enabled, a message will be sent to inform them that their app passwords will need to be changed because of resetting their password.

[Asset 4493727]

If the customer cannot complete the above process for any reason, see KB 3123259 Process: The Recover your Microsoft account form for details on how to use the “Recover your Microsoft account” form.

4. Additional details

4.1 Two-step verification

Two-Step Verification, or TSV, utilizes an authenticator app and app passwords as an additional security measure against attacks.

Refer customers to the Two-Step Verification FAQ at http://windows.microsoft.com/en-us/windows/two-step-verification-faq for more info.

Return to top

4.2 App passwords

Some apps and services do not support TSV and the Authenticator App. In these cases, an App Password must be configured for that app or service. Refer customers to the App Password and Two-Step Verification FAQ at http://windows.microsoft.com/en-us/windows/app-passwords-two-step-verification for more info.
