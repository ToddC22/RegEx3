Key notes

  • For users with Rave read-only access, join the ravereadonly security group via http://idweb. Note: A business justification must be provided otherwise the request will be denied. Read access allows users to:
    • Search for Office 365 customer by using a domain / tenant id.
    • Search for Office 365 Rave support cases.
    • Add internal notes to any support case. Your name and timestamp of the note will be logged automatically.
  • Rave uses a combination of hard and soft skills to match the case details to the next available advocate best suited to handle the customer’s request and offer it.
  • Hard skills are skills that advocate possess that are non-negotiable when locating the best resource for the case. The hard skills will ensure the customer is supported by the correct team program, with the correct technical skills for the issue.
  • Soft skills are preference-based skills and are negotiable when locating the best resource for the case. The soft skill checks will, for example, attempt to route the case to an advocate that is located in the same region as the customer, so the advocate will be more likely to speak his or her native language.
  • If the case has been created after-hours and the customers preferred language support queue is closed, the customer will be asked to confirm willingness to work with Global English support. If agreed to, the case will be transferred. Otherwise, they will be notified to contact support during business hours.
  • DAO stands for Direct Agent Offer. When DAO is turned on, the tickets are routed and offered directly to the available ambassador to accept. The description of the ticket is not made available before the ticket is accepted.

Queue Management

Unassigned queue displays the cases that matches your Support Area + Skills combinations which are set by an Admin. Not all case information is displayed in the Unassigned view for QM/DM to properly assign cases to the right resources in the best order. QM/DM need to switch between multiple Support Areas and determine which cases should be assigned first, look at all Skills and all Support Areas and ignore the cases that are not relevant. Also, for cases misrouted to the general Enterprise Support Area, DM/QM need to monitor and route them to the appropriate support area. 

            Note: DAO tickets will not be displayed on the Unassigned queue.

FAQs

Q: How to review an engineer’s skills?

A: To check an engineer’s skills, on the Admin Dashboard – queue view, enter the alias of the engineer to the Ambassadors filter and search -> right click the account and click Edit. Then you can check the skills assigned to the engineer and you can correct the skills and re-assign them.

Q: How to use the Quality view for real time case load management?

A: Enter values to the filters to see the cases under the specific settings. For example, if you support in Exchange, English, and want to see cases taken today, then select language: English, enter your team name: Microsoft – Premier (please pay attention to the team name format), select the time period: today, then search, you will see all the cases taken by your team in the time period.

Q: What PowerBI report is available for the delivery team to use?

A: Delivery team can use the PowerBI report to monitor and report support cases. Access the OpenCases report for the current opened cases (refreshed on an hourly basis). The SIP (Support Insights Portal) report is also being adjusted to support the new Premier skill filter.

RAVE Team Names for CSS

This guidance is targeted at CSS suppliers that support the Concierge, Enterprise, Partner and Premier workloads. All CSS supplier teams, including Enterprise and Microsoft FTE is now under the CSS-Concierge Organization in RAVE. The Supplier field is a hard-coded list of all CSS Supplier/Sites. Team Name is a free-form text entry field.

Structure: Organization > Supplier > Team Name

Example: CSS-Concierge > Convergys – Pune > TeamName

Due to reporting needs and current limitations in the Quality View, we must standardize how the Team Name is input. Below is the standardized team name structure.

All sections besides the last one, are abbreviated to 3 characters. The last section will be the combination of the word “team” and the alias of that team’s manager as they would show in OneTeant (filas). The below example assumes that John Doe is the team manager.
