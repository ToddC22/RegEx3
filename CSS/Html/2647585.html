Overview

This article describes how ASMS can convert a customer subscription from one offer type to another. Use the following document to assist a customer in converting from one Azure subscription type to another using the Commerce Management Agent Tool (CMAT).


Applies To: All customers who want to convert from one Azure subscription type to another offer type.
Example: A customer currently has an Azure MSDN benefit subscription, but then decides that they want to change to an Azure Pay-As-You-Go subscription. 

Policy

  • For Azure in Open (AIO) conversions, ASMS only supports converting AIO subscriptions to Enterprise Agreement (EA) subscriptions.
  • No subscription type should be converted to an Azure Pass subscription, without direct approval from the Azure Pass team.
  • No Azure subscriptions should be converted to the TechNet Cloud Essentials Benefit (-0024P). This is an active offer, just not for Azure. Because it is an active offer it will show up in CMAT, but no Azure subscription should be converted to this offer type. 
  • An Azure account is only allowed to contain one MPN subscription (even if suspended). You should not attempt to convert a subscription to MPN if one already exists (even if suspended) under the account. If you do, you will receive the following error message in CMAT: "User limit is exceeded for Offer id 'MS-AZR-0025P".
  • Some offer conversions may fail in CMAT, with one of the following error messages: 
    • "Get Product Eligibility Failed. Status invalid, both sponsored subscription, commerce entitlement is not in enabled state."
      • This is due to the subscription being in a suspended state. A subscription can only be converted or transferred when in an active state. To resolve this issue, you must first active the subscription. 
    • "4011- Line1 required"
      • This is due the customer's profile in the Destination Account or the "Ship to" Order Address is missing mandatory details.
      • To resolve this issue, ask the customer to complete the mandatory fields under order "ship-to" address. Once they have updated the fields, attempt the transfer again. 

Support Scenarios

Before converting any subscription to another offer type:

  1. Check if the subscription has pending charges: 
    1. If the subscription has pending charges, follow step 2.
    2. If the subscription doesn't have pending charges, you can convert the subscription after checking the prerequisites in this TSG.
  2. Inform the customer they still will have pending charges after conversion, and they have to pay the pending charges.

Example: If the customer wants to convert from Pay-As-You-Go (PAYG) to Sponsorship, the Sponsorship credits will not cover the charges from the PAYG subscription. The customer has to pay by the payment method they had before converting. If the customer wants to pay the pending charges with the Sponsorship credits, engage AIRS team (don't guarantee the customer that they can use credits as this is subject to the AIRS team approval) to deduct the pending charges under the Sponsorship credits.

Scenarios

ASMS Support Engineers may assist customers who wish to convert their Azure subscriptions in the following scenarios:

Scenario #1 – Customer wants to convert from a Pay-As-You-Go subscription to a Microsoft Developer Network (MSDN) subscription. This option is only available for MSDN-qualified customers.

  1. Use the CAT tool to verify the origin customer has an active subscription.
    1. First time users should select CSSSubscription Validation Tool from Microsoft Dropdown box.
  2. After verification, go to How to Change an Offer Type in CMAT and follow the instructions there.

Scenario #2 – Customer wants to convert from an MSDN offer to another offer. Although this scenario is unlikely, this request does not require escalation.

  • Ask the customer to go to My Account at azure.microsoft.com, remove the monetary cap, and enter a valid credit card as the new Payment Instrument (PI).
  • Once the new PI is in place, you can complete an offer conversion to Pay-As-You-Go. Go to How to Change an Offer Type in CMAT and follow the instructions there.

Scenario #3 – Customer wants to convert from a non-partner offer to a partner offer.

This option only applies to partners who exist in the Microsoft Partner Network (MPN).

Note: If a customer needs assistance purchasing an entitled benefit subscription except for BizSpark Plus/MS4SU, use the steps below to resolve the issue. For BizSpark Plus/MS4SU, customers must be properly entitled and use that entitlement to make requested purchase through the official process. ASMS may not attempt to bypass the BizSpark Plus/MS4SU process.

  1. Try to get the user to make the purchase via the correct link first (you can find offer details at http://azure.microsoft.com/en-us/support/legal/offer-details/).
  2. If the customer is still unable to activate, work with the appropriate partner team (MSDN, MPN, BizSpark, etc.) to make sure the user is entitled before doing an offer conversion from a Pay-As-You-Go subscription to an entitled offer. Manual conversion can cause significant problems when the systems connect later and this person may lose the benefit because it wasn’t started properly.
    • If the customer is entitled:
      Convert the customer’s subscription from a Pay-As-You-Go account to their appropriate benefit subscription. Go to How to Change an Offer Type in CMAT and follow the instructions there.
    • If the customer is not entitled:
      Inform the customer that they do not have an active entitlement for the requested offer. The customer will need to purchase a new subscription.

Scenario #4 – Customer wants to convert a Pay-As-You-Go/paid subscription to an Azure MPN subscription so that they can use MPN benefits.

Often, the customer has an existing Pay-As-You-Go/paid subscription which they want to activate their MPN benefits on, but they end up creating a new Azure MPN subscription instead. They request that we convert their Pay-As-You-Go/paid subscription to Azure MPN so that they can use MPN benefits.

If data migration from the paid subscription to the new MPN subscription is not an option, you may convert the Pay-As-You-Go/paid subscription to an Azure MPN offer, but things must be done in the proper order, as an Azure Account can only have one Azure MPN subscription. 

  1. Deprovision the incorrect Azure MPN subscription by following the instructions in TSG 2646764: “Request to Cancel Now - Only to be done by TA”
  2. Go to How to Change an Offer Type in CMAT and follow the instructions there.

Scenario #5 – Customer wants to convert an EA subscription to an EA MSDN Dev/Test subscription.

  1. Log in as the enterprise administrator.
  2. Click Manage.
  3. Click the Account tab.
  4. Click to select the row to enable Dev/Test access.
  5. Click the Edit (Pencil) icon of the row.
  6. Check the Dev/Test checkbox.
  7. Click Save.
  8. After they have checked the box, you can do an offer conversion for them to the new -148P (EA Dev/Test) offer. (Go to How to Change an Offer Type in CMAT and follow the instructions there.)

Note: As per the agreement signed by the customer, conversion from EA to EA Dev/Test is not supported for Production environments.
 
Please confirm with customer that it is indeed a Non-production Test Environment and only then convert the subscription to EA Dev/Test (provided they are made aware of the new Quota Limitation which will be in place after the conversion to EA Dev/Test).

Scenario #6 – Customer wants to convert any type of subscription to an EA subscription.

  1. Verify that the customer has a valid EA enrollment by verifying it in the EA portal (ea.azure.com).
  2. Confirm that the customer wants to move all subscriptions on the account under that EA enrollment. This is an all or nothing process. If the customer wants to move a select few subscriptions, then a subscription ownership transfer will be required so that only the desired subscriptions are converted.
  3. Request that the user pull the subscriptions under their EA enrollment by adding the account admin to the EA subscription. This will convert them all automatically. 
  4. If the customer is in the EA portal and does not see the subscriptions listed as an EA subscription, then you may need to complete an offer conversion to an EA offer. (Go to How to Change an Offer Type in CMAT and follow the instructions there.)

Note:  Subscriptions should only be converted to EA after it is verified that the account with the correct authentication account (Microsoft Account / Work or school account) is Active in the EA Portal. If the account is still shown as Pending, first work with the customer to activate it before converting to EA.

Scenario #7 – EA Customer wants to convert a non-EA account.

  1. Convert the EA subscription to a BizSpark offer and then transfer this subscription to a non-EA account.
  2. Once transferred, convert the BizSpark subscription to the desired offer type that the customer chooses.

Note: The above workaround will not work if the EA subscription account uses an invoice method for payment. The Invoice PII blocks the action when you try to convert the EA subscription to a BizSpark offer. The error message states, “Invoice is not allowed for this order. Amount is above credit check threshold.” Increasing the threshold on the account will not work, and the conversion will still fail with the same error.

Scenario #8 – Customer wants to purchase or convert a paid support plan.

Purchase or conversion of a paid support plan can be performed by the customer directly via self-serve action in the Account portal, however, if they have issues or problems, ASMS can assist with this process.

  1. Gain customer approval for the paid support plan change.
  2. Verify the required support languages are available (only Pro-direct is limited).
  3. The paid support conversion can then be processed through CMAT.

Note: For more information, see the Paid Technical Support TSG (TSG 2777322).

Customer facing content - How do I purchase Azure support?

See http://www.windowsazure.com/en-us/support/plans/ for additional support plan details.

Workarounds for converting EA subscription with Invoice PII:

  1. Perform Subscription Ownership Transfer (SoT) of the EA Subscription (tied to Invoice) from the current EA Account to an EA Account under the same enrollment which does not have any PII associated with it.
  2. Make sure to select “Without payment instrument” before doing the transfer. This will ensure the invoice tied to the EA sub removes after SoT under the destination account.
    1. If there is no eligible account without a PII, then the customer can add a new account to EA and then purchase an EA subscription under that account.
  3. Convert the EA Subcription (now without a PII) from EA to Sponsorship offer.
  4. Perform SoT of the Sponsorship subscription from EA Account to an account which is non-EA.
  5. Convert the Sponsorship subscription to the desired offer type that the customer chooses.

Loss of Billing History After MOSP Offer Conversion to AIRS or EA

Before converting a Microsoft Online Subscription Program (MOSP) subscription to an Azure Internal Registrations System (AIRS) or EA subscription, you should instruct the customer to back up their billing history. After conversion, the entire path to get billing details will change and the customer will lose access to their previous billing history. If they stay within MOSP, they will still be able to see their billing details.

How to Change an Offer Type in CMAT

  1. From the Commerce > Subscriptions screen in CMAT, click the existing offer under Offer Name.
  2. On the Convert Subscription Plan screen, choose Select a new plan and click the blue arrow.
  3. On the next screen, choose the new offer, then click the blue right arrow.

    Note: Always verify the offer number of the target offer before clicking the blue arrow. Some offers have similar names, but the underlying offer number will always be unique.

    Secondary Note: Not all offers can be changed to other offers. See the list of approved scenarios above before moving forward.
  4. On the next screen, select the payment instrument and enter comments as needed, then click the blue right arrow.
  5. On the next screen, add the SR number and confirm the subscription offer conversion by clicking the blue check box.
    [Asset 4015716]
  6. After converting a subscription, always verify the OfferCategories string as explained in the next section.
  7. Let the customer know that changing the offer type will not change the subscription name. They will need to change the name in the accounts portal.

How to Verify the Offer Category in Jarvis

After a subscription is converted from one Offer Type to another Offer Type, you should verify in RDFE that the subscription's Offer Category has been updated. There are issues with some conversions where the Subscription will appear to be converted successfully in CMAT but is still considered to be the old subscription type in RDFE. This can cause several issues, such as subscription-specific options not being available to the converted subscription, or the new subscription type missing benefits it should have. For example, converting a Pay-As-You-Go subscription to BizSpark Plus/MS4SU and not having the BizSpark Plus/MS4SU credits showing up after conversion.

  1. In Jarvis, go to AzureRT/Subscription Management/Get Subscription with Details
  2. Enter the subscription GUID of the subscription you're verifying and click Execute.
  3. In the output, look for the string OfferCategories: near the top of the output. It will look like this:

    OfferCategories:Azure_Platform_All;Azure_Paid;Azure_Consumption;Azure_MSDN;Azure_Credit;Azure_MS-AZR-0064P
  4. If the Offer Type (the MS-AZR-00xxP string) does not match the intended Subscription type, or if anything else seems incorrect with the whole Offer Categories string, the Offer Categories may need to be updated. To get the Offer Categories of a subscription updated, please escalate a CRI to the ASMS Service Engineering (ASE) team - http://aka.ms/asecri

Unable to Remove the Spending Limit/Offer Conversion

If the customer receives the error "The service use location is incomplete," please make sure the “Ship To” address is complete. This can be done two different ways.

  1. Go to the Billing > Order tab in CMAT and click Update Order Address. This is the same as the “Ship To” address displayed in other places.
  2. Have the customer update the “Ship To” address at https://commerce.microsoft.com/PaymentHub/Profile or by going to Subscription in the accounts portal, selecting the subscription, and clicking on Change subscription address.
