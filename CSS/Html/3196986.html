1. Instructions for all advocates 

Important! When advocate tools go down, it is against Microsoft Privacy Guideline to record customer's personal information (including name, telephone number, address, email address, credit card information, PID, product key, and other information) anywhere other than in the appropriate advocate tools. Specifically, do not store this information, even temporarily, in documents on the computer or with pen or paper.

For any of the outages for tools or services listed below, contact your manager to report the problem. The Team Manager should engage the tools escalation procedure through standard escalation (to CSS Command Center (CSS CC) or TPI, for example) submission so that the appropriate team can be contacted to resolve the problem.

See KB 3038858 Procedure: How and when to engage CSS Command Center (CSS CC) all regions

Do not attempt to create an SR, escalate, or transfer the customer without direct instruction from your supervisor/lead or this article.
If a support tool is down, most scenarios dictate the customer should call back later.

2. Sales and Accounts & Billing

2.1 Contact handling

If a tool outage prevents you from assisting the customer:

1. Check the Scenarios table below for possible workarounds.

  • If you are still unable to assist your customer, use the messaging below to notify the customer.

Messaging:

Tool down:  

" We are currently experiencing unanticipated difficulties accessing our tools. At this point, I do not know the exact time when the system will be back online."

Call-back later:

"To continue helping you, I need to access to your customer account information. Please call us back any time after [approximate end of downtime]. I will be happy to help you once we regain access to our system."

3.3 App services down

Microsoft Store

A customer might think the service is down and contact us for help. Here are a few examples of what a customer might say:

• Is the Store app down?
• I'm unable to log into Store, are you experiencing any issues today?
• Is the Store down? I'm unable to chat with anyone.
• The customer may see an error message that Store is not available.
Office.com Store

A customer might think the service is down for Office.com Store and contact us for help. Here are a few examples of what a customer might say:

• Is the Store app down?

• I'm unable to log in to Store, are you experiencing any issues today?

• They may see an error message that Store is not available.

Try other web sites to make sure the customer can see and log in using other sites (For example, live.microsoft.com). If the customer can log in on other sites but not Office.com Store, contact your Team Manager or Tech Lead. The Team Manager will engage the tools escalation procedure through TPI submission.

If they cannot login using their Microsoft Account, start troubleshooting Microsoft Account using existing [Asset 4346155] articles.
Xbox apps - Xbox Live Games, Video, and Music

Server Outages XBOX apps connect to services. A customer might think the service is down and contact us for help. Here are a few examples of what a customer might say:

• Is Xbox app down?
• I'm unable to log in to Video, are you experiencing any issues today?

Check the service status at http://www.xbox.com/status. If the server status is showing as offline, the expert will ask the customer to try again in a few hours.

Messaging:

“We are aware and are working to resolve the issue. Please continue checking the status of the service at http://www.xbox.com/status where we will post updates on the availability of the service.”

If the server is running, troubleshoot the issue.

4. Product activation

4.1 Contact handling

If the PA application is down, try to troubleshoot the tool. The PA application troubleshooting steps will help reduce escalations to Microsoft IT should you resolve the issue on your own. If you find the need to escalate an event to CSS Command Center (CSS CC) around one of the three identified PA application scenarios below, the PA application checklist will help Microsoft IT investigate application related issues with more efficiency and speed. The requested detail outlined in this checklist will hopefully help identify elusive root causes.
  • Follow your standard escalation procedures to CSS Command Center (CSS CC), which may include submitting an incident using EMP https://microsoft.service-now.com/ccsp/. See KB 3038858 Procedure: How and when to engage CSS Command Center (CSS CC) all regions.

Immediately identify the application template (in accordance with the event's issue type) from the below and completely fill it out. Team Leads should send the completed template to CSS Command Center (CSS CC) (csscc@microsoft.com), referencing your EMP incident number as a part of their escalation email.

CSS Command Center (CSS CC) ensures this information reaches PACTHELP ASAP. The information requested in the checklist should always be included in the initial escalation to ensure we get as much information as we can to the correct teams. It is important to include all information being requested in the application templates, including the output details of the TraceRT, as this helps IT determine the cause. The goal is to minimize time lapse once PA tool issue has been identified.

4.2 Troubleshooting steps for PACSR application issues

Action Screenshot
1. Make sure the user is using https://mspacsr.microsoft.com like the screenshot to the right. 
  • You should not be using a URL like https://mspacsr.microsoft.com/1033/retail.aspx. Correct your bookmarks.
2. If MSPACSR tool is appearing like this for you, follow the below steps to resolve the issue.

For IE 9, 10, 11:

1. In IE, select F12 on your keyboard.

2. Select Emulator from the Tools menu.

3. For IE 11: under Mode, select Microsoft Edge (default) from the User Agent String drop-down menu.

4. Try the tool now.
 
For IE 7, 8:

1. Make sure that you have installed all latest updates for Internet Explorer

2. Make sure that ‘Microsoft.com’ is not in the compatibility settings (Refer to above screenshots)

3. Press the F12 key on the keyboard and select Browser Mode and Document Mode as IE8 Standards

4. Refer to the screenshot
3. Make sure that you are not logged into PACSR tool from another computer, else you would get this error message.

Even if you are using the same browser, close the tool and clear IE cache and try the tool again.
4. If you are getting this error, go to https://mspacsr.microsoft.com/claimstestpage.aspx and copy paste the results. If you are a Partners user (partners\alias), reach out to visops@microsoft.com with the error details.
5. Do not click on Internet Explorer back button while working with PACSR tool. Once you activate an IID, click on Done and go for another IID activation.  
6. The Copy, Paste, Clear buttons from the old tool are replaced with new buttons in the new tool.
7. If you didn’t do any activations through PACSR tool for more than 15 minutes, refresh the page and then start doing further activations.  

8. If the user is still facing the issue, ask the user to follow below steps:

1. Close all the IE browser windows.

2. Start the IE browser but do not start PACSR or any other applications.

3. Go to Tool >Internet Options on the General tab. Select Delete (under browsing history).

4. Unselect Preserve Favorites website data and then select the following options:

Temporary Internet files and website files, Cookies and website data and History. Select Delete (see screenshot to right)

5. Click on OK

6. Close and reopen the browser (in regular mode, not in INPrivate mode)

7. Try accessing the PACSR Application by using the following URL: https://mspacsr.microsoft.com

Instructions for TraceRT

Some issues may require you to generate a TraceRT. The following steps should be followed:
  1. Start > All Programs > Accessories
  2. Right click on ‘Command prompt’ and choose ‘Run as administrator’
  3. IT engineer enters their credentials (Command Prompt windows opens up)
  4. Enters: ping <space> website address without entering https/http
  5. Selecting Enter will give the IP address (For e.g.: 207.46.84.246 for mspacsr.microsoft.com)
  6. Enters: ping <space> IP address (Ping Response may be rejected due to ICMP protocol is disabled)
  7. Once the TraceRT details are displayed, a screenshot is taken and sent as an attachment to the requested client

VLK: TraceRT vlk.one.microsoft.com
PID Plus: TraceRT pidplus.one.microsoft.com

4.4 Scenarios

PACSR Inform the customer that we are currently experiencing “unscheduled/scheduled” maintenance and are unable to assist them at this time. Ask them to attempt activating their product again online at a later time, or to please call us back at a later time (provide a timeline).  Apologize to the customer for the inconvenience and tell them that we are happy to help them once our tools become available again.
VLKCSR Inform the customer that we are currently experiencing “unscheduled/scheduled” maintenance and may not be able to assist them at this time. 

If the scenario qualifies, ask them if they have attempted locating their information using the Volume Licensing Server Center site (or other relevant sites according to license type).  If they haven't, assist them with accessing the relevant site. 

Otherwise, ask that they please call us back at a later time (provide a timeline).  Apologize for the inconvenience and tell them that we are happy to help them once our tools become available again.
RDSLMCSR Inform the customer that we are currently experiencing “unscheduled/scheduled” maintenance and are may not be able to assist them at this time. 

If the scenario qualifies, ask them if they have attempted activating a license, installing CALs, or reactivating a license server using the Remote Desktop Services Licensing website https://activate.microsoft.com/. If not, assist them with the procedure. If they have already attempted, ask that they please call us back at a later time (provide a timeline). Apologize for the inconvenience and tell them that we are happy to help them once our tools become available again.
KIT/KIS If the scenario absolutely requires the output of KIT/KIS to determine next steps, inform the customer that we are currently experiencing “unscheduled/scheduled” maintenance and are unable to assist them at this time. Ask them to please call us back at a later time (provide a timeline). Apologize for the inconvenience and tell them that we are happy to help them once our tools become available again.
Use ORST to generate replacement Office PINs and tokens. 
PID Plus If you are attempting to generate a key, inform the customer that we are currently experiencing “unscheduled/scheduled” maintenance and we are unable to assist them at this time. Ask them to please call us back at a later time (provide a timeline). Apologize for the inconvenience and tell them that we are happy to help them once our tools become available again. 
Digital River Global Commerce Tool (DGRC) - (Activation only)  Inform the customer that we are currently experiencing “unscheduled/scheduled” maintenance and we are unable to assist them at this time. Ask them to attempt activating their product again online at a later time, or to please call us back at a later time (provide a timeline). Apologize for the inconvenience and tell them that we are happy to help them once our tools become available again.

6. Surface

6.1 Contact handling

There may be times when a Surface agent cannot log in to the tools provided or certain functionality within the tool is not functioning. Since customer contacts continue to come in regardless, this procedure is meant to provide guidance on what to do when this happens. Here are some of the scenarios that are covered:
  • You cannot login to Assisted Support Desktop (ASD) OR some functionality within a tool is restricted (planned or unplanned)
  • You cannot access [Asset 4346155]
  • You cannot access the Surface advocate portal (blog) or the Support Boundaries site
If the tools are down or malfunctioning and the Surface advocate cannot entitle the customer (for example, determine support eligibility), the expectation is that best effort support is provided within the boundaries that exist. We should verbally validate with the customer the state of their warranty and proceed with providing support. Transactions that require the customer's personal data to proceed will need to be halted, but any troubleshooting that can be performed with currently accessible content to resolve customer issues can still take place. Not being able to create an Incident within ASD should not block the Surface advocate from supporting the customer.

Return to top

6.2 Scenarios

  1. If a contact reaches you while you are having issues with tools, set expectations with the customer that you are facing some difficulties with tools.

Messaging:

"We are currently experiencing difficulties accessing our systems and this may impact how I am able to assist you fully. I appreciate your patience."

  1. Consult a site technical lead and report the tools issue. They will follow the Tools Performance Issues (TPI) procedure to report the issue to Microsoft.

  2. Verbally validate the state of the customer's software and hardware warranty. In a case where the customer is outside of 90 days (software warranty) and/or 365 days WW, except for EU/730 days and except for UK (hardware) of purchase, advise the customer that even if they are outside of the free period for receiving support, you will still provide the best effort troubleshooting the systems that are down and that you do not have a way to validate the entitlement.

Messaging: 

"Since we are currently experiencing difficulties accessing our systems, I cannot validate the state of your warranty. Because I cannot validate, I will do my best to assist you now with the resources that are still accessible."

If the scenario requires fully recovered tools access (for example, creating a repair/fulfillment, refunding a Windows Marketplace content purchase, etc.), use the messaging below as guidance to set the right expectations.

Repairs/Fulfillments messaging:

"Unfortunately I cannot create a repair/fulfillment order for you at the moment due to difficulties accessing our systems. If you prefer, you can visit http://surface.com/support and create your own."
