Rave is used to validate support eligibility for Commercial O365 & Cloud/Dynamics support. In addition, it provides the ability to create support cases for customers who require technical support with these products.

This article details how to use Rave to determine support eligibility for new support requests, and to review and update Rave cases for continuing service requests.

All Office 365 Premier subscribers with standard, Unified and O365 GCC accounts are to be entitled to cases created using Rave. 

How to Access Rave

  1. Using the Edge browser, go to: http://rave.office.net
  2. Use your Microsoft alias and password to login
  3. Once connected, the main page will appear

IW (Individual Worker) Elevation

An Individual Worker Elevation relates to a scenario where the caller is an Office 365 User, but who needs to be granted Administrator rights on the tenant in order to conduct maintenance or configuration tasks. Use Rave for an IW Elevation request using steps shown below.

  1. Ask the caller if there is another Admin who can provide them with Admin rights
  2. If no other Admin is available, inform the caller that for security purposes, we will need to forward their issue to our Data Protection team
  3. For IW Elevation requests create a Rave case and route it to the Data Protection team. Inform the caller that the procedure can take as long as 48 to 72 hours to gain access to the account.

Follow Rave Data Protection steps below to route IW Elevation issues to the DP team.

Self-Service Purchase Exceptions (Power BI, Power Apps, Power Automate, Project, Visio)

Self-Service Purchase is an individual purchase pattern being introduced in the Microsoft 365 admin center which will enable Information Workers (Non-Admins) to procure software licenses for their departments, as well as manage their own licensing and billing independently from IT admins in the Microsoft 365 admin center.

These Information workers will be exposed to a light-version of the Admin Center enabling them to manage their asset(s), assign and revoke license(s), billing account and profile.

Global and Commerce admins will have a view of all Self-Service individual purchases and billing accounts within the tenant, such as how many licenses were purchased, which Azure Active Directory enabled users those licenses have been assigned to.

Rave Exception Handling

  1. Create a Case for a Subscription not listed as Premier

Some Premier customers may have an Office 365 subscription that has not been associated with their Premier contract, as indicated by the Premier Tenant field showing “No.” In this case, a Premier toggle is available to enable advocates to route the case to Premier level support engineers. This toggle only applies to non-Premier tenants, and applies on a per case basis. Additionally, it does not change the subscription to a Premier account. Follow the steps below to apply the Premier toggle.

Qualify the caller for Premier Routing:

Show me how
  1. The caller must have called into the Premier phone queue, requesting Office 365 assisted support. If the caller did not call the Premier phone queue, then process the case according to broad commercial procedures
  2. The caller must be an Admin and accurately verified their contact information. If the caller is not an Admin, suggest that they contact an Admin. If another Admin is not available, then refer to the Caller is not an Admin procedure.

Applying Premier Routing for a Standard Support account:

Show me how
  1. Since the caller dialed the Premier phone line, select the Premier “Toggle” located at the upper right corner of the Create new case dialog box.

[Asset 4478012]

  1. After the toggle has been selected, the gold “Premier Support” badge will appear, along with Premier Data Protection and Critical Issue checkboxes

[Asset 4478014]

  1. In the case notes enter: “Confirmed Premier Tenant”
  2. Process the case according to existing Premier case procedure

Note:  The Premier Toggle is only available for new cases, it cannot be used for existing cases. If a customer created a Standard case online, but requires Premier routing, create a new case and include the original case number in the case notes, use the Premier Toggle to route the new case as a Premier incident. Finally, open the original case and send a note to the engineer indicating the issue, refer to the new case number and request to close the case.
