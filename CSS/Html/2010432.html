Event Timing in Microsoft Advertising

[Asset 4509089]

This article contains the current average timing for common Microsoft Advertising events. When sharing with advertisers, indicate that these times are guidelines based on current optimal operation. These times may vary depending on external factors (such as server maintenance or tool upgrades) and should only be escalated if there is reasonable concern that observed delays are well beyond these posted times.

FAQs


Q: My customer made changes in his account, but these are not showing on Bing and/or there is still no delivery.  What should I do?

A:  The Event Timing listed above is intended to show the optimal performance of the Microsoft Advertising system.  If a customer has made changes to ads/keywords, they may be subject to additional Fraud/Editorial checks that will prevent changes taking place within the usual expected timeframe. 
Before escalating, check in AdInquiry whether this customer is in .Fraud status, or has a ‘Pending’ flag applied to their account.  (Listed above in the ‘Ad Delivery’ section, under ‘Editorial Review’.)   For more information, see the Fraud Status VKB
