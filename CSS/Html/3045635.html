ASMS – Credit Reason Codes

Overview

The Business Group Reason Code document (http://Aka.ms/azurecreditpolicy) provides the reasons, rules, and limitations on when a particular credit or refund can be provided. 

It also outlines the empowerment levels to be followed by ASMS. A high level breakdown table of ASMS Credit Approval Empowerments can be found below.  

Policy

  • When reviewing the reason codes, be sure to pay close attention to the difference between codes 9, 10 and 30, which may appear to be the same, but they are in fact very different. A quick E-learning session has been created to explain the differences between these codes. 

  • In cases where the policy states "You can only use this policy one time per customer.", but you feel it is justified to apply the policy to a customer again, use the code CS30.

    • If the credit amount is above your empowerment, escalate to the team noted in the credit reason policy and inform them that the amount is being requested for CPE reasons. If you receive pushback for using out-of-policy codes, escalate to your manager for assistance.

  • To check if a Field Led or Enterprise Agreement (EA) customer has already been credited one time per the policies, do the following:

  1.  Go to the Azure Ops on Visual Studio.
  2.  Search for past credits in the system. using the EA Agreement number.
  3.  If a past credit already used the code you intended to apply, use CS30 and follow the empowerment guidelines to escalate to the correct team for approval.
