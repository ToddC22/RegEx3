1.1 Order status

Use the following table to view how an order status in MST will appear online to the customer:

Order state in MST Status on accounts.microsoft.com Customer handling
AuthorizationFailed Failed or canceled

The order didn’t complete due to payment authorization failure. A new order will have to be placed by the customer or advocate. Suggest using a different payment instrument (PI) for placing the new order.

Offer to assist the customer in placing a new order.
Canceled Canceled

The order or line item has been canceled and won't proceed further. Any item in the canceled state cannot be recovered or altered.

Review Events tab and System Activities to verify the reason for cancellation
PendingCancelation   This is a temporary state while a product or order is being canceled. It will move to Canceled state once cancellation is complete.
ChargeBack Disputed charge Explain to the customer that since they have disputed the charges with their financial institution we are unable to process a refund until the investigation is completed.
CheckingOut  

This is a temporary state and orders should flow past this state.

The order was attempted for placement by the customer, but still hasn't completed purchasing or has failed to complete.

If you see this state, the order is likely stuck in the purchase flow. Review the Risk and Events tab for potential blockers.
Editing

Physical goods: “Processing”

Pre-orders: “Pre-ordered”

 Products have been added to a cart, but the checkout was never completed, and an order never fully placed.
Failed Declined

The order has been declined and not processed. The customer wasn't charged for this transaction, but authorizations may appear on the customer's account.

Review Risk Results tab, and Risk notices at the top of the customer account. Perform replacement if all risk items have been resolved, or if CP Risk has approved a replacement.
InDunning  

The customer’s paid subscription has lapsed with no valid payment instrument provided for renewal.

Follow the guidance in KB 4092700 Informational: Modern platform subscriptions.
RefundFailed Failed refund

A refund was attempted for this order but was not successful. For some orders, the refund can be retried, but may still fail if the payment instrument is unable to receive the refund.

If blocked in performing a refund for a customer, and they are eligible according to Microsoft Store policies, consult with ACA for a PayOps inquiry.
Pending

Physical goods: “Processing”

Pre-orders: “Pre-ordered”

The order is currently being processed for fulfillment but hasn't completed yet.

Review the Events tab for each line item or System Activities to dive deeper into the order state.
PendingFulfillment Processing

The order has moved from a pending state and is currently being processed for shipment.  This is a temporary state.

For trouble shooting, or orders that aren’t processing, please see KB 2988925 Procedure: Microsoft Store: Physical Products/Software Products: Returns & Exchange.
PendingRefund  

A refund has been requested correctly but hasn’t been fully processed from Microsoft’s side.

Customers should expect the funds back into their account within 5 to 7 business days.

If refunds stay in this state for longer than 7 business days, consult ACA for a PayOps review.
PendingReturn Pending return

An RMA has been generated for the order, and we are waiting for the product to arrive in the warehouse and be checked in.

Inform customers they should ship their package using the RMA label provided, or if already shipped, use the tracking information to identify the status and location.
PendingReview  

This is an uncommon order state. Orders in this state are likely in a manual review before being processed (either charged or declined).

If the order stays in this in this state, see KB 2896711 Procedure: How to handle risk rejections and engagement with Risk Ops.
Pickedup Completed Order is a Buy Online, Pick Up In Store (BOPIS) order. The customer has retrieved the order from the physical store.
PreOrder Pre-ordered The order has been created as a pre-order. The product is not yet available for delivery. A pre-order date may be visible in the Events tab, and in the customer’s confirmation email. For more information, see KB 2889231 Procedure: Microsoft Store: Pre-order scenarios.
PreparingForPickup Pending The order is a BOPIS order. The customer’s order is being processed for pick-up at the physical store. As soon as the order changes to “ReadyForPickup,” the customer will be notified and can go to the store to pick up their order.
Purchased Completed

The order has completed and been fully purchased.

Physical products go into the Shipped stated instead of the Purchased state.

Digital products go into the Purchased state.
ReadyForPickup Completed The order is a Buy Online, Pick Up In Store (BOPIS) order. The customer’s order is available for pickup at the appropriate physical store. 
Refunded Refunded

The product or line item has been refunded and should be reflected in a negative dollar value (-$).

For example: -12.50 USD.

Note: Miscellaneous credits onto customer accounts will always appear as Refunded because we have credited an account with funds. These aren't seen as charges.
RiskRejected Failed The order has been rejected by Risk Ops. Please review the Risk tab for the order, and follow KB 2896711 Procedure: How to handle risk rejections and engagement with Risk Ops.
Shipped Shipped

Physical Products only.

There will be a ship date and tracking information for this state.
Tracking information is found by line item.
ShippedNotCharged Processing

Physical Products only.

The product has been processed for fulfillment but is still pending a charge. A charge will be attempted again at a later date.

3. Detailed information found in the orders section

Under each individual order, there are five sections with additional information.

Details section: Shows the date/time of the order, the order number, where it was purchased (Partner), the price, tax and total amount.

Billing Records section: Shows the date/time of the order, the line item ID, the billing record ID and any balance owned.

Events section: Shows the date/time of the order events, the Event ID, the Event type, status, payment and amount. If there is an outstanding balance, this will also show the Decline Count number. For more details about the subscription decline cycle and write off process refer to KB  2846004 Procedure: Microsoft 365 subscription billing was unsuccessful.

Expand for images

If the Event Type displays "TRANSFERBALANCE_IN OneTimeTransaction", then the order wasn't charged, and instead a balance transfer to another payment instrument occurred, which cannot be refunded.

[Asset 4488675]

If the customer claims it was charged, then the agent must request the relevant evidence to sustain a refund request.

[Asset 4470020]

Risk Status section: Shows the date/time of the order, the Tracking ID, the Transaction type, the Risk Decision and Reason Code.

Transactions section: Shows the date/time of the order, the Transaction Guide, payment, status, reason code and amount. This is where you can see the status of a refund.

Cancel Order items for modern purchases: You will be able to cancel orders that are eligible for cancelation, this does not check the order state and is not dependent on it.

11. Complete purchase (Editing or checking out status)

Use this process for orders that show a status of Editing or Checking Out. 
If you see this status, the order is incomplete, and the customer has added products to a cart, but never went through checkout and completed the order. Orders in these states can be completed via MST POBO by selecting the Complete Order button.

See POBO section of this side nav TUG for more information.

Note: This means that Build-A-Bundles, and Buy M, Get N orders could be started by the customer and completed by the agent. Only perform this process if the customer specifically requests that you close the order for them. The process still requires that the customer create the order on the storefront. Ideally, you should guide the customer through purchasing rather than completing the sale via MST.
