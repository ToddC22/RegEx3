Support Scenarios

Admin Console

The Admin Console tool gives an insight of subscriptions data or properties from the OMS end, as well as pinpoint synchronization issues that occur between OMS and MINT. UST (PayOps) use this tool while investigating to confirm synchronization between MINT and OMS and confirm OMS data matches what RDFE displays. 

TAs can use the Admin Console to find the following information:

  • DPOR update status on a subscription.
  • Subscription transfer history for both Web Direct/EA and CSP.
  • CSP subscription cancellation history.

Note: This helps identify who canceled the subscription which CMAT does not provide.

  • OMS CAID account details.
  • The reason a subscription is in “sync” state pending.

Access (TA Only)

  1. Navigate to https://tools.cp.microsoft.com/ManagementPortal/
  2. Click New Request
    1. Choose Request Type: Request Application Access
    2. Application Name: OMSAdminConsole
    3. Business Group Name: SubscriptionConsole
    4. Role Name: SubscriptionConsoleReadOnly
    5. Users to Add: Name of the analyst will be populated automatically.
    6. Approver: Name of the immediate manager will be populated automatically.
    7. Business Justification: I work as a Technical Advisor with ASMS and need access to resolve customer issues.
  3. Confirm

Once submitted, your reporting Manager needs to approve the request post which the engineering team will provision access to the tool.

AME Accounts (FTE Only)

FTEs 
FTEs will need an AME account for elevated privileges in Jarvis and JIT.  To obtain this card do the following:

  1. Request an AME account here.
    • Once your manager approves, you will get a second email about ordering a Smartcard. You can only order the card via a SAW machine. If you have a SAW machine, you can request the card to be sent to your office. 
    • Should you not be able to access the office due to uncontrollable circumstances, you should be able to have the card and/or card reader shipped to your current residence. If the system allows you to do so, please proceed. If you are blocked please reach out to your manager.
  2. If your computer does not have a built in card reader, you will need to order one following the steps below:
    1. Navigate to MyOrder.
    2. Click on the Shop Staples Office Items tile.
    3. Search for and purchase a HID OMNIKEY 3021 Computer USB Smart Card Reader for yourself.​

  3. After receiving the smartcard, you may need to unblock your pin. To do so follow the steps outlined here.

    You should also note the following about this step. 

  • On step 4 of the Unblock PIN request, a form will appear. Enter  "Need assistance unblocking Pin on AME card" (or similar) and click submit. You will then be reached in Teams by Engineering. 
  • Engineering will provide instructions by using the MS Pin tool (in Offline mode) to get a challenge code and the SmartCardguid so that you can set a PIN on your AME smartcard. 
  • After the PIN is set, your machine will automatically receive the AME Certificate.  
  • You can check which AME certificates have been issued in Windows, by going to Control Panel > Manage User Certificates. Expand the section Certificates Current User > Personal > Certificates to see certificates issued by AME SC CA 01. 

A guided video has been put together to help you navigate using your AME credentials for Jarvis actions.

Jarvis

Access

You must have passed your cloud screening (http://aka.ms/screening) prior to requesting access to Jarvis or your request will be rejected.

You only need one of the three different levels of Jarvis access. Please don't request more than one level of access at a time and if you are requesting a new level of access, be sure to cancel the existing access, first.  

  • ASMS Read/Write: Advanced teams can request via //MyAccess - ASMS ACIS ReadWrite - 15221
  • Q&A, Wipro and New hires can request via //MyAccess - ASMS ACIS QA Team - 17197
  • ASMS Read Only Anyone needing read only access (Ie. managers, Case Insights, Etc..) can request via //MyAccess - ASMS ACIS Read Only Team - 18321

Tool Down

If there is an issue with Jarvis that is affecting more than just your personal account, an incident should be created using IcM (https://portal.microsofticm.com/imp/ ). This will engage the on-call person, so the issue can get triaged and taken care of.

For high-severity issues (for example, Jarvis is completely down for all users) severity should be set to 0,1 or 2. If this happens in the middle of the night, people will be woken up to address the issue. You should also contact the acishot alias.

TSG: https://internal.support.services.microsoft.com/en-us/help/2782736

Port 25 - How to Request Access to Enable

Membership to the MyAccess group azfraudportunblock is required to remove the port 25 restrictions from a customer subscription.

Note: All ASMS SEs will be auto approved for the MyAccess group: azfraudportunblock.
If you do not have access you may submit an access request:

  1. Navigate to MyAccess.
  2. Click Request Access.
  3. Search for azfraudportunblock.
  4. Select azfraudportunblock.
  5. Click Next, then Submit.
  6. Select the Permission field button and provide the following justification:
    1. I am a member of the CSS Azure Subscription Management Support team, which needs the ability to set an override subscription feature flag AllowTcpPort25Out which implements an anti-fraud measure to avoid platform abuse.
  7. Click Next.
  8. Review the Request Summary.
  9. Click Submit Request.

You will receive a notification email after the request has been submitted and another when it is approved.These instructions are also available in TSG 4511085 (Port 25 Enablement).

Timeline (TA/PTA Only)

Timeline is a read-only tool used by PayOps to check Subscription, Billing and Payment details. It provides details from both the Legacy (Classic) and Modern platform. In CMAT, we cannot find the details of Modern transactions. To troubleshoot the service requests (SRs) related to these areas, PayOps team may need to use Timeline tool. Access is now being expanded to ASMS TA/PTA’s and we mostly use this for investigating PI related issues.

Access

  1. Navigate to the Management Portal. 
  2. Click New Request.
    1. Select Request Application Access and click the Next arrow.
    2. Fill the items in the form as follows:
      • Application Name: Timeline
      • Business Group Name: ASMS_Timeline
      • Role Name: ASMS_Timeline_Readonly
      • Business Justification: I work as a Technical Advisor/Partner Technical Advisor with ASMS and need access to resolve customer issues.
  3. Click the Confirm checkmark.

Troubleshooting

Troubleshooting MyAccess/Jarvis access issues

If you are having MyAccess/Jarvis access issues:

  1. Verify you are a member of all of required groups for your role as listed in the table below. If not, request membership to those groups.
  2. If you have verified your group memberships are correct but the access issue continues, take a screenshot of the exact error message and/or unexpected behavior you are experiencing.
  3. Include this screenshot along with detailed information on the steps in which this issue occurs when escalating the issue as detailed above.
