Procedure: M365 TS: LogMeIn Usage Guidance

[Asset 4462485]

Who should use this article?

All O365 support resources, both Microsoft FTE and Contact Center Suppliers relying on remote assistance tools.

What is this article about?

This article establishes the guidelines for O365 support working to resolve a customer's issue by using the LogMeIn (LMI) remote access tool. The procedure should be followed when you need to use remote assistance to investigate or resolve a customer support request.

Assumptions

  • Customer’s device is not Windows 10. LMI can be used for any other OS environment or limitation not covered by Quick Assist.
  • If the device is a Windows 10 machine, Quick Assist is the default official CSS remote assistance tool. More information on Quick Assist can be found in article 4025638 TUG Quick Assist.
The person in the O365 support:
  • Is trained to use LogMeIn correctly.
  • Understands the LogMeIn guidance and procedure as it relates to their role.
  • Must understand and always comply with 2622104: Procedure: M365Clear Desk Clean Screen Guidance. This includes saving and storing LogMeIn recorded sessions.
  • LogMeIn cannot be used to view, change or record customer passwords.
  • LogMeIn is the permitted remote assistance software available for support personnel. 

If you are not able to log into LogMeIn (or failed), follow below instructions to gain login access:

  1. If you are a non-CSS support partner, file a request through http://o365supporthelp.azurewebsites.net/Home/SubmitIssue.
  2. If you are in CSS and using OneTenant for LMI, work with your site-sponsor delegate to file a request.
  3. If you are in CSS and not using OneTenant for LMI, work with your site-sponsor delegate or tools manager to engage LMI team for assistance.

Procedure


LogMeIn use requirements
  1. Before you start a LogMeIn session, ask the customer to do the following:

    "Please take the precaution of removing any personal, private, or confidential information from view on your screen before you start the Remote Assistance session. Minimize or close documents and applications unrelated to this service request."
  2. Use the SSO link to start the version of the tool that was installed. Important: Use the save the SSO link to save the SSO to your favorites. Note that you can't save it yourself after you've started the tool as the SSO will redirect to the tool URL. 

    For support engineers with @microsoft.com domain account:

    Web SSO (Internet Explorer only) - https://CSSRemote.microsoft.com/CSSEnterprise/LMISSO
    link to save the Web SSO link - Web SSO shortcut link

    Desktop SSO (Internet Explorer or Edge) - https://cssremote.microsoft.com/cssenterprise/lmisso/home/desktop
    link to save the Desktop SSO link - Desktop SSO shortcut link
     

    For support engineers with @office365support.com domain account:
    (The SSO for Azure is Web ONLY.)

    https://cssremote.microsoft.com/LMIAzureSSO
    https://aka.ms/AzureLMI

  3. To begin using LogMeIn, direct the customer to go to http://support.microsoft.com/help. On this page, the customer must review the "Receive Remote Assistance Support from Microsoft" agreement and software license terms.
  4. If the customer does not accept any of the conditions of use, or asks to use a different tool, work with the customer based on CSS Remote Assistance implementation rules. However, feel free to use the following script: 
    • "LogMeIn is our support group’s tool for remotely accessing our customer’s computers. LogMeIn provides you with extra protection as we cannot use the tool without your direct acceptance when the tool is installed."
  5. To use LogMeIn, the customer needs to do the following tasks:
    1. Click the I Accept button.
    2. Enter the 6 digit LogMeIn session ID on the support.microsoft.com/help page.
    3. Click Connect to technician to proceed.
  6. The support.microsoft.com/help page will prompt the customer to download and run the LogMeIn application. Direct the customer to disable pop-up blocker if the option to run the application does not appear.
  7. When the Chat window opens, remind the customer of the following:
    • "Quick reminder, please remove any personal, private, or confidential information from view on your screen."
    • “At any time, you may choose to end this session by clicking X in the session chat window”

      Note: The preferred support model is to communicate with the customer over the phone while conducting a LogMeIn session. The chat window is used when it is necessary to provide clarifying instructions.
  8. Minimize and/or lock screen if any of the following occur:
    • The customer must step away from the computer during a session. (Although closing the application is preferred, it’s acceptable to minimize the application in this scenario).
    • You need to step away from the computer during a session.
  9. Immediately stop the session when LogMeIn is no longer required to troubleshoot the customer’s issue.
Note requirement for engineers outside of FTC

After a LogMeIn session concludes, you must fully document your usage of LogMeIn by adding non-customer facing internal case notes in the SR with the following information:
  • The customer’s explicit acceptance of using LogMeIn.
  • Purpose for using LogMeIn.
  • Session activity – how you helped the customer during the session.

Guidance

Tools usage

Please note use of this tool is based on a couple of simple, but important policies. Continued failure to follow these policies can result in your account being disabled.

  1. You need to provide a valid Service Request number for the Incident Management system you utilize.
  2. You need to access the tool via the SSO (Single Sign On) processes in place.

If you are not an Administrator on your desktop or unable to install software, please contact your Administrator for assistance in setting up the Remote Assistance tool. If your Administrator needs information or details, please have them contact the Vendor Infrastructure Team (VIS).

If you are an Administrator and able to install software, please refer to the information below to set up the Remote Assistance Tool. Below are links to install two versions of the LogMeIn Rescue Technician Console: the web-based version and the desktop console. You can use either version, but you must start the tool using the appropriate SSO link that is also included below.

Install either or both of the versions:

  • Web TC: Technician Console Installer
    • Please note that Edge is not a supported browser. If this option is chosen, you need to use Internet Explorer for the remote assistance tool.
  • Desktop TC: Technician Console Desktop App
    • Please note installing this will give the option of starting the tool immediately. Please do not do this. Please also do NOT install the desktop icon. If installed, please delete the desktop icon. If presented with the option to provide a login email and password, please click cancel/exit and use the appropriate SSO link.

Please check this support site for known issues, helpful information and FAQs. It is also available via the web and desktop Technician Console.

 

Special Considerations
  • During a call, if the customer needs help using PowerShell, you may use LogMeIn to talk the customer through the appropriate steps.
  • If the issue is especially about hardware or networking or something to do with the environment at the customer side, you can use LogMeIn Lens feature to utilize the customer mobile device to show the agent something that would help with the support call.
  • Office 365 Contact Center support engineers may not have "Launch Remote Control Session" feature enabled in LMI.
  • In addition to the standard customer approvals required by LogMeIn, you still need to receive explicit customer approval either in a message or verbally over the phone before beginning a LogMeIn session on the customer’s computer.
  • LogMeIn will not work on the Microsoft Surface RT.

  • Commercial support engineers must continue to start with Microsoft-approved tools, see below approved tools listed from the CSS Remote Assistance implementation rules. However, support can use Microsoft Teams or Skype for Business for voice meetings or conference calls if customer insists.

  • While the Remote Assistance implementation rules is being reviewed, for Commercial Support (FTE/Contact Center Suppliers) , if a customer insists on the use of a particular technology such as WebEx or Microsoft Teams, then it is acceptable to use those tools without seeking an exception from TrIP (formerly,  IPG). However, support cannot compromise customer confidentiality when using these tools, for example:
    • Do NOT add unauthorized parties to the conversations.
    • File sharing via Teams is expressly prohibited. Files are classified as Highly Confidential and must be transferred via DTM due to transport, storage and retention requirements.

LogMeIn conditions
LogMeIn may only be used on a customer’s computer per all of the following conditions:
  1. After performing all due diligence, you determine that LogMeIn is the most appropriate option for better helping the customer with their issue. For example:
    1. The customer is unable to follow basic instructions during a support call.
    2. The support call requires use of advanced features beyond the expertise of most users such as PowerShell.
  2. Customer explicitly approved receiving support using a LogMeIn session.
  3. Customer remains present at the computer during the entire LogMeIn session.

Screenshot Conditions
Take appropriate steps to minimize sharing of non-pertinent information from the customer’s machine. When using LogMeIn to take screen shots from a customer:
  • Ask permission before taking a screen shot.
  • Focus only on data necessary for troubleshooting. Ensure the customer either closes or minimizes any background information that is unnecessary for resolving the issue including:
    • Social media applications
    • Email applications
    • Sensitive documents
    • Websites
    • Any other application unrelated to the issue you are helping the customer resolve.
Prohibited screenshots – high value data
If high value or medium value data (HBI, MBI) will be captured in the screenshot, do not use LogMeIn to take a screenshot. Direct the customer to capture the screenshot themselves and then use Data Transfer Manager (DTM) to upload the file to Microsoft. See 2895143: Tool User Guide: M365: DTM Usage in Rave. For examples of HBI and MBI, refer to the Appendix below.

Appendix

High Business Impact
Unauthorized disclosure may cause severe or catastrophic loss to the organization. Impact may reflect loss or theft of assets, organization productivity, damage to reputation, or increased legal and regulatory liability.
 
HBI category Examples
Authentication credentials Passwords, private cryptographic keys, and hardware tokens.
Highly sensitive business material Financial data and intellectual property.
Personally identifiable information (PII) Any information that would allow an attacker to identify customers or employees or know any of their personal characteristics.
Financial transaction authorization data Credit card numbers and expiration dates.
Medical profiles Medical records or biometric identifiers.
Financial profiles Credit reports, personal tax records.
 
Moderate Business Impact
Unauthorized disclosure may cause moderate loss to the organization disrupting normal organizational functions.
 
MBI category Examples
Internal business information Employee directory, purchase order data, network domain designs, information on intranet sites, internal business use data.
