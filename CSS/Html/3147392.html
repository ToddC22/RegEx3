1. What is a blocked product key and how to block a product key?

Blocked Product Key: Some product keys that show high activation history on different hardware IDs are blocked from activation. In most cases the user knows the product key has been abused. However, end users and partners who have been victims of software counterfeit, or the owners of the original genuine key might contact Microsoft for assistance.

To Block a 5x5 Product Key. PIN/Token: If a customer has been provided with a refund or is requesting a product key replacement, you may need to request a 5x5 product key, PIN/Token to be blocked.

2. Products affected

2.1 If a blocked product key

Retail and OEM versions of:

  • All Microsoft products requiring activation
  • Exceptions:
    • All Microsoft products that have reached End of Support. See KB 4018070 Procedure: MPA: Activation for legacy Windows version.
    • Keys that are blocked by design (OEM:SLP; Default Key, as blocked reason)
    • PIN/Token

Volume License versions of:

  • All Microsoft products requiring activation
  • Includes all Microsoft products that have reached End of Support.
    • Keys that are blocked by design (GVLK)
    • Bypass Keys

Return to top

2.2 To block a product key or redemption key (PIN/Token)

Retail, OEM, Volume License MAK, and Volume License KMS versions of:

  • All Microsoft products requiring activation, using a 5x5 product key
  • All Microsoft products requiring redemption, using 5x5 PINs/Tokens
  • Excludes: RDSLM CAL, SBS CAL, XBOX

3. Contact handling – If a blocked product key

3.1 Contact Handling – overview

First, follow the guidelines for 45090 or Block Product Keys handling provided here: KB 3147768 Procedure: MPA: Activation blocked due to blocked product key.

If after, you have determined this situation is eligible for escalation follow these instructions:

Customer Scenarios:

  • Scenario 1: The customer insists the software is genuine; however, the PACSR Tool shows the Error Code 45090, Digital River Commerce Tool shows Error Code Message Black Listed Serial Number (3010) or Deactivated (3005), or the KIT tool shows the key as blocked.
  • Scenario 2: System Builder or OEM Partner downgrade.

3.1.1 Scenario 1: The customer insists the software is genuine; however, the PACSR Tool shows the Error Code 45090, Digital River Commerce Tool shows error code message Black Listed Serial Number (3010) or Deactivated (3005), or the KIT tool shows the key as blocked

Inform the customer that you can escalate their request, but we require them to provide us with personal information. If the customer accepts:

  1. Provide them with your call center's email address or fax number. (If you are not sure what the e-mail address or fax number for your call center is, contact your supervisor.) Request they provide the following to Microsoft:
  • Customer's first and last name, telephone number, and email address (include company name, if applies)
  • Image of proof of purchase (for example, receipt with date and customer name)
  • An image of the COA label (if available for W7 & 8)
  • An image of the product key label or email
  • The reason the customer is reinstalling Windows and how many times it was done so far.
  • A visual image (picture) of the product key sticker, and the Certificate of Authenticity (COA). If the product key is on the COA, the image of the COA is enough.
  1. Inform the customer that once Microsoft receives their email or fax, it will be reviewed, and a response will be provided within 72 hours. If all the information is acceptable, a new product key will be provided.

Note: Regional processes may vary between having all escalations centralized through the Customer and Partner Advocacy Team, or going through a Product Activation supervisor. The following two steps may differ from region to region.

  1. Once the customer's e-mail is received, a supervisor verifies the information and checks the details in KB 3150228 Tools User Guide: Key Information Tool (KIT). If the key is blocked, selecting the Details link will provide the supervisor with the Tenant ID of the organization that applied the key block. 3.1.2 Scenario 2:  System Builder or OEM Partner downgrade

This service is intended for Microsoft Registered Partners. Confirm that the System Builder or OEM has provided different customers with the same product key to execute downgrades to Windows 7 (Professional or Ultimate), or Windows Vista (Business or Ultimate), or to Windows XP.


Inform the System Builder that you can escalate their request, but we require them to provide us with some information.

  1. Supervisor should copy the following table into an email and send the email to PKSBET@microsoft.com after entering all required information:
     
    Important!  These email addresses should never be provided to the customer.
     

    Data Information

    Customer Details
    Date <date>
    Case (SR) Number  <number>
    Have you reviewed info posted at http://support.microsoft.com/kb/2832566/ ?  
    Has OEM been contacted? (OEM should be able to assist with downgrade)  
    Is the customer a system builder? <y/n>
    Example Product ID of the current version (displayed in: Control Panel > System): <example> 00180-20022-XXXXX-AAOEM
    Machine Serial # (For multiple machines provide a few examples):  
    Is this a Volume Licensing Customer?  
    Company Name <company name>
    Customer Name <contact name>
    Contact Email <contact email>
    Supervisor Requestor (full name) <advocate contact name>
    Supervisor Requestor (full email address) <advocate contact email>
    Current product version (must be OEM Pro)  
    Product version needed <example> Windows 7 Professional
    Attached proof of purchase <y/n>
    License Quantity: # of keys to be downgraded  
    Details on why downgrade is needed, and how downgrade will be deployed  
  2. Inform the System Builder that once the escalation is received, it will be reviewed and a response will be provided within 48 hours. For lists larger than 50 expect more time. New monitored keys will be provided to them for downgrades and will have a higher activation limit so that they does not encounter this Blocked PID issue again.
  3. Once the escalation has been closed, emails should be deleted.

4.1 Contact handling – To block a product key

Ensure there is proper approval from your Supervisor or Operational Lead for the product key to be blocked. Blocking a key can result in a high CPE issue so we need to ensure we are blocking a key for the right reasons.

  1. A supervisor for each site is to collect all product key blocking requests and consolidate all requests into one email. This consolidated request should be sent at the end of the day, ensuring all details are copied and filled out using the template below for each individual 5x5 Product Key.
