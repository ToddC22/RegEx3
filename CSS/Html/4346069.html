Procedure: Documenting Solutions

[Asset 4462485]

Who should use this article?

Support Engineer | Escalation Engineer | Technical Advisor

What is this article about?

This procedure covers what needs to be documented following the identification of a solution to the customer's/partner's problem.

1. Steps

How to Communicate the Solution

  • Send the Customer or Partner an email through the existing ticketing system.
  • Request confirmation of the solution provided and permission to close the SR.
  • If you cannot receive confirmation of the solution and approval to close the SR:
    • Try to reach a commitment on when to check back in with the Customer or Partner to confirm the solution worked and gain agreement to close the SR.
    • Send a follow up email summarizing the call, the request for solution confirmation and permission to close the SR.

Disposition and Case Status

  • If Customer or Partner confirms the solution did not resolve the issue or does not work, it's important to always change the Disposition or Case Status back to Waiting on Support (CAP), Troubleshooting (RAVE), or Troubleshooting (Service Desk Case) before offering a new solution.
  • Set SR Disposition (CAP), Case Status (RAVE), or Status (Service Desk Case) appropriately:
    • Delayed validations use SD - Pending Confirmation (CAP), Pending Customer Verification (RAVE), or Waiting for Confirmation (Service Desk Case).
    • Delayed closures for monitoring requests, that have initial confirmation of solution use SD - Solution Confirmed (CAP).

Completing a Customer Facing Comment in CAP

  • Always create a Customer Facing Comment with the solution that you have provided the customer. This enables the customer to see the solution provided in the web portal.
  • If the solution did not work, continue working the case and create another Customer Facing Comment when next solution is offered. Include previous solution steps if is determined that they were necessary to solve the issue.
  • Complete the Customer Facing Comment by using this template:
    • Symptom: Use this to describe the problem and how it presents itself.
    • Cause: Use to describe what the problem was. Be specific as possible, but keep it as short as possible.
    • Resolution: What you did to fix the problem. Be specific as possible as this may be used by others to resolve their cases.

Completing a customer email communication on proposed solution in Rave

  • Send an email to customer with the solution that you have identified.
  • If the solution did not work, continue working the case and communicate with the customer.
  • Complete the solution proposal email by using this template:
    • Symptom: Use this to describe the problem and how it presents itself.
    • Cause: Use to describe what the problem was. Be specific as possible, but keep it as short as possible.
    • Resolution: What you did to fix the problem. Be specific as possible as this may be used by others to resolve their cases.

Documenting Solution in Service Desk Case

  • Send an email to customer with the solution that you have identified.
  • If the solution did not work, continue working the case and communicate with the customer.
  • Complete the Notes by using this guideline:
    • Cause: Use to describe what the problem was. Be specific as possible, but keep it as short as possible.
    • Symptom: Use this to describe the problem and how it presents itself.
    • Resolution: What you did to fix the problem. Be specific as possible as this may be used by others to resolve their cases.
