Procedure: Collaboration

[Asset 4462485]

Who should use this article?

Support Engineer | Escalation Engineer | Technical Advisor | Business Process Manager | Duty Manager

What is this article about?

This procedure is to be used when assistance is required to progress, a case, either across technology or within the same technology.

Collaboration is when one engineer requests another engineer with a different skillset to help solve a customer issue. When implemented through current case management tools, collaboration requires dispatching an object to a queue.

When to Swarm or Collaborate?

Use 4464163 Procedure: Swarming to step through the collaboration procedure  in real time with engineers that support a different but relevant technology, where available. Work with the engineer to assist case progression while retaining ownership and managing the customer experience end-to-end.

If swarming does not enable real time collaboration with engineers in the relevant technology, or that technology is not enabled for cross-technology swarming, then continue with collaboration procedure below.

See article 4488309 Informational: Case mangement tool product support list to understand which product is being supported by which ticketing system.

Rave Assistance Request Notification

  • Assistance Requests are a key way premier engineers are seeking help from experts in the different technologies. The Assistance request tab allows engineers to either accept or update the request without assigning it to themselves.
  • Service Desk Engineer needs to monitor within Rave for Assistance Requests (Collaboration Requests) from Rave Engineer (refer to Step 4 under Collaboration from Rave to Service Desk). There is NO alert functionality from Rave when a new collaboration comes in, so Service Desk Engineer will need to be logged into Rave to check for new Assistance Requests. Rave does NOT automatically refresh when new data is available, so users will need to click the Refresh button in the upper right corner of Rave. Check with your Manager or Team Lead if you should be monitoring Rave for Assistance Requests.
  • TIP: The Case Ninja for Outlook value-add app can be configured to pop up a desktop notification when a new Rave Assistance Request arrives (Show Incoming Tasks) for a Support Area that you monitor.  Case Ninja for Outlook can also be configured to alert you when an Assistance Request is assigned to you. For more information on Case Ninja for Outlook click here.

Collaboration within Service Desk
  1. Complete the 4346609 Template: Summary and put in Notes.
  2. Transfer a new Collaboration task to a queue with the transfer reason set to Administrative Reroute:
    • Follow routing rules for selecting queue. If you determine the incorrect queue was selected, then check that Area Path, Support Language, and Country of Origin are correct.
    • Expected Response time for a collaboration request matches the initial response goal by service level and severity.
  3. Follow up on all customer/partner commitments while you wait for a response to the collaboration request:
    • If your task is not picked up by an engineer within the expected response time, email queue Duty Manger to request an update.
  4. As soon as you have made contact with the receiving engineer:
    • Discuss the issue with the receiving engineer, bringing the customer/partner into the conversation as necessary.
    • It remains the responsibility of the case owner to ensure that all customer/partner commitment goals are met.

Note: If a collaboration is needed from a user or team that doesn't have access to Service Desk, use Rave. 

Collaboration within Rave

If a collaboration is needed in Rave, Assistance Request feature in Rave can be used.

Request assistance by using Rave Assistance:

  1. Open the Rave case and select Request assistance from the Actions pane.
    [Asset 4346570]
  2. Select the Team you need assistance from, add Region and Time Zone such as NOAM | UTC-5 |15:30*| Need help with mailflow in the Description section and then click Request.
    *Callback time is optional but please include if agreed to with the customer.
    [Asset 4346571]
  3. The Assistance pane will show the information of the sought-out collaborative engineer.

Note: Please make sure you create an internal note in the case so that the collaborative engineer could have a clear understanding on what kind of help you are looking for.

If it is an incorrect collaboration request, Rave engineer should cancel the assistance request from the Rave case, select Cancel on the Assistance request to cancel out this request. 
[Asset 4498790]

Provide assistance by using Rave Assistance:

  1. Click the Unassigned Cases tab and then click the Assistance Requests tab.
  2. Click Accept to start providing assistance on the case.
    [Asset 4346582]
  3. Review full details of the case history and notes by the case owner, use internal notes to assist them and contact the customers directly if needed.
  4. The case is added to the My cases view, and it is marked as an ASSISTANCE case.
  5. When the work is complete, select Close assistance from the Actions pane.
    [Asset 4346583]

Note: An Assistance Request needs to be closed before the case can be closed. If a user starts collaborating on a case and then moves to take ownership, he/she should first close the assistance request via case actions. For more information, please refer to Rave Assistance Q&A.

Rave Assistance Q&A

Q: When should I use the Assistance feature?

A: When you need to collaborate with someone and don’t know the collaborative engineer’s name.

Q: Can I use the Assistance feature to request assistance from a specific engineer rather than a team?

A: Currently no. When you request assistance, you specify a team. Any user who has the appropriate skills will see the assistance case and will be able to accept it.

Q: How many people can I request assistance from at one time on a ticket?

A: There is no limit. You can request assistance from as many teams as needed.

Q: When I create an Assistance request, is there another ticket number?

A: From the Rave side, there’s just one case number. This case will appear in the My cases view for the case owner and any user who has accepted the case to provide assistance.

From the SD side, there will be a SD case and a corresponding Rave case (the numbers will be different). 

Q: What skills will a user need to see assistance cases?

A: There is no additional skill required. All users can see assistance cases. However, you must have the appropriate skills to see the case. For example, if the case is a Premier Escalation case and the case owner needs assistance from the EnterpriseOneDrive team, you will need to have both the Premier Escalation and EnterpriseOneDrive skills to see the case. The same is true for the language of the case.

Q: What case information will a user see when they accept an Assistance case?

A: The full Rave case history can be found in Rave.  However, if the request is initiated from SD, the case communication will be copied from SD case to Rave case and also whatever was written in the SD collaboration task will show up in the Assistance Request Description filed.    
[Asset 4498795]

Q: How can I tell which cases I own vs. those for which I’m providing assistance?

A: In your My cases view, you’ll see the blue ASSISTANCE stamp on cases for which you are providing assistance.

Q: How can I tell who the main case owner is?

A: The case owner (agent) is displayed in the Case Summary pane.

Q: How can I tell if my assistance request has been accepted?

A: The Assistance pane will show the name of the user who has accepted the assistance request.

Q: What happens if I close the case before the Assistance case has been closed?

A: Rave won’t allow the case to be closed until the Assistance case has been closed.

SD cases CAN be closed with an open Rave assistance request.

Q: Can I transfer or reassign an Assistance Request that I accepted?

A: Yes, the Assistance Request owner can transfer or re-assign the current Assistance Request to another Support area or Agent by clicking on Update button in the Assistance pane. In order to reassign an Assistance Request to another user, you need to have one of the following roles (Premier Lead, Group Admin, Site Admin) and be within the same Rave org as the Assistance request. Requests for Service Desk users are sent to support areas within the CSS-C&AI Rave organization, so users within Modern Workplace will not be able to assign requests to Service Desk users.

Q: Can I accept or make update to the Unassigned Assistance Requests?

A: Yes, an Accept or Update button in the Assistance pane allows users to Accept or Update the assistance request by moving the request to another support area or assigning it to a user. In order to reassign an Assistance Request to another user, you need to have one of the following roles (Premier Lead, Group Admin, Site Admin) and be within the same Rave org as the Assistance request. Requests for Service Desk users are sent to support areas within the CSS-C&AI Rave organization, so users within Modern Workplace will not be able to assign requests to Service Desk users.

Q: Can a Collaboration Task from SD be rerouted to update Assistance Request support area in Rave?

A: No, updating the Collaboration Task from Windows to Office as an example will not work in Assistance Request in Rave. A new Collaboration Task for Office should be created in order to have a proper Assistance Request generated in Rave.

Q: What notifications will I see?

A: You can see when an Assistance case has been accepted or closed by looking in the Assistance pane. You can see when case notes have been updated by checking the case communication history.

Q: Is the Assistance feature available for all users?

A: Assistance Request is available to all support, however O365 Broad Commercial support engineers can accept RA, but not submit a RA.

Collaboration with Azure CXP ACE

Some US Government cases will route to the Azure CXP ACE team initially. At times the Azure CXP ACE may need technical expertise from a CSS engineer to solve their customers problems. When this happens the ACE engineer will open a new case (just like a normal customer would), that CSS will use to provide assistance. These customers have special confidentiality agreements that require CSS to proceed as explained below.

Expectations for these cases:

  • Cases will be created as a generic customer and will have the Azure CXP ACE engineer as the primary customer contact (contact details).
  • Cases will be identified as being an assistance request from Azure CXP ACE and contain a reference to this knowledge article.
  • Cases will include the question or information the ACE Engineer needs.
  • Cases will include limited information about the issue and the real customer behind the questions – ACE will determine what information can be shared.
  • Case severity will be set based on the customer scenario, so respect the severity and response SLAs accordingly – only ACE will have full details of the customer scenario.
  • Cases will be used for a single request for assistance – they will not be kept open to ask questions for different customer engagement that ACE is handling.

Customer data expectations:

  • No information about the real customer and its cloud subscription/tenant will be available to CSS in Service Desk. This means that tools that rely on customer accounts like ASC will not be usable.
  • Some data that comes from the technology itself can be shared. Example: Error messages, the status of a field in a log file, etc.
  • You can engage other CSS engineers to help with the collaboration case. Any data that is shared in the case to CSS can be shared with any engineer in CSS necessary to address the question from ACE.

Expectations for the CSS Engineer:

  • Treat these cases as you would cases from an external customer. This will impact case prioritization, labor logging, and what channels you respond with.
  • Answer questions to the best of your ability. Keep your mindset on walking the Azure CXP ACE through your troubleshooting steps. Provide guidance and suggestions for next steps and things the ACE should look for.
  • Do not ask questions about the customer themselves.
  • Do ask about the technical errors being encountered, and specific results that may be seen in a log file.
    • Example: You advise the ACE Engineer to access ASC for the customer account, run a specific test, and let you know the results.
  • Do share data queries that need to be performed with the ACE engineer.
    • Example: You need to know a specific result from a Kusto query, so you share it with the ACE engineer that works with the customer to generate the additional data needed.
  • If you do not hear from the ACE engineer or feel the severity has been set incorrectly, do not contact the account’s IM. Instead contact the ACE OneGov Leads team: <aceonegovleads@microsoft.com>.
  • Once you have helped, get confirmation the ACE Engineers question are answered, then close the case.
  • You will not be asked to submit an ICM escalation to engineering nor participate in a root cause analysis.

Frequently asked questions

Q: What is the difference between Swarming and Collaboration?

A: Collaboration is when one engineer requests another engineer with a different skillset to help solve a customer issue. When implemented through current case management tools, collaboration requires dispatching an object to a queue.

Swarming is a method of collaboration, established in real time, that enables multiple engineers with different skill sets to work together as a virtual-team to solve a customer issue.

Q: When to use either Swarming or Collaboration？

A: Swarming should be the preferred method of collaboration for all situations where there is a live customer issue and a customer waiting for resolution. Swarming enables a shift in Microsoft support where customers don’t need to wait for the right engineer to be found, but instead engineers with the right skillset are engaged in real-time to immediately continue solving the customer issue.
