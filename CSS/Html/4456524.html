Procedure: Resolution Codes

[Asset 4462485]

Who should use this article?

Support Engineer | Technial Advisor | Escalation Engineer | Team Manager

What is this article about?

This document defines the Service Desk Case resolution codes for service requests. In order to understand how Service Desk Case incident management system is working, it is important to understand the reasoning behind each incident closure. The system used to identify the reason for closure is called the Resolution Code. Resolution Code data is used to analyze the types of incidents that are being resolved and the end result of the resolution activities.

1. Steps

It is the responsibility of the Support Engineer (SE) who owns the SR at closing to accurately code the incident resolution based on the definitions in this document.

Service Desk

Resolution Dropdown

Definition

Resolved

Select when Technical Solution has been provided and verified.

Resolved External

Select when Technical solution for the problem was provided by a third party, external to Microsoft.

Resolved by Customer

Select when customer found technical solution for SR problem before the SR owner.

Un-Resolved

Select when Technical solution has not been provided for SR problem.

Duplicate

Select when you close an SR that is a duplicate of another SR.

Disconnect/Hang-up

DO NOT USE.

Rave

Resolution Dropdown

Definition

Resolved

Select when technical solution has been provided and verified by the customer.

Unresolved

Select when there is no technical solution or the request is outside of the support boundary; customer is unresponsive; or the technical solution is not accepted or does not work.

Duplicated

Select when you close an SR that is a duplicate of another SR.

Transferred

Select when the SR needs to change ownership or to be moved to a different support team.
