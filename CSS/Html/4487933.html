1. Assets + subscriptions blade

Assets are containers for ownership and management of various products, for more detail see the Modern Assets training On Success Factors​ or Learning​.

The Assets Tile gives an overview of all Assets in a customer’s account. The View All  experience has a number of filtering options (for example, Asset ID, Subscription GUID​, Reservation Order ID and Project) that will allow a support agent to quickly identify and perform actions on an Asset. The operations available in this experience are Asset Transfer (for example, Subscription Ownership Transfer by doing a Usage Asset transfer) and auto-renewal management using the Manage operation for subscription based assets (for example, Azure Support Plan or Marketplace Dev Services).

1.1 Azure subscriptions + Assets

Azure subscriptions are represented as Usage Assets​ in CST. Support Agents can filter out the Asset associated with an Azure Subscription by Azure Subscription GUID to perform various actions.​​ While not all Usage Assets are necessarily Azure Subscriptions, all Modern Azure Subscriptions (17G and 148G offer types) have a corresponding Usage Asset in a 1:1 relationship.

In the Assets Tile or Assets View All, Usage Assets can be filtered by the behavior of Usage. Currently, only Azure subscriptions create Usage Assets.  You can search and filter by the follow sets of data:

  • Project – Project that is tied to the Asset
  • Behavior- Defines the Asset relationship to Commerce
    • Values - Ratable, Usage, Future Billing Term Based, Quantifiable
  • Custom Search Attribute – Asset ID, Subscription, GUID, Reservation Order ID
  • Custom Search Value
  • Asset ID- Unique Identifier that ties the Asset to the Billing Group
  • Asset Name – Name of Asset (for example, Azure Standard, RI and VM)
  • Type – Same as Behavior (Ratable Usage, Future Billing, Term Based, Quantifiable)
  • Subscription GUID – Subscription tied to the Asset
  • Start Date – Asset ID start date
  • End Date – Date Asset ID ends (Free Trial End Date)
  • Is Free Trial – True or False based on the Asset
  • Quantity – Number of Assets
  • State – Active or inactive

[Asset 4525774]

There are two types of transfers a customer can perform. An Asset Transfer (Transfer Asset) and/or a Billing Ownership Transfer (BOT).

Asset transfer: The customer can transfer their Asset (for example, Saas) along with the Billing Ownership to another Entity. You would perform this transfer like a normal Asset Transfer, but then select the checkbox that states to transfer billing.

Ownership transfer: The customer has the option to transfer just the ownership on an Asset without converting the offer. 

You can transfer one or more subscriptions from one customer to another.

For example, Azure subscriptions are typically bought by individuals and not tenants. When employees change jobs, customers need to transfer an Azure subscription from one employee to another or from one partner to another.

Things to consider:

  • The customer that you’re transferring the subscription to, needs to have a billable account with a valid PI.
  • The financial liability present before the transfer stays with the original customer (the existing financial liability isn’t transferred with the subscription).

To perform an operation on an Asset:

  1. Go to the Assets Section and select View All
  2. Select the Corresponding Asset
  3. Select the Blue Dial and you will be able to perform the following (we will be covering in more detail)
    1. Transferring an Asset – Allows customers to move an Asset from one entity to another.
    2. Returning an Asset
    3. Manage (auto renew and cancellation operations)
    4. Converting an offer
    5. Transferring Ownership - Allows customers to change ownership of their subscriptions  from an existing owner to a new owner.

[Asset 4525775]

Alternatively, a specific subscription can be filtered by GUID as the search parameter:

  1. Copy Subscription GUID.
  2. Select Subscription GUID from the Custom Search Attribute drop down.
  3. Paste Subscription GUID in Custom Search Field.
  4. Select Apply to filter the list.

  • Project - Project/Invoice section to which the subscription is associated.
  • Description - the Azure subscription name.
  • Delegated - Whether the Asset has been delegated to another project.
    • Whether or not the customer’s entitlement is tied to two assets/projects.

  • Subscription GUID - Azure Subscription GUID
  • Start Date - Subscription creation date.
  • End Date - Specifies whether the subscription has a termination date.
  • Updated - Date when Asset was last changed by the system (for instance, the transfer date).

The Related tab displays the different types of Rating Assets (first Party, third Party, Marketplace, Reservation) that are tied to the customer’s usage Asset. If you have a customer that has multiple Assets tied to their usage Asset, it will list out each one (by Asset ID) in this section.

[Asset 4525779]

The Events tab displays a log of all system operations performed on this particular Usage Asset. This will include when the Usage Asset was created, which was captured below.  

  • Event ID – Unique identifier that is created at the point the Event occurs.
  • Date – The date of the Event
  • Type – This is the type of Event (for example, Usage Asset Created, transferred and deleted)

[Asset 4525780]

The Subscription and Offer Details tab displays the following:

  • Subscription ID - Commerce Identifier for Azure subscription.
  • Entitlement Status​ - This is the status detail of the asset in Commerce. Same as Entitlement Status in CMAT.
  • Fulfillment Status -This is the ARM status of the subscription from the Azure side. If the Status and Entitlement Status show suspended, but Fulfillment Status is Active, the Azure subscriptions and services are still active and editable.
  • Purchase Date - Date when subscription was provisioned with resources.
  • Start Date - Date when subscription was provisioned in Azure.
  • End Date - Date when subscription will expire.
  • Billing Cycle - Cadence of billing for subscription (Annually/Monthly).

Billing/Offer displays the following:

  • Offer GUID - Indicates the subscription's offer type:
    • Standard = f8e068a9-d288-4056-96f8-16cd485c4353
    • Dev/Test = 45dea93b-65c2-431a-a6e8-41316b9f6e82​
  • Product SKU - N/A
  • Revenue SKU - N/A
  • Order ID - This is a Legacy concept and is the same from CMAT. Represents the relationship between.
  • Payment Instrument This is a Legacy concept and is the same from CMAT (Credit Card/Invoice).

2. Legacy subscriptions blade

The legacy Subscription tile shows non-Modern subscriptions that do not have any Marketplace purchases tied to the Tenant in context. This means the Legacy Subscriptions do not have a Modern presence.  If a customer is partially converted to Modern and partially in Legacy then you will see Legacy subscriptions under Legacy Subscription Blade and the remaining Modernized under the Assets +Subscriptions Blade.

This section provides details on the following below. This is where can also perform Subscription Transfer, Offer Conversion and Transfer Ownership which is detailed in the below section.

  • Offer Name – Name of offer
  • Offer Type – Free Trial or paid offer
  • Admin Email – Email address of admin assigned to the Legacy subscriptions
  • Plan ID – Unique ID that is tied to the subscription plan
  • Start and End Date – Start and End date of the subscription
  • Status – Active or inactive

[Asset 4525782]

Details + Offer/Billing tab displays:

  • Subscription ID – Same as the Subscription GUID
  • Enrollment Status – Displays active or inactive status
  • Purchase Date – Date offer was purchased
  • Billing Cycle – Type of Billing Cycle (Monthly, Annually)
  • Auto Renewal – True or False
  • Offer GUID – GUID assigned to the specific offer
  • Product SKU – If applicable, Product SKU will display here
  • Revenue SKU – Revenue SKU tied to this offer
  • Order ID – Unique identifier tied to this order
  • Payment Instrument – Type of PI assigned to this offer (for example, CC, Check/Wire or Invoice)

3.Subscription GUID

Expand the Subscription GUID to be able to see the following:

Details

  • Subscription ID – Same as Subscription GUID
  • Entitlement status – (New, active or disabled)
  • Fulfillment status – (New, active or disabled)
  • Purchase Date – Date subscription was purchased and activated
  • Billing Cycle – Date billing is due, if applicable
  • Auto Renewal – (True or False)

Offer/Billing

  • Offer GUID - Unique identifier tied to the offer
  • Product SKU – Where applicable, SKU of products purchased
  • Revenue SKU – Where applicable, will show N/A if not applicable
  • Order ID – Unique ID that is generated when order is place
  • Payment Instrument – Default PI that is tied to the billing group

From this section, you are able to perform the following actions which are called out in the Step By Step GUIDe in this document

  • Transfer Subscription – When you transfer subscription ownership from user to another.
  • Offer Conversion – When you convert an offer from free-trial to paid.
  • Transfer Ownership – When you transfer ownership from owner to another.
