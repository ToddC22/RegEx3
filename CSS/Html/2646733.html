ASMS – How to Download Billing and Usage Files

Submit a ModTrack request to update this TSG.  

Overview  

This TSG provides guidance on how customers can download their invoice and usage reports, and how ASMS can troubleshoot any download issues the customer may have.  

 Applies To: All Microsoft Online Subscription Program (MOSP) customers with an Azure Subscription. For EA customers refer to TSG EA Pricing, Credits, & ABD/ROC Escalations 

Policy 

  • For MOSP customers, only the Account Admin (AA) of the subscription and Role Based Access Control (RBAC) users enabled by AA can gain access to billing via this link.

    • Customers who are not the Account Admin but are requesting invoice and usage details should be directed to contact their AA. Only if written approval from the AA is provided should copies of billing and usage details be provided.   

  • Important: When assisting a customer, you may be required to provide them with a copy of details that contain Personal Identifiable Information (PII) (Ex: copy of invoice). In these scenarios, you must always be sure to delete these downloads from your computer’s hard drive as soon as you are done using them. This is to avoid accidently exposing the customer’s PII.

Support Scenarios  

Customer Unable to Download Invoice  

Ensure that the customer who is trying to access the invoice and usage is the Account Admin. If not, refer to the Policy section above.  

However, if the customer is the AA, follow the below steps to download the invoice and usage from CMAT and provide the customer a copy via email:   

  1. Search and select the customer's account/subscription in question. 
  2. Go to the Subscriptions tab and select the desired subscription.
  3. To download the Invoice:
    1. On the Subscription tab, go to the Billing section and click on Order ID number. 
    2. Click on the down arrow on the right side of the page.  
    3. Select the Billing Period and click on Invoice URL to open and see their detailed report.  
  4.  To download the usage (CSV file): 
    1. On the Subscription tab, go to Export Detail Usage at the bottom of the page. 
    2. Select the billing period. 
    3. Detail Usage report will open.

Customer Unable to Find Invoice  

  1. Check if there are any known bugs or workarounds related to this scenario.
  2. If there are no bugs, check the billing period and customer’s Account Anniversary Date.
  3. Determine if the Account Anniversary Date falls within the current billing period:
    1. If the Anniversary Date does not fall within the billing period in the scenario, then the invoice has not generated yet. Educate the customer on the billing model.
    2. If this satisfies the customer’s questions; Go to step 6 below and close the case.
  4. If the current billing period and Anniversary Date match, troubleshoot as follows:
    1. Ask the customer if they see a billing statement link in the Azure Portal:
      • If the link is not available, reference TSG 2651579 to engage PayOps and submit a bug. If they don’t have a solution, then reference TSG 3093263 to engage ASE.
      • If the Link is available, but the customer can’t download the billing statement, reference the support scenario “Customer Unable to Download Invoice” and engage PayOps.

Note: In addition to engaging PayOps or ASE, you can also provide the customer a copy of the invoice, only if they are the Account Owner.

  1. Once you receive a response from the Internal Engagement Team:
    1. Update the customer with the next steps.
    2. Ensure they are now able to find and download their invoice.
    3. If the customer is still unable to find and download their invoice, re-escalate to the ASE team and/or PayOps as appropriate.
    4. If you are unable to get resolution from either of these teams, reference TSG 2929022, to engage your TA for assistance. 
  2. Upon customer confirmation of successfully finding and downloading their invoice, code and close the case as appropriate.

Investigating usage missing in usage report / recon file scenario:

Commerce bills the resources which are reported within the billing period. The usage policy extends to 9 days beyond the billing period which means if the period is 5th of every month, commerce can go ahead and bill the usage emitted by the Resource Provider for the same period till 14th of that month. If there is any delay in emitting beyond this period, it will not be billed and does not display in usage report / recon file.

Any usage missing scenario should be investigated by ASMS by downloading the aggregate report for that billing cycle, if the aggregate report shows the usage but usage report / recon file does not, then it is due to the above issue and customer will not be billed for this usage.

If the usage is missing in both the aggregate report and usage report / recon file then contact ASE Team for further investigation as this is an issue with upstream team not reporting the usage to commerce.

Customer Not Receiving Azure Invoice Email After Opting in 

If the customer has opted in but is not receiving the invoices via email, check Jarvis to ensure the system has their correct information. 

  • In Jarvis: Billing/Invoice Communication Management/Get Invoice Communication Details from SubscriptionID 

  • If the output says the following: "Metadata not found. Use the Update operation to add metadata." It means that the user has not opted in as we do not have any metadata for that subscription. 

  • If the output is as noted below, then they have opted in and should be getting the invoice via email.  

    • InvoiceAttachmentOptedIn Value: True 

    • InvoiceEmails: "UPN":"email@contoso.com","ListType":"To"

Customer Wants to Add, or Has Been Added as a Billing Reader In RBAC via the Ibiza Portal 

Customers may contact Support for Billing Reader RBAC questions. In those instances, ASMS will only be able to assist with the following:  

1. Unable to add user to the role. (Contributor, Reader, or both.) 

2. Unable to perform billing operation. 

a. Unable to delegate user. 

b. Unable to download invoice. 

Walk the customer through the steps below to add users to the Billing Reader role. If needed, more detailed steps can be found here. 

1. Select your subscription from the Subscriptions blade in Azure portal. 

2. Select Access control (IAM) and then click Add. 

3. Choose Billing Reader in the "Select a role" page. 

4. Type the email for the user you want to invite, then click OK to send the invitation. 

Note:  The user must be present in the directory before adding to any role in RBAC. Once added, the user will only have read access to billing, meaning they can download invoices, access cost management views by resource/service, and access the subscription properties API. None of the roles will have the ability to delegate access to new users, or access the Accounts portal. Only the Account Admin will have access to subscriptions via the Accounts portal.
