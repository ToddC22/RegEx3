What is this article about?

This article explains how to use LogMeIn Rescue (LMI). Guidance on appropriate usage and application is not within the scope of this article.

  • Note: This cannot be used on a device running Windows RT, such as Surface RT/Surface 2, or devices running Windows 10 S, such as Surface Laptop.


LogMeIn is a tool that enables an agent to view the customer's device and chat at the same time to assist with troubleshooting. The customer will need to log in and access a web page to start the LMI session. If a connection cannot be established for various reasons (for example, the customer's Wi-Fi does not work), return to standard troubleshooting methods without using LMI.

If a connection cannot be established for various reasons (for example, the customer's Wi-Fi does not work), return to standard troubleshooting methods without using LMI.

1.1 Create a New Session

Provide the following verbatim scripting to the customer before continuing:

  • "Once we've started our session, I will have the ability to view your desktop. Please close any windows that may contain sensitive data. At any point if you feel uncomfortable, you can end the remote session by selecting the red X button.”

1. Verify the customer in ASD. If the customer is not authenticated, follow the process to verify the customer. See KB 4506610 Procedure: Microsoft account verification.

2. Select LogMeIn (LMI) in Solutions and Tools. If the customer is not authenticated, a padlock symbol will appear next to the Launch LogMeIn (LMI) link in Solutions and tools.

3. On a new Incident form select New to generate the SR number. The incident form does not need to be completely filled out to generate an SR number.

  • Note: The steps need to be followed in the order listed. If anything is selected in the Product field on the Incident form prior to authentication, the padlock will remain and the LMI tool will not launch. If this happens, cancel the case and open a new case.

4. The Technician Console will display. Initiate a session by selecting New Session.           

[Asset 4517660]

5. The Create New Session window is displayed.                                                                         [Asset 4517662]

6. Enter the customer name in the Customer Name field, and the SR number in the Service Request ID field. The customer email field is optional and does not need to be populated. Select enter, and a six-digit PIN Code will be created. Do not select the Close button as you will need to provide this PIN in a subsequent step.

 [Asset 4517663]

7. Provide the customer this URL: http://support.microsoft.com/help. This opens a browser window with the terms and conditions the customer must accept in order for the LMI session to be initiated.

8. Make sure to tell the customer: "Please take a moment to review the terms on this page. If you accept the terms, please select I accept."                                                                                         

[Asset 4517664]   [Asset 4517665]

9. Provide the PIN to the customer.

10. Once the customer accepts the terms, they will need to enter the six-digit PIN in the space provided and select the Connect to Technician button.
 [Asset 4517666]

Important: If the customer does not accept the terms, close LMI and continue with the standard troubleshooting steps.

11. A pop-up window will open on the customer's device. Ask the customer to follow the displayed directions and choose Run.

12. Once the install is completed, the customer will see a chat window open on their device. 

 [Asset 4517667]

13. You will see the active session on the Technician Console.                                                        [Asset 4517668]

14. You have two choices – Explain the two options to the customer and select the customer's preference.

  • Launch Remote Control Session: You can see and control the mouse.
  • Launch Desktop Viewing Session: You can see but the customer controls the mouse.

15. The customer will see a pop-up window requesting permission to either view the desktop or share control. They have 30 seconds to select OK before the request times out. If this happens, simply press Launch Remote Control Session or Launch Desktop View Session button on the Technical Console as shown in the previous step.  

[Asset 4517669]

Customer sees: Launch Remote Control Session

Customer sees: Launch Desktop Viewing Session

16. Once the customer clicks OK, the customer's desktop appears in the Customer Desktop tab of the Technician Console.
[Asset 4517670]

There are two features which can be used to aid the troubleshooting.

[Asset 4517671]

  • Laser Pointer Tool: The Laser Pointer is a small red dot that you can use to help your remote user identify screen elements. When you turn on the Laser Pointer tool a red dot appears on the remote desktop and follows your mouse.
  • Whiteboard: Whiteboard activates a pencil that you can use to draw free hand images on the remote desktop.

1.2 Pausing the Session

  1. Use the Hold Session button: If there is ever a possibility that a customer's personal data or other sensitive information has become viewable, you must use the Hold Session button on the Agent Console. This pauses the session.

[Asset 4517672]

Note: If at any point, you discover content that is not appropriate, follow the policy documented in KB 2626212 Procedure: Discovery of Child Pornography.

  1. To resume the session, either press the green Start Session button, or double selectin the status field. The status will change to Active. To restart the session, press the Launch Remote Control Session or Launch Desktop View Session button.                                                                                  [Asset 4517673]
  2. The customer can revoke permission to view their desktop at any time by clicking the red X. Even though the viewing session is stopped, the chat window will still be open. To restart the session, press the Launch Remote Control Session or Launch Desktop View Session button.

1.3 Rebooting the customer's PC remotely

1. Ensure the customer understands that you are about to reboot their PC.

2. Open the Reboot tab from the Technician Console.                                                              [Asset 4517674]

3. Press the Green button to do a normal reboot.

Note: Always use the Normal Reboot and Reconnect buttons. This closes all the processes first then reboots the device.

4. A pop-up box on the Technician Console will display to confirm that you want to reboot the device. Select Ok to continue the Reboot process.                                                                             

[Asset 4517676]

5. A pop-up box shown below will display on the customer's screen. They have 30 seconds to accept and select yes. If the pop-up disappears due to time out, press the Green Button to send another request.   

[Asset 4517677]

6. Once the device has completed the Reboot Process, the device will restart, and the chat box will reopen on the customer's device.

Return to top

1.4 Ending the Session

  1. To end the session completely, instruct the customer to close the application by pressing the X at the top of their chat window. Since this also removes the applet, this session cannot be reestablished, and a new session will need to be initiated.
  2. The session can also be ended from the Technician Console by pressing the red X on the top tool bar, or on the customer desktop tab. Once the session has been ended, it cannot be reestablished, and a new session will need to be initiated.

1.5 FAQ

Scenario

Solution

The Customer does not accept the Terms, what should I do?

Close the LMI application and continue using standard troubleshooting.

The Customer wants to know why they need to download software.

The customer downloads a small applet that lasts for the duration of the session to allow for the connection, then the applet deletes itself when the session is closed. No software is installed.

3. LMI Rescue mobile user interface differences

Mobile icon

You should all be familiar with the LMI Rescue console used to connect to PCs and MACs. The same LMI Rescue console is used to connect to mobile devices. When you log in to the console, you will see some subtle differences when the mobile license is applied to your account.

When you launch LMI Rescue, you will see a new mobile icon under the Option menu. This simply means the mobile license has been applied successfully.

[Asset 4517679]

Click2Fix tab

The Click2Fix tab is where you will see most of the information related to the mobile device. In this section you will see:

  • Mobile network information
  • Battery status
  • System information
  • Device view (where the view of the mobile screen will appear)
  • Connection (GPS, Wi-Fi or Bluetooth)

[Asset 4517680]

Device information on the click2fix tab will only be populated in Android sessions

4. How to connect to a mobile device

Irrespective of how you connect to the customers device, the customer will need to download the appropriate mobile app for the platform you will connect to. If you use the PIN code method, the customer will need to manually download the app using the links below. All other methods will send a link to the customer. When the link is selected, the app will automatically be download and connect to the session. Note: Windows Mobile Devices is not supported.

Android: https://play.google.com/store/apps/details?id=com.logmein.rescuemobile

iOS: https://itunes.apple.com/us/app/rescue-mobile/id661269562?mt=8
 

To connect to a mobile device, select New Session at the top of the console.

There are four methods below which will allow you to connect to the user’s device:

  • PIN Code
  • Email
  • Link
  • SMS

5.1.2 Turn on iOS Screen Recording feature:

To connect to the iOS device, the screen recording option must be enabled.

[Asset 4517684]

To access the control center the customer needs to swipe up from the bottom of the screen. If the screen recording icon (see image below) is missing, you will need to enable the feature. 

[Asset 4517685]

To enable the screen recording feature:

  1. Select the Settings icon
  2. Select the Control Center
  3. Select Customize Controls
  4. Select the Screen Recording icon (the icon is red in Customize Controls)
  5. The screen recording feature should be available in the Control Center.
  6. Ask the customer to open the Control Center by swiping up.
  7. Ask the customer to hold their finger down on the Screen Recording icon. A new screen will populate showing which applications can use this feature.
  8. Select LogMeIn Rescue.

6.2 Color Quality

There is an option to change the color quality the agent sees during the remote session. When changing this option, the color quality only affects what the agent sees, it does not affect the color quality on the device. The reason for using this option is two-fold. If the customer is using their mobile network instead of a high-speed internet connection (not connect to Wi-Fi) and the customer is concerned with the data usage on the LMI session. Reducing the color quality reduces the information sent from the device. The second reason for lowering the color quality is to speed up the screen refresh. High quality allows you to see what the customer see on their screen, but this can impact the time it takes for the LMI screen view to update.

[Asset 4517696]

Device information such as Mobile Network, Battery, System information or Connection method is not available (This is a restriction Apple places on third-party applications)
