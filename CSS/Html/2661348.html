ASMS – Add or Remove EA Administrator

Warning! Do not add or remove EA Admins on the customer's behalf.

Effective immediately, ASMS should not add or remove EA Admins. Instead, encourage the customer to follow the self-serve guidance outlined on the Get Started with Azure Enterprise Portal page.

New processes on how to handle these requests are being discussed and will be updated in the respective TSGs as more details become available. See Message Center Post for the latest updates.

Overview

This TSG provides guidance on how to assist customers in adding and removing Enterprise Agreement (EA) Administrators.

If you are looking for guidance on how to add EA Accounts, refer to TSG 4537343.

Note: This TSG applies to SEs and TAs who are monitoring the following queues:

  • MSaaS ASMS EAL MCA (Global English and non-APJ LL; includes USG)
  • MSaaS ASMS EAL MCA - APJ Local Language (Mandarin, Korean, Japanese)

Policy

  • An existing EA Administrator (Admin) must approve additions and removals of other EA Admins. This can be in the form of direct approval from the existing EA Admin, or in the form of an approval email attached to the case file.

    Note: The EA Admin giving approval cannot be a Read-Only EA Admin. 

    If you receive a case to update or to remove the Read-Only permission of an existing Enterprise Administrator, approval from another existing EA Administrator must be provided before proceeding with the change.
     
  • The Notices Contact and Online (NTC) Admin is the enrollment’s default EA Admin.

Customer Experience and Provisioning

Steps to Add or Remove an EA Admin

Adding an EA Admin

  1. Navigate to the EA Portal.

  2. If managing more than one enrollment, select the desired enrollment.

  3. Navigate to Manage > Enrollment.

  4. Confirm that the Authentication Level is correctly set for the type of user being added. Change it accordingly if it is not correct.

    • This setting will only affect the next user added. It will not impact any users that have already been added.

    • If adding a Cross-Tenant Work or School account, change the Auth Level to Work or School Account Cross-Tenant.

  5. Click +Add Administrator.

  6. Select the desired Authentication Type.

  7. Input the desired Email Address and Confirm Email Address. Also note the following:

    • Notification Contact is optional and may be left blank.

    • Notification Frequency defaults to Weekly.

    • Lifecycle Notification Suppression is not checked by default, allowing the system to send a notification when the enrollment is approaching its end of life.

  8. Read Only defaults to No. If the new EA Admin should only have read rights, change this option to Yes.

  9. Confirm that all information is correct and click Add.

Removing an EA Admin

An EA Admin cannot be removed if they are the only EA Admin on the enrollment. To remove an Admin, do the following:

  1. Navigate to the EA Portal.

  2. If managing more than one enrollment, select the desired enrollment.

  3. Navigate to Manage > Enrollment.

  4. Hover over the EA Admin to be removed and click on the blue X icon.

  5. Confirm that the desired EA Admin is selected and click Yes.

Note: If a customer removes an account as an EA Admin but then sees that the account has been added back as an EA admin automatically by the system, check the EA Enrollment in Explore.ms to see if the affected account is a part of the participant list. If so, the customer should reach out to their software advisor to get the participant list updated and remove the requested account. Then, the system will not auto-sync and the unwanted EA Admin will not be re-added.

Adding a Partner Admin (Existing Partner Admin)

  1. Navigate to the EA Portal.

  2. Select the enrollment and navigate to Manage > Partner.

  3. Confirm that the Authentication Level is correctly set for the type of user that you are about to add.

    • Change it accordingly if it is not correct. This setting will only affect the next user added.

    • If you are adding a Cross-Tenant Work or School account, ensure that you have changed the Authentication Level from Mixed Account to Work or School account Cross-Tenant.

  4. Click +Add Administrator.

    • Select the desired Authentication Type.

    • Input the Email Address and Confirm Email Address.

    • Read Only defaults to No. If the new Partner Admin should only have read rights, change this option to Yes.

  5. Confirm all information is correct and click Add.

Adding a Department Admin (EA Admin)

  1. Navigate to the EA Portal.

  2. If you manage more than one enrollment, select the desired enrollment.

  3. Confirm that the Auth Level is correctly set for the type of user that you are about to add and change it accordingly if it is not.

    • This setting will only affect the next user that you are going to add and will not impact any users that have already been added.

    • If you are adding a Cross-Tenant Work or School account, ensure that you have changed the Auth Level to Work or School account Cross-Tenant instead of Mixed Account.

  4. Navigate to Manage > Departments.

  5. Navigate to List Enrollments and search by Enrollment Number.

  6. Select the enrollment in question and navigate to Manage > Department.

  7. If the department in question does not yet exist, click +Add Department.

    • Department Name is a mandatory field and must be unique.

    • Cost Center and Spending Quota are optional.

  8. Select the department in question and click +Add Administrator.

    • Select the desired Authentication Type.

    • Input the desired Email Address and Confirm Email Address.

    • Read Only defaults to No. If the new Department Admin should only have read rights, change this option to Yes.

  9. Confirm all information is correct and click Add.

Support Scenarios

Customers who have the proper credentials can complete the actions listed in the Customer Experience and Provisioning section above. However, in some cases customers may need assistance from support. In those instances, determine which of the following scenarios best applies to the customer’s request and follow the guidance accordingly.

Request from an Existing EA Admin

In some cases where the EA enrollment status is not Active, the EA Admin will not have the +Add Administrator option and will need to contact support for assistance. In these instances, the requestor will need to provide approval to add an EA Admin on their behalf. Be sure to verify the customer before adding.

Request not from an Existing EA Admin

Support can add EA Admins on the customer’s behalf once the customer has provided the following:

  • Approval from an existing EA Admin on the enrollment

  • Confirmation of the desired Authentication Type of the user to be added as an EA Admin.

Once the desired customer has been added as an EA Admin, be sure to inform the requestor of this update to their portal.

Requests from Direct, Indirect Organizations, and Partners

Direct

Direct customers may contact support for updating the Admins for their organization. In these instances, inform the customer that they will need to contact their Software Advisor to update the NTC Admin (using either the manual CICR method or the electronic CICR method detailed below).

The customer can also reach out to their Bill-to-Contact to provide the approval. In return, support can help add the EA Admin once approval has been provided.

Note: Refer to TSG 4463565 for information on how to view the Software Advisors, NTC Admin, and other contacts on the enrollment.

Indirect

Indirect EA customers may contact support for updating the Admins of their organization. In these instances, inform the customer that they must contact their Partner to update the enrollment’s NTC Admin (using either the manual or electronic CICR method detailed below).

Partner

Inform the customer that as a Partner/Software Advisor, they can update the NTC Admin using either the manual CICR method or the electronic CICR method (see details below).

CICR Methods

  • Manual CICR method: A Partner can submit a Customer Information Change Request (CICR) form to update the NTC Admin. The form should be submitted to the Regional Operations Center (ROC) through a Call Logging Tool (CLT) ticket in Explore.ms. In return this will update the default EA Admin.


Note: If the EA Portal is not automatically updated after the NTC Admin has been updated, SEs can manually add the new default EA Admin to the EA Portal since the information has been updated at the agreement level.
 

  • Electronic CICR method: A Partner can update the NTC Admin via eAgreements. This will update the default EA Admin. If the EA Portal is not automatically updated after the NTC Admin has been updated, SEs can manually add the new default EA Admin to the EA Portal since the information has been updated at the agreement level, or SEs can request that Electronic Communications Information Technology (ECIT) re-push the agreement details for the enrollment by following the below steps:

  1. Compose an email message addressed to OrderL3@microsoft.com and orderhlp@microsoft.com.

  2. In the subject line, enter: Order re-push [Enrollment Number]

  3. In the email body, provide the following information:

    • Enrollment Number

    • Request that ECIT re-push the enrollment because details at the agreement level have not been updated in the EA Portal.

If there is a request to re-add an existing EA Admin with another authentication type for an indirect enrollment that is in Pending status, and the requestor is not one of the contacts from the participants list, the change can be done without a CICR. Be sure to verify that the requestor is from the Partner company/organization.

Request from an Internal Team

Internal teams at Microsoft may contact support on a customer’s behalf and request to add or remove EA admins. In these instances, provide the internal team with the following action options:

  • Contact the Partner/Software Advisor to update the NTC Admin.

  • Select someone from the agreement participants list to be included in the email thread in order to provide approval.

In urgent cases where the above two options are not possible, the SE can add the requested user as an EA Admin if the internal team has provided assurance that they have completed the necessary checks and the internal team agrees to take full responsibility for authorizing this action.

Request to Add a Partner Administrator

Note: A Partner Admin may be set up as Read Only.

Email approval from an existing Partner Administrator is required to add a new Partner Administrator. Once this approval has been received, perform the following steps:

  1. Verify that the email address that the customer requests to add is a valid email address and authentication type combination.

    • If the email address is valid, proceed to step 2.

    • If the email address is not valid:

      • If the email address is for a Microsoft Account, notify the customer that they will need to create a new Microsoft account.

      • If the email address is for a Work or School Account, notify the customer that, if they have Office 365/Azure Active Directory (ADD), they need to add the user to the Azure Active Directory. Otherwise, they may want to consider using a Microsoft Account instead.

  2. Check if the request is from an existing Partner Admin using step 1-5 under Adding a Partner Admin (MS Admin User. If the request is from an existing Partner Admin, notify the customer that they can add Partner Administrators using the steps under Adding a Partner Admin (Existing Partner Admin)

  3. If the request is not from an existing Partner Admin, notify the customer that any of the existing Partner Administrators on the enrollment can add a new Partner Administrator using the steps under Adding a Partner Admin (Existing Partner Admin).

  4. Alternatively, the customer may provide email approval from an existing Partner Admin on the enrollment. In this case, confirm the email address and Authentication Type of the user that the customer would like to add. Once this approval is obtained, add the Partner Admin using the steps under Adding a Partner Admin (MS Admin User). Notify the requestor once this is done.

Adding a Partner Admin (MS Admin User)

  1. Navigate to the EA Admin Portal.

  2. Select List Partners and search by Partner Name or Partner Number.

  3. Click on the Partner Name. This will display a list of enrollments under that partner.

  4. Select an enrollment. You will land on Manage > Enrollment.

  5. Navigate to the Partner tab, to see all of the current Partner Administrators.

  6. Click +Add Administrator.

    • Select the desired Authentication Type.

    • Input the Email Address and Confirm Email Address.

    • Read Only defaults to No. If the new Partner Admin should only have read rights, change this option to Yes.

Confirm that all information is correct and click Add.

Adding a Department Admin (MS Admin User)

  1. Navigate to the EA Admin Portal.

  2. Navigate to List Enrollments and search by Enrollment Number.

  3. Select the enrollment and navigate to Manage > Department.

  4. If the department in question does not yet exist, click +Add Department.

    • Department Name is a mandatory field and must be unique.

    • Cost Center and Spending Quota are optional.

  5. Select the department and click +Add Administrator.

    • Select the desired Authentication Type.

    • Input the desired Email Address and Confirm Email Address.

    • Read Only defaults to No. If the new Department Admin should only have read rights, change this option to Yes.

  6. Confirm all information is correct and click Add.

Frequently Asked Questions about Departments

  1. Does deleting a Department delete the associated accounts? 

    • No. When a Department is deleted, the associated accounts are returned to the Unassigned category.

  2. Can a deleted Department be restored?

    • When a Department is deleted, it is marked as Inactive. Inactive Departments cannot be returned to Active status.

  3. I have deleted a department and I want to create another department with the same name. Why do I encounter an error when I try to do so?

    • Department names can only be used once. To work around this, modify the department name slightly by adding a space, symbol, number, or dash so that the system will recognize it as a different name and allow creation of the new department.

  4. Is it possible to completely delete a department so that it does not show up at all?

    • For record-keeping purposes, it is not possible to completely remove a Department. Please ensure that the Active checkbox at the top right corner of the Department list view is checked if you would like to hide the inactive departments from view.

  5. What is the default view for Departments?

    • The Active checkbox in the Department tab is checked by default. It is unchecked automatically if there are only Inactive Departments on the list.

  6. What can a Department Admin see if they are also an Account Owner?

    • They will see all that would normally be seen by a Department Admin as well as an Account Owner.

  7. Can a Department Admin simultaneously be an Account Owner on the same enrollment?

    • Yes.

  8. Can a Department Admin simultaneously be an EA Admin on the same enrollment?

    • No. The rights of an EA Admin conflict with those of a Department Admin.

  9. Can Department Admins see everything on the Usage Summary tab?

    • Department Admins cannot see the graph on the Usage Summary tab since the graph contains information pertaining to the entire enrollment.

  10. How are the Department Spending Quotas calculated?

    • Only first-party usage is considered in the calculation. Marketplace purchases are not part of the calculation.

  11. How can I change/stop notifications about Department Quotas?

    • There is no way to stop notifications about Department Quotas. The frequency of the notifications is fixed and cannot be changed. The only way to stop the notifications is to remove the Quota or remove the Department Admin.

  12. After an account is removed from a Department, what is shown under the Usage Detail Report?

    • In this case, the account will appear under the Unassigned category. It will remain so until the account is added to another department.

Add or Change a Billing Administrator

Billing Administrators have limited edit access. They can edit PO numbers and download usage reports.

Billing Admins cannot be added or changed in the EA Portal. They are added to enrollments automatically based on the user that is set up as the Bill-To contact on Explore.ms. To change the Billing Admin, the Bill-To contact must be changed by issuing a request to the Regional Operations Center (ROC).

If the Billing Admin is an existing EA Admin, an information icon will be displayed beside the EA Admin status. An EA Admin who is also a Billing Admin cannot be deleted from the EA Portal.
