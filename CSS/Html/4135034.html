Tool User Guide: Data Transfer & Management (DTM)

[Asset 4049349]

Who should use this article?

  • All Consumer

What is this article about?

The DTM tool is a secure file transfer platform used when Microsoft support needs to securely send or receive highly confidential/sensitive personal data or sensitive business information from customers or other Microsoft support advocates.

See KB 3064766 Procedure: Privacy: Handling Consumer customer personal data for the definitions of highly confidential/sensitive and confidential personal data.

1. Creating a workspace and Service Request (SR)

  1. Open a new or existing SR/child SR, as a reference for the transferred files. DTM will reference this SR for access control.
    • For engagements with Advanced Capability Advocates (ACA), Accounts & Billing, Empowered Resolution Advocates (ERA) team, and Xbox Rangers should use the child SR as the reference.
    • Anyone with the DTM link can upload files to the workspace. Only the customer, current and previous case owners, and those msncs.com aliases included on the DTM workspace shared on your tools will be able to see or have access to the files in the workspace.
  2. Launch the DTM tool here: https://client.dtmnebula.microsoft.com/msa/Landing.
  3. Enter the SR number in the search field and select Go within the DTM window.
  4. Select the External tab, then Customer access to enable collaboration with a customer.

[Asset 4493331]

Note:

  • Internal tab: Microsoft to Microsoft.
  • External tab: Microsoft to customer/Customer to Microsoft.
  1. A pop-up will appear stating “No workspace exists for case (case number).” Select Create, as seen below. 
  2. To provide a customer with a location to attach or upload file, select Customer access. You will be provided a link, which may be shared in a chat or pasted into an email. Include instructions on How to use Secure File Exchange to exchange files with Microsoft Support.
  3. Include the DTM workspace link in the case notes.

 [Asset 4493333]

Note: The customer may need to open a text editor or word processing application and save the requested information to upload a file to DTM.

  1. Ask the customer to select the DTM link you have provided. After selecting the link, the customer will need to log in with their Microsoft account. Once logged in, they can select Add Files, then locate the file they are looking to attach. The file will then start to upload. Once the upload is finished, they can close the DTM window.

[Asset 4493335]

  1. Once the customer has uploaded the requested information, the case owner can review what was uploaded by logging back into DTM using the same SR. It may take a couple minutes for uploaded files to populate in the workspace.

Note: If you are engaging/escalating a case, make sure that all the files have been uploaded by the customer. Engaging/escalating prior to collecting all required information may result in delayed resolution or de-escalation.

Retrieve/Add files

  1. Access and take ownership of the SR where the DTM file(s) are attached. 

[Asset 4493336]

  1. Open DTM and add files by using the Upload files button. Or if the workspace has already been created, view the files as needed.

    Note: Each workspace can contain up to 1000 files, each up to 100GB in size.

2. Using DTM Collaboration feature inside Assisted Support Desktop (ASD)

The field DTM shared with internal users, allows for specific msncs aliases to access the DTM workspace without them having to take ownership of the service request.

By adding the msncs credential of the person into the service request, you can share the DTM workspace with them. You'll find this field under the case disposition & closing section of the ASD form.

[Asset 4493337]

Use a semicolon (;) to add more than one user to the case, and share the internal workspace link with them. Note: Be sure not to send msncs users the customer-facing link, or they'll only be able to add files.

All internal msncs users have two options to access a case DTM workspace that has been shared with them:

  1. Access the regular DTM link, enter the case number, and log in using your msncs credentials.
  2. Access the case workspace directly at: https://client.dtmnebula.microsoft.com/msa/Home?srNumber=xxx, where "xxx" is the case ID.

3. Troubleshooting DTM issues

To troubleshoot tool issues (after access is requested and granted), try the following:

  1. Have the site forget your ASD credentials, close the tab, then manually enter your ASD credentials (@msncs.com).
  2. Confirm you own the case.
  3. Restart your browser and clear the cache memory.
  4. Restart your computer.
  5. If these steps do not resolve the issue, engage your team lead as a tools issue. See KB 3038858 Procedure: How and when to engage CSS Command Center (CSS CC) all regions.

4. Handling payment instrument (PI) information via DTM

  • Full PI information (full digits of credit card numbers and associated CVV numbers) can’t be sent or stored under any circumstances (never request images of the customer’s credit/debit cards) including with the DTM tool.
  • Highly confidential/sensitive information can only be handled through the DTM tool. See KB 3064766 Procedure: Privacy: Handling Consumer customer personal data for a list of highly confidential/sensitive personal data type examples.
  • For phone calls, you must pause the call recording when payment information are being collected (full 16 digit credit card numbers or CVV), and resume after collecting that information.
    • See KB 3102066 Tool User Guide: OneCloud for Advocates for more details on how to pause/resume.

5. FAQ

Q: Does Microsoft follow GDPR guidance globally?

A: The General Data Protection Regulation (GDPR), which sets requirements for protecting customer data for EU residents, became enforceable on May 25th, 2018. Microsoft believes all its customers’ data should be equally protected and has extended the requirements for everyone globally.

Q: Does ACA follow the same guidance to use DTM to collect any additional customer personal data with a child SR? Will it be linked with the previously collected information in DTM?

A: Yes. If the customer needs to provide additional information or access a document provided by ACA, they will use the same workspace link.

Q: In the training, it says that DTM tools will be used for the account takeover scenario for Accounts and Billing. Can you clarify any other scenarios where the DTM tool should be used or for whenever it needs customer personal data (per KB 3064766)?

A: Account takeover is in scope for all advocates, see KB 930018 Procedure: How to handle account takeover on a Microsoft account. However, anytime a customer needs to send highly confidential/sensitive personal data or sensitive business information, the DTM tool MUST be used.

Q: Can customers send documentation via email?

A: Make sure the customer always uses DTM to send sent documents like PoP.

  • Should the customer accidentally send documentation via email, ask the customer to upload the file again, but use DTM.
  • If customer emails accidentally, agents should ask the customer again to upload the file using DTM.
