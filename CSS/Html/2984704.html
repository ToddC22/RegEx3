Procedure: Microsoft account: Handling temporarily suspended (terms of use violation) accounts

[Asset 4049349]

Who should use this article?

  • All Consumer

What is this article about?

This article provides guidance on assisting customers with temporarily locked accounts.

There are two scenarios for which accounts may be locked:

  • Compliance 
  • Suspend for Abuse

Be it for a compliance issue or an abuse issue, it appears that the customer’s account has violated the Microsoft Services Agreement.

1. Suspended for Abuse – Next button

1.1 What the customer is seeing

The customer will see the following error message during Microsoft account sign in:

"Your account has been locked" with a Next button.

1.2 If the customer asks why their account was suspended

Explain that we cannot determine the specific reason why their account was blocked, but we've detected questionable activity. We suspended the account to make sure it was safe. For more information the customer can see the Code of Conduct at https://www.microsoft.com/servicesagreement.

1.3 If the customer asks you to reactivate their account

The error message during sign in should also include instructions on how to reactivate the account. Following those instructions is the fastest and easiest way to get back into an account after it has been suspended.

Note: If asked to reactivate their account, you can only offer to walk them through the process.
 
1. Have the customer go to the error message or, if they closed it, have them try to sign in again which should bring up the message.

Once there, select Next.		 [Asset 4501183]
2. Have the customer enter a phone number to any phone they can get text messages on. This does not have to be a phone listed in their security info.

Once a phone number is entered, select Submit.

If the customer does not have access to a phone that accepts SMS text messages, refer to section 2.1.		 [Asset 4501184]
3. Another screen will appear while the code is sent. The customer should leave the window open and check their phone for the code. [Asset 4501185]
4. If the code does not work, make sure the customer copies the code inside the text message and not the number the text message was from.

If the code wasn't received, have the customer make sure the phone number entered was correct, that the phone accepts SMS messages, and try disabling any message blocking features.
If these steps fail, recommend that the customer try another mobile device.

If the customer does not have a phone that receives SMS text messages, have them ask a trusted friend or family member for theirs. This will be a one-time code.		 [Asset 4501186]
5. Once the code is received, have the customer copy the code into the field, and select Submit. [Asset 4501187]
6. While this message appears to suggest that the process is complete, there may be one more thing to do...

Once customer selects Continue, they might be prompted to change their password.		 [Asset 4501188]

7. If prompted, walk customer through changing their password. Suggest using a password they can remember but is not the same as other passwords they might use. Sometimes a phrase is better than a single word.

Once they are done, select Submit.

 [Asset 4136863]
8. Once the password has been changed, have them sign in and they should be done.  

Important note: There is no escalation path for this issue other than following the steps outlined above.

2. Compliance issue- Navigate to “aka.ms/compliancelock”

2.1 What the customer is seeing

The customer will see the following error message during Microsoft account sign in:

“Your account has been locked” with no Next button.

[Asset 4345238]


Selecting "aka.ms/compliancelock" brings the customer to the following page:
https://support.microsoft.com/help/13997/microsoft-account-temporarily-suspended-private


[Asset 4091459]

Return to top

2.2 If the customer asks why their account was suspended

Inform the customer of the following:

"If Microsoft finds that your account was used to violate the Microsoft Service Agreement/Code of Conduct, Microsoft reserves the right to close your account without warning. To read the Microsoft Service Agreement, click here. To read the Code of Conduct, click here."

Return to top

2.3 If the customer asks you to reactivate their account

Explain that you don’t have authority to do that, but there is a formal process to request a review that they can use:
  • Microsoft Service Agreement/Code of Conduct violation review requests must be submitted via the online form at the bottom of the page they were referred to: https://support.microsoft.com/help/13997/microsoft-account-temporarily-suspended-private.
  • If asked why you can’t do it since it says Contact Support, inform the customer of the following:

    "For security and privacy reasons support advocates cannot reset a password, make any changes on an account or unlock an account. The only solution is to follow the steps prompted when trying to sign in and submit the account reinstatement form. The form is fully automated in order to keep your sensitive information secure."

Important note: There is no escalation path for this issue other than following the steps outlined above.

Note: This procedure does not apply to customers who want to recover their Microsoft account. For account recovery refer to KB 3123259 Procedure: Microsoft account: The "Recover your Microsoft account" form.
