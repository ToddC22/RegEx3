Event ID 1202 logged when adding the Active Directory Domain Services role in Windows Server 2008 R2

An Event ID 1202 may be logged in the DFS Replication log when the Active Directory Domain Services role is added to a Windows Server 2008 R2 computer. The error message reads, "The DFS Replication service failed to contact the domain controller to access configuration information. Replication is stopped. The service will try again during the next configuration polling cycle, which will occur in 60 minutes."

If the event is logged on the first domain controller in a new domain and the Distributed File System Replication (DFSR) service event 1206 is logged after the first reboot of this domain controller you can safely ignore the error message.

This event is logged whenever a domain controller is added using the Active Directory Domain Services role feature in Windows Server 2008 R2 and another domain controller cannot be queried for domain information. This event will always be logged for the first domain controller in a domain since it is not yet joined to the domain and there are no other domain controllers to query. After rebooting this first domain controller, the Distributed File System Replication (DFSR) service will log an event ID 1206 when the domain has been joined successfully; thereby, resolving the event ID 1202 error.

If this error is logged on other than the first domain controller, this event can be caused by TCP/IP connectivity, firewall, Active Directory®, or DNS errors or misconfigurations, see the TechNet Events and Errors Message Center at http://www.microsoft.com/technet/support/ee/ee_advanced.aspx for more information as well as the following:
