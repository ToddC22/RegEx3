The following agent roles: All roles

A partner’s credit request has been approved and is ready to be applied to their account.

2. Policy

  • The monetary thresholds listed in this article are per request.
  • CSP agents are authorized to process approved credits in the Commerce Management Agent Tool (CMAT) for CSP partners up to $2,500 (or local currency equivalent) per request without creating an escalation to PayOps Execution, except for discounts and special pricing and withholding tax credits. Any approved credit/discount above $2,500 must be escalated to PayOps Redmond to be applied in CMAT.
  • If the approved credit is ≤$650 USD or local equivalent, CMAT will apply the credit to the partner’s account without triggering the approval workflow. 
  • If the credit is > $650 USD or local equivalent, CMAT will prompt the agent to select approvers in the Refund Transaction workflow. The approvers will be notified of the pending credit needing their approval. Once the transaction is approved, the credit is applied to the partner’s account.
  • This process is for CSP subscriptions sold to Direct CSP partners or Indirect Provider CSP partners only.

3. Process 

Note: Credit threshold varies on the type of credit request. Credit policies and scenarios can be found in the following articles:

  • Cancellation credits, please see article 4023102 CSP: Microsoft Online Services cancellation policy and process.
  • Vat credits, please see article 4037109 Supporting Value-Added Tax and Tax Exemption for CSP (EMEA only)
  • Tax credits for US, Canada and Puerto Rico, see article 4132854 CSP: Processing a Sale Tax Exemption for Partners in the USA, Canada, and Puerto Rico
  • Tax credits for APGC, see article id 4133860 CSP: Tax exemption process for APGC
  • Withholding tax credits, see article 4535497 Procedure: Partner: Self-service capability for CSP partners to upload WHT certificates
  • SLA credits, see article 4346704 CSP- Processing a credit request due to a service outage (SLA credit)
  • CPE credits, see article 4517737: Informational: Partner: CSP - Processing a CPE credit request.

3.1.  What to do if an incorrect credit is applied to an account

If you make a mistake applying a credit to an account, create a credit reversal, navigate to CMAT and select Transaction: verify that the line item to be reversed displays as a negative amount. If the amount to be reversed is positive, process as a standard offset. 

Note: Reversals can only be completed one Line item at a time and must be 100% reversed. Ex. A credit was submitted for $50.00, so the reversal must be entered for $50.00.

3.2. Steps for applying a line item credit

1. Is this an SLA credit?

If this is an SLA credit, first check the Message Center Alerts to see if there is an alert about the outage. If the alert states that automatic or proactive credits will be issued, do not request or apply a credit.

Please review article 4346704 CSP- Processing a credit request due to a service outage (SLA credit) for specific instructions on how to handle this type of credit.
2. Is this a cancellation credit?

If this is a cancellation credit, first check the Commerce Management Agent Tool (CMAT) to see if an automatic credit has been applied to the account since the cancellation. An automatic credit may affect the credit amount you plan to apply.

If the partner did not receive an invoice yet since the cancellation was processed, then ask the partner to wait until their next invoice and to create a ticket again if the credit is not automatically processed.

Note: Credits for test accounts and integration testing are not allowed, Partners should use the integrated sandbox account for testing purposes (see steps to create on https://docs.microsoft.com/en-us/partner-center/develop/set-up-api-access-in-partner-center#set-up-your-accounts)

Please review article 4023102 CSP: Microsoft Online Services cancellation policy and process for specific instructions on how to handle this type of credit.
3. Is this a CPE credit? Please review article 4517737 Informational: Partner: CSP - Processing a CPE credit request for specific instructions on how to handle this type of credit.
4. Is this a Tax credit?

Please review the following articles

  • VAT credits, please see article 4037109 Supporting Value-Added Tax and Tax Exemption for CSP (EMEA only)
  • Tax credits for US, Canada and Puerto Rico, see article 4132854 CSP: Processing a Sale Tax Exemption for Partners in the USA, Canada, and Puerto Rico
  • Tax credits for APGC, see article 4133860 CSP: Tax exemption process for APGC
  • Withholding tax credits, see article 4535497 Procedure: Partner: Self-service capability for CSP partners to upload WHT certificates
for specific instructions on how to handle this type of credit.

3. In CMAT, search for the customer’s subscription.

4. Once the customer’s subscription is expanded, click on the Transactions icon [Asset 4023187], then click on the link View all.

5. Determine the line item to which you will apply the credit.

6. Expand the line item. Can Refund must be “true” in order to use this line item. Click Refund Transaction.

[Asset 4023188]

7. Change the Refund Amount, if needed. The amount entered should be Pretax.

8. Select the credit reason from the Reason drop-down menu. 

[Asset 4023189]

9. Select the Refund to billing account.

[Asset 4023190]

This will apply the credit to the partner's next Microsoft invoice. Refund to billing account is the only Refund Option available for CSP partners (because their payment method is Invoice Pay).

10. In the Comments field, enter the following information:

Date
Your alias
Service Request #
Invoice #
CMAT
Subscription ID
Service Plan
Reason for credit (from the table: Credit reasons for common credit scenarios)

11. If the credit/refund amount is greater than $650 USD (or local currency equivalent), the Approval field will show Required. Click the arrow icon to go to the next page.

12. a. For approved refunds/credits ≤$650 USD, enter the Service Request Number and click the checkmark icon to apply the credit.

[Asset 4023191]

b. For refunds/credits >$650 USD up to $2.500 (Approval field = Required):

  1. Select the aliases of the credit approvers for your region from the Approver drop-down list. While still in the Approver drop-down list, scroll to the bottom of the list and click OK.
  2. If this is the only or final refund transaction for the SR, check the Send email to Approver(s) box, enter the Service Request Number, and click the checkmark icon to submit.
  3. If you have multiple refund transactions for the same Service Request (SR), submit the request but do not click Send email to Approver(s) until you are submitting the last request for the SR.
  4. Once the request is submitted with Send email to Approver(s) checked, an email will be sent to the approver and the agent’s email alias will be cc’d. (See Sample email sent to the approver and copied to the requester.)

[Asset 4023192]

13. You can view your requests by clicking the Requests hyperlink in the Title Bar of CMAT. The number to the left of Requests (“0” in this screenshot) indicates to approvers how many outstanding requests they have. It does not indicate how many requests you have submitted.

[Asset 4023194]

14. The default filters on the Requests page are “Assigned to me” and “Pending”. Change the values to “My requests” and “All”. Possible values for the status field are: All, Pending, Approved, Rejected, Cancelled, Expired.

[Asset 4023195]

[Asset 4023196]

15. If the request is approved, the credit will be applied automatically in CMAT. You will be able to see the Offset Transaction on the Transactions page.

[Asset 4023198]

3.5. Steps for applying a discount – ONLY for estimated discount value below $2,500.

These steps should be followed by escalation agents only and the only allowed scenarios are described in VKB id 4517737 –scenarios 4 Special/Exceptional Circumstances and 5. System Error, if the estimated discount value is below $2,500. No other scenario is allowed.

CMAT steps to view, apply, or remove subscription discounts.

Note

  • O365: All Syndication Partner and CSP discounts are processed at the partner tenant level. Discounts and comments can be viewed at end customer tenant level.

To view or apply a subscription discount:

  1. Go to the Subscriptions page.
  2. Find the appropriate subscription and click at the end of the row.
  3. In the Billing and Price section, click Discount.

[Asset 4056717]

     4. If the subscription has a discount applied, the following info is shown:

· Start Date: The date the discount began

· End Date: The date the discount ends

· SKU: The subscription SKU

· Amount: Amount or percentage of the discount (see Type)

· Type: The discount type

· Reason Code: The reason for the discount

[Asset 4056718]

5. To apply a new discount, click Apply Discount.

6. Click the Calendar. Choose a Start Date from the calendar on the left and an End Date from the calendar on the right, then click OK.
Note If you want the new discount to override the current discount, the date range of the new discount must overlap the date range of the previous discount.

7. Select the appropriate SKU from the dropdown.

8. Select a Discount Type:

· Amount off Unit Price: Amount to subtract from the current per seat price.

· Percentage off Unit Price: The percentage taken off of per seat price

· Flat Amount for Unit Price: New price that replaces the original per seat price.

· Flat Amount off Total Price: The amount taken off the total cost.

9. Type the discount amount or percentage amount.

10. Choose Reason Code “Other”, type an explanation in the Comments box, and then click Next. [Date – Agent Alias – SR# – Subscription ID – Offer Name – Discount Amount/Type – Discount Start/End Date – Explanation]

[Asset 4056719]

11. Confirm the change, add the SR#, and click Done () to apply the subscription discount.

[Asset 4056720]

To remove a subscription discount:

Complete the Steps 1-4 and take note of all existing discounts in Step 4. Then in Step 6, choose tomorrow as the Start Date (current date + 1 day) and for the End Date choose the Start Date + 1 day (or current date + 2 days). Complete the remaining steps and be sure to type 0 (zero) for discount amount or percentage in Step 9.

Notes

  • This operation is NOT available for O365 subscriptions that have a “discount based” promotional code.
  • Sequential discounts are applied to the same SKU so the discount start and end dates can’t overlap (if the dates overlap the new discount replaces the previous discount).

3.6. Steps for applying credits on multiple line items using multi refund

To enable this functionality apply the MultiRefundTransactions roleset to user. IMPORTANT: advocates can use the multi refund feature as long as the threshold of $2,500 per credit request is respected.

  • There isn’t any approval workflow for this.
  • Each lineitem has a threshold of USD 2,500 that the agent can process, however agents should follow the rule described at the beginning of the article and only process $2,500 credit per request (not per line item).
  • Works best in IE & chrome.
  • Loads only 100 lineitems at a time (this is due to the technical limitations on the system architecture).
  • Should launch the tool again after performing few multi line item refund operations (due to system limitations).
  1. When entering the Transaction tab to start the refund, select the Multi Refund button.[Asset 4535135]
  2. Like in one line refund process, fill the “Reason”, “Refund option” and “Comments” field (Mandatory) then select all line items that needs to be fully refunded.[Asset 4535136]
  3. In the verification page, make sure that all the transactions were selected properly and enter the Service Request Number.[Asset 4535137]
  4. Once the refund is proceeded, you will be sent back to the main Transaction tab and a header will appear with the details of the refund.[Asset 4535138]
