Procedure: Microsoft account verification

What is this article about?

This article describes the procedure for account verification and should be followed when you receive a call from an unauthenticated customer, where they need help with their Microsoft account.

1. Overview

Account verification is the method used to validate that you are speaking to the true owner of the Microsoft account. This occurs by sending a code to a customer’s phone or email address.

After locating the customer’s Microsoft account in your support tool, use the send code button to send a code to the customer.

  • If sending to an email, the customer must be able to receive emails at the email address listed in your support tool.
  • If sending to a phone, the customer must be able to receive text messages on the phone number listed in your support tool.

Notes:

  • Customers who don’t have access to the email or phone, but do know their password, can sign in at https://support.microsoft.com/verifyme with their password.
  • Click2Call/Click2Chat: Some customers may come in already verified via Click2Call or Click2Chat. These customers should be treated as already having passed verification.

Important reminder on personal data:

  • Never share or provide the customer with any personal data, masked email, phone information, or any information shown in your support tool. This information can only be used to confirm the information provided by the customer.
  • Always receive personal data from the customer and then verify what you see on the tool.
  • You can only share some information without the customer being verified. This information is provided to help the customer locate the correct account only when the customer provided a Microsoft account: You can share the status of the account as seen under the Issue and Description section or the last two digits of the phone number or email domain under Security Info. Do not share anything if the customer only shared a phone number, gamer tag, first/last name, or account found when searching by credit card.
  • If a customer provides just a Service Request (SR) number, you can’t provide any additional information regarding steps taken, or resolution status of the escalation, without customer passing account verification.

2. Procedure

2.1 Setting expectations

  • Always set the customer’s expectation that you’ll need to secure the account before continuing to assist them.
  • Explain to the customer that you need to verify their account to protect their personal information from phishing and social engineering. You can use the following messaging to set expectations:

"Before I move forward, I need to send you a code, which you will need to read back to me to verify your Microsoft account. I can send you the code with three options: website, email, or phone text message. If by phone, you may be charged for the text message, depending on your service plan. Which option is most convenient for you?"

If the customer worries about the account verification process or does not recall having done the process before, you can reassure them by explaining:

“We take your security very seriously. To protect you and the information on your account, we do that by sending a code to a place you’ve already registered within your account. This is like the way you confirm a password reset request.”

2.2 Email, phone, or web personal data verification

  1. Work with the customer to select the best method to verify them: Either receiving a code to their email or phone number or generating a code through the web sign-on method.
  • In case the customer chooses the email or web verification method, inform the customer that it will require them to be near a device/PC that can access the internet while still on the call.
  • In case the customer chooses the web verification method, also instruct the customer that they will need to sign in with their Microsoft account. The option to validate through a web sign-on is always available regardless of the information you have available in your support tool.
  1. Determine which email or phone number to send the code to.
  • You should first look at the redacted email or phone number to make sure it matches the Microsoft account you just searched in your support tool (comparing the first and last characters/digits).
  • If you still cannot determine the correct account, you can use the customer’s first name and state of residence to verify the appropriate account.

Note: You may want to send the code to all the emails and phone numbers associated with the account. However, prior to doing so, the customer must be asked if they would like to utilize this option and be proactively provided an explanation as to what the send all option entails – sending a code simultaneously to both their e-mail alias and phone via text.

  • A customer utilising this option may incur a cost for receiving an SMS message depending on the plan they are using.
     

Following this explanation, the customer must confirm their choice to use this option before you proceed.

  • If several results appear in your support tool (several email addresses, several phone numbers), though the information is masked/hidden, compare the first and last characters or digits (for example, 4XXXXXXXX0, exxxxxxx@mxxxxxx.com).
  • If the customer is not sure what Microsoft account they used, you can help them by asking questions like “Is there a different email you may have used or potentially did it have a different provider name such as hotmail.com or live.com?” or guiding them to use the user name hint flow.

Once the account has been verified, you may confirm that the information they provided matches that in the profile.

  • If a customer can pass account verification using a phone number as an account alias and has forgotten their Microsoft account, you can provide it at their request.

Note: When directing non-English speaking customers, inform them that the website is supported in 41 languages. Customers can select their language from a drop-down menu on the website.

2. Explain to the customer that they must sign in using the Microsoft account they are contacting us about.

Note: If you need to send another code to the customer, make sure you tell the customer that you’re sending another code since they haven’t received the previous one.

Messaging:
“I’m going to send you another code since you haven’t received the last one. My apologies for the delay. I just need to make sure you’re getting this code. We can also try verifying through a website as well.”
