What is this article about?

This article provides information on supporting customers who have enabled two-step verification (TSV) and will be prompted to enter a security code. This usually occurs after signing into the Microsoft account on any device not registered as a trusted device.

1. What is two-step verification?

Two-step verification adds an extra layer of security to an account by requiring a security code in addition to a password to sign into the account. This increase the difficult for hackers to take over an account since hackers would need to know the account password and have access to the customer’s security info, instead of just the password.

While a typical customer may find the added step of using security info and a password for sign in inconvenient, this feature is especially useful for customers with high-profile accounts (celebrities for example) and for technically savvy customers who value increased security with their accounts.

2. Common issues

2.1 I have enabled two-step verification and tried to recover account, but account recovery page (https://account.live.com/acsr) doesn’t work. What should I do?   

Explain to customers with two-step verification enabled that account recovery (ACSR) will not work. To regain access to a two-step verification enabled account, customers need to verify at least two pieces of security information on their account. If the customer does not have those, they cannot access their account. Note: It is very important for customers to maintain at least three pieces of security info so that they are still able to access to their account if they lose one. (See section 2.4 for more details.)

Return to top

2.2 Customer knows password but lost all security info

When prompted for a security code at sign in, the customer must select I don’t have these anymore and follow the instructions to replace their security info. After doing so, the customer will not be able to sign in again until the security info is replaced after a 30 days waiting period.

Return to top

2.3 Customer forgot password and no pieces of security info

In this case, the customer will not be able to recover their account. The Recover your account form does not work on two-step verification enabled accounts and you cannot sign into a two-step verification account without at least two pieces of information (PW+SEC INFO or 2 SEC INFO). If this happens, explain to the customer that they should create a new account with at least three good pieces of security info to prevent this from happening in the future.

4.2 Authenticator app

The Authenticator app is a smartphone app for Windows, iPhone, or Android phones that generates time sensitive security codes for Microsoft account or other accounts that support these security codes. This app allows the customer to bypass having a code sent to their security info by using those pre-generated codes presented in the app instead.

To configure the Authenticator App:

  1. Go to the Security basics page and sign in with your Microsoft account. Select More security options.
    [Asset 4130933]
  2. Scroll down to the security verification apps section and select Set up identity verification apps.
    [Asset 4130934]
  3. Select which type of smartphone you want to install the app on, then select Next.
  4. Follow the instructions to install and configure the app on your device. Select Get it now.[Asset 4130935]

  5. Select the Google Play or App Store link or enter your phone number to install the app on your device. This should take you to the appropriate store page for that device. (Sign in may be necessary).
    [Asset 4130936]

  6. Launch the app and pair it to your account by adding an account, then select the camera icon. Align the bar code on the instruction page in the viewfinder, and then select the save icon once the account info is shown.

  7. Once the app is configured you can use the security code presented in the app to sign into your account.

Note: The code will change every 60 seconds, but there is some leeway between changes.
