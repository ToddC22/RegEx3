This document provides troubleshooting information to help you respond to error messages that you may receive from customers when they deploy the Extended Security Updates (ESU) solution in their environment.  Refer to the ESU blog for end-to-end deployment instructions for ESU.

The update is not applicable to your computer.

 

[Asset 4541398]
The package (SSU or the ESU monthly rollup/security only/standalone) you are trying to install is not applicable to your Windows OS edition/architecture.
  1. Make sure that the package is meant for your OS edition and architecture.
  2. Reboot the system and try installing the package again.​
  3. If you still see the error message then please visit Windows Update troubleshooting under section “The update is not applicable to your computer” for a list of the most common reasons for this error.

 
Contact Microsoft Support and provide the CBS log file (C:\Windows\Logs\CBS\CBS.log).
 
Installer encountered an error: 0x80096010 The digital signature of the object did not verify.

 

[Asset 4541399]

 
The SHA-2 updates are missing on the machine.
Install SHA-2 updates. List of prerequisites and SHA-2 updates can be found here under the Installation prerequisites section.

Section 3. ESU key activation errors

This section is divided into 3 subsections based on the type of activation being performed: slmgr, VAMT, or phone.
 

Section 3a. slmgr activation errors

State of the user machine - All of the prerequisite updates for ESU mentioned here have been installed on the machine and the user is trying to deploy and activate ESU Year1/Year2/Year3/PA key has been successfully installed on the machine.
 
 Error
 Possible Cause
 Action to Take
If the issue is not resolved, gather this for further investigation
 
Product activation failed while trying to activate ESU product key.

 

[Asset 4541415]

 

[Asset 4541416]

 
This Windows OS edition/architecture is not eligible for ESU.  
Make sure your OS edition/arch is in the list of supported ESU editions/architectures. See here for supported ESU editions/architectures.
Contact Microsoft Support

 

0xC004C020 the activation server reported that the Multiple Activation Key has exceeded its limit

 

 
The MAK has exceeded its activation limit. By design, MAKs can be activated a limited number of times.
If you require additional activations, contact the Microsoft Licensing Activation Centers.
 
See internal KB 4538322 - How to Add Additional Activations to ESU MAK Key

 

 

Product not found while trying to activate ESU key

 
The activation ID used in the activation command is not correct.
To get the activation ID for the ESU program -
  1. Open command prompt in elevated mode​
  2. Run cscript /h:cscript​
  3. Run slmgr /dlv (If WS 2008 SP2, slmgr /dlv all)​
  4. Copy the Activation ID for the ESU key​.
  5. Run slmgr /ato <Activation ID> from step 4.

 

 

 

0xC004F025 access denied: the requested action requires elevated privileges.

 

[Asset 4541417]

 
The command prompt window is opened in non-elevated mode.
Open command prompt in elevated mode –
Start-> Command Prompt->right click and select run as administrator

 

 

Error: 0x80072EE7
 

[Asset 4541418]

 
The machine is not able to communicate with the AVS server to activate the ESU key.
Make sure that the machine is connected to internet or has the Activation URLs white listed and try again. VAMT Proxy activation or Phone activation could alternately be used for machines not directly connected to the internet. Refer to the blog.
 
See Section 3b for the VAMT Proxy Activation URLs.
 

 
 
0x80072EE2 The operation timed out

 

 
 

 
Machine is not able to connect to the Microsoft Activation service, possibly disconnected from internet or having issues with internet connectivity.
Make sure that the machine is connected to internet or has the Activation URLs white listed and try again. VAMT Proxy activation or Phone activation could alternately be used for machines not directly connected to the internet. Refer to the blog.
 
See Section 3b for the VAMT Proxy Activation URLs.

 
 
Activation command succeeds but the ESU key is still in Unlicensed

 
ESU Activation ID is not passed to the slmgr /ato command.
 Activate ESU key using: Slmgr /ato <ESU Activation Id>. Please refer to the blog.

 
 
License Status: Unlicensed
Machine state – All pre-requisites installed and ESU key is activated. Installed the standalone package after which the ESU keys license status shows Unlicensed.
[Asset 4541419]
 

 
Activate the installed ESU key again using slmgr/phone activation/ VAMT(Online or Offline) activation.

Unable to verify product key
The specified product key is invalid, or is unsupported by this version of VAMT. An update to support additional products may be available online.
 
[Asset 4541407]
 
 
 
pkconfig files needs to be replaced.
 
 
  1. Download the pkconfig files from here and replace the files at the following location - C:\Program Files (x86)\Windows Kits\10\Assessment and Deployment Kit\VAMT3\pkconfig\ .
  2. Restart VAMT and try to add the key again.
 
 
 
 
 
 
Unable to connect to the WMI service on the remote machine while activating the remote machine using VAMT online/proxy activation.
 
[Asset 4541408]
 
 
 
  1. The WMI (Windows Management Instrumentation) service is not enabled on the client machine.
  2. Windows firewall is not configured properly on the remote machine to allow VAMT access.
 
 
  1. To enable WMI service, click Start -> Services -> Right click Windows Management Instrumentation and select restart​.
  2. To configure windows firewall on client and host computer , follow instructions here.​
  3. Please visit install and configure VAMT for detailed information on how to install the VAMT tool and configure client computers .
 
 
 
 
 
 Error: Access is denied.
 
[Asset 4541409]
 
 
The user doesn’t have permissions to access the client computer.
 
 
  1. Make sure that the client has access/permissions to the client computer if it is domain joined.
  2. If client computer is domain joined, make sure that the user is a part of list of user accounts on the client computer. Steps to create a new user account – Start ->Control Panel -> User Accounts -> User Accounts and Family Safety ->User Accounts -> Manage Accounts->Create a new account-> fill in the account name and select Administrator and put in the domain name if required.

Error: 0xC004F04D The Software Licensing Service determined that the Installation ID (IID) or the Confirmation ID (CID) is invalid. 
 
[Asset 4541410]
The Confirmation ID is incorrect.
  1. Call the Microsoft Licensing Activation Centers again and they will walk you through the steps to get the confirmation ID.
  2. Run slmgr /atp <Confirmation ID> <ESU Activation ID> to activate the ESU SKU using the confirmation ID obtained in step 1.

Section 4. ESU security update installation errors

State of the user machine - All of the prerequisite updates for ESU mentioned here have been installed on the machine and the ESU Year1/Year2/Year3/PA key has been successfully deployed and activated on the machine.  The user is trying to install the February 2020 or later extended security update on the machine.
 
 Error
 Possible Cause
 Action to Take
If the issue is not resolved, gather this for further investigation 

 

The Windows Module Installer must be updated before you can install the package. Unable to install the security update on my machine after installing ESU product key.
 
[Asset 4541402]
 
The SSU (Servicing Stack Update with AI changes) package is not installed on the computer.
Make sure that the SSU package is installed on the machine. Go to Start -> Control Panel -> Programs -> Program and Features ->View Installed updates. If not, then install the SSU package and restart the machine after successfully installing it.
Contact Microsoft Support and provide the CBS log file (C:\Windows\Logs\CBS\CBS.log).
 
Some updates were not installed while trying to install the security update.
 
[Asset 4541403]
ESU Specific: -
  1. A valid ESU key is not installed on the machine.​
  2. For Desktop Client/server, the ESU key is installed but not activated on the machine. Link to Possible causes and error codes.​
  3. For embedded systems (FES), potential causes can be found in the row below.​
  4. The Windows OS is not in the list of ESU supported editions/ architectures. See here for the list of supported editions/architectures.
  1. For Desktop Client/Server machines, check if a valid ESU key is installed and activated on the machine - 
    1. Open command prompt in elevated mode​
    2. Run slmgr /dlv or slmgr /dlv <Activation ID>​
    3. Verify ESU add-on key is licensed​
    4.  [Asset 4541404]
    5.  NOTE: For Windows Server 2008: Running "slmgr /dlv" doesn't show the licensing status of add-on SKU. Run "slmgr /dlv <Activation ID>" or ”slmgr /dlv all” to view the licensing status of add-on SKU.​
  2. For non-embedded machines, install and activate the ESU key if not done already via -
    - VAMT Online/Proxy Activation​
    - Phone Activation​
    - Using command prompt (slmgr /ato)​
    a. Open command prompt in    elevated mode​
    b. Run slmgr /ipk <ESU key> and wait for the success message.​
    c. Run slmgr /ato <Activation ID>​
  3. For embedded machines, actions to be taken can be found in the row below.
  4. For more potential error codes, refer to the Windows Update error reference.
 
Contact Microsoft Support and provide the CBS log file (C:\Windows\Logs\CBS\CBS.log).
For embedded machines (FES) (continued from above)
Some updates may not install and the CBS log may contain entries like these:
 
Condition 1:
 ESU: NO ESU KEY FOUND
 ESU: not eligible:HRESULT_FROM_WIN32(1633)
[Asset 4541426]
 
Condition 2:
 ESU: Windows is not activated
 ESU: not eligible:HRESULT_FROM_WIN32(1633)
[Asset 4541427]
 
Condition 3:
 ESU: Windows is not activated
 ESU: not eligible:HRESULT_FROM_WIN32(1633)

 

[Asset 4541428]

 

 
 
 
 
 
Condition 1: If Windows key is IN the range (of keys for Embedded editions) and activated, then ESU key is not installed on the computer .
 
 
 
Condition 2: If Windows key is IN the range (of keys for Embedded editions) but Windows is NOT activated and ESU key is installed (but not activated) on the computer.
 
 
 
Condition 3: If Windows key is NOT in the range (of keys for Embedded editions) and ESU key is installed (but not activated) on the computer.
 
For Condition 1: Install a valid FES ESU key on the computer and attempt to install the ESU package
 
 
 
 
 
For Condition 2: Activate Windows key OR activate ESU key and attempt to install the ESU package. Either of these two solutions will work in this case.
 
 
 
 
For Condition 3: Activate ESU key and try the ESU package.

 
Error: 80070643 – prep-check KB installation fails with this error code.
  • KB 4528081 for Windows Server 2008 SP2
  • KB 4528069 for Windows 7 / Windows Server 2008R2
The CBS log may contain entries like these:
 ESU: Product = 36 (0x00000024).
 ESU: Is IMDS check needed:FALSE​
 ESU: Pre IMDS checks failed, Not Eligible:HRESULT_FROM_WIN32(1605)
 
1605 = ERROR_UNKNOWN_PRODUCT
 

 

  1. The OS edition is not supported by the prep-check KB. Non supported editions are all *V and *Core editions.
  2. Latest Servicing Stack Update (Feb 11, 2020 or later) and Monthly Rollup (Feb 11, 2020 or later) is not installed on the machine.

 
Install the latest Servicing Stack Update (i.e. Feb 11, 2020 or later) and monthly rollup (Feb 11, 2020 or later) and try again.

Failed install update on Azure VM with 12029 Error(ERROR_WINHTTP_CANNOT_CONNECT)

2020-08-12 22:23:03, Info CSI 00000001 ESU: Product = 7.
2020-08-12 22:23:03, Info CSI 00000002 ESU: Is IMDS check needed:TRUE
2020-08-12 22:23:03, Info CSI 00000003 ESU: Checking IMDS
2020-08-12 22:23:03, Info CSI 00000004 Azure
2020-08-12 22:23:03, Info CSI 00000005 ESU: check failed HRESULT_FROM_WIN32(12029).

Case 1. Azure endpoint is blocked. It needs to be allowed for IMDS check work

Case 2. There can be issue with network configuration, need to update SSU

Case 1. 169.254.169.254 is virtual IP of host node.

https://docs.microsoft.com/en-us/azure/virtual-network/security-overview

https://docs.microsoft.com/en-us/azure/virtual-machines/windows/instance-metadata-service

It should be allowed to have GET operation from 169.254.169.254 ip address. Otherwise it’ll fail.

Case 2. Need to install SSU later than 2020 10B. depending on Customer configuration, network interface may need more time to show up. 10B added retry logic to overcome this issue.

  

Failed install update on Azure VM with certificate chain error on Azure

2020-05-18 09:34:48, Info CSI 00000001 ESU: Product = 10 (0x0000000a).
2020-05-18 09:34:48, Info CSI 00000002 ESU: Is IMDS check needed:TRUE
2020-05-18 09:34:48, Info CSI 00000003 ESU: Checking IMDS
2020-05-18 09:34:48, Info CSI 00000004 Azure
2020-05-18 09:34:48, Info CSI 00000005 The chain does not seem valid
2020-05-18 09:34:48, Info CSI 00000006 ESU: check failed CRYPT_E_NOT_FOUND.

Customer may not allow access to certificate renewal site.

The customer will need to make sure intermediate certificates can be installed on the VM by allowing network connection to the CRL sites shown in following page.

https://docs.microsoft.com/en-us/archive/blogs/kv/azure-tls-certificates-changes

These certificates are required for using various Azure Services. If these are not installed, then other services also may fail.

If they don’t want to allow network connection to the CRL sites, they may download and install certificates from following page.

https://www.microsoft.com/pki/mscorp/cps/default.htm

If the intermediate certificates are present, then please check next the root certificate is available:

Connectivity issues may occur when the Baltimore CyberTrust Root certificate is not installed on client computers that use Microsoft Intune

https://support.microsoft.com/en-us/help/2831435/connectivity-issues-may-occur-when-the-baltimore-cybertrust-root-certi
