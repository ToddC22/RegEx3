ASMS – Azure Internal Registration System (AIRS) Accounts

Submit a ModTrack request to update this TSG. 

Overview

This article provides guidance on how ASMS can assist Microsoft employees in setting up their Azure Internal Registration System (AIRS) accounts, and common workflows associated with AIRS accounts.

Policy

ASMS can help internal customers with their AIRS accounts in several ways:

  • ASMS can help with issues related to the Azure customer user interface.  

  • ASMS handles usage validation cases (not the AIRS team).

    • Engage your technical advisor (TA) or partner technical advisor (PTA) prior to engaging the concerned team to investigate.

    • Create a task to the TA or PTA using the below template:
      Task Title: Internal 15P Subscription Usage Report

      Account Owner Live ID:

      Subscription GUID/s:

      Date Range: Write out the date (i.e March 3, 2016 to May 5, 2016) rather than the short date (i.e. 3/3/2016 to 5/5/2016)

      Service:

  • ASMS should transfer cases with pricing related issues to the AIRS team.

  • ASMS should engage the appropriate team to conduct a technical investigation to determine if AIRS needs to process a credit or refund. 

  • The AIRS team processes refund and credits for Internal Subscriptions due to technical or systemic issues only.

  • Azure Security does not recommend the use of domain accounts as account owners.  

    • ASMS should not recommend this scenario to customers. 

  • All Internal Full-Time Equivalent (FTE) employees are eligible for the 15P offer. 

    • Note: No details about the 15P offer are available on our public-facing website. 

  • On the billing side, AIRS downloads usage information and then applies Microsoft’s internal service rates. AIRS can apply both internal rates and external rates to this usage. Because of the way AIRS works, ASMS is unable to generate any usage files. 

  • There is a common misconception around usage, as the customer may feel that "AIRS is capturing incorrect usage as I don't have that service running," but AIRS is only applying rates to usage information provided by Azure, so further troubleshooting needs to be done to correctly identify and deprovision running services. 

  • The AIRS team provides contact information and a billing FAQ on the AIRS SharePoint site.

  • AIRS entitlements usually fall under one of the following offers:  

    • Azure for Students (170P) 

    • Azure CSP Sponsorship (36P) 

    • EA sponsorship (136P) 

    • Microsoft Azure Sponsorship (36P) 

    • Microsoft Azure Sponsorship 2 (143P) 

    • Microsoft Internal Consumption (15P) 

  • Without an approved AIRS registration, internal customers will not see any AIRS offers and will be blocked from activating their account.

Searching for an AIRS Subscription

  1. Navigate to the AIRS Registration portal, and sign in.

  2. Click on List Registration (BIOS) in the left-hand navigation pane.

  3. Set the Status drop-down menu to All.

  4. Use the Account Owner Live ID field to search for the Account Administrator’s email address. (See the following image.)

  1. To view and confirm the AIRS/Sponsorship subscription under the registration number/Account owner Live ID, visit the Custom Azure Billing System (CABS).

Migrating Internal Account Subscriptions

For account ownership transfers or data migrations for internal subscriptions, follow the process outlined in the TSGs for Subscription Ownership Transfer TSG (3040383), Simple Migration (2856526), and Forced Migration (2892044).

Warning: Failure to follow these instructions can cause a subscription to be moved to an account that won't properly bill an internal profit center code.

  • Subscriptions should only be transferred to registrations that have their status set as Approved. (StatusId=30).

  • The AIRS team can update the profit center of a subscription. Please provide the registration number and necessary details.

  • If the customer is requesting a billable registration to be moved to a non-billable registration, this request should be denied.

Setting Up AIRS

All cases from internal customers wanting to set up an AIRS account should be sent to the Azure Internal Offer (MSFT only) queue. They will work with your internal customer to get them set up. You may also direct your internal customers to the following Quick Start guide provided by the AIRS team. If the customers main objective is a subscription ownership transfer or any other ASMS related topic, please cut a collaboration task to the AIRS team and do not sent over the entire case.

If a customer has a non-Microsoft account and wants to receive Microsoft employee rates (or they are receiving charges when they think they should not be), they should make sure that they have registered an account at the AIRS Registration portal. You can work with the AIRS team to be sure the registration is set up properly, and that any necessary transfers are communicated to the customer. They can also ensure that any refunds or discounts that the customer may be eligible to receive are applied.

In general, customers will not be able to receive internal pricing and billing for subscriptions registered to a non-Microsoft ID.

Moving AIRS (Internal Subscriptions) to a Non-Microsoft Tenant
To create an AIRS (Internal subscription) under a different tenant, or move a subscription to a different tenant,  ask the customer to perform the steps below:

Note: Microsoft owned/internal tenants are: Microsoft.com, Ame.gbl, Gme.gbl

  1. Create a new registration with the destination account email address.

  2. While creating a registration, select the Account Type as External Demo Account.

    1. The prompt will ask for a start and end date; set a reasonable timeframe.

    2. The prompt will ask for a sponsorship cap (amount); estimate the expense and enter an appropriate value.

Note the following:

  • The internal customer can update the end date and cap amount in AIRS at any time.

  •  If the account hits the end date or the cap, the subscription will be disabled immediately.

  1. Once the registration is approved, go to the Azure website and activate the "Microsoft Sponsorship 2" subscription.

    • Note: If the customer needs additinal assistance, engage the ACE-IPO team (formally Microsoft Internal and Sponsorship Team (MIST)) via a task. See TSG 2927087.

Customer Cannot Move the AIRS Tenant
Sometimes, the customer may be either unable to perform the steps above, or they prefer to not create a demo/sponsorship subscription.

In these cases, do the following:

  1. Send a collaboration task to the AIRS (MSaaS Azure Internal Offer) team for approval.

    • ​​Justifiy why they are unable to follow the steps above.

  2. When the collaboration task is approved, work with the customer to determine if there is an Express Route virtual network is present within their subscription. 

  • The Express Route virtual network extends the on-premises CORPNET network into Azure.
  • The Express Route network has a specific naming convention “NAME-TYPE-LOCATION-VNET-NUMBER". 
  • There are several types of express routes (DB, InetApp, Lab and PvtApp) with trailing numbers of 1 to 5 digits. For example, “Contoso-LAB-WUS2-VNET-1234”.

Customer Confirmation
The customer can confirm the information by following the steps below:

  1. From the Azure Portal, select the subscription needing the tenant move. 
  2. Select the Resource and sort the Type backwards (Z-A). Check if there are Types like Virtual network or ERNetwork that match the naming convention specfied above.

  1. If a vnet is present, the requestor has two options, as Express Route networks are not permitted outside the microsoft.onmicrosoft.com:
  • Delete all resources behind the vnet, then delete the vnet, then move.

Troubleshooting Sign-in Issues

Be sure the customer is using the correct links to sign into the Azure system. While we have used multiple portals in the past, internal customers should now be using the main Azure portal (https://portal.azure.com) to sign in.

Use the existing sign-in troubleshooting steps to work through any sign-in issues reported by AIRS customers.

Internal Billing Refund Query

Refund requests must be $5,000 or more for the AIRS team to approve, as amounts below $5,000 will not be considered.Initial troubleshooting will be initiated by ASMS, and once we confirm that it is a valid refund request we can then move the entire case to AIRS Team. 

An example of a valid request (5K and above) which can be sent directly to AIRS is when the SR creator has already attached the email thread which has the approval to charge against the PC Code from the responsible member. Declined requests will be handled by ASMS.

Internal Customer Requests to Upgrade Support Level

All Internal customers have Developer level Technical Support for free.

If these customers want to upgrade to a higher tier of Technical Support, they will need to do so in AIRS. Rates for Internal customers are the same as they are for the general public.
