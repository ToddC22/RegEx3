What is this article about?

For BOS advocates, this article covers how to process Uploads (Bulk Campaign Management) requests. The Back Office Support (BOS) team executes these requests as part of Advertiser Enablement Services (AES) program, using the Editor tool or Microsoft Advertising UI to make bulk changes or assignments to an advertiser’s Bing Ads account quickly and easily.

2. Definitions

  • Import - Loading the csv Excel file of ads into the tool.
  • Upload - Loading the csv Excel file to a live server. The ads will go live once the campaign is in a live status and not paused. Uploads to active status should be explicitly listed in the request, otherwise uploads (ad groups and keywords) should always be paused.

3. Best practices and policies

The following section provides best practices and reminders for BOS advocates when processing an Uploads request.

3.1 Uploads status

  • When uploading new entities that have their own statuses (for example, campaigns and ad groups), always pause the new entities unless otherwise stated by the requestor. This is to give the requestor/advertiser the opportunity to review the changes prior to activation.
  • Ad Extensions and Targeting Settings will follow their parent status thus, there is no need to clarify the preferred status before posting unless otherwise stated by the requestor.

Note: If the requestor remains unresponsive even after following the Total Case Ownership (TCO) process, retain the current status of the affected campaigns/ad groups and send your final email including a notification that the status of the parent campaigns/ad groups remain unchanged.

Return to top

3.2 Clarification

  • Clarify to the requestor if the required information in the import file has not been provided.
  • Clarify if the requestor’s confirmation is required before making changes, such as ad group bid being set to default due to missing bid amount. It would be best to recommend the minimum search bid for quick resolution.
  • Do not clarify for issues/errors that can be resolved without the requestor’s assistance or confirmation (for example, duplicate items).
  • If there are missing required details that require clarification to the requestor (for example, the import will not push through without these details (for example, campaign budget), the best practice is to ask for any other missing details as well. This is to enable us to set the correct settings during the initial upload, and minimize back and forth communication with the requestor, extending the final turnaround time. For example, if the uploads request requires updates to the budget, budget type and daily budget options for a campaign, and the budget information was not provided, the advocate should verify with the requestor what the preferred budget option types are. Otherwise default settings will be set.
  • If pushing back is necessary, send an email to the requestor on how to resolve the error, if in scope. For a list of in-scope items, refer to the Communicating with AES section of see KB 2023008 AES: Process: Microsoft Advertising: AES Requests.

3.3 Uploads trials

To check the possible outcome of an Uploads request, you should replicate the advertiser’s account using a test account and do a test run of the uploads. This will enable you to anticipate possible issues and/or errors before making changes to the advertiser’s actual account. Doing trials in a test account also allows you to investigate an issue and find resolutions without modifying an advertiser’s live account.

Uploads test run steps:

  1. Download the advertiser’s account in Editor.
  2. Export the whole account.
  3. Access your test account in Editor.
    • Delete all the contents of your test account if it has existing data. The objective is to replicate the advertiser’s account as accurately as possible, so there should be no other data in the test account.
  4. Import the downloaded advertiser’s account data into your test account.
    • If you encounter issues, errors, or skipped items but these items are not involved in the request or will not affect the entity types the requestor asked you to upload, you may skip resolving the issues.
    • Ensure everything that will affect the entities in the request gets uploaded into the test account successfully to ensure accurate results.
  5. Proceed with processing the uploads request using your test account.
  6. Assess the results.
  • Check all requested changes will be posted successfully.
  • Identify the cause of skipped items, errors, and warnings (if applicable).
  • Resolve all issues that do not require additional information from the requestor.
  • Take note of all issues encountered and if additional information is needed from the requestor.
  1. If there are errors or issues, process the uploads request using the advertiser’s account. This is to check to see if you will receive the same issues/errors when the changes are applied to the advertiser’s account, as there are some factors that may alter the outcome of an upload, such as entity IDs.
  • Do not post changes if the resolution to the issues/errors requires confirmation or additional information from the requestor.
  1. Resolve all errors that do not require the requestor’s assistance.
  2. If there are no errors or any issues, proceed with completing the uploads request following all uploads process guidelines.

3.4 Backups

Make sure to always export the whole account to generate a backup file. This may be used if the requestor/advertiser wishes to restore the account to its previous state. This also serves as a reference when comparing the original structure and contents of the account with the imported/changed items.

The best practice is to save a final export of the account that contains your completed work. While a change history report may be pulled up to verify the changes made, an export of the account provides more details.

5. Uploads Process - Import from a file

The Import from a File function allows you to import an Excel sheet into Editor to upload new campaigns and other items to a Microsoft Advertising account. This is also used to bulk-edit existing items and/or settings.

Note: If you have the bulk sheet open on your desktop, close it. You cannot import a spreadsheet that is already open.

1. Before importing the file, you must first Gather Requirements. To do this, see KB  4493123 AES: Process: Microsoft Advertising: Gathering Requirements for a change request and submit for Requestor Review.

2. In the Primary Navigation ribbon, select Import > Import from a file.

  • Note: Make sure that the import file contains only one sheet, which will prevent Editor from mixing up data if there are multiple sheets in the import file. If the import file has more than one sheet, move the other sheets to a different book/file first. Save a copy of the original file as a backup.

3. Browse to the location of the import/upload file, then select Next.

4. Check if each column of your data has been mapped appropriately. Select the correct data field in the drop-down menu for columns that haven’t been mapped properly.

  1. Some columns with no corresponding heading, such as ParentStatus, will be tagged as Not Importing by default.
  2. Scroll to the right to check the rest of the columns.
  3. Select Next once everything has been properly set.

Notes:

  • If you don’t want to import a column, you can choose Not Importing from the column heading drop-down menu.
  • If the file uses Microsoft Advertising Headers, specific headers do not need to be called out specifically. If the file is from Google Ads, due to the discrepancies between Google and Microsoft Advertising column headings, the headers that do not have an exact match with Microsoft Advertising Headers will be recognized as Not Importing (for example, Criterion Type, Headline, and Networks).

5. Select the import options. You may modify the default settings or select the default values for new account data by selecting the link Change your default settings.

  1. In the Defaults tab, choose all the correct settings. Select OK.
  2. After choosing the appropriate settings, select Next.

[Asset 4497096]

Note: Make sure to set the correct Time zone and Language. While time zone can be edited at the account level, it cannot be edited at the campaign level unless deleting it then, re-upload.

6. Check imported items, resolve any errors (if necessary), and upload your changes.

  • For how to do these processes, see section 7.

7. Submit the output file to the requestor to do a requestor review. To do this, see KB  4493123 AES: Process: Microsoft Advertising: Gathering Requirements for a change request and submit for Requestor Review.

In Editor, the Make multiple changes wizard is an easy way to make multiple changes, if the changes are all at one level of the hierarchy (same entity type). For example, if an advertiser sends you a bulk sheet with only keyword changes, the Make multiple changes wizard would be an efficient way to handle the task.

6.1 Adding or updating data via the Make Multiple Changes Wizard

The following instructions describe how to add or update multiple items of the same entity type using the Make Multiple Changes functionality.

  1. Before adding or updating data, you must first Gather Requirements. To do this, see KB  4493123 AES: Process: Microsoft Advertising: Gathering Requirements for a change request and submit for Requestor Review.
  2. Navigate to the desired entity tab in the browser pane.
  3. Select Make multiple changes in the Action bar.

[Asset 4497097]

  1. Select Add/update multiple <entity type>.
  2. In the Make multiple changes wizard, check the column headings for the specific entity type selected.  

[Asset 4497098]

  • For a list of required data for each entity type, see Required Columns and Details.
  • Make sure to maximize the Make multiple changes window to view all column headings, as some headings to the right may not be visible when the window is in its default size.

6. Access the Excel file containing your data and check if all required columns are complete and all data is correct.

7. Press CTRL + A to highlight all data.

8. Press CTRL + C to copy all data.

9. Go to the blank field in the Make multiple changes wizard in Editor and paste all data by pressing CTRL + V.

  • If your data doesn’t have column headings, you may insert column headings directly into the Make multiple changes wizard by clicking Insert headings before you paste the data from the Excel file.
  • When you use the Insert headings function, make sure the columns of data you’re pasting match the column headings inserted.

10. Select Next. You will be presented with a summary of the imported changes.

11. Check imported items, resolve any errors (if necessary), and upload your changes.

  • For how to do these processes, see section 7.

12. Submit the output file to the requestor to do a requestor review. To do this, see KB  4493123 AES: Process: Microsoft Advertising: Gathering Requirements for a change request and submit for Requestor Review.

7.2 Resolving errors

  1. Check to see if the import contains errors/warnings.
    • Errors are denoted by the error icon (red circle with vertical line): [Asset 4497105]
    • Warnings are identified by the warning icon (yellow triangle with exclamation point):  [Asset 4497107]
    • You will know if your import contains errors or warnings when you see the corresponding icon in the manager pane. These icons will appear for all affected levels of the hierarchy.
    • An error indicator on the ad group level means there could be an error at the ad group level, keyword, or ad level.
  2. Troubleshoot any initial errors. You may use to the following file as a reference for known errors and their corresponding resolutions: Known Errors and Resolutions.
    • For more information, see section 11.
  3. If there are no errors or skipped items, proceed with verifying that all items in the bulk uploads file have been imported into Editor.
    • Filter the import file by entity type and count the number of items per entity type.
    • Countercheck if the number of items in the import file tally with the number of imported items.

7.3 Uploading changes

  1. Once the initial troubleshooting steps have been completed, or if there are no skipped items or errors, select the Post Selected Campaigns button.
  2. Editor will display a window detailing the summary of changes per entity type. This also counts the number of items that will be successfully posted and won’t be posted due to further issues.
  3. You will have the option to post changes either to All Campaigns or Selected Campaigns. Select All campaigns in the Summary of changes window to post all changes you made.
  4. Select Post.
  5. Check the Microsoft Advertising UI if the changes are reflecting in the customer’s account.

9. Google Ads import

If the advertiser is already using Google Ads, their ad campaigns may be imported directly into Microsoft Advertising using an import file. Not all information will be imported, but that doesn't mean that it's not supported in Microsoft Advertising.

A Microsoft Office Excel Worksheet (.xls, .xlsx) file, Comma-delimited (.csv) file, or Extensible Markup Language (.xml) file may be used to import data into Microsoft Advertising s. Available imports from Google Ads to Microsoft Advertising include the following but not limited to:

  • Campaigns (Search, Dynamic & Shopping)
  • Ad Groups
  • Keywords
  • Negative keywords. Note: Broad Negative keywords will be uploaded as ‘Phrase’. Microsoft Advertising automatically convert it from Broad to Phrase match type.
  • Expanded Text Ads
  • Ad Extensions except Image Extensions
  • Location targets such as:
    • Country/Region
    • State/Province
    • City

For the total number of items that can be imported from Google Ads, see Microsoft Advertising help article: What gets imported.

Note: Some data is not included, such as radius targeting or zip code targeting information. Be aware that you may not see a warning about the data that is not imported. After the import, be sure to review the campaigns and ensure that targeting, bids, and all other data is configured as you expect and add the missing information back to the campaigns within Microsoft Advertising.

9.1 Requirements and best practices for Google import

  • The import file must be an Excel or CSV (comma-separated values) file.
  • Each campaign or ad group is limited to 10,000 of combined location and negative location. For example, you can set a maximum of 10,000 city location criteria or 1,000 state location criteria and 9,000 city location criteria.
  • Always use the Import from Google Ads option when importing Google Ads files.
  • Map any invalid locations using the latest Microsoft Advertising Geographical Location Codes. This will reduce the number of invalid locations after the import. Make sure to have the requestor’s approval before posting them.
  • The data you import must not exceed the system limits. See the below link for full details.  
    • Note: This is also the same with Microsoft Advertising Import. Refer to Microsoft Advertising Hierarchy and Limits for the complete list.

10. Determining net-new data

10.1 De-duping data in the Import file

If the advertiser already has existing data in the account and requests to upload only new data from a large import file, follow the steps below to de-dupe the import file against existing data in the account:

  1. Export the whole account from Editor.
  2. Open the import file that the requestor provided.
  3. In the import file, add a column to the right of the last column, and perform a VLOOKUP comparing the campaign names in the import file with the campaign names in the exported file. See KB 4040847 Tool User Guide: Microsoft Advertising: Excel Features and Functions on how to perform VLOOKUP.
    • All entities that return as #N/A are considered as net-new and will be uploaded into the account.
    • All those that return a distinct value based off the VLOOKUP process are considered as existing campaigns.
  4. Determine which entities under the existing campaigns are already in the account and which entities are net-new:
    • Start with one entity type under the existing campaigns. Concatenate all applicable fields for that entity type both in the import file and in the exported file.
    • In the import file, perform a VLOOKUP comparing the concatenated column values in the import file with the exported file, to find out which of these entities are net new. See below for steps.
  5. Repeat these steps for each entity type under the existing campaigns.
  6. Once done for all types of entities, send the file to the requestor for review.

10.2 Determining net-new data in Editor

If the advertiser already has existing data in the account and requests to upload only new data from a large import file, follow the steps below to determine new data using Editor:

  1. Open the import file that the requestor provided and save it.
  2. Pull up the account in Editor.
  3. Select Import from a file and browse for the import file.
    • Note: Lookup value has a maximum character limit of 255. Make sure to check any #VALUE error, aside from #NA value.
  4. Go through the Import Wizard window and look out for the data that will be returned on the New and Skipped columns.
  5. Export all entities tagged as new by Editor to an Excel file.
  6. Determine what caused the skipped items if those are net-new entities.
  7. Send all net-new entities to the requestor for review.

11. Troubleshooting errors

  1. Navigate through the tabs in the browser pane with error/warning icons (from keywords through to campaigns). The error/warning icons will indicate which of the tabs need to be worked on.  [Asset 4497115]

Note: For large accounts with many line items, you may use the Advanced Search feature below the Primary Navigation ribbon to quickly search for an error notification specific to a selected entity type and filter for all items with that error notification. Use Advanced Search for filtering specific error notifications:

  • Under Add search criterion, choose the level of hierarchy or entity type you wish to search in (for example, campaigns, ads, keywords, ad groups) in the drop-down menu.
  • In the next drop-down menu, select Notification Message.
  • In the next drop-down menu, choose the specific error alert you wish to filter.
  1. Review the errors present in Editor and resolve the errors that can be solved without the need for clarification to the requestor (for example, duplicate items).
  2. If there are errors that require the requestor’s assistance, before posting changes, do either of the following:
    • Ask via IM how to resolve the error, if in scope. For a list of in-scope items, see the Communicating with AES section of the KB 2023008 AES: Process: Microsoft Advertising: AES Requests.
    • Send an email to the requestor for errors that are not in-scope to resolve via IM.
  3. Examples of errors that require assistance from the requestor include:
    • Campaign budge
    • Missing entity details, such URLs
  4. Refer to the following file for a list of known errors and their corresponding resolutions: Known Errors and Resolution.
    • Note: Clarify to the requestor only if the necessary information is not in the import file, instructions, and details they provided.
  5. When the requestor responds with the needed information, proceed with resolving errors using the details they provided.
  6. Post changes.
  7. If there are still errors after the changes have been posted, copy all remaining errors that cannot be resolved from our end (for example, Editorial errors). Errors may be downloaded into Excel per entity type.

1. Highlight the account name in the browser pane.

2. Choose Errors from the View drop-down menu on the upper right-hand portion of Editor.

[Asset 4497116]

3. In the browser pane, select the entity tab with errors.

4. In the manager pane, press Ctrl + A to highlight all items.

5. Select Export to CSV file.

6. Save the downloaded Excel file.

7. Repeat steps c-f for all entity tabs showing errors.

9. Copy and paste all errors into the Error Report tab of the Output File and send to the requestor.

12. FAQ

Q: Why are Sitelink Extensions being skipped, when the skipped file is downloaded, there are no details of the skipped sitelinks?

A: When the same unassociated Sitelink, Location, and Call Extensions are imported again, the download summary page will display these items as Skipped because they are already downloaded in the Editor's local database. However, when the skipped files from the Import Completed summary page are downloaded, these items will not be shown. See here for more information about importing unassociated Sitelink, Location, and Call Extensions.

Q: What if the whole account cannot be exported to CSV file?

A: Split the download by campaigns or ad groups depending on the size of the campaigns in the account.

Q: A requestor would like assistance in re-uploading standard text ads due to various reasons including error in language selected for campaign/ad group.

A: Inform the requestor that a re-upload of the existing ad copy will not be possible due to the mandatory cutoff date and migration of all newly created ads to the expanded text ads format. What we can offer is an assistance in creating new expanded text ads based on the existing standard text ads content. By doing this, make sure to ask them the details for the creation of Expanded Text Ads (such as Capitalization Preference, Path 1 and 2). Then, advocates should refer to KB 3180960 AES: Policy: Microsoft Advertising: Ticket handling and KB 4043521 AES: Process: Microsoft Advertising: Ad Copy Creation for guidelines.

Note: Advocates should check our supported languages first before offering this option. See Markets, Languages and Hours of Operations under KB 2023008 AES: Process: Microsoft Advertising: AES Requests for reference.

Q: Where can I download the latest area location targeting list?

A: Go to Microsoft Advertising Geographical Location Codes and download the correct localized list.

Note: The location codes provided here are for your organization's direct use of the Microsoft Advertising platform. These may not be shared to any individual outside your organization or another organization besides yours.
