1. Introduction

To create a new Microsoft account, customers must visit the Microsoft account sign-up page. Customers can use one of the following options to have their Microsoft account created: 

  • Use an existing email address (preferred method) 
  • Use a phone number 
  • Create a new email address

1. Navigate to account.microsoft.com, and select Sign in with Microsoft

[Asset 4294818]

2. Select Create one!

[Asset 4294819]

3. Customers select one of the 3 options presented to create their new Microsoft account (sign up for a new Microsoft account)

  1. Type in an email address of their choice
  2. Use a phone number
  3. Get a new email address

[Asset 4294822]

4. After selecting Next, the sign-up process will start. Refer to section 2 to review the appropriate flow and guide customers in creating their Microsoft account.

2.4 Using a username or phone number already used as an existing proof

It may happen that customers try to sign up using a username (email/EASI account) or phone number that already matches an existing proof on an existing Microsoft account.

During the sign-up process, Microsoft will detect, in the background, that the email/EASI account or phone number is already being used on an account as a proof, which is a strong indicator that customers may already have an account that they’ve forgotten about.

As a result, the following prompt will appear telling the customer that they might already have an account. 

[Asset 4464796]

The screen gives the user 4 options to continue:

  1. Select Sign in
  2. Select the Recover it now flow
  3. Select the Create the account anyway flow
  4. Go back to the sign-up username collection screen by selecting the back arrow ([Asset 4464797])

 

2.4 .1 Sign-in flow

The customer decides to continue by selecting Sign in.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

This option will direct the customer to the usual Sign-in process, prompting the customer to enter their correct username and password.

[Asset 4464798]

Example: During the sign-up process, the customer enters an email/EASI account/phone number which does not match an existing alias but matches a proof on an existing account. (Phone or email.)

Scenario:

The customer’s parents have asked her to sign up for Skype so they can video chat with her. She downloads the Skype app and starts to sign up for a new account.

  1. She doesn't connect that her Outlook email account is also Microsoft and could be used for Skype. She enters her email in the create account field.

(In the background, Microsoft sees that this email is not a Microsoft account but is being used on an account as a proof. This is a strong indicator that she may already have an account that she's forgotten about.)

  1. The customer remembers that Outlook and Skype are both Microsoft products and she can use the same account for both. So, she selects Sign In.

  1. Back at the sign in screen, the customer enters their correct email address/username, and starts using Skype.

The customer decides to continue by selecting Recover it now.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

This option will direct the customer to the Forgot username flow, which will automatically send a code to the proof that was entered during the sign-up process.

[Asset 4464799]

Example: During the sign-up process, the customer enters an email/EASI account/phone number which does not match an existing alias but matches a proof on an existing account.

Scenario:

The customer just bought a new Office365 subscription at their local big box electronics store and wants to create an account, so the customer can apply their code.

  1. At the sign-up screen, the customer enters their email address in the create account field and selects Next.

(In the background, Microsoft sees that this email is not an Microsoft account but is being used on an account as a proof. This is a strong indicator that he may already have an account that he's forgotten about.)

  1. The customer is interrupted with a message saying they may already have an account. The customer remembers that they have an account that they use on their Xbox, but doesn't remember what it is. They select Recover it now.

  1. Microsoft sends a security code to the customer’s email account to validate that they have access. When the code arrives, they enter it in the field provided.

  1. The customer is shown a masked version of the email address they created to sign up for Xbox.

2.4.5 Frequently asked questions

Q. Does security information have to be verified to be in the database?

A. No, but customers must be able to round trip a code to the proof before they can see the list of associated usernames. By doing so the security information becomes verified.

Q. Does recover option return primary AND secondary aliases?

A. Yes

[Asset 4464802]

Q. Do aliases have to be marked to be allowed for sign in?

A. Yes

Q. Is there a way to reduce the amount of masking to see more of the email?

A. Not currently.

Q. The customer doesn't recognize any of the emails shown.  What should I advise them?

A. Walk the customer through the tips for remembering a forgotten username (KB 2979990 Microsoft account: That Microsoft account doesn't exist) or recommend that they continue to create a new account
