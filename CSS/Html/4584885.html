Timeline FAQs

 

  • The Timeline card captures a chronological order of events that occur at the entity/record level
  • Timeline card has three functionalities:
    1. shows events that occurred on a case
    2. provides ability to add case information (Notes/Posts)
    3. executive customer interaction activities such as: appointment, email, phone call, etc.
  • Each record has its own Timeline view for example: cases, users, account contact, etc.
  • The Timeline events captured and type of activities that can be performed will vary depending on which record type you are viewing.

Case Events captured in Timeline

  • Notes
  • Posts - (including manual and system generated)
  • Email – inbound & outbound (including manual and auto email)
  • Appointments
  • Phone Call Log
  • Key case events will show up as "Auto-Post" in the timeline: Case Created, Attachment Added/removed, SLA Completed/Expired, Severity Updated, Status Changed, Contact Created/Updated, Ownership Changed, Task Assigned, Case closed, SAP Changed, Internal Title Changed, etc.
  • When new DfM functionality is deployed, additional case events/actions will be displayed in the Timeline view, as an example:
    • Task: follow up task, collaboration task, and escalation task
    • Phone – inbound & outbound
    • Chat (transcript)
    • Customer web portal post
    • Social – Facebook, Twitter, Forum, Blog post, Community post, etc.

Add Case Information: Notes and Posts

  • Navigation: To add an activity go to the Case Summary tab > Timeline card bottom left corner> select the "+" icon in top right corner of the Timeline card
  • Links to more details on Case Notes and Posts

Customer interaction activities: Appointments, Emails and Phone Calls Log

  • Navigation: To add an activity go to the Case Summary tab > Timeline card bottom left corner> select the "+" icon in top right corner of the Timeline card
  • Links to more details on case emails .
    Tip: Only generate case related emails from the Case Timeline card to ensure the customer interaction stays linked to the Case. There are other places in the D365 UI that you can generate emails, however they do not provide the expected customer support experience and may generate an orphaned email.

  • The User Timeline shows:
    1. Events the user executed events across entities (i.e. cases, contacts, etc.)
    2. Events that occurred on their assigned cases
