Dispatching a RAVE Assistance Request and there is no response

When dispatching an Assistance request, it creates a record in the other Organization. You will see that it has been sent to another Support Area, but the ownership is pending. You will not be able to see or access the Assistance Request.

Please use the below site to find and then request assistance through the Duty Managers or Managers listed on the site.

Collaboration between Teams or Tools

Regardless of the support tool you’re using, all collaboration will occur within RAVE. The below will give you a sense of the scenarios, and areas of common confusion:

  1. When collaborating with a team’s request that is primarily in Service Desk, you will need to go into RAVE to access the request. Things to know:
    1. RAVE - A collaboration request will not be created in Service Desk and will have to be pulled from the RAVE Support area.
    2. If you do not see the case listed in the support area, you will need to be skilled under their profile in RAVE to see the assistance request (Have your manager ensure that all the skills are setup per https://aka.ms/RaveCollabAdminGuide).
    3. If an Assistance request (AR) is misrouted, there is no way to transfer the request.  The requestor will need to close out the AR request, create a new one and then dispatch to the appropriate Support Area in RAVE.

Transferring between Teams or Tools

When transferring a case, please ensure that you’re 80%+ sure it needs to be worked by that Team. When in doubt, please create and dispatch an Assistance request. When a case is transferred to another ticketing support system, we will not be able to reopen / use the case.

  • Note: There is a request into the Tools teams to provide the ability to transfer multiple times, but will take some time.

I am a Service Desk

Engineer and I transfer

a case to Rave and it

doesn't get assigned to

OffAgent.
  • Check to see if there is a Resource URI generated for Rave in addition to case details.
  • Check to see if the case was created in Rave.
  • If there is no Resource URI and the case was generated by a Rave transfer, you can't transfer it back. The case needs to be recreated in Rave. Contact a Tech Router for help.
  • If there is no Resource URI and the case was not generated by a Rave transfer, file a bug report with Rave.

I need to change the

severity of a case.
  • Make the change in the system you are working the case in.
  • If you are working in Service Desk and there are open collaborations within Rave, you need to go to the Rave assistance request and raise the severity manually. Changes in the Service Desk case will not sync to the Rave collaborations.

I created my Rave

collaboration against

the wrong support

area/support area path. 

  • Assistance Requests cannot be rerouted in Rave. Close the Assistance Request and create a new one against the correct support area/support area path.
  • If you are a Service Desk engineer, canceling your collaboration task will not cancel the assistance request in Rave. You need to cancel the assistance request or leave a case note so the receiving engineer can close it.
