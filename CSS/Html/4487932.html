1. Billing profile

This section provides the customer billing profile information including Details, Payment Instruments (PI), Users, and Events. This section is where you can issue monetary credits and flush invoices. Both are outlined below.

Details tab:

  • Billing Profile Name – Full name of partner
  • Billing Instructions – Single PI
  • Primary PI – Main PI on account (CC, check/wire, invoice)
  • Backup PI – If there is a backup PI, it would display here (MC)
  • Billing Day – Billing cycle date
  • Flush Policy – Monthly flush policy
  • Billing Profile ID – Unique identifier for the billing profile. This is generated at the point of billing profile creation.
  • Default PO – The default PO that is created at the point of transaction.
  • Date Created – Date the billing profile was created.
  • Date Updated – Last update made on the billing profile.

[Asset 4525755]

Payment Instruments:

This section will display the partner’s PI, including their MC.

  • Name on Card – If they’re a credit card (CC) customer, the name on CC.
  • Commerce Account ID – Unique identifier associated to the Commerce Account.
  • Type – Type of PI (CC, check/wire, invoice).
  • Description – If applicable, further description of PI.
  • Status – Status of PI (active, inactive).
  • Expiration Date – If they’re a CC customer, the expiration date on card.

[Asset 4525756]

Users:

This section will display the users that are tied to the partner’s billing profile.

  • Name – First and last name of the user
  • Email – Primary email address
  • Phone – Primary phone number
  • Roles – The assigned roles within Azure portal
    • For example: Company Admin, Admin Agent, Billing Group Owner, or Billing Group Reader.

[Asset 4525757]

Events:

This section will display all events that have taken place on this partner account, including dunning:

  • Event Date – The date the event took place
  • Type – The type of event. 
    • For example: Manage Invoice Assets, Summary State Changed, and Email Sent.
  • State – Display the state of the event (for example, healthy, or dunning.)
  • Document IDs- If a document is produced and sent to the customer, a unique identifier is created to be able to track.

2. Issuing monetary credit

Support has the option to issue monetary credit(s) to a customer’s account in CST. To perform this action, you follow these steps:

  1. After searching for the customer, go to the Overview Blade.
  2. Select the blue arrow that is associated with the Org. Id of the customer.
  3. Scroll down to the Billing Profile section, and select Issue Monetary Credit.

[Asset 4525760]

  1. Once the new page loads, you will enter the following information:
    • Amount – Total amount of monetary credit
    • Expiration Date – The date the monetary credit expires
    • VSO ID – Unique VSO ID entered for monetary credit
    • Credit Scenario – Select credit scenario reason (30)
    • Customer – Select the customer
    • Recipient – Select the recipient
    • Project – Select the Project ID
    • Comment – Enter comments which are required
  2. Select Next.

[Asset 4525761]

  1. Confirm the information entered is correct and select Submit for processing.

[Asset 4525763]

3. Flushing an invoice

Customers have the option to request an immediate flush on their invoice, which will produce an invoice that has charges calculated up until the date of the request. To perform this action, you follow these steps:

  1. After searching for the customer, go to the Overview blade and select the blue arrow that is associated with the Org. Id of the customer.
  2. Scroll down to the Billing Profile section and select Flush Invoices.

[Asset 4525764]

  1. A new page will load where you will select the reason:
    • Customer Request – Customer requested the invoice to be flushed.
    • Overpayment – The account was overpaid, and a new Invoice needs to be created.
    • Billing Issue – If there is an issue on the invoice and credits/refunds were issued, a new invoice needs to be generated for the customer.
    • Test – Testing purposes only.
  2. Confirm information and select Next.

[Asset 4525765]

  1.  Confirm information is correct, select Submit for the invoice flush.

4. Billing profile blade

This is where you can see the customer’s billing profile (previously Billing Group), payment instrument information and the users that are tied to the profile.

Billing Profile details:

  • Billing Profile Name – Name given on the billing profile at the point of transaction
  • Billing Instructions – Single payment instrument
  • Primary PI – Main payment instrument on account (CC, check/wire, invoice)
  • Backup PI – Where applicable the backup PI on the account
  • Billing Day – Billing cycle based on type of customer
  • Field Led – 5th of month
  • Customer Led – 8th of month
  • Partner Led – 9th of month
  • Flush Policy – Monthly
  • Billing Profile ID – Unique identifier tied to the billing profile
  • Default PO – Default PO created at the point of transaction. Customer can create their own in store for business.
  • Date Created – Date the billing profile was created
  • Date Updated – Last date the billing profile was updated

Payment Instruments:

  • Name on Card – If there’s a CC on the account, name displayed on CC. 
  • Commerce Account ID – CAID Tied to the billing profile
  • Type – Type of PI (CC, Check/Wire, Invoice)
  • Description – Further description on the PI
  • Status -Active or Inactive PI status
  • Expiration Date – Date PI expires
