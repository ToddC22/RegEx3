Overview   

Azure Security Center (ASC) is a service used is to prevent, detect and respond to the threats of customer Azure resources. With increased visibility and control this First Party service is available for all MOSP, EA and US Gov offers. 

This TSG provides information about this service and how to assist customers with any questions they may have. 

Customer Experience and Provisioning  

  • This service can be enabled via the Azure portal. 
  • The Free tier of this services is enabled by default when the customer enables security center on a particular subscription.
  • Pricing tier (Free and Standard) can be selected for each subscription in an Azure account.
  • Standard pricing can be enabled at the workspace level. Configuring pricing for a workspace does not enable Just-In-Time (JIT) VM access, adaptive application controls, and network detections for Azure resources.  
  • Customers can upgrade an entire Azure subscription to the standard tier, which is inherited by all resources within the subscription.
  • Customers can enable threat protection for the following services at the following levels:
    • Azure Storage Accounts at either the subscription level or resource level.
    • Azure SQL Database SQL servers at either the subscription level or resource level.
    • Azure Database for MariaDB/ MySQL/ PostgreSQL at the resource level only.
       
  • By default, all the VMs under a resource group are billed. Customers can choose to move the VM off of a resource group to avoid charges. It is recommend to move VM’s to a different subscription if the customer wants to have Azure Security Center in Standard tier for some and have other VM’s in free tier to avoid charges in another.
  • Data migrations are not supported for this service Customers will need to enable Security Center on the destination subscription once the data is transferred.

ASC will emit a daily count of the number of nodes that were active at any point during the day. This will be the monthly rate divided by the number of days in the month. Stopped resources, such as VMs that have been stopped but not deleted, will not be included in this daily count.

For example:

  • Customer protects 10 nodes every day for the first 15 days of the month. Service will emit 10/31 every day for the first 15 days.
  • Customer protects 20 nodes every day for the last 16 days of the month. Service will emit 20/31 every day for the last 16 days.
  • Total price paid by customer at the end of the month = ((10/31) *15 + (20/31) *16) * price.
  • Meters will be emitted for Free and Standard tiers.

ASC saves data to a Log Analytics workspace. Workspaces created by ASC while configured for Azure Monitor logs per node billing, do not incur Azure Monitor logs charges. Security Center billing is always based on your Security Center security policy and the solutions installed on a workspace.

Additional data uploaded over the included daily data and additional retention beyond one month for virtual machines only are based on Azure Monitor data ingestion pricing. 

Security Center uses Azure storage to save security data generated from your protected nodes. Costs associated with this storage is not included in the price of the service and will be charged separately at regular Azure storage rates. Storage charges apply even during the trial period.

Portal workflow for changing the pricing tier

Customers can follow the steps below to enable and update the Azure Security Center pricing tier. Additional information for upgrading tiers can also be found on the public facing page here. 

Enable Security Center

  1. Log into the Azure Portal.
  2. Select Azure Security Center from left navigation pane.
  3. Select Enable Security Dashboard.
  4. Review the Security Policy.
  5. Select the subscription that you want to update the pricing tier for.
  6. Select a Pricing tier (Free or Standard).
  7. Click Save

Toggle between Standard and Free Tiers

  1. Log into the Azure Portal.
  2. Select Security Center from left navigation pane.
  3. Select Pricing & settings on the Security Center main menu page.
  4. Select the subscription that you want to update the pricing tier for.
  5. Select a Pricing tier (Free or Standard).
  6. Click Save

Upgrade from Free to Standard:

  1. Log into the Azure Portal.
  2. Select Security Center from left navigation pane.
  3. Select Pricing & settings on the Security Center main menu.
  4. Select the subscription that you want to upgrade to standard.
  5. Select Pricing tier.
  6. Select Standard to upgrade.
  7. Click Save
     

Support Scenarios 

  1. Unable to upgrade or downgrade the tier.

Customers will need to upgrade or downgrade their tier via the Azure Portal. This action can be done at any time. Should a customer encounter any issues engage the Azure Security Center team as outlined in the Engagement and Escalation Path section below.

Be sure to collect and include the following customer details in your engagement:

  • Service Name
  • Accurate Date range of dispute
  • Usage file from CMAT
  • Brief description of the request
  • Subscription GUID
  • Any other relevant information
  1. Billing & Usage validation

Usage details are available via CMAT as part of the CSV file. If a customer has a validation question, engage the Azure Security Center team as outlined in the Engagement and Escalation Path section below.

Be sure to include the following details in your engagement:

  1. A full copy of the detailed usage file.  
  2. Collect the following customer data.  
  • Service Name
  • Accurate Date range of dispute
  • Usage file from CMAT
  • Brief description of the request
  • Subscription GUID
  • Any other relevant information
     
  1. Credits and Refunds
    Follow the standard credit and refund policy.
  1. Who turned on ASC?
    Customers may contact support asking how to track who performed a pricing tier change? In these instance leverage the public facing page here.
