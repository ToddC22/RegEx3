What is this article about?

Data Transfer Management (DTMv2) is a secure, highly reliable, fast, and modern file exchange tool that is used to exchange files containing HBI, MBI and personal data between customers and Microsoft Support Professionals. DTMv2 is an ISO 27001 certified system that has been independently verified as meeting the industry standards for security and protection of data.

1. Overview

  • Secure file transfer and storage in the Azure public cloud.
  • Customers sign in using their work or school, or personal Microsoft account.
  • Restricted access to only those Support Engineers who own the case or are added by the owner. 
  • Compliance with Microsoft HBI standards and contractual, regulatory and certification requirements.
  • Files loaded to the workspace do not need to be encrypted.
  • Permanent deletion of files 90 days after the case is closed.
  • Up to 1,000 files each 100GB (100TB total) per case workspace.

1.1 What is the DTMv2 Engineer Portal?

The DTMv2 Engineer Portal is an internal-only portal that is used by Support Engineers. In the DTMv2 Engineer Portal you can:

  • Share workspaces with customers and support engineers. 
  • Upload, Download, and Delete files. You can also Cancel uploads that are being added.

2. Sharing the workspace

2.1 Sharing the workspace with a customer

The Secure File Exchange is an externally facing portal hosted on support.microsoft.com. The following article can be used by customers to understand the Secure File Exchange: How to use Secure File Exchange to exchange files with Microsoft Support.

  1. In the DTM Engineering Portal, select the Customer Access button on the home page. The popup dialogue should show all contacts that can log into the DTM workspace to manage and retrieve files.

    [Asset 4056790]
  2. Copy and send the File Transfer – Case # link to the customer. This is a unique, case-specific URL and there is no automated notification delivered to the customer.
    1. With the case link, no login is required, and anyone can load files to the workspace.

      [Asset 4295073]
    2. From the same link, customers who have emailed the ASD ticket will see the Microsoft account sign in option. They can then sign into or create an Microsoft account using their email to retrieve or delete files.  

      [Asset 4295074]

2.2 Sharing the workspace with another internal user

DTM can be used to share documents with other msncs users. To do so, simply add their msncs credential into the ASD ticket. You'll find this field under the case disposition & closing section of the ASD form. 

[Asset 4332266]

 

Use a semicolon ‘;’ to add more than one user to the case and share the internal workspace link with them (be sure not to send msncs users the customer-facing link or they'll only be able to add files).

All internal msncs users have two options to access a case DTM workspace that has been shared with them:

  1. Access the regular DTM link, enter the case number, and log in using your msncs credentials.
  2. Access the case workspace directly at https://client.dtmnebula.microsoft.com/msa/Home?srNumber=xxx, where "xxx" is the case ID.

For most escalation partner teams, (SANOPS (aka AdOps), Global Support, Fraud, etc.) encrypting files and emailing/loading them directly to UCMT is sufficient. However, DTM is preferred and will be adopted as the standard for document transfer in the future. DTM should be used to store documents that may contain sensitive information (WHT certificates, account statements/invoices, etc.) or documents that are intended for the customer consumption.

All internal partners are approved to use a shared ASD credential, cssba_msft@msncs.com, to access DTM workspaces.

3. DTM Guidance for MAST Escalation Partners

Data Transfer Management (DTM) is a tool that can be used by all CSS Support advocates to create an HBI-secure workspace in which customers and enabled internal users can store and share documents related to a specific ASD case.

See KB 3174641 Process: Microsoft Advertising: Handling Personal Data team guidance for MAST policies on data protection, and KB 4056786 Process: Microsoft Advertising: Data transfer management for key features and tool functionality information.

Note: Unless DTM is used, documents from Tier 1 support teams must be encrypted at rest and in transit, regardless of storage location or method of transfer.

4. Internal vs. External tabs

  • The Internal tab’s purpose is for Microsoft Support Professionals to exchange files with other Microsoft Support Professionals. Files in this tab will never been seen by the customer. This tab can only be used if an ASD field is created that writes to the “DTM Workspace shared with” property in CAP. At launch, this will not be available for Microsoft Advertising Support teams and only the case owner will be able to access files loaded by the customer. 
  • The External tab’s purpose is for Microsoft Support Professionals and customers to exchange files. All files in this tab are scene by both the Microsoft Support Professional and the customer.
