1. Introduction: What are ads and where are they shown?

An ad is a visual advertisement for a customer’s business. Types of ads vary, but usually contain descriptive text, a clickable link, and a URL that takes a search user to the customer’s website. Some ads also use images or logos.

Ads managed in the Microsoft Advertising UI will show on a combination of locations:

  • Microsoft Owned and Operated (O&O) websites such as Bing or MSN.
  • Syndicated Partner Websites.
  • Yahoo! O&O properties and partner sites.
  • Microsoft Audience Network pages.

Bing’s Search Engine Results Pages (SERP) is the most common location for Text Ads to surface, and these are shown in one of three locations:

  • Mainline (Top three-four ads shown on SERP, above organic results).
  • Sidebar (Up to five ads shown but cannot display rich data).
  • Bottom of Page (BOP) – one-four ads shown.

Note: In the following markets, sidebar ads are not shown for PC/Tablet traffic on Microsoft O&O websites:

  • AU, CA, DE, FR, UK, US

See What is "ad position"? for more information.

3.1 Expanded Text Ad vanity URLs for pharmaceutical advertisers

Expanded Text Ads removed the concept of Display URLs for all customers. We are re-introducing the Display URL concept for select advertisers but will not be publicly documenting or proactively promoting this feature. It is only available to pharma advertisers due to industry regulations requiring them to use a Display or Vanity URL which differs from their landing page URL.

If you receive a request from a pharma customer who wishes to use an Expanded Text Ad Vanity URL, follow the below enablement process:

  1. Review their ads, keywords, and website to ensure they are advertising pharma content.
  2. Create an escalation to Global Support. In your notes, ask for the escalation to be assigned to the campaign FTE.

Note: If the customer is an agency with mixed business models (pharma and non-pharma) under their customer shell, the reviewer will decide on a case-by-case basis. Exceptions can be made to enable only the pharma accounts under the agency shell.

3.3 Ad customizers

Ad customizers provides advertisers with a way to completely customize their ads based on various attributes such as keyword, device, location, time of day, and others. This allows for hundreds of variations for a final ad, based on just one ad copy in the UI.

When the ad delivers, it will pull data from a feed file to create the advertisement. (Similar to {Param} insertions, except the param is a feed attribute, and the text to insert is stored in a feed file instead of the keyword level.)

The feed attributes are generated based on machine learning, so that the most relevant attributes will be applied.

You can find a complete setup guide in the About ad customizer feeds help article.

Ad customizer setup process

There are three steps to setting up ad customizers:

Step 1: Create the feed file
Advertisers create a feed file (.csv, .tsv, .xls, and .xlsx format) and choose their feed attributes.

Step 2: Upload the feed file into Microsoft Advertising
Accessed via Shared Library > Business Data, advertisers can go to the Ad customizer feeds section and select Manage Ad Customizer feeds.

[Asset 4460084]

Step 3: Reference the feed file and attributes
Advertisers can use ad customizer parameters in any part of their EXTA other than the final URL and tracking template (text, path1, path2, title1, title2).

Advertisers need to use the following syntax for ad customizer parameters: {=feedname.attribute:default value}

Note:
There are a few system requirements to be aware of:

  • Ad customizers can only be used in EXTA.
  • Only one ad customizer feed can be applied per ad.
  • Ad customizer parameters cannot be used in the final URL.
  • Default text is recommended when using the feed parameter. If default text is used for one parameter, then default text must be applied for all parameters used in the ad copy.
  • If no default text is applied, we recommend at least one static ad per ad group, to prevent delivery errors.
  • Up to 100 attributes (column headings) can be applied per feed file in Excel.
  • Up to 5M feed items can be used across all feeds in an account.
  • When creating a feed file, the Action and Item ID columns are required for the Feed import to be successful. All the other attribute columns are optional.

4. App Install Ads (AU, CA, FR, DE IN, UK, US)

App Install Ads are dedicated to driving customers to install an app on their device. Unlike App Extensions, which require a destination URL website for the main ad, App Install Ads do not require a destination URL.

App Extensions only require a final URL, which can link directly to an app store, and allow app installs to be counted as conversions (via third-party tracking).

Notes:

  • Although App Install Ads are available in many regions, Android apps are only supported via United States Google Play store.
  • App Install Ads are shown only on mobile and tablet devices.
  • Users can choose to exclude tablets if they wish, by selecting Device Preference = Prefer Mobile Ads, and using Device bid adjustments in the Advanced Targeting settings.
  • There will be one app platform per ad, so the user will need to create a different ad if they have the same app across multiple app platforms.
  • It is not possible to import App Install Ads from either Google Ads or via MAE. 
  • Microsoft Advertising/Bing has several different publishing SLAs, based on how new the app is, and what attributes were recently updated:
    • Newly created apps: Up to 30 days
    • Existing apps: Three to four days
    • Updated app icons: Up to 10 days
    • Updated app URLs: Three to four days

Read What is an App Install Ad? for more information on setup.

5. Responsive Search Ads

Responsive Search Ads (RSA) use AI machine learning to dynamically select headlines/descriptions, based on manual information entered by the customer, in order to create ad copy that is more relevant to customers and also to help achieve a higher CTR and conversions.

Customers provide the following combinations:

  • Ad title (15 maximum)
  • Ad descriptions (four maximum)

Their ads are then generated and will display on Bing as normal, using a title and description combination generated by Bing to ensure the best possible relevancy and quality.

[Asset 4518546]

Bing AI automatically tests different combinations of headlines and descriptions and learns which combinations perform best. Over time, Responsive Search Ads will serve the best message to different searchers depending on the keyword they search for, their device, their past browsing behavior, and other signals.

  • Headlines and descriptions are treated as text assets.
  • Text assets are deduplicated - If the same string is used in two different ads, the ads will share that asset.
  • Assets are reviewed individually for policy violations and disapproved.
  • Dynamic Keyword Insertion (DKI) will be supported for RSA but not Ad Customizer.

If they would like to, advertisers can also pin titles and descriptions to guarantee that these will be shown (though this is not recommended, as it prevents the AI from fully optimizing the ad copy).

Advertisers can pin headlines and descriptions to specific positions in RSA.

  • Multiple assets can be pinned to a position. If multiple assets are pinned to a position, they are rotated.
  • One asset cannot be pinned to multiple positions in an ad.
  • A pinned Ad Asset cannot be displayed in other positions.
  • Unpinned assets cannot be displayed in pinned positions.

Best practice guidelines:

  • Advertisers need to enter a minimum of three headlines and two descriptions.
  • Ideally aim for 8-10 headlines.
  • Use standard EXTAs as well, to act as a baseline for performance.
  • Provide at least five distinct headlines, avoid repeating words/phrases. Ideally aim for 8-10 headlines.
  • Use Dynamic Text in the headers.
  • Include branding in the headline to build trust.
  • Use two distinct descriptions at a minimum.
  • Vary the description lengths, to allow ads to be customized based on screen size.

See the About responsive search ads help article for guidance on setup and editing these ad types.

Asset suggestions in RSA Recommendations

When creating RSAs via Recommendations in the UI, advertisers will also be presented with the option to view suggested headlines/descriptions based on existing campaign data, to help improve ad quality.

They will also see an Ad Strength indicator of how impactful we believe their ad to be.

Ad Strength and Headline/Description recommendations can be accessed in the following way:

  1. When the has RSA suggestions via the Recommendations tab, they should select Edit to modify the suggested RSA.
  2. On the page that loads, the Ad Strength indicator is immediately visible. Selecting View Ideas next to this indicator will show the headline/description suggestions.
    • Recommended headlines/descriptions will also be available via a pop-up menu when the advertiser is active in the relevant asset field, during the RSA editing workflow.

6. Dynamic Search Ads

Dynamic Search Ads (DSA) are currently available in the following markets:

AT, AU, BE, CA, CH, DE, ES, FR, IE, IT, NL, SE, UK, US

DSA setup

Advertisers will need to create a new campaign and select Campaign Type > Dynamic Search Ads. (Unless they are part of the Mixed Mode pilot – see the Mixed Mode Ad Groups section of KB 3183363 Tool User Guide: Microsoft Advertising UI for more information.)

Advertisers enter their website details, select Targeting, and create Ad Description information. No keywords are required. When a user searches on Bing, Microsoft Advertising will recognize the user’s intent, and use the website content to dynamically create an ad title and choose the best landing page for the DSA. Advertisers can also set static ad headlines via a feed file. See section 6.3 for more information.

When creating DSA’s, advertisers will need to choose autotargets and set autotarget bids. An autotarget tells Microsoft Advertising which areas of the website to create DSA for, and what the Bid should be for those pages. (Additional autotargets can be created by advertisers using "Title = ", "Category = ", "Page title =" or "Page = “.) See Target searches automatically with dynamic search ads for more information. Advertisers can also manage autotargets via DSA Page Feeds, as outlined in section 6.3.

[Asset 4471738]

After creation, an inline alert in the UI will inform the advertiser that it can take up to one week for their new DSA campaigns to begin serving.

Note: Customers outside of the supported markets can be opted into this feature, if they are only targeting the markets listed above.

For more assistance, see the help article on how to Target searches automatically with dynamic search ads.

6.1 DSA troubleshooting

When an advertiser reports an issue with their DSA campaign, start your investigation with the 12 steps listed below before escalating to Global Support.

1. Verify the DSA campaign entities are enabled, active, eligible and that the campaign budget is high enough and not being placed on budget pause. (Check via Change History report).

2. If the campaign is Eligible, make sure it was created four to six days before it was expected to start serving.

  1. Select the Change History tab to see the date the campaign was created.
  2. In the date text box, select Entire time. When the history loads, you might need to scroll to the bottom of the pane to see the date the campaign was created.
    • It takes four to six days for the DSA campaign to start serving.
    • New auto targets added after a DSA campaign starts serving may take up to 72 hours before showing.

3. Check if targeting is set to one of the Supported Markets.

4. Verify the DSA campaign is created within an existing Text Ads account.

This is to ensure that Exact Match wins. If the DSA campaign in the same account as Text Ads, it will share all account settings including negative keywords, negative shared library, and exclusions.

  1. In the Campaigns tab, under Campaign Type, check that there are campaigns with the type Search & content.
  2. If the DSA campaign is in a new account, check the settings in the Settings tab to make sure they match the Text Ads account settings.

5. Check that auto targets are scoped and relevant to the targeted web site content.

  1. If the DSA campaign is in a new account, check the settings in the Settings tab to make sure they match the Text Ads account settings.
  2. If there are a variety of ad targets, check for ones that have:
    • No impressions in Impr. column.
    • Value of 0 in the Website Coverage column (see section 6.2 for more information).
  3. If the advertiser has a Google Ads campaign, advise them to use Google Import to set up the DSA campaign.

6. Check that auto target bids are comparable to non-Exact Match campaign bids for Text Ads.

  1. In the Auto Targets tab, check the DSA campaign's bid in the Bid column.
  2. Compare it to the bids for related non-exact match keywords in a different campaign.
  3. DSA bids should not be lower than the non-exact match campaign bids.

7. Check for exclusions and negative keywords that might be blocking DSA from serving.

  1. In the DSA campaign, select Auto Targets tab.
  2. In the View text box, select Exclusions (or Negative keywords).
  3. Look for exclusions (or negative keywords) that might be blocking targeted content.

8. Verify ad descriptions are generic and relevant for auto targets.

This helps drive clicks and conversions.

9. Verify that the advertiser is using appropriate Ad Extensions to help drive click volume.

  1. On the Ad Extensions tab, select Campaign (or Ad Group).
  2. In the View dropdown box, make sure extensions such as Sitelinks, Call Out, Review, Location, or Structured Snippet are enabled.

10. Make sure the advertiser is using Enhanced CPC (eCPC) or Manual CPC.

[Asset 4471739]

In the campaign's Settings tab, check the selection for Bid Strategy.

11. Check that the budget is not set too low or getting paused.

  1. In the campaign's Settings tab, check the Budget amount.
  2. Check Change History to see how frequently the campaign is pausing the budget.

6.2 Website coverage troubleshooting

Website coverage = number of pages associated to an autotarget, divided by number of pages indexed/stored in Bing.

You can troubleshoot using the following steps:

  1. In the Auto Targets tab, check the Website Coverage column for a value.
  2. If you don’t see the Website Coverage column by default, select Columns > Modify columns > Performance. Then select Website Coverage column > Apply.
  3. Check the campaign is older than seven days, and that autotargets were added more than three days ago. Latency can cause lack of impressions to begin with.
  4. Check campaigns/ad groups/ads are created, eligible, and unpaused.
  5. Check the autotargets match up with the website content.
  6. If an autotarget has multiple conditions (for instance, if ‘URL contains book’ and ‘URL contains science’, confirm with the advertiser that URLs which meet those conditions exist on the website).
  7. Confirm the website language in the Campaign settings matches the actual language of the website. You can do this by right-clicking on the webpage and selecting View Source, then search for the Lang parameter as shown.
    Note: If the website has pages with multiple languages, the advertiser can create multiple campaigns for each web page.
    [Asset 4490244]

  1. Lastly, check the Robots.txt for any crawling exclusions for the targeted landing pages via auto targets.
    1. The robots.txt file can be accessed from the advertiser's website by entering the domain name/robots.txt, for example, jet.com/robots.txt or Contoso.com/robots.txt.
    2. The list of disallow directives lists the contents/directories that would be blocked.

Escalate to Global Support for further assistance if these checks do not help resolve the problem.

6.3 DSA Page Feeds (AU, CA, DE, FR, UK, and US only)

Advertisers in these markets can use Page Feeds to upload all the URLs they want to target via .csv, .tsv, .xls, and .xlsx format. This avoids having to manually create individual auto-targets for each URL.

Advertisers can also apply static ad headlines at the URL-level via Page Feeds.

Note:

  • Ad titles are applied across all DSA campaigns, across all accounts, within a customer shell. If an ad title is provided for a URL and associated with a campaign in one account, that ad title will be used for that URL if it’s targeted by a DSA campaign in another account (whether or not the feed is associated with that campaign explicitly) until the ad title is updated or removed.
  • The most recently updated [GlobalHeadline] will be used. If more than one is supplied in a feed file, only the first one will be used.
  • Each URL can have one static headline.

Advertisers can also use existing Google Ads Page Feeds, to save time and effort.

Page feed requirements

Feeds must be provided in a standard spreadsheet format, (csv, tsv, xls, xlsx) and include three columns:

  • Page URL - Contains the URLs that you’d like to include in the feed.
  • Ad Title - Contains the headline you’d like to use for ads using the URL in that row.
  • Custom Label - Allows you to group URLs within the feed and create auto targets by custom label. You don’t need to provide a custom label for each URL. You only need to provide a custom label for each URL if you want to create auto targets using custom labels or apply a [GlobalHeadline]. A single URL can have multiple custom labels.

If you apply the custom label [GlobalHeadline] to a row, the headline contained in the Ad Title column of that row will be applied to all URLs that don’t have an ad title specified in the feed. See the examples to the right for more detail.

For example:

[Asset 4488402]

Up to 100 feeds can be added per account, and up to 5M feed items (rows) per feed or account.

See the About dynamic search ads page feeds help article for walkthrough instructions on how to create and associate feeds.

For reporting, advertisers can run one of two reports:

  1. Dynamic search ad auto target report: Understand how custom label autotargets are performing and where adjustments may be useful.
  2. Dynamic search ad search term report: See how ads perform against keywords and identify new positive/negative keywords. A new column will be added to show if a URL is a page feed or not.

6.4 DSA page feeds troubleshooting

Use these steps to solve for the following problems:

  • Ads serving for URLs not included in the page feed.
    • Check Campaign Settings and confirm targeting source is set to Use URLs from my page feed only. If not, recommend that the advertiser selects that option. The default setting is Use Bing’s index of my website. The customer can also choose Use URLs from both Bing’s index of my website and my page feed.
    • If this doesn’t solve the issue, escalate to Global Support.
  • No website coverage for custom labels.
    • Confirm campaign, ad groups, ads, and autotargets are created and enabled. The system won’t process paused items.
    • Confirm the page feed hasn’t been deleted.
    • Check the page feed includes custom labels, and that the URLs have been tagged with that custom label. If no URLs have been tagged, website coverage will be 0.00%.
    • If the autotarget includes specific conditions like “URL contains XXX”, confirm that there are URLs in page feed that meet the condition.
    • Check for capitalization - Custom labels are case sensitive.
    • Autotargets should be created using single custom labels.

8.1 Best practices for managing and optimizing Mobile Ad delivery

Use incremental bidding to increase or decrease the likelihood the ad will appear on specific devices. Refer to the Target Ads with UDT section above for more detail.

  • Use ad extensions to give your customers more ways to interact with your Mobile Ads.
  • Sitelink Extensions: Add links to up to four deeper pages on your website.
  • Call Extensions: Add a clickable phone number to your ads.
  • Location Extensions: Add a business address and phone number to your ads.
  • Use all three keyword match types (exact, broad, and phrase) to improve campaign performance.
  • Do not use exact match exclusively because it limits reach.
  • Use broad match if reach is a key campaign goal.
  • Know that time-of-day and day-of-week targeting options may reduce available ad impressions and clicks.
  • Reach relevant searchers by targeting customers in specific countries/regions, states/provinces, metro areas, Nielsen Designated Market Areas in the US, or cities.
  • For advertisers who use call extensions, historic campaign data shows that there is a higher success rate with phone numbers that connect to LiveEngage or customer service representative.

9. Ad suggestions

Ad suggestions (formerly Ads by Bing) is a feature providing customers with automatically generated ads using AI and machine learning to improve ad group performance, based on variations of existing campaign content.

Ads generated by Bing will have the Automated property applied in the UI. Users can add the Automated column to grids by using the Column Chooser. Note: Ads generated by Bing will still have the Automated status even if a user made edits to them before adding. If a user edits the ad again once they’ve been added, the Automated tag will be removed.

 

[Asset 4522135]

Users are notified on the Ads tab of the Campaigns page, or see an Ad Suggestions card on Recommendations tab (if enabled), when suggested ads have been created for them (maximum of 50 suggested ads at one time).

If a user has enabled Auto-Apply Mode, they will have a 14-day review period in which to apply, edit, or decline any suggested ads. If no action is taken by the end of the review period, the UI automatically applies the ads to an ad group.

Review Ad suggestions can help boost performance for more information on how to apply, review/edit, or opt-out of ad suggestions.  

Eligibility criteria:

Note: These criteria are for internal awareness only, and not to be shared with customers. The engineering team will use these to determine whether a customer is eligible for the pilot.

The following criteria must be met for a customer to participate in the ad suggestions program:

  • Must be Small/Medium Business (SMB).
  • Advertisers must fall into categories/services supported by our AI logic. Our AI is continually learning, so we do not have a comprehensive list of what is or isn’t supported, but be aware that ads will not always show, as we cannot guarantee the AI will be able to generate a suitable ad.

Opt-out process:

Customers who would like to opt-out of auto application may do so by going to Shared Libraries > Account Level Options > and turning off Auto-Application of Ad suggestions.

FAQ:

Q: Are the automated ads reviewed for quality?

A: Yes, we are checking the work done by our AI systems prior to inserting them into the customer’s accounts.

Q: What if the ad performance is worse?

A: As this is an experiment, we advise customers to manually monitor the success of automated ads, especially over the first few weeks of being added. By opting in, customers consent to this and confirm their awareness of our recommendations. It may be helpful to tell customers that performance improvement should be monitored at the ad group level, not the individual ad level.

10. Ad management

Editorial policy compliance

All ad types must comply with the Microsoft Advertising policies guidance for their market, to avoid being rejected. Which will prevent the ad from serving on Bing.

Ad rotation

Advertisers with multiple ads in a single ad group can control how Bing will choose which ad to deliver, using the Ad Rotation settings. Located in the Ad Group settings.

  • Optimize for Clicks - Will predominantly show ads that have the highest click-through rate (CTR).
  • Rotate Ads More Evenly - Will rotate between ads on an equal basis, based on ad relevance. Ads with poor relevance may not show.

See the Ad rotation explained article for more insights here.

Notes:

  • Only apply to EXTA and legacy Text Ads.
  • Works across both O&O and syndicated partner networks.
  • Device preferences do not impact ad rotation.

12. Suggested Search

Suggested Search is a way of serving relevant keywords (KWs) at the bottom of an article or email, making it easier for a user to continue their search journey, particularly on mobile. When a user clicks on a KW within a Suggested Search implementation – which looks like a block of KWs – a search results page will open. From there, they can click through on an ad.

Suggested Search implementations can appear both on Owned and Operated (O&O) networks, and on syndicated search results (for example, Yahoo!’s ‘Trending Now’ searches).

The featured KWs will be based on the content of the website or user information if the user is logged into Bing. For Trending Now, the KWs are both editorially curated and based on trending searches on Yahoo.com.

[Asset 4343881]

Selecting a Suggested Search implementation will trigger a search based on the keywords within it and return sponsored result suggestions as normal. Note: The look of the search results may differ depending on which O&O site is running the search. For example:

[Asset 4343882]

When the search user clicks on one of these sponsored results, the advertiser will be charged for the click, using the same bidding logic as a standard search on Bing.

As this is a two-click process (the user clicks twice before landing on the advertiser’s page) the traffic is strongly qualified.

12.2 Opting out

Note: Customers cannot opt out of Suggested Search completely – we can help them opt out syndicated partners at the domain-level, from Yahoo! ‘Trending Now’ implementations, or from MSN Suggested Search; however, advertisers may still receive Suggested Search clicks from other Microsoft O&Q websites, which cannot be excluded.

Help advertisers reduce unwanted Suggested Search clicks as follows:

MSN Suggested Search opt-out

If a customer is unhappy with the performance they receive from MSN Suggested Search implementations, educate them on the fact that Suggested Search reflects natural search behavior, and that our standard process is that we would not separate out or block traffic coming from this implantation type.

If the advertiser insists on being opted out of this functionality, this can be done by Global Support at the domain-level. For them to take this action, Global Support must find evidence of ‘volatile traffic’ (for example, traffic that is outside of the usual expected performance).  

  1. Escalate to the Global Support team to request a traffic investigation and D2AS opt-out. You will need to provide the following:
  • Account Number
  • Account ID
  • List of Advertiser Domains (ad display URL from the ad group the keyword is in). Note: Only root domains are needed. For example, we don't need nike.com/jordans and all variants, just nike.com.
  1. Global Support will review the traffic and confirm whether volatile traffic was found and take the appropriate next-actions to block Suggested Search delivery on O&O channels in future.
  • If Global Support does not find evidence of volatile traffic, they will not be able to opt the customer out of O&O Suggested Search traffic.
  • In this scenario, educate the customer that the traffic review was found to be valid and performing as expected, and set their expectations that we would be unable to exclusively block Suggested Search traffic.
  1. If the customer continues to insist on opting out of O&O, we can propose that they add the msn.com as an Excluded Website in the Microsoft Advertising UI, but make it clear that by doing so they will be losing out on the following implementation types: 
  • MSN Suggested Search
  • MSN Arbitrage (Syndicated partners driving traffic through to their website by advertising their own site on other providers.)
  • MSAN for MSN

This will prevent all traffic for these implementation types from delivering for the customer on O&O channels.

Yahoo!/Verizon Trending Now Opt-Out

If a customer is unhappy with the performance they receive from Yahoo!’s ‘Trending Now’ Suggested Search implementations, and insists on being opted out of this functionality, this can be done by Global Support. The advertiser will be opted out of all Suggested Search implementation on Yahoo! and Verizon.

For them to take this action, Global Support must first find evidence of volatile traffic. 

  1. Escalate to the Global Support team to request an opt-out, and they will be able to support your request. You will need to provide the following:
  • Account Number
  • Account ID
  • List of Advertiser Domains (ad display URL from the ad group the keyword is in). Note: Only root domains are needed. For example, we don't need nike.com/jordans and all variants, just nike.com.
  • Annual Spend of the advertiser (required for prioritization).
  • Impacted Keywords
  1. Global Support will review the traffic and confirm whether volatile traffic was found and take the appropriate actions to block Yahoo!/Verizon Suggested Search delivery in future.
  • If Global Support does not find evidence of volatile traffic, they will not be able to opt the customer out of Suggested Search traffic.
  • In this scenario, educate the customer that the traffic review was found to be valid and performing as expected, and set their expectations that we would be unable to exclusively block Suggested Search traffic.

Syndicated partner opt-out

If an advertiser wishes to prevent specific syndicated partner websites from running Suggested Search implementations, this can be done by either the customer or CSS advocate, using Website Exclusions. 

See KB 3183354 Process: Microsoft Advertising: Targeting for more information on how to apply exclusions.

12.3 Suggested Search FAQ

Q: I want to provide a goodwill gesture to my advertiser, but I don’t have any performance data to pull from. What should I do?

A: As mentioned earlier, the standard policy is that we should not issue refunds due to valid traffic. If your TA/SME approves additional goodwill, discuss with them how you should best determine the refund amount (for example, a month-on-month comparison to see spend variation, partnered with a review of Change History to ensure this isn’t the result of campaign optimization). Remember that goodwill is only ever up to 50% of the total spend, for more information see KB 3173012 Process: Microsoft Advertising: Credits, Refunds, & Rebills.

Q: Can you bid for Suggested Search traffic separately from Search bids?

A: We’ve received requests for this, and it is something we are currently exploring, but we will continue to use a single bid value across all Search advertising at this time.

Q: Can we opt out of all Suggested Search implementations?

A: We are thinking through possible solutions to address this concern. Currently we only offer the opt-out options listed above.

Q: What is keywordblocks.com? I see it pop up within the publisher performance report.

A: This is a Suggested Search implementation within syndication. You can exclude this by simply adding “keywordblocks.com” to site exclusions.

Q: Why do I keep seeing different versions of this domain in reporting, such as 1233.keywordblocks.com or 98633.keywordblocks.com?

A: Each instance of this domain within the report is a distinct domain featuring the Suggested Search implementation. To make it easier for you to block this traffic, the publisher is passing the serve URL in this format. This way, instead of blocking each individual domain, you could just block the root domain — keywordblocks.com — to opt out of this traffic.

Q: Can you investigate if my account is impacted because of Suggested Search?

A: Suggested Search drives valuable, quality traffic to our advertisers. We are always happy to investigate if there is a specific concern relating to clicks or impressions but cannot provide guidance around implementations and associated volume outside of that.

Q: Can you give me a split by account on what % of traffic that we receive is driven by Suggested Search?

A: We currently do not offer reporting by implementation type within the Microsoft Advertising UI. You can use reporting by domain to see which sites your campaigns are delivering on and use website exclusions to refine your performance and targeting.

13. Standard Text Ad (STA) deprecation

On March 31st, 2020, we began to ramp towards full deprecation of the STA format of ads (targeting a completion of October 22nd, 2020). During this time, advertisers may still see performance for STA campaigns; however, performance will steadily decline.

If CSS advocates observe STAs being used by customers, they should explain the deprecation and help the customer replace these with EXTAs or other ad types.

Auto-Generated EXTAs (AU, CA, UK, US)

In addition to manually converting STA to EXTA (see Upgrade your standard text ads to Expanded Text Ads) we’re assisting customers in making the move to EXTA, and to avoid the risk of accounts going dark, by providing an auto-generated EXTA solution, which will be offered in two phases:

Phase 1 – July 24th to August 23rd

1. Advertisers will see a blue message banner in the UI for any account that includes at least one ad group containing only standard text ads.

[Asset 4575810]

2. Advertisers can choose to make inline edits, add some/all ads to their campaigns, and they will also be able to select Download All Ads to edit these in Excel, prior to reupload.

Phase 2 – Variable, and may start from July 24th

1. Advertisers will see a yellow message banner in the UI for any account that includes at least one ad group containing only standard text ads.

[Asset 4575811]

2. Advertisers can choose to make inline edits, add some/all ads to their campaigns, download and edit ads in Excel prior to upload.

  • Additionally, they can also Dismiss some/all suggestions, and use the Options button to opt out of future suggestions.
  • Unless an advertiser opts out of the suggestions, we will automatically apply all Suggested EXTAs on 1st October, ahead of the full STA deprecation.
