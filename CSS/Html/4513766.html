2. Definitions

Action Text- A localized text displayed on the call-to-action button.

Ad Schedule - The list of days and time ranges that are displayed with the ads.

Landing page URL - The webpage where people end up after they click an ad. There are two choices: Final URL or destination URL. The URL of the page is usually the same as the ad's final URL.

Final URL - The upgraded version of the URL address of the page on the website that people reach when they click the ad from a desktop or laptop. The specific webpage link that corresponds to the action text for Action Extension.

Destination URL - The URL address of the page in the website that people reach when they click the ad.

Mobile URL - The upgraded version of the URL address of the page in the website that people reach when they click the ad from a mobile device.

Tracking template - The fields where tracking information is added. When an ad is clicked, this information will be added to the final URL to create the landing page URL. When this is defined at the account, campaign, or ad group level, it is called a shared tracking template because all ads, keywords, and extensions (if applicable) will inherit the tracking parameters.

URL parameters - URL parameters that are defined by Microsoft Advertising that can be added to the tracking templates. To learn how to add to a tracking template, see How do I create an account tracking template?

Custom parameters- URL parameters that can be created and can be added to the tracking templates. For more information, see Can I use custom parameters?

3. Policies and guidelines

Advertisers choose from a pre-defined actions list and enter the URL for the page which the end-user will be taken to after clicking the Action Extension. The Action Extension is clickable and will appear next to the ad’s description.

[Asset 4513763]

  • Action Extensions can be set up and served in all Microsoft Advertising markets.
  • Action Extensions can be enabled for all Microsoft Advertising supported languages.
  • Action Extensions can be enabled using Microsoft Advertising UI or BAM tool. As of May 2019, this feature is not yet available through Microsoft Advertising Editor.
  • Action Extensions are served in PC and mobile devices on Bing owned and operated sites.
  • Action Extensions appear at mainline only.
  • Action Extension can be associated at account, campaign, or ad group level.
  • A maximum of 20 action extensions can be associated at campaign or at the ad group level where the lowest level takes precedence.
  • Action Extensions can appear with other Ad Extensions.
  • Action Extensions can have Ad Extension schedule.
  • Action Extensions are supported to all Ad Extensions reports.
  • There are 69 predefined action types available to all markets. See list here.
  •  Action Extensions are available for the following languages. The language determines in what language the action extension will serve.

4. ASD coding

Ticket status (choose the one that best fits your case): Completed, pending, or cancelled.

Customer issue: Optimization Tactic Request

Topic: Extensions

Scenario:  Action Extensions Creation or Update

Problem:  Action Extensions Creation or Update

Solution delivered (choose the one that best fits your case): Action Extensions Created or Updated, Action Extensions Request Cancelled, or Solution in Progress.

Requirements: Enabling Action Extension requests (upload and associate) are coded as Action Extensions Creation or Update. These requests are for net-new Action Extension.

The BOS advocate is responsible for creating a new, separate ticket in the following scenarios:

  • Delete existing Action Extension then, enable new Action Extension. The BOS advocate is responsible for creating a new, separate ticket for deleting existing Action Extension (total of two separate tickets for the request). The deleted portion of the work follows the requirements in KB 4039244 AES: Process: Microsoft Advertising: Uploads (Bulk Campaign Management).
  • Edit or delete existing Action Extensions. This request is coded as Uploads and follows the requirements in KB 4039244 AES: Process: Microsoft Advertising: Uploads (Bulk Campaign Management).

The BOS advocate is responsible for providing the new case number for each portion of the work to the requestor in the original request to set proper expectations. Then, the newly created ticket(s) must have a separate email communication and proper documentation in the ASD Case Notes.

5. Creating Action Extensions proposal

The following steps will need to be followed to successfully complete the Action Extension creation request.

5.1 Gathering requirements

The following section provides instructions on how you can gather all requirements necessary before proceeding with creating action extensions.

Account Managers/Coordinators or frontline agents will submit a request to the BOS team. Once received, the following steps should be completed:

1. Review the submission form. The requestor will specify which level they want the Action Extension to be created – Account, Campaign or Ad group.

Note: There is no minimum suggestion when creating the Action Extension. However, requestors can provide their preferred number of suggestions. In case the requested number cannot be met due to website’s limited content and/or due to Microsoft Adverting policy compliance, the BOS advocate should inform the submitter to set proper expectations.

Also, the BOS advocate can suggest changing the level of suggestion to campaign or ad group level, if deemed applicable.

For example, the advertiser has a different website per campaign or ad group; the campaign or ad group has specific offerings.

If the requestor has not provided all the information needed, you will need to send a follow-up email (asking for additional information) before you can proceed to the next step.

2. Access Microsoft Advertising UI and search for the requested account. Right click on the account number and select Go to Campaigns.

[Asset 4540730]

3. Search for the specified Campaign/Ad group and select that Campaign/Ad group.

[Asset 4540606]

Note: If the Campaign/ad group has no existing ads, ensure that you inform your submitter since creation of extension without existing ads is out of scope.
4. Once the specified Campaign/Ad group has been selected, select the Ads tab.

[Asset 4540607]

5. Select the box underneath the Create ad to select all the ads, then select Download to download all the existing ads as a file.

[Asset 4540608]

6. Use the following settings in downloading the ads as a file:

[Asset 4540609]

7. The Ad report will be downloaded and can serve as a reference when suggesting Action Extensions to ensure relevancy.

[Asset 4540610]

1. Review the ad copies in the campaign/ad group. Look for key callouts in the advertising, such as product types, offers, and services where you will base your suggestions.

2. Go to the advertiser’s destination URL or final URL and review the landing page.

Note: When suggesting Action Extensions, you should review the customer’s website in detail to get insights into their product/service/offerings. This will help to ensure that the Action Extensions you suggest are appropriate and highly relevant.

3. Open the standardized Output File template and input the Output File Headers for creating new Action Extensions. Refer to the below for the required headers:

  • Type: Indicates the level of action extension. This field should be populated with one of the following to identify if the action extension is to be applied at the account, campaign or ad group level:
    • Account Action Ad Extension
    • Campaign Action Ad Extension
    • Ad group Action Ad Extension
  • Campaign: Indicates the name of the Action Extension’s corresponding campaign.
  • Ad Group: Indicates the name of the Action Extension’s corresponding ad group, if the Action Extension will be added on the ad group level. Remove this column if the Action Extensions will be added at the campaign level.
  • Action Text: Indicates the corresponding action text for the chosen action type.
  • Final URL: Indicates the final URL of the action extension. This is optional. If there’s no URL for the Action Extension, Microsoft Advertising will use the URL at the lowest level as the landing page when the action button is clicked.

[Asset 4540611]

4. Create up to a maximum of 20 Action Extensions at account, campaign, or ad group level, depending on the requestor’s instructions. Place these in the Action Text column in the output file.

5.3.1 Quality Check

Before sharing your Output file, perform a Quality Check. Go to Quality Check section of this article for details. It is important that everything has been checked to ensure accuracy with the suggestions provided.

  1. Check the proposal whether it is Account, Campaign, or Ad group level Action Extensions.
  2. Ensure that the suggested Action Text matches what we have in the list of predefined action types.
  3. Suggested Action Extensions should be highly relevant to the client’s website and campaigns/ad groups/ads’ structure.
  4. Make sure to check if the account has existing Action Extension association. This is to avoid duplicates when creating your suggestions.
  5. Output file should be formatted correctly.
  6. Send your output file to the requestor for review/approval, following guidelines from KB 3180960 AES: Policy: Microsoft Advertising: Ticket Handling.

If you are asked to process an upload based on your suggestions approved by the requester, there is no need to create a new and separate ticket for it. This is because uploading/associating and creating proposals for Action Extensions use the same ASD coding -- Action Extensions Creation or Update. You may continue with the new request (upload) under the same ticket.

Should the ticket need to be reopened, see KB 3180960 AES: Policy: Microsoft Advertising: Ticket Handling for reopening tickets.

Note: Do not upload the action extensions you created without the requestor’s prior approval or unless it was specified in the request.

7. FAQ

Q: What happens when you don’t provide the final URL for Action Extensions?

A: The ad’s URL will be used as a landing page unless a URL is provided at the keyword level. In which case, the keyword URL is used since the lowest level takes precedence.

Q: What happens when an account/campaign/ad group has multiple Action Extensions set up? How does our algorithm determine which to serve?

A: Our algorithm will decide based on user query and other ad data to determine which Action Extensions can best improve the ad performance and serve that action.

Q: Do mobile Action Extensions compete with other mobile extensions?

A: Yes, they can compete with Call Extensions and Sitelink Extensions.
