1. Microsoft accounts

Any email can be used to create a Microsoft account. The Microsoft Advertising workflow guide users through the sign up or sign in process seamlessly.
 

For customers desiring to use their work or school account, emails hosted in Azure Active Directory (AAD) are now being supported for direct login. For more information see Signing in with a work account.

New advertisers are required to use a Microsoft account to set up their Microsoft Advertising account. With the roll out of Multi-Factor Authentication (MFA) and the related two-factor authentication process, advertisers are required to have and utilize a second proof (email or phone) to access their account. More than one Microsoft Advertising account can be created using the same Microsoft account email.

Legacy credential users were forced to migrate to a Microsoft account login no later than September 2019. If a customer does not migrate by the September deprecation of managed credentials, they will miss the window for converting their user credential to an email. Once this occurs, users who still have username credential (non-email) will need to be manually verified by support, create a Microsoft account email login, and be invited as a new user.
  • If an advertiser already has a Microsoft account for a PubCenter or Digital Marketing Center account, a different Microsoft account must be used for any Microsoft Advertising accounts created separately. However, when a customer creates a PubCenter account, personal data and credit card details are saved under a Microsoft Advertising account created automatically.

Microsoft account closure

Closure of the Microsoft account has no impact on the Microsoft Advertising account, but access will be lost. Advertisers should be sure to pause all campaigns and resolve outstanding billing transactions before closing their Microsoft account, especially if it is the only user on the Microsoft Advertising account. If a Microsoft Advertising account is deemed inactive (currently identified by no spend within 15 months) Microsoft Policy states that we may deactivate or delete the customer’s account. Once deleted, an account would be deemed non-recoverable.

2. Troubleshooting login issues

The guidance below is for existing user account access issues. When a customer cannot access their account, the following best practices should always be followed:

  • Initial troubleshooting should be provided by CSS, any subsequent escalations should be directed to Global Support.
  • CSS should avoid directing advertising customers to Microsoft account (unless that is the customers preference to self-serve). Login issues that require Microsoft account review or engineering assistance will generally go through Tier 2.
  • Where available, recommend users download the Microsoft Authenticator App. The app provides easy verification for all your Microsoft accounts by simplifying how codes are received.
  • Assuming that a customer cannot auto-authenticate, always follow the manual authentication process (see KB 3171493 Process: Microsoft Advertising & Bing Places: Reactive Support Ticket Handling Policy) before assisting the customer with account changes or account specific information.
  • Ensure that user is active and not suspended* (via UI Support dashboard) prior to providing login assistance or account information.
    • *If the user is suspended or there is a QBR – Compromised alert on the account, the user could be related to an account compromise. Identify the Customer ID (CID) for the account and check in UCM to verify. If it is confirmed as compromise, see KB 3169740 Process: Microsoft Advertising: Compromised Account/Unauthorized Access for rehabilitation eligibility and guidelines.
  • Avoid recommending or sending a new user invitation as a workaround to account access issues without proper authentication, troubleshooting, and escalation protocol (unless indicated by process workflow or escalation solution).
  • Recommend good account hygiene, including:
    • Regular password updates.
    • Not using the same password across multiple accounts.
    • Use of Microsoft Authenticator.
    • Adding both an email and phone number for MFA security proofs.

The screenshot below indicates the customer updated their security proof and will not have access to their account for 30 days (or less, in some instances).

Or it may indicate their identify could not be verified and that the code cannot be sent to the existing email address or phone number:

[Asset 4558608]

Resolution:

  1. In the instances above, the advertiser has likely updated a security proof and may be blocked by the MFA feature. When you receive an escalation where the customer is having login issues related to MFA, but has not yet encountered the 30-day lockout, instruct them to update their security proof information (for example, “I don’t have any of these” in error code above), so that they place themselves in the 30-day new security proof review. This will ultimately ensure that their security information is up to date.

    Messaging:
    You can use the below talking point and reference the next steps:


“Dear <advertiser name>,

This experience that you are going through has been brought about by our user security feature known as Multi-Factor Authentication (MFA). This feature was implemented to add an additional layer of security protection to our users’ accounts. The 30-day security review is to allow our Microsoft teams to review the user’s new security proof and to ensure that the user’s account is protected. To understand more about this feature, please see our MFA Blog.”

  1. If they are not able to login due to being locked out of their account for 30 days:
    1. Follow guidance in section 2, including a check on user status to ensure the user is not suspended.
    2. Assuming auto-fortification is not possible, use a manual fortification strong proof to assess customers eligibility for early unblock. Follow the guidelines in KB 3171493 Process: Microsoft Advertising & Bing Places: Reactive Support Ticket Handling Policy.

Note: If you or the account owner have any reason to believe an unauthorized third party may be trying to access the account, get in touch with your TA to discuss options for reporting your concern to the Fraud team.

  1. Collect the following customer information via DTM for escalation to Global Support*:
    • Screenshot (like above) after attempting to login at ads.microsoft.com.
    • Email address (login used) of the user in question.
    • Full unmasked email address or phone number of the replacement security information.

Note: Screenshot requirements - Ideally this will be a screenshot with current security info and new security info details. To encounter this error, the customer must attempt to sign into Microsoft Advertising. The Microsoft account support team is requiring this because this is the only place where they can leverage the user’s information to validate them.

*If customer provides sensitive information within the ticket, including chat, phone, or email (that is not within DTM) ensure the ticket is scrubbed of sensitive information related to privacy incident.

  1. Escalate to Global Support providing the screenshot, user details above, and template Issue Type: Campaign set up Technical Issue.
  2. Set customer expectations that we will investigate (no resolution guaranteed) and to expect a follow up in approximately seven business days.
  3. If the customer is determined to be legitimate, Global Support will work with appropriate partners (including Fraud or Microsoft account) to allow the customer to use their existing credentials while we monitor the update of the user’s new security proof information for the duration of their 30-day lockout.

    Messaging:

“Dear <advertiser name>

Thank you for your patience as we continue to review your security information. We take the safety and security of our users and our platform very seriously. At this time, we have unlocked your existing security information for your use while we await the completion of the 30-day review of your new security information. Upon successful completion of the 30-day review of this update, you will receive an email notification indicating that your security information has been replaced.”

Upon completion of the customers new security proof information update of 30 days, they will receive an email notification indicating that the new security proof has replaced their previous one and that they can utilize that going forward.

  1. Internal only: Under special circumstances, certain (high-spending unmanaged customers [as deemed by Engg]) customer may go through an expedited unblocking process like the managed customer users. At this time, the spend threshold is being determined by Engg.
    1. Upon completion of the investigation with Global Support, the customer will receive an email notification indicating that a new security proof has been added to their account and that they can utilize that going forward.
    2. Before the customer can use the new security proof, they must check on their security proof by attempting to sign in to login.live.com to get to the screenshot they previously shared and at the bottom of it, select Cancel this request. Now that their security proof has been updated, this will get them out of the 30-day lockout window as the first step towards being able to use their new proof.

[Asset 4558609]

  1. They will be prompted to enter a code that will be sent to the security proof of their selection (see screenshot below).

[Asset 4558610]

  1. Upon going through that successfully, they can now use their new security proof.

If any review from Global Support results in a rejection of expediting the user’s security proof review, use the following:

Messaging:

“Dear <advertiser name>

Thank you for your patience as we investigated this account access issue. We take the safety and security of our users and our platform very seriously. We are unable to expedite the update of your multi-factor authentication information at this time and advise that you wait the remaining days for your information to update. For privacy and security reasons, we cannot provide additional details around the factors which led to this decision.

We apologize if this impacts your business in any way. Please let us know if there is anything that we can do to support you while your security information update is pending.

Sincerely,

Microsoft Advertising”

2.2 Code is not received if the customer is not able to receive their code and does not meet the above criteria from section 2.1 (for example, no error code)

Follow the steps below to help troubleshoot before sending to Global Support:

  1. Recommend the customer use the Microsoft Authenticator app to simplify the access approval and verification codes.
  2. Ensure customer checks that code is being sent to a phone or email that they have access too. If this information is outdated, it must be updated now. Afterwards, this update will likely take them to a 30-day lockout experience. If so, follow the steps listed above in section 2.1.
  3. Recommend that the customer has both phone and email added to two-factor options.
  4. Have the customer check their junk email and/or spam folder for the missing email.
  5. Have the customer add the email to their safe sender list by adding the @accountprotection.microsoft.com or account-security-noreply@accountprotection.microsoft.com alias to the safe sender list (see https://support.microsoft.com/office/add-recipients-of-my-email-messages-to-the-safe-senders-list-be1baea0-beab-4a30-b968-9004332336ce?ui=en-us&rs=en-us&ad=us#ID0EAACAAA=Office_2007).
  6. Review Microsoft Account Security info for troubleshooting verification code issues here: https://support.microsoft.com/help/12428/microsoft-account-security-info-verification-codes.
  7. If the issue persists, escalate to Global Support, confirming the troubleshooting steps taken above and (if applicable) providing a screenshot if there is an error being encountered.

2.3 Forgot username or password

Resolution:

  1. If the advertiser forgot their Microsoft Advertising username or password, advise them to select the link Forgot your username/password? from the ads.microsoft.com sign in page.
  2. Select the appropriate radio button option for username reminder or password reset.
  3. Advise the advertiser to enter the username or full email associated with that username. An error will not be provided if the email is entered incorrectly.
  4. Direct the caller to check their email for the mail including either their username or further instructions on how to reset their Microsoft Advertising username and password.
    • If the advertiser can complete the password reset, guide them through accessing their Microsoft Advertising account.
    • If the advertiser is unable to complete the self-serve username reminder/password reset steps above, manually authenticate them for access to the account and send a new user invite.
  5. Once the user receives the new user invite email, they will be guided through the new user invite setup process, which will allow them to use their preferred email address to log into Microsoft Advertising going forward.
    External help page:     https://help.ads.microsoft.com/#apex/3/en/50823/0.

2.4 Error messages

Error message: “There is no Microsoft Advertising account associated with this email address.”

Issue: If the user sees this error message, the email entered is not yet associated with a Microsoft Advertising account or the user was suspended.

Resolution:

  1. Confirm which Microsoft Advertising account the user should be associated with.
  2. Look up the user in the Microsoft Advertising UI to check the user status. If the user is inactive or there is a QBR – Compromised alert on the account, the user is compromised. Do not fortify, see KB 3169740 Process: Microsoft Advertising: Compromised Account/Unauthorized Access for rehabilitation guidelines.
  3. If the user is pending, guide the user to retrieve the pending invite from their email and complete the activation steps. If they cannot find an invitation email, cancel any pending invitation in the UI, verify the caller, using strong-proof manual verification, and re-invite them.

Note: If the issue persists, escalate to Global Support.

Error message: “That Microsoft account doesn't exist. Enter a different account or get a new one.”

Cause: If the user sees this message, either they entered an incorrect email (typo), the alias was deleted from the Microsoft account, the Microsoft account was manually closed/deleted, or the user may be suspended.

Resolution:

Look up the user in the Microsoft Advertising UI to check the user status. If the user is inactive or there is a “QBR – Compromised” alert on the account, the user is compromised. Do not fortify, see KB 3169740 Process: Microsoft Advertising: Compromised Account/Unauthorized Access for rehabilitation guidelines. Otherwise, see KB 2979990 Procedure: Microsoft account: Handling "That Microsoft account doesn't exist" error and I forgot my username or password.

Error message: “It looks like someone else might be using your account”

Cause: This may mean the advertiser's Microsoft account has been compromised. This does not mean their Microsoft Advertising account has been compromised/modified.

Resolution: See KB 930018 Procedure: How to handle account takeover on a Microsoft account or refer the advertiser to Account temporarily suspended.

If the advertiser is unsuccessful in resetting the password, then direct the advertiser to complete the Recover your Microsoft account form.

If the advertiser is unsuccessful in resetting the password using both the reset your password process and the Recover your Microsoft account form, and the user does not want to create a new Microsoft Advertising user, instruct the advertiser to follow the steps below:

  1. Go to the Windows Live ID Support page.
  2. Select I'm having trouble signing in.
  3. Since the advertiser can’t sign in with a current Microsoft account, have the user select Sign Up to create a new Microsoft account and follow the steps needed to create a support request.

If the advertiser wants to create a new user, conduct complete manual authentication before assisting them using the new user invite process.

Error message: “This email is part of a reserved domain. Please enter a different email address.”

Cause: The email was likely already used to create a Microsoft account.

Resolution: Create the Microsoft Advertising account by logging in with the existing Microsoft account or create a new Microsoft account using a different email.

Error: “Signing into Microsoft Advertising now requires an email address…”

Cause: Legacy credential login is no longer supported. Microsoft account email login is required.

Resolution: Manually authenticate the caller for access to the account and send a new user invite to their email address of choice.

2.5 Other issues

Issue: Microsoft account creation limited to three per day.

Cause: Users are limited to creating three Microsoft accounts per day. This aligns with customer-facing information available on support.microsoft.com in Troubleshoot errors creating a Microsoft account.

Resolution: Customers should simply use multi-account access to gain access to multiple accounts or wait 24 hours to create further Microsoft accounts.

Issue: Unable to add new user.

Cause: Microsoft Advertising user invitation email not received.

Resolution:

  1. ​​​​Have the advertiser check their junk folders to ensure the Microsoft Advertising email was not received.
  2. If the caller is a super admin on the account and is attempting to add another user, walk the caller through cancelling and resubmitting the invitation. See How do I give someone access to my Microsoft Advertising account?
  3. Manually authenticate the caller for access to the account. If not authenticated, refer them to contact a super admin on the account.
  4. If manually authenticated, proceed with cancelling and resending the invite to the new user.

Note: Not all support advocates are able to submit new user invitations. Consult a SME/L2 for assistance with inviting the new user.

  1. If the invitation is still not received, try sending again. If the customer still does not receive it after 10 minutes, escalate to Global Support.
Issue: Activation link in email is not working.
 

Resolution:

  1. Ensure the caller is logged out of any alternate email accounts or is using an InPrivate browsing session when attempting to activate their user.
  2. The advertiser should then copy and paste the link into their browser address bar.
  3. If the activation link is still not working, have the advertiser create a new Microsoft account and try again. If the advertiser does not want to create a new Microsoft account, request assistance from Global Support.

Issue: Customer didn't receive an email from the Microsoft account team during the process of creating a new account.

Resolution:

A customer will only have to go through this step if they need to create a new Microsoft account using a non-Microsoft owned email domain (for example, yahoo.com or gmail.com).

In this scenario, after selecting Create account in the Create an account Microsoft account workflow, they will be required to enter the code sent to the non-Microsoft email to complete setup using their preferred email. This email should be received within five minutes.
