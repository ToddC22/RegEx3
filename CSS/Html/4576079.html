Surface Hubs do not work with Intune Management Extension.

Intune Management Extension is also known as Sidecar.

Workflow of Intune Management Extension:

 Intune Management Extension gets initialized.

 Discovers the Endpoint of Intune.

 Gets AAD token via impersonation which logs using User account.

 Starts application to query Required and Available Win32 app.

 Gathers Application policies.

 IME verifies if there are any dependencies declared.

Detection logic starts:

 MSI : Checks WMI query for MSI code.

 Path : Checks the path defined.

 Registry : Checks the registry key path.

 Custom script : Triggers the script.

If the detection logic fails, then Application installation fails.

 If success, then proceed with download of application.

 After download job is completed, IME creates installation process in User or Machine session.

 Then runs Detection rules to confirm successful install.

 Now IME creates Application report and Compliance state to be sent to Intune service.

 Report is sent and locally saved to cache.

 App polling phase will be stopped post these activities.

Check from Kusto:

First of all check if the Intune Management Extension agent is installed on the end user client using below Kusto query:
‎ DeviceManagementProvider
‎ | where env_time >= ago (24h)
‎ | where * contains "<deviceId>"
‎ | where * contains "IntuneWindowsAgent"
‎ | project env_time , name , applicablilityState , reportComplianceState
‎ | summarize max ( env_time ) by name , applicablilityState , reportComplianceState

Possible causes of Failures and troubleshooting:

 IME will not re-evaluate application status if it is an Available app.

 Application report status to IME may fail due to Network issue.

 If there is detection of User session change or log off on device before final app results are shared.

 Device moves to Hibernate\sleep due to inactivity.

 Improper detection value set from the Intune portal.

 Due to Antivirus which blocks download.

 Content Delivery Network (CDN) used times out.

 Any unknown error while downloading content.

 Unavailability of Disk space or Disk size constraint.

 If status is success but app is not even installed, then detection rule might not be sufficient.

 If the device is not Azure AD joined or Hybrid domain joined and MDM scope is not set.

 If there is application install timeout or if the application detection fails , customer will need to contact the application vendor to determine the correct installation parameters to use for a silent install.

 Use Procmon and gather ProcMon dump to see process execution flow of app installer. Refer: https://docs.microsoft.com/en-us/sysinternals/downloads/procmon/

 IME cannot be installed on Co-managed devices if the workload is not switched to Intune, due to state HostNotApplicable .

 IME fails if incorrect proxy gets configured in AutoProxy key. Verify proxy from powershell: netsh winhttp show proxy . Remove proxy if there is change using netsh winhttp reset proxy . Restart IME to force new proxy and sync IME service.
