What is this article for?

This guide covers how to accomplish various tasks within the new tool, Modern Commerce Support Tooling (CSTv2). It is not intended to replace existing guidance – refer here for help understanding how to execute a task. The new CST UX was designed to make it easier to access customer information and this guide will provide details on how to not only Navigate the new Tool, but how to perform various Modern Scenarios.

1.1 Request CST Portal Access

For Production access, submit your request here: https://tools.cp.microsoft.com/ManagementPortal/requests

  1. Select My Requests and select New request

[Asset 4530945]

  1. Select Request Application Access and select next: 

[Asset 4530947]

  1. Fill in the required details:
  • Application Name as “CST”
  • Business group as CST_BG4BizOps

Note: Select CST_BG4BizOps as business group (BG), the other BGs you will see in the list are not configured yet and will not work.

  • Role you want to request – to get read-only access you need to select SD_CST_Read role
  • User to Add – By default logged-in user Id shows here, however you can change the Id
  • Approver of the request.

Note: Approver must be the user’s manager or skip level manager. Select the Change button to switch.

  • Business justification.

2. CST Permissions 

There are three states to an operation in CST. The below defines how you will see the different operations:

  • Available: The field will appear white and can be selected for action
    • Action is available for the selected asset type
      • Example: Amend Invoice, Download Invoice, Download Azure Usage
  • Unavailable: The field will be grey and cannot be selected for action
    • Action is unavailable for the selected asset type
      • Example: Pause and Resume Dunning, Pause and Reset Dunning, Extend Due Date
  • Permission Not Granted: The field will be grey, cannot be selected and a lock icon will appear
    • Action is available, but the logged in user does not have access to perform the action
      • Example: Update PO
