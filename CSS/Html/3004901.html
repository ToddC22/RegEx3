ASMS – JIT Access

Submit a ModTrack request to update this TSG.

Overview

This TSG provides information about the Just-in-Time (JIT) tool and guidance for submitting a JIT request for Jarvis write operations.

Policy

  • Support engineers (SEs) cannot complete administrative level commands without submitting a write operation request (AKA JIT request) via Jarvis.
  • Write operation requests have a duration of eight hours

Guidelines

Note the following before submitting a JIT request:

  • It is mandatory to mention the justification for obtaining a JIT request.
    • Justification should contain the operation in Jarvis that needs to be performed by ASMS.
  • One JIT request is needed per case/per operation.
  • Do not submit multiple JIT requests for the same operation.
    • Multiple JIT requests may be rejected.

How to Request a Jarvis Write Operation

To request JIT access, use the following steps:

  1. Navigate to Jarvis and click Get Access.
  2. Enter the following items into the Just in Time Access prompt:
    1. Work-Item Source: Other
    2. Work-Item ID: <The Service Desk Case number>
    3. Resource Type: ACIS
    4. Scope: RDFE
    5. Access Level: CustomerServiceOperator
    6. Justification: 
      https://servicedesk.microsoft.com/#/customer/cases?caseNumber=<Service Desk Case Number> 
  3. Click the Submit button.

Note: Do not ping the individuals noted in the JiT approval group to approve. If your JiT request isn't getting approved in a reasonable time, you can email the wasmsesc DL and let them know you need the JiT request approved. 

  1. After you see that your request was approved:
    1. Work and complete your case as needed.

Engagement and Escalation

If you experience issues while requesting access to JIT, escalate to the JITsupport alias.
