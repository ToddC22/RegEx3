INTUNE: ZtdDeviceAlreadyAssigned and ZtdDeviceAssignedToOtherTenant errors

[Asset 4476087]
  • Content Idea (CI): 117123

Support Topic:

  • Cloud and Online Services\Intune\Technical Support

When importing a CSV in to the Windows Autopilot devices blade in the Intune portal (Microsoft Intune | Device enrollment | Windows enrollment |Windows Autopilot devices), you may receive errors similar to these:


Internal error

Error code:

806 - ZtdDeviceAlreadyAssigned

CSV line numbers affected:  1


Internal error

Error code:

808 - ZtdDeviceAssignedToOtherTenant 

CSV line numbers affected:  1


[Asset 4561338]



These errors indicate that an admin is trying to import an Autopilot device that is already present in the Store for Business or Intune service. 

Customer bought either bought a refurbished device or OEM put hardware such as Motherboard from a device that was already present in Intune service but belonged to a different customer. Ideally, OEM should have asked previous owner to remove the device from Intune tenant but this generally isn’t a feasible option for customers so Microsoft asks for proof of ownership and based on this may take a decision to manually dissociate the device on their behalf. Customers can see this issue again and when that happens, they would need to contact Microsoft. Microsoft may not be able to share details of previous owner because of data privacy reasons but will be happy to help if customer can provide proof of ownership.


Important Note: IT Level 2 teams in Charlotte Yarkoni's org supporting OEMs, please use Declaration form to provide proof of ownership. If you have any questions or to get access to form, please reach out to Chris Liao (chrliao@microsoft.com)


Begin by asking the customer to search the Azure AD devices blade for the serial number (remove any leading zeros). Once you have verified with the customer that the device is not visible in their tenant, use these steps to troubleshoot this issue.

  1. Send the customer this template email.  Delete any requests for data that the customer has already provided to you.

    Hi <CUSTOMER NAME>,

    We will now begin our investigation of your Autopilot CSV import issue.  To best assist you, please upload these files to the secure file transfer site at the bottom of this email:

    • Device CSV.  A copy of the device CSV file that generated the error.
    • Proof of ownership.  Typically, this will be a bill of sale or an invoice in PDF format.
      1. The document must be in its original form.  Unfortunately, we cannot accept screenshots.
      2. The serial number, chasis ID, or other unique identifier should be present in the document.  For some vendors, you may need to request additional details about the purchased device to see this information.
      3. Your company name must appear in the document.
    • Diagnostic logs.  Please log on to the device and run the command Mdmdiagnosticstool -area Autopilot -cab c:\out.cab.  The out.cab file contains important diagnostic data that will help us troubleshoot your issue.

    Once you have shared these documents with me, I will verify the status of the device and escalate a request to delete the record, if needed.  The deletion process may take a short while (typically less than a few days), and I will keep you notified of any changes and let you know when it is safe to retry importing the CSV file.

    Secure file transfer link:
    <INSERT DTM LINK HERE>

    Thanks,
    <SIGNATURE>

  2. Have the customer try to delete the device from Store for Business (SfB).  This will remove the device record in the case that a device record exists in SfB has a record for the device but Intune does not.
    1. Navigate to https://businessstore.microsoft.com/en-us/manage/devices/all and locate the device(s) you wish to delete.
    2. Select the device and select Remove devices
    3. Return to the Intune portal and navigate to Home\Device enrollment - Windows enrollment\Windows Autopilot devices to try to re-import the CSV file.
  3. If #1 fails to locate or delete the device, the most likely reason is that the device is registered to another tenant.  Please follow these instructions to gather the required data:
    1. Device CSV.  A copy of the device CSV file that generated the error.  This contains two critical pieces of information: the device serial number and hardware hash.  Verify that both are present in the file.
    2. Proof of ownership.  Typically, this will be a bill of sale or an invoice in PDF format.
      1. The document must be in its original form.  Please do not collect screenshots.
      2. Verify that the serial number or other unique identifier is present in the document.
      3. Verify that the customer's company name appears in the document.
    3. Diagnostic logs.  Log on to the device and run the command Mdmdiagnosticstool -area Autopilot -cab c:\out.cab.  Have the customer upload out.cab to DTM.
      1. If the machine is still in OOBE, you can open an elevated command prompt by typing shift + F10 to run this command.
  4. Search Rave for the serial number.  In Rave, navigate to Applications\Intune\Troubleshooting and select Device from the list.  Under Device Search, select Serial Number and enter the serial number from step 2a above.  In most cases, no record will be found.  [Asset 4561339]
  5. Run the following queries in Kusto and record the results.  In most cases, no records will be returned.  

    CorporateDeviceDataService_ZTDDevice

    | where ZTDSerialNumber contains "123456"

     

    Note:  Use contains instead of == as serial numbers may not be stored in the same format especially if you see leading zeros.  For example, if the serial number is 000123456, search using | where ZTDSerialNumber contains"123456".

     


    Find the ZTDID from the cab file collected in step 2c above. Open the MdmDiagReport_RegistryDump.reg file in Notepad and search for ZtdRegistrationId.   

    [Asset 4561340] 

    Copy the GUID and run following queries.

     

    CorporateDeviceDataService_ZTDDevice

    | join AccountService_Account on AccountId

    | where ZTDRegistrationId == "<ZtdRegistrationId GUID from .reg file>"

    | project AccountId, ContextId, Name, IntuneDeviceId

     

    DeviceService_Device

    | join AccountService_Account on AccountId

    | where DeviceDataStoreServiceRegistrationId == "<ZtdRegistrationId GUID from .reg file>"

    | project AccountId, ContextId, Name, DeviceId

     



    IMPORTANT: Many customers will have multiple tenants.  If you do find records in Rave or Kusto, investigate the Domains tab in the tenant's properties in Rave.  If the domain is in the Select domain: drop-down list, ask the customer to investigate in that tenant.  Please do not share contact details of tenant admins where the device is currently registered and/or attempt to reach out to other customers to ask them to delete the device(s) in their tenant.
