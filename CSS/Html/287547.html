PKI: Object IDs associated with Microsoft cryptography

Microsoft Internal Support Information

BUG #: 47247 (Content Maintenance)

This article describes the object ID numbers (OIDs) that are defined for Microsoft Windows Server 2003, Microsoft Windows XP, and Microsoft Windows 2000. Some OIDs do not exist in all of these operating systems.
Object identifiers are numeric values that enable programs to determine whether a certificate is valid for a particular use. The object identifiers can be used to represent components such as X509 extensions, PKCS #7 extensions, and PKCS #7 contents. Each subtree in the Microsoft object identifer is assigned to a specific area.

This list contains the OIDs that are defined for the operating systems that are listed in the "Introduction" section. One operating system may not contain all these OIDs.

The Microsoft Outlook Express OID 1.3.6.1.4.1.311.16.4 seems like it should appear in the following IMC Web site:
http://www.imc.org/ietf-smime/other-smime-oids.asn
It is an OID that Outlook Express uses to include the sender's preferred encryption key among the bunch of certificates that are sent in a PKCS7 signedData structure. The OID is 1.3.6.1.4.1.311.16.4 (seems to be in Microsoft's arc). It is used as an authenticated attribute; the idea seems to be the same as SMIMEEncryptionKeyPreference (OID = 1.2.840.113549.1.9.16.2.11). The syntax is slightly different; it "points to" the actual certificate with only IssuerAndSerialNumber; i.e. no CHOICE; no IMPLICIT tag. No idea whether it's peculiar to Outlook Express or also used in Outlook or any other details. For more information, visit the following Microsoft Web site:
http://bugcheck/default.asp?URL=/Bugs/Exchange/215676.asp
