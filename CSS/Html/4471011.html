Summary

This article lists all the Support Areas in RAVE currently. The list will continue to change as new support areas are added. Please see the most up-to-date list in RAVE itself. Some Rave support areas and all Service Desk support area paths can be searched from the MSaaS portal: https://msaas.support.microsoft.com/.

Support Area

Note: All the Enterprise – XXXXXXX support areas are only for M365 (O365) technologies (Exchange Online, Cloud Identity, SharePoint Online, etc.), not for OnPrem. Cases that arrive in ExchangeOnPrem support area should be transferred to OnPrem support areas. If there is not a support area in RAVE for the corresponding OnPrem technology, then we need to work to find the proper solution for the customer issue to get the case transferred to the proper team. Only if the customer confirms the issue is with M365, then we can transfer the case to the Enterprise – XXXXXX support area.

Here are some available information for Rave case traction: http://aka.ms/reactiveescalation.

For duty manager contact information, see article 4345311.

Engineers who wish to initiate a collaboration request, refer to the Collaboration from Rave to Service Desk section from article 4346610 Procedure: Collaboration.

DeveloperMessaging supports cases that have to do with Exchange, Exchange Online, and Outlook extensibility, as well as cases to do with Office 365 APIs (the likes of Graph API, Outlook REST API and the Office 365 Management APIs), and requires Premier subscriptions.

DeveloperOffice supports cases related to Office development (WXP, OneNote, Visio, etc.) using Visual Studio languages or VBA, and requires Premier subscriptions.
  • This support area handles issues where customers are developing applications that integrate with Office On-Premises and Online, customizing Office using Visual Studio, or are otherwise programming using .Net technologies/VBA.
  • This would include automation, extensibility, integration and development of Word, Excel, PowerPoint, Visio, MapPoint, Access, Project & Publisher using Visual Studio or Visual Studio .NET (VB, C#, C/C++/ATL, MFC, ASP/Scripting) & Visual Studio Tools for Office (VSTO), Apps for Office (programming issues only with Word, Excel and PowerPoint apps), Office VBA (Word, Excel, PowerPoint), Access VBA (PRO only), Excel and OneNote Graph APIs, Word Automation Service (programmability only), Excel Services (programming only), Visio Service (programming only), PowerPoint Service (programming only).

Management Activity API – SPDev Supports ONLY the API. Any questions about compliance or the data returned from that queries should be addressed by the O365 Security and compliance team.

  • In scope: Connection problems or errors; authentication problems or errors; and setting up/configuring use with the API.
  • Out of scope: Helping analyze data already pulled from the Management Activity API; and incomplete or inconsistent data retrieved. These are handled by the O365 Compliance team.
  • Out of scope: 3rd party products using the Management Activity API. These are supported by the 3rd party vendor, who can work directly with appropriate Microsoft support team if needed.

This queue also supports issues related to Security, only if:

  • Not related to an O365 account compromise. This is handled by the O365 Security & Compliance team (Product routing is EnterpriseSecurityCompliance).
  • Not related to a Brute Force attack/account lockout. This is handled by the AD/Azure AD team.
  • Not related to Azure DDoS. This is handled by the Azure Network team.
  • Related to a confirmed compromise of the environment. Suspicious behavior have to be investigated by the related support team first. If confirmed to be malicious, Security team can be involved.

In addition, RCA for Security issues are only ensured in the context of a Premier contract. For a Professional contract, the customer will have to involve a vendor or a partner.

Common misroutes are:

  • O365 ATP – Do not mix Azure ATP, WD ATP and O365 ATP. These are three different products and O365 ATP is supported by O365 Security & Compliance team (Product routing is EnterpriseSecurityCompliance).
    • Before routing an ATP case, please ask the customer which one is involved.
  • OCAS – OCAS is related to Office 365 Security & Compliance. Please see: What are the differences between Microsoft Cloud App Security and Office 365 Cloud App Security? (Product routing is EnterpriseSecurityCompliance).
  • Windows Defender Application Guard & Windows Defender Application Control (Device Guard) are supported by the Windows team.
  • Windows Defender Firewall is supported by the Network team.
  • Microsoft Security Graph API has nothing to do with AAD Graph API which is supported by the AAD team. Please be sure to clarify which API is used by the customer before routing the case.

For more information, please check: https://cssstatus.linqto.me/PodDetails?eID=46.
