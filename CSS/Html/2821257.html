Identify if the licenses were received as an MPN benefit

For VL advocates, transfer the case to partner center by following the transfer process in article 4564983-Procedure: VLS/DEV/WWL: Transfer to Partner Center.

For Commerce, follow below procedure:

License keys received as a benefit of the Microsoft Partner Network (MPN) are not eligible for replacement following the procedure in this article.  

How to identify if the licenses were received as an MPN benefit:

  1. Check if the license has "IUR" in the product name.
    - If yes, please go on to follow Step 3.
  2. If not, ask if the company is a Microsoft Action Pack or competency partner.
    - If yes, please go on to follow Step 3.
    - If they say no or don’t know, search for the company name on https://ocpinsights.microsoft.com/.
  3. If you find their company is a partner,
    - please ask the partner to open a ticket with Partner Frontline Support (PFS) directly via https://aka.ms/pfs.
    - PFS will determine if the partner may be eligible for a replacement. However, you should set expectations that MPN licenses have different usage rights from VL licenses.
  4. Keys replacement can be provided within 30 days of activation, if the request comes in after 30 days, the request should be denied with the following message: “A token Replacement cannot be approved in this case as per policy, since more than 30 days have passed (since the seats were added / since the token was redeemed on the wrong tenant). As per the MPN Benefits usage guide, program administrators should pay close attention during the activation of a key because there is no token replacement provisioned for wrongly activated tokens.”

Guidance

Replacement key scenarios

In some scenarios support is allowed to provide replacement keys – read the replacement key scenarios in article 4566340.
After provided the replacement keys scenarios, VL support don’t need to deactivate subscription on the wrong tenant or correct expiration dates, however Commerce support does.   

Customer late with renewal:

When the customer contacts support needing extra time to purchase a new product key, if the customer’s subscription status is active, inform the customer that they have 30 days beyond their subscription end date to get a product key. If their subscription is already past the end date and in the status of “InGracePeriod”, please extend the current status end date by 30 days to allow the customer to acquire a product key by using "extend current status end date” for any prepaid subscription.
The customer will remain in “InGracePeriod" before being moved to "Disabled". Please follow the steps below to extend the current state end date:
 

How to extend the current state end date:
  1. Navigate to the Subscriptions page in CMAT. Click the Current Status End Date (the date is a link).
     
  2. A change form will be opened. Change the date and enter the reason for the change in the Comment field.

    [Asset 4090623]
  3. Click the next button.
  4. Confirm the date is correct and enter the case number. Click the check button to make the change.
    [Asset 4090625]
  5. The advocate will receive a message that the change is successful.

    [Asset 4090626]

Do not suggest to a customer that they may be entitled to a "refund" when redirecting them to their Retailer or Partner. This can set the wrong expectations for the customer as each Retailer and Partner sets their own return policies.

b. Customer activates keys on the wrong tenant

Issue

Action plan

The customer has activated product keys on the wrong tenant.

Troubleshoot in CMAT where the key was activated.
If the tenant is not known to the customer, the key can be replaced without having to disable the subscription.

For Commerce:

Request permission to disable the subscriptions on the wrong tenant and, if permitted, disable.

Add the tenant the keys (last 5 digits) that should be used and the reason in the case.

Engage Tier 3 in the POD for a replacement key using the escalation template.

Tier 3 needs to verify that the subscription is disabled from the incorrect tenant in CMAT, then give out the new product key.

Contact the customer with the new product key. Ask the customer to activate the key.

Note: With this process, support has already asked/verified/disabled the old/wrong key before given customer a new key.

For VL team, there is no need to disable the keys on the wrong tenant – exception see article 4566340.

c. Customer needs keys on multiple tenants

Issue

Action plan
The customer has only one key but wants to activate subscriptions on multiple tenants.

Advice the customer to contact the partner to cancel the order and place a new order with different line item which will generate multiple keys.

d. Customer extends their subscription instead of adding additional seats
 
Issue Action Plan
The customer used their product key to extend their subscription when they meant to add users. The customer would now like to be able to remove the time and add users.
  1. Inform the customer that you will be removing the extra time from the subscription. Using the Subscription Status Change History in CMAT, find what the old end date was prior to the product key being added. Set the end date for this (the original end date) date.

    Note: There is no action needed on the old key. Old key information is not needed.
  2. Add the tenant and the reason for the product key request in the case.
  3. Engage Tier 3 in the POD for a replacement key using the escalation template. 
  4. Contact the customer and provide the new product key. Please walk the customer through how to add seats using a product key. Refer to 2821797: Information about Prepaid Plans section titled “Add more seats to a prepaid subscription”.

For VL team, there is no need to make changes to the original end date.

e. Customer adds licenses instead of extending their subscription
Issue Action Plan
The customer used their product key to add more licenses when they meant to extend the subscription term. The customer would now like to be able to remove the additional licenses and extend the end date of the subscription.
  1. Engage Tier 3 in the POD to request a replacement token with the old number of seats using the escalation template. 
  2. Provide the replacement token(s) to the customer and ask them to click renew in the Microsoft 365 admin center to extend the subscription with the new token. (See 2821797: Information about Prepaid Plans for steps to extend/renew a prepaid subscription.)

    Note: Inform the customer that you would not be able to extend the end date until the new key has been redeemed.

    Note: Using Renew in the Microsoft 365 admin center will lower the seat count to the number of seats used. For example, if the customer has 50 seats and they extend with a 40-seat token, their seat count will reduce to 40.

    Important: After the customer extends the subscription with the new token, the end date in CMAT may be incorrect. Follow step 3 to determine whether the end date needs to be adjusted in CMAT.(Old key information is not needed.)
  3. When their seat count is correct, adjust the end date in CMAT, if needed.

For VL team, there is no need to make changes on the end date.
