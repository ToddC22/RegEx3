Overview  

This TSG provides detailed information about the Azure Enterprise Agreement (EA) Microsoft Developers Network (MSDN) Dev/Test offer (148p). 

The EA MSDN Dev/Test offer is designed to enable EA customers to make a single monetary commitment to run all their cloud workloads, production or dev/test, on Azure at the best rates.  

Applies To: EA customers with active MSDN subscriptions.

Policy 

EA MSDN Dev/Test (148p) key features: 

  • Offer restrictions. A MSDN benefit is required in order to enable the EA MSDN Dev Test subscriptions. 

  • No separate payment. The EA MSDN Dev/Test offer uses the funds from the Azure Monetary Commitment. EA customers are billed monthly for every SKU, based on the usage of both the standard EA Offer and EA MSDN Dev/Test offer. Under the EA MSDN Dev/Test offer, customers should not enable any other Azure SKUs, even though they may be visible in the Licensing Solutions Partners (LSPs) on the EA price list. 

  • No impact on the MSDN Azure offer. MSDN subscribers have a monthly credit for Azure use, which is separate from the EA MSDN Dev/Test offer. See the details on the MSDN Azure benefit offers page. 

  • Development and testing only. No production workloads should be implemented under the EA MSDN Dev/Test offer. 

  • Use is restricted to active MSDN subscribers, and to end users of the application (to provide feedback or perform testing). There are currently no entitlement checks on the EA portal to prevent non-MSDN subscribers to activate EA MSDN Dev\Test subscriptions due to the "trust" relationship that we have with EA customers.  

    • Note: The above is internal information only, do not communicate lack of entitlement checks to customers. 

  • We strongly discourage customers from converting existing MSDN Azure benefit offers (59p and 63p) into EA MSDN Dev/Test (148p).  Existing MSDN subscriptions should remain separate from that of the MSDN Dev Test offer. 

  • Standard EA offer rates apply to most services under the EA MSDN Dev/Test offer. 

  • Centralized management. Customers can configure and manage this subscription offer via the EA portal, while billing is managed in the Azure Account Portal. 

  • No SLA.  There is no Service Level Agreement (SLA) for any of the MSDN Azure offers, including the EA MSDN Dev/Test offer, and no financial compensation if Azure does not meet the guaranteed uptime for a service. 

  • Developer’s end user access.  End users of an application (i.e. developers) who are added by MSDN subscribers can access the application to provide feedback and performance acceptance test. This is consistent with our MSDN subscription terms of use, which are the only scenarios where a non-MSDN subscriber can access resources running in Azure subscriptions based on this EA MSDN Dev/Test offer. 

  • Single EA monetary commitment. The existing Azure Monetary Commitment prepaid funds are used to pay for the Azure resource usage within subscriptions, so there are no separate payments needed for the EA MSDN Dev/Test offer. 

  • Cloud Services, HDInsight, and Websites are charged at the EA Linux VM rate. In all MSDN Azure offers (MSDN Benefit, Direct MSDN Dev/Test PAYG, and EA MSDN Dev/Test), we charge for Windows VMs—including those running Microsoft software such as SQL Server—at the Linux VM rate. Customers are not paying for the implied software charges since they are already licensed to run this software via their MSDN subscription. Additionally, in all MSDN Azure offers we charge for Cloud Services, HDInsight, and Azure Websites at the Linux VM rate—based on the size and type.

Customer Experience and Provisioning

How to enable MSDN subscriptions under EA accounts

To enable MSDN, the Enterprise Admin will need to select the checkbox in the EA portal. In return, Admins will be able to create EA MSDN Dev/Test subscriptions. Enterprise Administrators add users called Account Owners to their agreement via the EA Portal.

  1. An EA Admin must log into the EA portal (ea.azure.com) and go to Manage Departments and Accounts.
  2. Access the account where they want to enable the Dev/Test MSDN access.
  3. Select the MSDN checkbox for each subscription you will be converting for them.
  4. After the box has been checked, ASMS will need to be contacted to convert the subscription to the 148P (EA Dev/Test) offer.

Support Scenarios 

Convert an offer from Microsoft Azure Enterprise to Microsoft Azure Enterprise MSDN Dev\Test

  1. After the customer has enabled MSDN (by checking the MSDN checkbox), the customer will then contact ASMS to convert any existing subscriptions from EA to EA Dev/Test.
  2. The SE will need to follow the steps outlined in TSG 2647585 on how to convert a subscription in CMAT.
