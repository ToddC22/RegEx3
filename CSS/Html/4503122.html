Summary


Who should follow this procedure?

  • All Microsoft and vendor resources supporting Office 365 Premier cases in Rave, such as Cloud Knows Premier (CKP POD) engineers, Project Beacon engineers, in-house Premier Support Engineers and Premier Escalation Engineers supporting Office 365 cloud and hybrid product.

What is this procedure about?

  • This procedure provides how to transfer case between Premier General and Premier Escalation in Rave.

When should this procedure be followed?

  • Transfer case between Premier General and Premier Escalation.

back to top

Procedure

This feature allows us to move cases between Premier General and Premier Escalation, so that we can ensure that both teams handle the correct customer-scheduled set by routing. The developer approach is to allow engineers to match Rave cases to the current engineer’s Rave profile and not change the user profile to match the cases, easing the overhead needed to assign cases. Any time the Quick Facts contains Performance Support, Mission Critical and Office Engineering Direct, then the case should be moved to Premier Escalation. All other Service names are part of the Classic Premier offerings.

Note: This is only available for Premier Skilled cases.

 

How it works

  1. From the case actions, select Transfer (only available once the case is assigned). 
    [Asset 4503133]
  2. At the case transfer pane, select Customer needs specialized support. 
    [Asset 4503134]
  3. After selection, we will be presented with the available Support Area for transfer. 
    [Asset 4503135]
  4. Then the optional Skills will be presented.
    [Asset 4503137]

    Notes:

    • DAO: The option is presented only to Premier Support teams in pilot phase. By choosing the option, cases will be offered to advocates through Direct Advocate Offer.
    • Non-DAO: The option is presented to PremierEscalation team (as DAO is not yet enabled).
  5. At the Skills selection, we can disable/enable DAO and select it if we want to laterally move tickets (callback, regional transfer, etc.) or vertically move misrouted cases, or a case is in need of another level of resource, such as out of support boundaries, Advisory, etc.
     

    Scenario 1: When a case is identified as in need of another level of resource, such as out of support boundaries, Advisory, etc. or an escalation is requested by Incident manager (IM) or Manager, we can vertically move the case to the Premier Escalation team.

    Note: If we are moving the case vertically to PremierEscalation, we should always select Non-DAO and PremierEscalation, no matter what the language or support Area is.

    Select:
    • Routing: non-DAO
    • Service Offer: PremierEscalation

    Scenario 2: When a case is identified as misrouted to one technology or language, we can horizontally move the case to the correct Support Area/Language.

    Note: Depending on the current Pilot phase, use below options for transfer.

    For WALK phase (current): 

    • If we want to horizontally move an English case to EnterpriseCloudIdentity, EnterpriseCommerce, EnterpriseEDU, EnterpriseExchange, EnterpriseOneDrive, EnterpriseOutlook, EnterpriseSecurityCompliance, EnterpriseSharePoint, EnterpriseSkype, EnterpriseTeams, EnterpriseYammer and Kaizala support areas, we should select:
      • Routing: DAO
      • Service Offer: Premier Support
    • If we want to move a case of any other language, we should select:
      • Routing: non- DAO
      • Service Offer: Premier Support
    For Run phase (ETA May 7th):
    • If we want to horizontally move any Language case (except Korean and Japanese) to EnterpriseCloudIdentity, EnterpriseCommerce, EnterpriseEDU, EnterpriseExchange, EnterpriseOneDrive, EnterpriseOutlook, EnterpriseSecurityCompliance, EnterpriseSharePoint, EnterpriseSkype, EnterpriseTeams, EnterpriseYammer and Kaizala support area, we should select:
      • Routing: DAO
      • Service Offer: Premier Support
    • If we want to move a case of languages Korean or Japanese, we should select:
      • Routing: non- DAO
      • Service Offer: Premier Support
  6. Provide a Reason for Transfer:
    1. We should add the transfer reason which will soon be shown on the Description of the case at the unassigned view (for non-DAO transfer cases).
    2. Hashtags are to be used to report transfer reasons. Some hashtags are suggested but we can create new ones as needed. Please sync with your TA on best practices and refer to the Transfers Do’s and Don’ts (Transfers Rules). 
      [Asset 4503138]
  7. Finally press Transfer to complete the process.
  8. A new RequestID will be created for the case, and the case will be routed to DAO or seen at Unassigned view (depending on selection at bullet 5).
  9. The new RequestID will contain the ticket Role, Language, Channel, Ticket Type, Region and contain selected Support Area, selected routing and selected Service Offer for NASE to pick it up.
