Azure Support - Creating A Grace Case

Overview

This TSG provides guidance about creating a grace case. A grace case is the process used when ASMS SEs need to create a ticket (or grace case) on their customer’s behalf.

Note: Before creating a grace case, be sure to follow Scenario 3 under Technical in TSG 3104062.


Support Scenarios

Create an ASMS Grace Case

To create an ASMS Grace case, perform the following steps:

  1. From the Service Desk Case (SDC) Home page, click +Case (in the top right corner).
  2. Search by the customer’s email address.
  3. If the customer has a pre-existing profile, select and attach it to the new case. If not, create a new customer profile to attach to the case.
  4. Update the Support Area Path (SAP) and fill in the required fields in the new SDC.
  5. Assign the relevant eligibility.

    Note: The case is automatically assigned to the SE who created it. If needed, the SE must reroute the ticket to the relevant queue.

Note the following about grace cases:

  • Only create SRs that you or your team will work: 
    • The Technical team creates its own SRs.
    • ASMS teams create their own SRs.
  • You can only create a professional case. If the customer wants to increase the entitlement, they will need to reach out to their IM+.

If the previous Note needs to be deviated from, collaboration with the team the SR would be transferred to is REQUIRED:

  • Explain why the SR is a grace case.
  • For queues outside of Technical/ASMS, ensure managers (of SR creator and collaborating engineer) are engaged if pushback for recovery purposes.
