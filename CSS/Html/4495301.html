Informational: Azure Partner Admin Link (PAL)

[Asset 4090831]

Summary

Who should use this article

This article applies to partner frontline advocate (RSC) teams from all regions.

What’s the article about

Introduce a new Azure feature of Azure Partner Admin Link (PAL) that allows partner organizations to associate their Microsoft Partner Network (MPN) ID with the credentials that they use to deliver technical services to customers. PAL provides a method for Microsoft and partners to better track partner engagements with customer Azure consumptions.

1. Product/Process Overview

  • What is PAL?

To demonstrate Azure-consumed Revenue impacted by a Partner doing essential work on behalf of other organizations, the Azure feature ‘Partner Admin Link’ is available now which can be applied in the Microsoft Partner ecosystem. Link MPN ID (aka: Partner ID) to the credentials or accounts that are used to create Resources in the customer’s Azure tenant (requiring an RBAC role*). Consequently, all accounts used to create resources on customer’s engagements must be tagged to Microsoft MPN ID. Through this association at the account level, Microsoft helps our partners to track the customer engagements and attribute our partner‘s influence on a customer’s Azure consumptions, relying on an enhanced reporting, insights and a product-based telemetry with optimized precision. 

  • Prerequisite for Partners using the PAL on the Azure Portal

    • A mutually-signed agreement between the customer and partner
    • Credentials granted to partner with the admin access (RBAC) to Azure resources
    • Partner ID linked to the customer’s credentials
    • Revenue generated out of the Azure consumptions

  • How to Link a Partner ID to Azure Account

    • Step 1, the partner gets access from customer to the customer's Azure resources. A role-based access control (RBAC) role must be assigned to the partner. For more information about the legacy admin model, and RBAC role model, and the difference, refer to Classic subscription administrator roles, Azure RBAC roles, and Azure AD administrator roles.
    • Step 2, After get access to customer Azure resources, partner then use either Azure portal, or PowerShell, or Azure CLI to link their MPN ID with Azure account.

For detailed steps and FAQ, refer to the article Link a partner ID to your Azure accounts.

Note: PAL association is not available for mainland China Partners but is available for Hong Kong and Taiwan partners. Also, the PAL attribution is unavailable for Government tenants.

  • Report

Reports will be provided for customers, partners, and Microsoft sales. Partner can find the report for customer usage attribution in Partner Center Analyze dashboard, MPN MyInsights, and Partner Insights BETA Azure Usage Report.

  • Incentives and Competencies

  • Digital Partner of Record remains the method for receiving financial rewards on an EA
  • Partner Admin Link based financial incentive went live March 2019
  • Competency credit: influenced revenue as measured by PAL counts towards Cloud Platform and competency. Frontline advocates should encourage partners to check the requirements under the desired competency on both Partner Center Dashboard and MS partner portal for understanding the achievement conditions applying for each competency (Partner association type, Offer type, TTM). Unless the partner association type is specifically mentioned as part of the requirements list under the claimed competency, this will not be recognized towards meeting the performance goals.
  • The PAL association applies at workload level, hence multiple partners receive azure consumption for competencies for the workloads that they provide hands-on services for their customers. This means that multiple partners will have PAL association for a single subscription, but for different workloads.

Important: Because of how PAL association applies, we should not ask for MS invoices for partners who claim missing PAL Azure consumption revenue.

Also, PAL is currently a partner association type considered towards meeting the performance thresholds under Azure Expert MSP Program and Advanced Specializations such us: SAP on Azure, Windows Server and SQL Server Migration to Microsoft Azure and PAL values are tracked in trailing twelve months (TTM).

2. FAQ

Q1: Who can create and revoke a PAL link?

A1: Any user from a partner organization who is managing a customer's Azure resources can link a partner MPN ID to the customer’s environment. The MPN ID can be added, changed, or removed only by the partner organization that created the link. The customer, or another partner, cannot change MPN ID associated via the PAL. However, a customer can revoke PAL access at any time.

Q2: How do I create a PAL link?

A2: A partner completes a PAL association when linking their organization’s MPN ID to the admin credentials granted to you by a customer to deliver services in the customer’s Azure environment. A customer may grant you admin credentials to Azure resources in three ways: a local domain account, a guest user account, or as a Service Principle.

The Azure Management Portal offers an intuitive user interface to link an MPN ID to a customer’s Azure environment: 

  1. Go to portal.azure.com and login to the customer tenant
  2. Click on the portal settings icon at the top of the page
  3. Select Link your partner ID to this Azure account
  4. Enter your MPN ID

Partners can also use PowerShell and Command Line Interface (CLI) to create a PAL association. Follow these instructions for PowerShell:

  1. Install the AzureRM.ManagementPartner PowerShell module to run PowerShell commands
  2. Login to the customer tenant with your user account or service principal
  3. Link your MPN ID to the customer tenant

Additional PAL-related documentation, including helpful how-to videos and step-by-step linking instructions, is available in the following locations: 

Partner Admin Link collection in CSP galleries

https://docs.microsoft.com/en-us/azure/billing/billing-partner-admin-link-started

Q3: Do I need to create a PAL association for every customer?

A3: Yes, Partners must link an MPN ID to their Azure admin credentials for every customer tenant in which they are delivering services. PAL associations are generated on a per-customer tenant basis and are not inherited by other customer environments serviced by the same partner. If more than one contact from a partner organization has admin credentials in a single customer environment, the MPN ID should be associated to each set of credentials in order to enable full credit for influence on that customer’s Azure consumption.

Q4: My organization uses multiple MPN IDs. Which one should we use for PAL?

A4: Only MPN IDs for Location or Headquarters are considered valid for PAL – virtual organization IDs are not accepted. A partner should use the MPN ID they wish to receive credit for when influencing the customer’s Azure engagement. It is possible to use different MPN IDs across the various customer Azure environments being serviced by the partner.

Q5: Can more than one partner influence the same subscription?

A5: Yes, PAL enables multiple partners to earn recognition and incentives for influence on Azure consumption within the same customer subscription. This capability, which is not supported in the Digital Partner of Record (DPOR) program, aligns with many real-world scenarios in which two or more partners are delivering services in the same subscription – for example, to support different LOB apps, or different specializations such AI, data management, or support.

Q6: Is PAL reporting available?

A6: Yes, PAL influence reports are available in Partner Center via the Partner Insights BETA Azure Usage Report. Click here to learn more. 

In this dashboard partners can view PAL attachment information by filtering on “Partner Attribution’s” and selecting Partner Admin Link. Results will include

  • Azure subscriptions
  • Active Azure subscriptions
  • ACR
  • Customer geography coverage

PAL reports are also available to the Microsoft partner account teams and field sales teams. The reports give Microsoft visibility into partner impact in customer engagements. They can help partners showcase the value of services delivered and drive conversations about collaborative or co-sales opportunities with Microsoft.

Q7: What happens if a customer changes a partner’s admin access rights?

A7: The PAL association is tied to a partner’s admin credentials within a customer’s Azure environment. If the customer changes a partner’s scope of access to resources, for example, or removes the partner’s admin credentials from the account, the PAL association ends. If the partner continues to deliver services in the account under a revised scope of access, the partner should create a new PAL association based on the revised admin credentials.

Q8: Will Azure consumed revenue (ACR) driven by PAL association be calculated retroactively, or will PAL association only be calculated from the date that PAL association becomes/became eligible for competency?

A8: ACR is tracked over trailing 12 months. Therefore, if a partner had already associated their MPN ID to PAL when PAL becomes/became eligible for competency, their ACR will be calculated retroactively, up to a period of 12 months.

Q9: How will PAL count toward competency attainment, and when does it start?

A9: Starting June 2020 we are adding Partner Admin Link (PAL) as an association method for meeting the Azure Consumed Revenue (ACR) requirement for the Gold or Silver Cloud Platform competency, Azure Consumption option.

Q10: When calculating the trailing 12-month revenue for a partner who has DPOR one month, and PAL another month, how is the calculation made?

A10: For a partner where both DPOR and PAL is associated, when calculating the 12-month trailing revenue, only PAL will be calculated. Until PAL is associated, DPOR will be used. Once PAL is associated, DPOR will not be counted, and PAL will be used. For example, a partner has DPOR in January 2019 through April 2019, then has PAL from May 2019 through December 2019. The trailing 12 months will leverage 4 months of DPOR revenue and 8 months of PAL revenue.  Both attribution types remain associated, however, only one attribution type will count on a monthly basis.

Q11: Is PAL available for partners within cloud Azure China by 21Vianet?

A11: Partners set up within cloud Azure China by 21Vianet are not able to associate their MPN ID to PAL to accrue ACR at this time. We do not have a timeline to share for when this will be available.

Q12: I added my MPN ID to PAL but I’m not seeing ACR being recognized in the Partner Center reporting I have access to.

A12: A partner receives revenue attribution and customer association if all of the following are true:

  • MPN ID is attached to credentials
  • Credentials have admin access to resources
  • Resources have consumed revenue

Common reasons that PAL association is not made:

  • Admin role from legacy model rather than RBAC
  • Credentials are member of CSP Admin Agents group (special role within the CSP tenant)
  • Customer subscription is sponsored or internal Microsoft
  • Credentials do not have admin rights within customer tenant
  • User does not authenticate into customer tenant before setting PAL
  • Resources within account scope have not generated any consumption revenue

There is a legacy admin model and the newer Role Based Access Control model, and PAL requires an RBAC role.

  • This article explains the admin role model, the RBAC model, and the difference.
  • This article provides instructions to add an RBAC role to an account.
Q13: Is PAL available for use by CSP partners?

A13: PAL can be used by partners to track their influence on Azure consumption, regardless of how a customer purchases Azure. Enabling PAL is not currently needed for partners who sell Azure to customers through CSP, because the partner association to the customer is established by default with the CSP purchase agreement. However, a second partner subcontracted by a CSP partner to deliver services in a customer subscription can enable PAL to track their influence on Azure consumption, but not to earn incentives.

Q14: If a CSP direct bill partner sells Azure and the customer engages a second partner for services, can the second partner use PAL?

A14: Yes, this is an important scenario that PAL supports. For example, an ISV partner that manages an application on an Azure subscription purchased from a CSP reseller can enable PAL to gain recognition for influencing consumption in the subscription, but not to earn incentives.

CRC Codes

 

 

Partners fail to associate PAL successfully

Confirm the following:

  • Partner is associating via Link to a partner ID in the Azure portal.
  • Partner user has Customer Admin role on Customer tenant.
  • Credentials are member of CSP Admin Agents group.
  • Partner user is authenticating into Azure portal with partner credentials (as per AOBO scenario).

Partner is not getting consumption recognition for associated customer

  • Customer subscription is not sponsored or internal Microsoft.
  • Resources within account scope have generated consumption revenue.
