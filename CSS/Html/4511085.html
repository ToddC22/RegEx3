ASMS – Port 25 Enablement

Submit a ModTrack request to update this TSG.

Overview

This TSG describes how and when to enable port 25 for customers who request it. Ports provide the interface computers need to communicate with a computer network. Port 25 is often used for unauthenticated email delivery. Therefore, its use is generally blocked on Azure subscriptions.

Example: A customer has setup a mail server in Azure that sends traffic outbound on TCP port 25, and has found that they are unable to connect to any outbound SMTP resources from Azure on port 25.

Policy

Since November 2017, sending outbound email directly to external domains (such as Outlook.com) from Azure virtual machines (VMs) or network virtual appliances (NVAs) using port 25 is only available to specific offer types. (See the Identify Offer Type section for more details .) Outbound SMTP connections that use TCP port 25 are blocked on all other Azure subscriptions.

Note: Port 25 restrictions do not affect subscriptions created before November 2017.

Before enabling Port 25, HIT must be engaged to validate the subscription. Customers' subscriptions must be on the list of types of subscriptions that can be enabled, and the subscriptions must be manually enabled via ASC. Only if the subscription is validated can the port be unblocked.
See sections Customer Refuses SMTP Relay and Enable Port 25 for a Subscription with Azure Support Center (ASC) for more details. 

How to Request Access to Enable Port 25 for Subscriptions

Membership to the MyAccess group azfraudportunblock is required to remove the port 25 restrictions from a customer subscription.

Note: All ASMS SEs will be auto approved for the MyAccess group: azfraudportunblock.

If you do not have access you may submit an access request:

  1. Navigate to MyAccess.
  2. Click Request Access.
  3. Search for azfraudportunblock.
  4. Select azfraudportunblock.
  5. Click Next, then Submit.
  6. Select the Permission field button and provide the following justification:

I am a member of the CSS Azure Subscription Management Support team, which needs the ability to set an override subscription feature flag AllowTcpPort25Out which implements an anti-fraud measure to avoid platform abuse.

  1. Click Next.
  2. Review the Request Summary.
  3. Click Submit Request.

You will receive a notification email after the request has been submitted and another when it is approved. Approval may take up to 24 hours.

Support Scenarios

Port 25 Case Overview

The following sections detail how ASMS receives port 25 cases, and how support engineers (SEs) should handle these scenarios.

Method 1 – Customer from Azure Portal

The customer submitted a support request using the Azure Portal and chose Virtual Network > Connectivity > Cannot send email (SMTP-Port 25).

  1. If the subscription qualifies for enablement, an automatic diagnostic runs a security check .
    1. If the security API check comes back as Validated: the customer is notified and directed to use the portal to stop and start all VMs and NVAs which handle outbound traffic, then test port 25.
      Note: Do not restart as this will not apply the port 25 enablement.
    2. If the security API check comes back as Manual validation required: The customer is notified that their subscription qualifies, but they must open a support request for enablement (see step 3a).
    3. If the security API comes back as Fraudulent: The customer does not receive a notification. An internal Azure Support Center (ASC) Insight directs the SE to close the case without contacting the customer. The subscription is disabled per fraud policy.
  2. If no Insight is shown, report an outage in ASC feedback.
  3. If the subscription does not qualify for enablement, the customer is notified and offered a workaround to change their offer type through this process:
    1. Verify the customer wants port 25 opened.
    2. Validate and update the support topic to Virtual Network > Connectivity > Cannot send email (SMTP-Port 25).
    3. Determine the customer’s offer type and eligibility with a Jarvis Action query: Azure Resource Manager> Subscription State Management> Get subscription.
    4. Identify the Offer Type and follow the applicable guidance.

Method 2 – Queue Transfer to ASMS

A support request is transferred from another team to the SMTP Port 25 queue. Check the Overview page in ASC for Insights. If no Insights for Port 25 are displayed, follow this workflow:

  1. Select Resource Explorer.
  2. Click Resource Providers.
  3. Click the Virtual Networks drop-down menu.
  4. Select the virtual network to investigate.
  5. Select the resource to investigate.
  6. Click the Insights tab.
  7. Click Run Insights.
    [Asset 4526125]

Results from Running ASC Port 25 Insights Manually

Follow the below guidance based on what the Insights header displays for SMTP Outbound Traffic:

  • Qualified and Automatically Enabled on TCP Port 25 – The customer has been automatically enabled. They should use the portal to stop and start all VMs and NVAs which handle outbound traffic, then test port 25.
  • Already Enabled on TCP Port 25 – The subscription was already enabled. Customers should follow the instructions in the Insights to stop and start all VMs and NVAs from the portal, then test port 25.
  • Denied on TCP Port 25 – Fraudulent activity determined. Insights instructs to close the case without contacting the customer. The subscription is disabled per fraud policy.
  • Qualified to be Enabled on TCP Port 25 – The subscription qualifies for enablement, but the manual approval process must be followed.

Note: Customers should not restart as this will not apply the port 25 enablement.

Test Traffic in Azure Support Center (ASC)

Test Traffic can be used to verify if network traffic is blocked from port 25.

  1. Find a VM resource using Resource Explorer in Azure Support Center (ASC).
  2. On the Diagnostic tab, select Test Traffic.
  3. Use the defaults, but make these updates (see image below):
    1. Destination IP to 8.8.8.8
    2. Destination Port to 25
  4. Click Run.

Pay-as-You-Go (PAYG), Microsoft Partner Network (MPN), Azure in Open (AIO)

For customers who request port 25 enablement and have PAYG, MPN, or AIO offer types, follow the below guidance.

Initial Response

Respond to the customer using their preferred communication method to let them know you understand their request.
Include the following information:

  • Port 25 is used to send email to external domains and is restricted due to the elastic nature of public cloud service IPs and the potential for abuse.
  • An SMTP relay service, such as SendGrid, is recommended as an alternative to port 25 enablement.
  • If there is a need that an SMTP relay cannot meet and requires port 25 enablement, it will take 3 to 4 business days for an assessment and response.

SendGrid SMTP Relay Configuration

Customers should contact SendGrid directly if they need assistance configuring it as their SMTP relay. Let your customer know there are additional costs associated with SendGrid.

Customer Refuses SMTP Relay

If the customer does not want to use an SMTP relay and still requests port 25 enablement, follow the steps below:

Note: The steps below must be completed before Port 25 can be enabled. The customers' subscription must be on the approved list of subscription types that can be enabled, and the subscription must be manually enabled via ASC (if that fails it must be validate by HIT). Only if the subscription is validated can the port be unblocked. 

  1. Submit the Port Unblock Request form.
  2. The Human Intelligence Team (HIT) will receive the request and initiate a fraud investigation. This may take up to 4 business days.
  3. The HIT team will respond with a Deny or Approval in the Status column on the Port Unblock Request submission page.
    • Deny – Fraud was determined, the subscription will be deactivated.
      • All communication with the customer should be terminated.
      • Close the support request (SR) with root cause: Subscription Management > Port 25 unblock request > Disapproved.
      • If the customer wants to appeal, they can open a support case through the portal. A specially trained agent will handle the appeal process.
    • Approval – Enable port 25 following the Enable Port 25 section.

Note: If you receive a Forbidden message on the Port Unblock site, verify that you are a member of azfraudportunblock MyAccess group. Request access if you are not a member using the process outlined in the How to Request Access to Enable Port 25 for Subscriptions section.

If you are a member of the group and still cannot access the form, submit an IcM request and email HIT a link to it.

Cloud Solution Provider (CSP)

For CSP customers who request port 25 enablement, follow the below guidance.

Initial Response

Respond to the customer using their preferred communication method to let them know you understand their request. Include the following information:

  • Port 25 is used to send email to external domains. It is restricted due to the elastic nature of public cloud service IPs and the potential for abuse.
  • An SMTP relay service, such as SendGrid, is recommended as an alternative to port 25 enablement.
  • If there is a technical need that an SMTP relay cannot meet and requires port 25 enablement, the request can be reviewed for an exception.

SendGrid SMTP Relay Configuration

Customers should contact SendGrid directly if they need assistance configuring it as their SMTP relay. Let your customer know there are additional costs associated with SendGrid.

CSP Customer Refuses SMTP Relay

  1. If the CSP customer does not want to use an SMTP relay and still requests port 25 enablement, send them an email which includes the following points:
    • The customer subscription may be approved for port 25 enablement.
    • Warning: Cloud Solution Providers are frequently targeted by fraudulent activity and can result in losses for both the CSP and Microsoft.
    • If the customer continues with the request to unblock port 25 they should acknowledge the above stated risk and liability of fraud which was agreed to in the CSP contract.
    • CSPs with port 25 enabled should proactively verify their customers through industry standards, such as credit identity matching.
  2. If the CSP customer agrees to the above, they must reply to the email to acknowledge the risks of port 25 enablement and continue their request.
  3. Ensure the email communication is added to the support case in Service Desk.
  4. Follow the steps in the Enable Port 25 for a Subscription.

EA and MS Internal Customers

Enterprise Agreement (EA) and Microsoft internal (AIRS) customers are not blocked by default. Follow these steps for customers with these offer types:

  1. Follow the steps in Test Traffic to verify if port 25 is disabled on the subscription.
  2. If it is disabled, follow the steps in Enable Port 25 to enable it.
  3. If this procedure fails, engage your technical advisor (TA), partner technical advisor (PTA), or team lead (TL) for assistance as appropriate.

Subscriptions Converted to EA

When the Jarvis Action Azure Resource Manager > Subscription State Management > Get subscription shows the subscription is EA but port 25 is still blocked, the subscription may have been converted to EA from another offer type. To verify, follow these steps:

  1. Use Jarvis Action NRP > NRP Subscription Operations > Get NRP Subscription Details with the relevant region and subscription.
  2. Find the value for key: /subscriptions/<subscription id>/resourceGroups//providers/Microsoft.Network/subscriptions (see image below).
  3. Inspect the value properties and quotaId (see image below).
  4. If quotaId is a blocked offer type, manually enable the subscription per the Enable Port 25 section.
    [Asset 4526129]

US Government (Fairfax) Cloud

US Government Cloud customers should be enabled immediately without fraud check or further questions asked.
Follow the below steps to enable port 25 for these customers:

  1. Verify the subscription as a US Government Cloud subscription.
  2. Once verified, follow the steps in the Enable Port 25 section .
  3. Ask your customer to use the portal to stop and start all VMs and NVAs which handle outbound traffic, then test port 25.
    Note: Do not restart as this will not apply the port 25 enablement.

Note: If you run into JIT failures when enabling Port25 for a Fairfax customer, submit an IcM to CloudNet Engineering Team: https://icm.ad.msft.net/imp/v3/incidents/create?tmpl=K2s42j.   

Global Azure in China (Mooncake)

Fraud checks for Global Azure in China (Mooncake) customers can be requested by submitting a Service Desk Collaboration task to the following queue:

  • Service Desk Site: Microsoft Management
  • Service Desk Team: RiskOps Service Team

In the task, be sure to explain that a Global Azure in China (Mooncake) customer is requesting Port 25 unblock. Ask that the RiskOps Service team review the customer for fraud and verify whether the unblock can proceed. Also include the customer's subscription GUID in the task.

This team has a 15-minute SLA. They will respond with one of the following:

  • If fraud concerns do not arise during the team's review, they will complete the task and reply in the task notes that the unblock can proceed.

  • If fraud concerns do arise during the team's review, they will complete the task and reply in the task notes that the unblock should not proceed. They will then take ownership of the case.

All Other Blocked Subscriptions

As of November 15, 2017, customers who create MSDN, Azure Pass, Education, BizSpark, and Free Trial subscriptions have technical restrictions blocking email sent from VMs directly to email providers to prevent abuse. No requests to remove the restriction can be made as they will not be granted.

See Outbound SMPT Issues in Azure for more information.

Customers may convert their subscriptions to PAYG as a workaround to have port 25 enabled.

Note: Customers on Free Accounts who upgrade to PAYG must remove the Azure spending limit for the PAYG subscription to be enabled in Jarvis.

Upgraded Account Showing as Free Trial

Follow these steps if the customer has upgraded their offer type, but Jarvis shows the offer type as: Azure_Platform_All;Azure_Direct;Azure_Consumption;Azure_Free_Trial;Azure_MS-AZR-0044P;Azure_Credit;Individual.

Subscription upgrades cannot be completed until any remaining credit has been used. There are two options to assist customers in this situation to enable port 25:

Option 1 – Wait

The customer may wait until they have used their credit and then submit another request for port 25 enablement.

Option 2 – Manual Upgrade

The customer can remove the spending limit and we can manually upgrade the customer’s subscription.

Note: Customers who choose this option will lose any remaining credit on the subscription and must provide the following information in an email:

  1. A screenshot of the subscription from the Azure Portal showing Pay-As-You-Go.
  2. A screenshot showing the spending limit has been removed.
  3. Written acknowledgement that all remaining credit will be lost.

Verify if Port 25 is Already Enabled in Jarvis

ASC may fail to enable port 25 on the subscription and generate an error message:

Error: Cannot set subscription feature flag to enable port 25. Please contact Cloudnet/VNET team for support.

Before you engage your TA, PTA, or TL to enable the feature in Jarvis, confirm whether port 25 is already enabled with this operation: Azure Resource Manager > Feature Management > Gets all features in subscription. If the output doesn't match the example below, please ask the customer to register the Microsoft Network resource provider. Once the customer confirms that it has been registered, please try the operation again via Azure Support Center.

Example Output:

{ "properties": { "state": "Registered" }, "id": "/subscriptions/1234ab56-7890-1234-567c-89d0e1f23456/providers/Microsoft.Features/providers/Microsoft.Network/features/AllowTcpPort25Out", "type": "Microsoft.Features/providers/features", "name": "Microsoft.Network/AllowTcpPort25Out" }

To check if the subscrion is registered for Microsoft.Network.RP use Jarvis and follow the path outlined below: 
Azure Resource Manager > Resource Provider Management > Get resource providers for a subscription
 

Enable Port 25 for a Subscription with Azure Support Center (ASC)

Note: CSP subscriptions are not in scope for fraud checks.

Before enabling Port 25, HIT must be engaged to validate the subscription. Customers' subscriptions must be on the list of types of subscriptions that can be enabled, and the subscriptions must be manually enabled via ASC. Only if the subscription is validated can the port be unblocked. 

 

Once the customer has been approved by the Human Intelligence Team (HIT), or an authorized exception per this TSG, follow these steps to enable Port 25 with Azure Support Center:

  1. Click Resource Explorer.
  2. Click the subscription GUID.
  3. Click the Diagnostics tab.
  4. Select Yes in the Unblock Port 25 box.
  5. Click Run.
  6. Once the operation is complete, ask your customer to use the portal to stop and start all VMs and NVAs which handle outbound traffic, then test port 25.
    Note:  Do not select restart as this will not apply the port enablement.
    [Asset 4526130]

Customer Requests to Block Port 25 on their Subscription

In such cases, the customer can re-enable themselves via NSG. The original block is just a system NSG. If the customer wants to block the traffic again, they just need to apply an NSG to their network(s). For more details on this process, reference this public-facing documentation.

Note: If the customer needs help getting the NSG configured, engage the Networking Technical Team.

Manual Enablement – TA Only

Follow these steps to manually enable port 25 if the process failed in ASC.

Ensure the area path is Azure/Virtual Network/Connectivity/Cannot send email (SMTP/Port 25) to prevent a “Site cannot be found” error.

Note: These operations require proper letter case. Ensure that your operation reads as AllowTcpPort25Out.

Submit an IcM – TA Only

  1. Submit JIT IcM for ACIS Elevation for both NRP and RDFE (if necessary).
    1. Wait 10-15 seconds, then refresh the IcM .
    2. Click Acknowledge.
  2. Azure Resource Manager (ARM) Deployments – Set the NRP feature flag on the subscription in Jarvis.
    1. Azure Resource Manager > Feature Management > Approve Feature Registration
    2. Resource Provider Namespace: Microsoft.Network
    3. The Feature Name: AllowTcpPort25Out
    4. Subscription: <Subscription Id>
  3. “Get Access ” for this operation.
    1. Select access level, auto approved: AFEC > PlatformServiceOperator.
    2. Work-item id: Enter the IcM number (from step 1 above).

RDFE Port 25 Enablement Process – TA Only

Skip the RDFE Port 25 Enablement Process if your customer has no classic deployments.

  1. Perform a Jarvis Query:
    1. AzureRT > Privilege Management > Update Feature Status
    2. Subscription: <Subscription Id>
    3. Feature Name: AllowTcpPort25Out
    4. Status: Active
    5. Approver: <CSS SE alias>,<TA approver alias>
    6. Approver Link: <ICM Link>
  2. “Get Access” for this operation.
    1. Select access level, auto approved for ANP TAs: Subscription > CustomerServiceOperator.
    2. Work-item Id: Enter the IcM number (from step 1 in the “Submit an IcM” section).
       

Unblock Existing Deployments – TA Only

After completing the above process, follow these steps to remove the port 25 restrictions from existing deployments:

  1. Have your customer stop and deallocate public-facing resources (firewalls, email servers, etc.), and start all outbound VMs/NVAs to enable port 25 (do not select restart as this will not apply the port enablement).
    Note: Customers must use a static public IP to keep their public IP addresses.
  2. Close the IcM.
  3. Notify the customer that port 25 has been enabled for their subscription and confirm the resolution.
  4. Close the SR with root cause: Virtual Networks > Virtual Networks > Port 25 unblock request > Approved

Confirm Subscription Feature Flag

For NRP

Check the feature registration here: Azure Resource Manager > Feature Management > Gets all features in subscription.

For RDFE

  • Use a Jarvis Query to search for AllowTCPPort25Out to check the subscription feature flag.
    • If the feature is blocked, you will see IsAuthorized:False.
      [Asset 4526131]

Operation Logs

Both NRP and RDFE operations create large log trails. Review these logs before filing an IcM for a feature flag not updated in NRP or RDFE.

NRP Logs

  • Namespace: NRP
  • Table: FrontendOperation
  • Recommended filters:
    • OperationName == PutSubscriptionOperation
    • SubscriptionId

RDFE Logs

  • Namespace: RDFE
  • Table: UpdateFeatureStatusOperation
  • Recommended filters:
    • FeatureName == AllowTcpPort25Out
    • SubscriptionId
