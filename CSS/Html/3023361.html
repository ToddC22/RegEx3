Summary

Who should follow this procedure?

  • Only those who have passed the Microsoft 365 Data Protection Submitter Training.

What is this procedure about?

  • This article describes when and how to perform a password reset or a multi-factor authentication (MFA) reset for a Company Admin (CA) or Global Admin (GA) who is unable to sign in.

When should this procedure be followed?

  • When a Company/Global Admin is unable to sign in to their Office 365 account.

Back to top

Important Notes:

  1. Support should NEVER reset a password on behalf of a customer who can do self-service.
  2. If there is more than one Global Admin/Company Admin on the account, direct the caller to the other Global Admins/Company Admins for assistance.
  3. If the only GA/CA is unavailable, a user or non-GA/CA may be elevated to GA/CA via 2887514: Office 365 Elevation procedure.
  4. Write case notes in English.
  5. Important note: All passwords must be delivered verbally – NO EXCEPTIONS.
  6. Additional information on Generic Admin can be found in article 3023334: DPET How to perform entitlement in Office 365.
  7. If the request is related to an MFA reset, the password will not be reset; instead, the system will allow a new phone number to be used as one of the forms of verification.

Procedure

Step 1: Entitle the caller

Entitle the caller to verify GA/CA’s status via 3023334: DPET How to perform entitlement in Office 365.

Step 2: Recommend Self-Service Password Reset (SSPR)

If the GA/CA status is fully entitled and there is a valid, alternate email address and a functioning phone number listed, confirm with the customer if SSPR was tried; if not, direct the caller to the “Can’t access your account?”. If already attempted, document why it did not work in case notes.

Step 3: Review the Tenant

  • If the Office 365 tenant domain is federated: See: Handling federation issues.
  • If a government tenant: Do not reset passwords for customers with a Government tenant, but engage the deleads@microsoft.com alias for transfer instructions.
    • In order to confirm if it is a government tenant, run a DiscoveryAPI check against AAD:
      • Copy and paste the following link in any browser: https://login.microsoftonline.com/contoso.onmicrosoft.com/.well-known/openid-configuration.
        • contoso.onmicrosoft.com should be replaced with the tenant (Initial Domain) provided by customer and hit enter.
        • You will get a code.
        • Using CTRL + F; Look for “tenant_region_scope” and check the next word: If “USG” = (Fairfax AAD), if “USGOV” = (Arlington AAD), or if anything else = (The tenant is Commercial and should be treated as any other tenant).
  • GoDaddy.com: Services purchased through GoDaddy have a domain of NETORG123.onmicrosft.com, where “123” is a multi-digit number. For GoDaddy customers, redirect the caller to GoDaddy support: http://support.godaddy.com.
  • DAP Partners: Check if there is a listed partner with DAP enabled; If so, ask the customer to get support directly from the DAP partner and provide the partner’s contact information listed on this tab.
    • If customer does not want to contact the partner and refers to a dispute, stop providing support and explain the terms and conditions when adding a DAP partner, you can refer to the following link: https://legal.office.com/en-US/docid28.
    • If there is no dispute, but the customer does not want to contact the partner, assist the customer with the request as you normally would.

Step 4: Verification Procedure

Single Point Verification (SPV)

SPV is used for scenarios when there is:

  • Only 1 GA listed.
  • Total assigned license count is less than 50.
  • Requester is fully entitled.

In the following order, obtain 1 point of verification:

  1. GA/CA’s primary telephone number
  2. GA/CA’s mobile phone number
  3. Telephone Number from Company Profile Page
  4. GA/CA’s primary email address and/ or GA/CA’s alternate email address
  5. Obtain permission from the Non-GAs (different from the requestor) OR the Technical Contact (different from the requestor)
    • Billing Administrator
    • License Administrator
    • Security and compliance Administrator
    • User management Administrator
    • SharePoint Administrator
    • Exchange Administrator
  6. DNS TXT Record

After obtaining 1 form of verification, initiate the password reset or MFA reset in the system.

If the account lists more than 1 GA:

Ask the caller to work with another GA for password reset. If the caller claims that he/she is unable to work with them, Data Protection will independently verify the availability of the other existing GAs/CAs as part of our security requirements. Refer to Availability Request for Multiple GAs.

If there is no response and/or there is a bounce back email, nor you are able to confirm via phone that there are other GAs/CAs available to reset the requestor GA/CA’s password, follow the proper Standard Verification procedure.

Standard Verification

Use Standard Verification in any of these scenarios:

  • If the tenant lists more than 1 GA/CA.
  • Total assigned license count is more than 50.
  • Requester is not fully entitled.

In the following order, obtain 2 points of verification:

  1. GA/CA’s primary telephone number
  2. GA/CA’s mobile phone number
  3. Telephone Number from Company Profile Page
  4. GA/CA’s primary email address and/or GA/CA’s alternate email address
  5. Obtain permission from the Non-GAs (different from the requestor) OR Technical Contact (different from the requestor)
    • Billing Administrator
    • License Administrator
    • Security and compliance Administrator
    • User management Administrator
    • SharePoint Administrator
    • Exchange Administrator
  6. DNS TXT Record

After obtaining 2 forms of verification, initiate the password reset or MFA reset in the system.

Verification failure

If there is none or only one form of verification obtained from the Standard Verification, additional verification is required.

For customers located in the United States: Office 365 Data Protection team members permissioned with approval rights may request (via LMI only) the customer to share a screenshot of their government issued ID, including passport or driver’s license. This is only applicable for named admin accounts.

For customers located outside of the United States, or request to validate generic admin accounts, please escalate the case to TA team for resolution. See Step 2 in 2714453: Office 365 How to get account ownership validation. If TA returns verification, initiate password reset or MFA reset. In case notes, use Admin Password Reset template. For MFA reset, use the same template as Password Reset and change “Admin Password Reset” to “MFA reset”.

How to initiate a password reset or MFA reset in the system

Complete the following template:

Use these templates in the non-customer-facing case notes used by the approver who will be reviewing the case.

Verification template note to approver:

  • Single point verification, Standard Verification or Verification Failure (choose accordingly)
  • Number of GAs
  • Number of assigned licenses
  • Form(s) of verification used

Deliver the temporary password per the GA/CA’s preferred phone number.

How to Verify Account Ownership

Phone Verification (primary, mobile, or company telephone number): A phone is considered verified when the Data Protection SE dials the phone number listed under the GA account or company profile and speaks directly with the requestor (Global/Company Administrator). Please note that only 1 of the phone numbers can be used for verification (primary phone or mobile or company telephone number).

Email Verification (primary or alternate): Inform the customer that Microsoft will send an email message to the alternate email address and primary email address on file and that a reply to that email is required for approving the password reset request or MFA reset, and should include the telephone number for delivery of the temporary password.

  1. Send an email message to the alternate email address and primary email address, using Standard Verification Email Template below (translated versions are available).
  2. When a reply is received, verify the following:
    • Received from the alternate or primary email address.
    • Contains the phone number for delivery of the temporary password.

Availability Request for Multiple GAs: inform the customer that Microsoft will independently verify the availability of the other existing GAs/CAs as part of our security requirements. Contact other GAs/CAs independently as required by security guidelines; this procedure could take up to 24 business hours to complete.

  • Call other GAs individually to verify their availability. If not reachable by phone, send email template using email template 4: Availability Request for Multiple GAs to all other GA/CA’s listed email addresses in the tenant and start the 24 business hours count.

Non-GAs: Inform the customer that Microsoft will send an email message to the Non-GAs email address on file and that a reply to that email is required within 1 hour approving the password reset or MFA reset request.

  1. Send an email message to the Non-GA(s) email address, using Email Template 2: Request Permission from the Authorized administrator (translated versions are available).

TC: Inform the customer that Microsoft will send an email message to the Technical Contact email address on file and that a reply to that email is required within 1 hour approving the password reset or MFA reset request.

  1. Send an email message to the Technical Contact email address, using Email Template 3: Request Permission from the Technical Contact (translated versions are available).

DNS TXT Record: Ask the GA/CA to add a code to their vanity domain in the form of a DNS TXT (today’s date).

  • If the domain was purchased through Microsoft, move forward with the next verification step.
  • If the servers are pointing to Microsoft, start a remote session to verify ownership (document in the case notes the LMI session ID).

Step 5: Closing steps

  1. Confirm that the GA/CA can log on to Microsoft 365 and can change the temporary password.
  2. For MFA/2FA reset, the GA/CA will be prompted to reconfigure MFA and enter information on the MFA proof page.
  3. Educate the global admin to:
    • Update their mobile phone and alternate email address so that the “Can’t access your account?” link is usable. Ensure all contact data is up-to-date (other admins, TC, etc).
    • Review Security best practices.
  4. In the ticket-handling system, close the case by using the Password Closure email template below to send a message directly to the GA/CA you were working with.

More information

Language support: see Office 365 incoming calls - how to engage a Language Line Services interpreter.

  • If the engineer needs to use Language Line Interpreter (LLI) support to reset a password or MFA, all guidance requirements must be met.
  • Do not engage LLI support during the delivery of the temporary password.

When the password reset is generated, send the temporary password to the GA/CA via DTM and perform Step 5: Closing steps above.
