Important comments:

             -Format must be precise: Spaces and format are important!

             -Exact wording of the SME area.

             -Position of the areas: first the Region, then the SME area, then Qualifier and then the title. (Do not change this order!)

             -Qualifiers are optional and go as a third category; this includes CO19, GCC-H, DCR, particular mentions if you need. This include Government special cases, like GCC-H (for the US), Black forest (Germany), etc.

             -Once a case gets an ICM, please use the Qualifier field to include it.

 

Examples:

APAC | Enroll | CO19 | Cannot enroll DEP devices

ATZ | MAM-App config | GCC-H | App protection policies are not working on iOS devices

EMEA | GUI-Reports | ICM 12345678 | Wipe report

ATZ | Certs-Resource | Wifi SCEP is not deploying

EMEA | App-Deploy | PowerShell script is not working 

 

Another big area is understanding how to classify a case and which area it belongs.

Under each SME Teams channel in the Intune Case Discussion one, you can see a Workload tab which will contain a link to a KB that describes every subject, troubleshooting, etc.

 

As an example, if we go to Device Actions channel you can find the following article: https://internal.support.services.microsoft.com/en-us/help/4532828

This explains in detail, what are device actions and elaborate on each one of them

SME Areas

Description

3Party

 

Cases related to 3rd party integrations, like Mobile Threat Defense partners (Symantec, Lookout, Sophos), Windows Defender ATP, JAMF, etc.

App-Deploy

 

Application deployment issues on all platforms, including VPP. This also includes areas as PowerShell scripts and Data Warehouse (for the deployment portion)

App-SDK

 

Application development, Intune SDK LOB wrapping. This also includes scripts failures, Graph API calls or any other coding scenario.

AutoPilot

 

All Autopilot scenarios

Certs-Resource

 

Cases involving certificate deployment (SCEP, PFX, and PFX import) and all Resource policies (VPN, Email and Wifi).

https://internal.support.services.microsoft.com/en-us/help/4530675

MEMCM

 

AKA Co-Management All Co-Management scenarios (MEMCM), including enrollment.

https://internal.support.services.microsoft.com/en-us/help/4569597

 

Cond-Access

 

All Conditional Access cases.

https://docs.microsoft.com/en-us/mem/intune/protect/conditional-access

https://internal.support.services.microsoft.com/en-us/help/4536008

Dev-Action

 

Are remote tasks issued on-demand by an Intune administrator to carry out a specific action e.g. Retire, Wipe, Reset Passcode.

https://internal.support.services.microsoft.com/en-us/help/4532828

Dev-Compliance

 

Compliance policies define the rules and settings that users and devices must meet to be compliant. Devices are required to adhere to these policies before they are allowed to enroll.

https://internal.support.services.microsoft.com/en-us/help/4532838

Dev-Config

 

Cases related to all device Configuration policies (with the exception of the one included on the certificate/Resource area)

Enroll

 

Englobes all cases with enrollment issues, except for Autopilot or Co-Management enrollment. This may include MDM authority reset scenarios.

GUI-Reports

 

This area includes issues with the UI and reporting (i.e. cannot export the “All managed devices” report, policy is not showing)

MAM-App config 

All cases related to Mobile application policies: Application protection polices, App configuration policies, etc.

Updates-Roles

 

Includes Windows update rings, iOS update, Intune RBAC roles, PC management

 

Win MDM

All Windows MDM process, including PM management and 3rd party MDM using Windows enrollment

https://internal.support.services.microsoft.com/en-us/help/4564658

 Endpoint Analytics

 

Includes all cases related to the new features available in the Endpoint analytics Workload: Endpoint Analytics, https://internal.support.services.microsoft.com/en-us/help/4569965

 

Tenant attach (transferring). https://internal.support.services.microsoft.com/en-us/help/4555985

 


 *If you have any question regarding where a case should be, please engage any TA.
