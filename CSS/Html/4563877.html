Servicing: 5B.20: LSASS crashes with status code c0000008 causing device reboot following user logon on Win10 1809+ (ICM 188881983)

[Asset 4476087]
  • Content Idea (CI): 118869

Abstract

LSASS crashes with status code c0000008 immediately following user logon after installing the following Window Updates released on 2020 5B on Windows 10 1809 and Windows Server 2019 1903, 1903 and 2004 computers. Updates known to cause this issue include: 

  • KB 4551853: May 12, 2020—KB4551853 (OS Build 17763.1217)
  • KB 4552930: May 12, 2020-KB4552930 Cumulative Update for .NET Framework 3.5 and 4.8 for Windows 10 Version 1809 and Windows Server 2019
This issue is resolved by Windows Updates being released in the 2020 7B listed in the resolution section of this article containing release note text similar to:
  • Addressed an issue where lsass.exe fails with the following error message: “A critical system process, C:\WINDOWS\system32\lsass.exe, failed with status code c0000008. The machine must now be restarted."

MSINTERNAL Root Cause
It appears that SCNotification fails to talk to LSASS when attempting to get a name, which requires SID translation
But this is likely a consequence of the LSASS crash – without an LSASS full process dump we will not be able to progress the investigation

  5. Guidance for case owners 

    Engage with the customer and clarify if setting up WER for a full process dump was at all attempted by the customer and if there have been any challenges with it
    - have the customer check if there are any lsass mini dumps available under %LOCALAPPDATA%\CrashDumps ; these would be better than nothing, and would possibly give us a hint on the issue while we are waiting for the WER LSASS full process dumps

    Possible follow-up plan:
    - if the customer can reproduce the issue at will on a machine, we could try to get an IDNA of LSASS to understand the dynamic of the issue, but the immediate priority is to get a full process dump of LSASS
