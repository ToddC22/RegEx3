Guidance on completing the template

Use:

  • This is an internal facing template.
  • When engaging with Windows - WDG Escalation Engineers in one of the following 3 scenarios:
    • Debug analysis.
    • Source code review questions.
    • Issues where SME (SEE) knowledge has been completely exhausted.
  • Complete the template in English (except Japan and Korea).

Subject/Title:

  • Prefix the subject with CRITSIT if its a CRITSIT.

Request Type:

  • Hang, Bugcheck, app crash, IDNA analysis, Perf, code review etc.):

Description of the scenario:

  • Please provide as much detail as possible.

Data Description, location and details:

  • What data do we have, what is the most relevant data and where is it?
  • What were the actions taken to collect the data?
  • What do you require the Escalation Engineer to do with the data?

Have you discussed this issue with anyone else and what was the outcome?

  • Include PG threads, technical discussions etc.

Action Plan:

  • When completing an action plan details in this section should be concise, but not contain vague references:
    • Action on Microsoft and Action on Customer/Partner should contain enough details for any reader to follow-up on them and progress the case.
    • Next Contact should contain date and time of expected next contact. If the customer said in a couple of days, then set a time for your follow-up. It is recommended you also create a commitment task if your incident management tool allows or an Outlook reminder
