1. Overview

1.1 Why is this happening?

If a customer's password is ever forgotten or stolen, they must use their security info to get back into their account. This security info is very important, therefore, so we occasionally ask customers to verify their security info to make sure they still have access to it, and that its current. 

You can find out more information about this in section 3.

Return to top
 

1.2 How do you verify security info?

Customers can verify security info by having a code sent to their email or phone. Once customers receive the code, they can copy and paste it into the security info verification window. This confirms that their security info is current and that they have access to it. (For more info, see section 3.3 - Additional resources.)

2. Things to try

2.1 Answer common questions 

Why are you making me do this? Security info is the key to getting back into an account if you ever forget your password or if someone tries to break into your account. Without that info, you could permanently lose access to your account, your games, apps, subscriptions, email, photos, and files so we occasionally ask customers to confirm their info is accurate so that doesn't happen. 
What if I don't feel comfortable sharing my other email and phone info with Microsoft?  Microsoft will never use security info for any reason other than to send security codes and send you notifications when there are problems with your account. Also, we will never share your security info with anyone else for any reason.
Why should I do this if I've never had any problems with my account before? While it's great to hear that you've never had any problems with your account, it's still important to have insurance against accidents and that's what security info is.

Also, while you might be good at remembering your passwords, you never know when someone might try to compromise your account. Having up-to-date security info helps protect against that too.  
Can I opt out of this or turn this off? No, but if doing this is inconvenient right now, you can select Skip for now, however that will only bypass the process for 7 days. After that, you will have to verify your info, and you cannot skip again. 

Verifying security info is mandatory because it is a such a vital part of keeping your account available to you in case you forget a password, or your account gets taken over. Without it, regaining access can be very difficult.

Think about all subscriptions, apps, games, music, videos, emails, and the files you have on OneDrive. We ask you to verify your info to help you protect your access to all that important stuff because we think it's important too.
  
Why do I have to wait 30 days after deleting my security info? If an attacker did replace your security info, they could permanently lock you out of your own account. The 30-day waiting period helps prevent attackers from changing your security info by giving you time to spot that change and to stop it. See KB 2984702 Procedure: Microsoft account: Handling "Security Info Replacement" contacts for more info.

2.2 I did not receive my code, or the code I got didn't work

1. Make sure the customer didn't close the “We just sent a code…” window before they checked for the code. Closing the window will invalidate the code that was sent, and customers need to restart the process to get a new code:

[Asset 4492952]

Note: To avoid closing this window, just open another browser tab or window before they check for their code.  

2. If the customer sent the code to an email address:                                                                                                                
   a. Check the Junk folder on their email account.                                                                                                              
   b. If they find it in their junk folder, add it to the Safe Senders list.                                                                                 
   c. If they did not find it in their junk folder, advise the customer email will come from account-security-noreply@accountprotection.microsoft.com which they could add to any email Safe Senders list they have and then try again.                                                                                                                                                              
   d. If the customer still cannot receive the code via email, ask the customer to try another security proof they have.                                                                                                                                                                                             
   e. If the customer has no alternate security verification method, the customer will need to follow the instructions at https://support.microsoft.com/en-us/help/12428/microsoft-account-security-info-verification-codes to remove all security info in order to update to one they can receive verification codes from. 

3. If the customer sent the code via text message (SMS):                                                                                                  
   a. Check the masked phone number listed and make sure the number is correct and not transposed.  

   b. Make sure that the customer opens the text message itself and enters the security code that is inside the text message, not the number that the message was sent from.

[Asset 4492955]

   c. If the customer still cannot receive the code, ask the customer to try another security proof they have.

   d. If the customer has no alternate security verification method. The customer will need to follow the instructions at https://support.microsoft.com/en-us/help/12428/microsoft-account-security-info-verification-codes to remove all security info in order to update to one they can receive verification codes from.

2.3 Limit reached, try again later

To prevent malicious activity, there is a limit to how many codes a customer can request within a period of time. If that limit is reached, customers are asked to try again later. If this happens, the customer must wait before they can try to send another code. 

Note: The time out period for this is variable and dependent on several factors so it's difficult to give an exact estimate on how long a customer should wait. If asked, suggest waiting 30 minutes, if the message still appears, try again in 4 hours, if it still appears after that, try again in 24 hours.

2.4 I lost access to my security info, or my security info isn't correct

  1. Help customers get back in touch with their security info:
    1. If an account they used was closed, try to reactivate it through the provider's site. Sometimes this is easier than resetting security info.  
    2. Use the masked info they see to make sure the problem isn't with a bad number or address instead of a blocked text, or email.
    3. Check to see if their security information is confirmed within the MSAS tool in ASD. Confirmed info means they have done this before and if they still have the account, it should work again.  
    4. Did the customer configure a 25-Character Recovery Code? If so, try to find it. Customers can use this to get into their account and update their security info.
  2. If the customer is sure their security info is no longer accessible: a. If the customer did enabled Two-Step Verification, it will be disabled and the customer will get an email that informs them they have to go through the process and wait for 30 days before they can re-enable Two-step verification again. Also, all devices using app passwords will stop authenticating after the 30-day period and the customer will need to create new app passwords for all devices and services that requires one. 
    1. When they lose access to all their security info, there is a way to delete it and to enter new security info (Step 3 below).
    2. However, deleting their security info starts a 30-day countdown. Once the 30 days are up, the new security info is automatically updated, no further action is needed.

      Note: If the customer believes they have waited a full 30 days, support agents can check MST in the Microsoft account Status section to view under Issue to verify if the customer still needs to wait more time.
    3. If the customer didn't enable Two-step verification, they will be able to continue using most of their Microsoft services and to access their account during the 30-day waiting period except for:
       
      • Managing their billing at https://account.microsoft.com.   
      • Accessing secured aspects of their account at https://account.microsoft.com.
      • Making online Xbox purchases or buying apps through the Microsoft Store.
      • Note: Previously purchased items should still be usable. 
  3. Walk through the security info recovery process:
1. When prompted to choose where to send a security code, select I don't have any of these.  [Asset 4508913]
2. If a recovery code was configured and recorded somewhere, the customer can enter it here and get back into the account. 
If it was not, this screen will not appear, or the customer can select No to continue.		 [Asset 4508914]
3. Select the type of security info (phone or email) and enter the details. 

[Asset 4508915]

[Asset 4508916]
4. The customer will be prompted to check that new security info (email or phone) to proceed (email shown in this example).  [Asset 4508918]
5. A message sent to that replacement security email or phone will contain info (a link if email or phone text) on how to continue (email shown in this example).  [Asset 4508912]
6. The customer will need to enter the name of the Microsoft account that is being updated to proceed.  [Asset 4508919]
7. The customer is informed that the change will be finalized in 30 days.  [Asset 4508920]
8. The customer will also receive notifications that the security info for their account has been replaced via email to their Microsoft account and email or phone notification to their original security info.

3. Additional Info

3.1 What is Security Info?

Security info is another name for the alternate email address and phone number that a customer associates with their account to receive security notices and security codes. Microsoft does not share security info with anyone and will never use security info for marketing purposes.

Return to top

3.2 What is a Security Code?

Security codes are numeric codes sent to an alternate email address or phone number that a customer has associated with their Microsoft account as a piece of "Security info". These security codes are sent as a method of verifying the identity of an account holder by "round-tripping", which is sending a code to a place only the account holder should have access to and then having a customer enter that code at a prompt to prove that they received it. In addition to using a customer's security info to round-trip a code, the customer can also use an Authenticator app which generates codes that a customer can use as security codes for identity verification.
