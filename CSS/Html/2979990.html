1. Overview

This can happen when:
  • The wrong Microsoft account name was entered by mistake, maybe due to a typo.
  • The sign-in name is a deleted Microsoft account alias.
  • The Microsoft account has been manually closed or deleted for inactivity (Microsoft account activity policy). 
  • An account hosted by a third-party, like Comcast or Google, not associated with the Microsoft account was entered by mistake.
  • A Microsoft "Organizational account" or "Azure AD account" was entered by mistake. (See Section 3.3 for details.)
Return to top

2. What to ask

1. What are you trying to sign-in to? Have the customer try signing in to account.microsoft.com. If that works, skip to Section 3.1.2
2. What have you tried so far? If the customer already tried any of the Things to try in section 3, just skip that pertinent section.
 
3. When was the last successful sign-in?

If the customer answers that it has been years, then skip to Section 3.2.2. If not quite as long ago, then focus on ensuring it's the right account name
4. What did you sign-in to when it worked last? Find out where the account worked last and have them try that. If it works, the account name may have been cached so have them see how it's different there or it could be a problem with the device or service they're trying to sign in to.
5. Do you have any account aliases? ASD does not recognize account aliases and will report those as unknown accounts. If the customer has aliases, ask them what the aliases are then try entering those in ASD to see if there is a match. If there is, have the customer try that account name.

3. Things to try

3.1 Check if this is the right account name

3.1.1 Check for common typos and mistakes

Ask "Can you confirm that the account name you entered is correct and double-check for typos?"

  1. If it still doesn’t work, refer to KB 4479728 Procedure: Microsoft account: Forgot my username to help customer in recovering their username. 
    Note: This process applies to both Web and Console.
     
  2. If the customer still fails, try the following:
  • Does the customer have an Xbox? If so, have the customer try the Find your Microsoft account email address wizard at https://support.xbox.com/my-account/microsoft-account/forgot-microsoft-account-solution.
  • If this is a new account or it's been a while, it can be easy to misremember the account name, so if the customer sounds unsure about the account name:
    • If customer hasn't tried a few variations, have them do so now. Example: If the account name includes the customer's actual name, have them try variations like adding a middle initial.
    • Ask if customer sent any email to friends or family. If so, maybe try asking them if they can look up the account name.
    • Ask if customer uses more than one Windows device to sign-in with. If yes, try one of those devices to see if the account name is cached and displays automatically.
    • If customer can sign in to one of their Windows devices, have customer check Credential Manager for their Microsoft account information.
      • If customer can’t find their account name under Web Credentials, have customer select Windows Credentials and look for MicrosoftAccount:user= and their account name will be listed in that entry if it exists.
      • Select Web Credentials and scan the list for https://account.microsoft.com and if it is there, the customer’s account name will be displayed next to it.
      • Click the Windows Start icon, then type Credential Manager and select Credential Manager when that result is displayed.
Return to top
 

3.1.2 Ask more about what the customer is signing into

  • If the problem is signing into email, sometimes customers using the Outlook desktop client are signing into an account that isn’t an outlook.com account. Ask if it’s a work account or maybe some other email service.
  • If it sounds like this might not be a Microsoft account the customer is trying to sign in with a work account, Google, or Yahoo, you may need to refer the customer to that service provider.

    Note: Any email address, such as an @gmail.com or @comcast.net account for example, can be registered as a Microsoft account name to access services like OneDrive, but this does not mean we host email for these domains.

3.2 Check for deleted aliases, manually closed accounts, or accounts deleted for inactivity

3.2.1 Check for a deleted alias

Ask: "Do you have more than one alias, email address, or phone number that you use as an account name?"

  • If no, then customer probably does not have any account aliases.
  • If yes, this may be an alias for the account that the customer deleted. Have customer sign-in to account.microsoft.com with one of their other account aliases. If successful, have customer check their account aliases while signed in:
    1. Select Your info then Manage your sign-in email or phone number
    2. Review the list and add any aliases that might be missing.
For more information on aliases, see http://windows.microsoft.com/windows/add-alias-account.

Return to top
 

3.2.2 Check when the customer last used their account

Ask: "When is the last time that you signed in with this account?"

If last sign-in was more than a few years ago

Reply:

Accounts are automatically deleted when they are inactive for a certain period of time. I’m sorry, but It sounds like your account may have been deleted due to inactivity.

If account is deleted, refer to section 4.1.

If last sign in was over 60 days ago:

Reply:

Is it possible that you requested to have your account closed? If so, accounts can be re-opened within 60 days of that request but are permanently closed after that.

If account was closed over 60 days ago, then refer to KB 2974112 Informational: Microsoft account Closure FAQ: All Products

If last sign in was less than 60 days ago:

This probably isn't a closed or aged out account.

Continue to the other steps.
Accounts that age out due to inactivity are permanently deleted. Microsoft account activity policy – effective Aug 30, 2019. 

The following chart may help you determine if an account may have been deleted for inactivity:

3.3 Make sure this isn't an Organizational or Azure AD Account

Most accounts provided by organizations, educational institutes, or businesses are actually "Organizational accounts" or "Azure AD (AAD) accounts" and cannot use normal Microsoft account sign in services. Organizational accounts and Azure AD accounts are not the same as Microsoft accounts. These are accounts that are hosted by Microsoft but managed by an organization or business for their employees to access Microsoft services like Microsoft 365 or Microsoft Azure. To see if an account is an Organizational account or an Azure AD account: 

1. Have the customer go to https://passwordreset.microsoftonline.com/ and type the account name along with the picture code, then select Next.

Note: If the customer asks why you're having them reset their password, say that we're just testing to see what type of account this is and we won't actual go through with a password reset.
2. If it IS an organizational account or Azure AD account, a message confirming that a password reset was sent will appear. Skip to the end of the section for next steps.

Note: If a reset is sent, instruct the customer to ignore the email so they don't reset their password.
3. If it IS NOT an organizational account or Azure AD account, a message will appear saying it isn't a valid account.
Note: Any support for issues with Azure AD or Organizational accounts should come from the employer or organization's IT department. However, if the customer indicates that isn’t possible, refer to KB 2769823 Procedure: Transfer codes for other lines of business for transferring customers to commercial support.
