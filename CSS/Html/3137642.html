ASMS – Manager Recovery Process

Submit a ModTrack request to update this TSG.

Overview

Engineers may encounter situations that require manager assistance. This TSG provides guidance on how and when to escalate to a manager, as well as what the manager should do upon receiving an escalation.

Policy

At times, it will be necessary to engage management (Pod lead) for direct customer contact, in the hopes of recovering the customer experience. Manager recovery scenarios include, but are not limited to the following:

  • The customer asks for manager/director 
  • It is a Microsoft bug (stuck at our partners/engineering)
  • It is a payment discrepancy due to Microsoft
  • It is a pro-active recovery 

Note: Manager recoveries can be requested if a case in a resolved or unresolved state. 

Manager Engagement Process

TA Escalation Process

  1. Determine whether a manager has been engaged or has communicated with the customer.
  • If a manager has previously been engaged on the case, re-engage the same manager for continued recovery to ensure continuity, and keep the number of customer-facing contacts to a minimum.
  • If no manager was previously engaged, create a customer recovery task using the Manager Recovery Engagement Template below, and transfer to the task to the ASMS PPP TA Consult – Internal Queue. 
  • Send an email to the managers with the task number and include case owner in Cc.
  1.  After sending the recovery task to the queue, change the SR Wait State status of the case to Recovery.

Manager Process

When a task arrives to the Pod queue, the manager will:

  1. Review the case notes provided in the task and engage the case owner or technical advisor (TA) involved with the case.
  2. Contact the customer and engage the TA if a technical decision is required.
  3. Update the task with the outcome from the customer engagement and update the associated case with relevant recovery triggers and comments .
  4. Once updated, transfer the task back to the TA or SE as appropriate.
  5. Maintain visibility and follow the case progression until closure, re-engaging as needed.
