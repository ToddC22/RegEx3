Procedure: Case Title and Case Notes

[Asset 4049349]

 

[Asset 4557567]

Who should use this article?

  • All Consumer

What is this article about?

This article provides guidance on how to create case title and case notes. Good case title and case notes will help others:

  • Easily understand what the case is about.
  • Understand what the customer was trying to do.
  • Understand what stopped customer from doing what they were trying to do.
  • Understand what you did to solve it, or what the next steps are to resolve the case.

1. Case titling guidance

All case titles are captured in the Subject field within the ASD incident form:

[Asset 4087231]

When creating a case title, please include a proper problem description. you add the problem description to the case title, include the problem activity and the specific feature or area that is impacted. The key is to use the word “issue” or “problem” or “error x” or phrase the customer activity using negative words.

Good examples of case title:

  • Activation - Error Code: 0xc004c008.
  • Install updates from Windows Update - Error Code: ABC.
  • Create a child user account – Does not know how to.
  • Using Microsoft Edge – Unable to scroll a web page.
  • Using Start Menu – Won’t open.

Also keep in mind the following:

  • Make sure you include a brief problem description in your case title. The description should include the activity as well as the area where we use a negative word (“unable”, “Error”, “Cannot”, “Won’t”).
  • Make sure you include the area or feature in your case title as well – as listed in the above examples.
  • You can continue to use keywords. But also provide the problem description after the keywords just like most of you are doing.

2. Where to create case notes 

Windows and Office support advocates:
Use the COMPLETED ACTIONS TAKEN AND RESULTS section to capture your notes in an A.B.S. Framework (see section 3. A.B.S. Framework in this article for more details).

[Asset 4058337]

For all other support advocates:
Use the pop out field on the left-hand side, click to open the case notes field. Capture your notes in an A.B.S. Framework (see section 3. A.B.S. Framework in this article for more details).

Note: Use the Better Case Note Tool to ensure documentation quality where deployed (Office, Windows, and Accounts & Billing – English and Spanish).

3. A.B.S. Framework

Case notes should be based on the core framework: A.B.S.

  • Activity - What was the customer trying to do?
  • Blocker - What is blocking the customer from achieving the activity?
  • Solution - How did you help the customer to resolve the issues?
    • What troubleshooting steps were performed?
    • What are the outcomes of those troubleshooting steps?
    • If you didn’t resolve the issue, what are the next steps that are to be performed, an action plan, an escalation, call back arrangements, etc.

Tips:

  • Use customer language when you document.
  • Be specific and brief – try to include details such as any error messages, app names, type of license, and action you took to solve the issue. If the same information exists in another field on the case, capture it in case notes, too.
  • Abbreviations – Avoid using abbreviations in your case notes.
    • Spell out product names , error messages, game titles, etc.
      • Good examples: Microsoft 365 Personal, Windows 10 Pro.
      • Bad examples: Microsoft 365, W10.
  • Avoid using special characters to format your case notes. Special characters should only be used if they are part of an error message. Examples of special characters, $ & # + |  Use professional language in your case notes. Examples: avoid using insensitive language even if it is a part of customer’s description.
  • Avoid putting customer’s Peronsal Data in the case notes.
  • KB 3064766 Procedure: Privacy: Handling Microsoft customer personal data
  • KB 2991467 Procedure: Privacy: Handling social engineering, fraud, and phishing incidents

5. Follow-up contacts

If a customer contacts support again with the same issue, case notes are still required. The rule is simple, for every interaction with the customer, a case note is needed.

6. Examples of case notes

 

Good example

Bad example

1- Windows

A: Customer wants to know what information is in the latest Windows 10 update.

B: Couldn’t find the correct information online.

S: Guided customer to the release notes and sent them a link via email.
 
  • A: Customer wants to know about updates.
  • S: Sent a link.

Why?

Didn’t include:
What the customer wanted to know about the update.
What link did we send and where to?

While you can still get this information by reviewing email and the link it adds time.  Having good case notes relieves having to do extra steps.

2 - Xbox

A: Customer wants to play Halo 4 on Xbox One game without crashing.

B: This game is returning to the dashboard every time I launch it.

S: We did a hard reset and the game launched successfully.
 

  • A: Game doesn't work
  • S: Troubleshooting

Why?

No game title.
Not sure why the game doesn’t work.
Not clear what troubleshooting steps were done already or if it was successful.

3 - Billing

A: Customer was trying to understand Microsoft charges on credit card bill.

B: Customer has a charge which they don’t recognize and don’t know what it’s for.

S: I looked up the charge in MST and let the customer know it was their Music subscription renewing automatically.

  • A: Question about their bill.
  • S: Subscription renew.

Why?

We don't know what the question was, it could be anything from a refund not landing to when he'll be billed for a pre-order. Subscription renew doesn't actually tell us what kind of data point it's trying to address.

4 - Office

A: Customer trying to launch Excel.

B: It gives a stopped working error message each time they try to launch the program.

S: Launched Excel in safe mode successfully, helped user disable add-in causing issues.

  • A: Cx is getting a stopped working error message whenever they run Office.
  • S: Launched in safe mode and disabled add-ins resolves the issue.

Why?

They look almost identical, but the next advocate doesn’t know which program in the Office suite caused the issue. If this customer called back with “I’m getting a stopped working error message in Word, the next advocate would have no idea if the problem came back or was new.
