ASMS — Capturing Network, HAR, and Fiddler Traces

Overview 

This TSG provides detailed steps for customers to leverage when requested by ASMS to capture a HAR file, Network trace, or Fiddler trace for further troubleshooting. 

Without providing a trace, it is difficult to provide help focused on the customer's specific needs. If a customer has questions about the use of a network trace, explain to the customer that the trace is needed even if there are no errors in the portal because it provides correlation IDs, session specifics, and details to specifically target the user's interaction with the systems. This will help us understand the issue and eventually reach a resolution.

Additional guidance can be found on the public-facing Capture a browser trace for troubleshooting page.

Procedures

Most issues that require troubleshooting can be solved by using the network trace capability within Internet Explorer. This is commonly known as the F12 trace data, because a user will push the F12 key in Internet Explorer to collect the data.

In other scenarios a more robust network capture tool such as Fiddler or HAR will need to be used. Use the options below to determine which steps your customer will need to use, so that further troubleshooting can be completed.  

 

If you ever need to capture trace from within the Azure Portal to troubleshoot an issue, here are the steps required to do that.

  1. Login to the portal here.
  2. Navigate to the step just prior to where the issue is happening.  Press F5 or Refresh.
    • It is important to only enable the tracing after you have already logged, because if logging is enabled when you login, then the log will include some sensitive data which would require some actions on your side to maintain the integrity of your account.  Also, pressing F5 or Refreshing will remove any unnecessary cached values.
  3. Press F12 to start the browser Developer Tools (IE and Edge)
    • In IE and Edge pressing the F12 open a console within the browser that provides some troubleshooting and debugging capabilities.
  4. Click on the Network, stop the tracing and click on the clear session button.
  5. Start the session and reproduce the issue.
  6. Stop the trace and save as an HAR file.

 

How to generate a capture file in Internet Explorer (IE)  

  1. Use Microsoft F12 Developer Tools and the Network Tool to capture the browser headers. Then export the captured traffic as HAR file (or XML if using an older version of IE).  

  1. Press the F12 key on your keyboard. This should open the Developer Tools panel in IE. If not, find it on the browser menu: F12 Developer Tools  

  1. In the Developer Tools panel, click the Network panel and then deselect the Clear entries on navigate option. (On by default).  

  1. Click the Network panel/button and then the Start Capturing button (or press Ctrl + E). Note: the icon looks like a green triangle  

  1. Click the IE Refresh button. The goal is to reproduce the problem you've been experiencing while IE is recording activity  

  1. The Developer Tool panel should now show a list of the URLs that are included in the page you have in your browser.  

  1. Click the Stop button when the issue has been reproduced (or press Ctrl + E). Note: the icon looks like a red square  

  1. Click the Export captured traffic icon and save the file somewhere convenient (or press Ctrl + S). Note: the icon looks like a floppy disk.  

  1. Please attach the file to your email or case with us so that we can assist further.  

  1. Click on the Console tab and look for any errors/warnings reported. If errors are shown, please right-click on the console errors and select Copy all or send a screenshot of them.  

  1. Close the Developer Tools panel and you're done!   

How to collect via Fiddler  

  1. Download Fiddler on your machine from http://www.telerik.com/fiddler    

  1. Close all browsing and windows, then run Fiddler.   

  1. Configure Fiddler with below options:  

  1. Within Fiddler, click Tools > Fiddler Options   

  1. Select the HTTPS tab   

  1. Check the checkbox to Decrypt HTTPS traffic   

  1. You will receive a few pop-ups that are required in order to install the Fiddler root certificate, which allows Fiddler to sit as a man-in-the-middle to your HTTPS session. Click Yes to these prompts.   

  1. Click OK to close Fiddler Options.   

  1. Start Capturing the trace as below: Click File > Capture Traffic to stop capturing.   

  1. Click in the Browse button to launch IE. Now reproduce the issue; go to the Azure portal and try to reproduce the error message once with the single user and another time with the group when they’re granted the “Cost Management Reader” role.   

  1. Save the trace: File Transfer - Case <insert case number>.   

How to generate a HAR capture file in Chrome  

  1. Start by opening Google Chrome. If you do not have Google Chrome and you are free to install additional programs, please download Google Chrome and install it.  

  1. In Chrome, go to the required page within ACM where you are experiencing trouble.  

  1. Select the Chrome menu (⋮) at the top-right of your browser window, then select Tools > Developer Tools.  

  1. The Developer Tools will open as a docked panel at the side or bottom of Chrome. Click on the Network tab.  

  1. Select the option Preserve log  

  1. The recording should auto-start and you will see a red circle at the top left of the Network tab. If not, click the black circle, to start recording activity in your browser.  

  1. Refresh the ACM page you are on. The goal is to reproduce the problem you've been experiencing while Google Chrome is recording activity.  

  1. Once you have successfully reproduced the issue while recording, right-click within the Network tab and click Save as HAR with Content to save a copy of the activity that you recorded.  

  1. Save the HAR file somewhere convenient.  

  1. Now click the Console tab and right-click anywhere in the console log. Select the popup option Save as and name the log file: Chrome-console.log  

  1. Close the developer panel and you're done! Please attach the HAR file and the Chrome-console.log to your email or case. 

How to generate a HAR capture file in Firefox  

  1. Start by opening Firefox.  

  1. In Firefox, go to the page within ACM where you are experiencing trouble.  

  1. Select the Firefox menu (Three horizontal parallel lines) at the top-right of your browser window, then select Developer > Network.  

  1. The Developer Network Tools will open as a docked panel at the side or bottom of Firefox. Click on the Network tab. Before starting the capture, be sure to enable the setting: Enable persistent logs in the Toolbox Options > Common Preferences (click on gear icon on the Toolbox toolbar to open the Toolbox Options pane).  Click on the Network tab after setting the option.   

  1. The recording will autostart once you start performing actions in the browser.  

  1. Refresh the ACM page you are on. The goal is to reproduce the problem you've been experiencing while Firefox is recording activity.  

  1. Once you have reproduced the issue and you see that all of the actions have been generated in the Developer Network Panel (should just take a few seconds), right-click anywhere under the "File" column and click on "Save All as Har"  

  1. Save the HAR file somewhere convenient and close the developer panel.   

  1. You're done! Please attach the HAR file to your email or case with us so that we can assist further.  

How to generate a HAR capture file in Safari   

  1. Start by opening Safari.  

  1. In Safari, go to the page within ACM where you are experiencing trouble.  

  1. Right-click anywhere on the page and select "Inspect Element". (If you do not see this option, enable the Web Inspector.  

  1. Go to the Console tab and check the option Preserve Log.  

  1. Go to the Network tab and check the option Preserve Log.  

  1. Refresh the ACMACpage you are on. The goal is to reproduce the problem you've been experiencing while Safari is recording activity.  

  1. Once you have reproduced the issue and recorded it, go to the Network tab and click Export to save a copy of the activity that you recorded.  

  1. Save the HAR file somewhere convenient.  

  1. Now click the Console tab and take a screenshot. Depending on how much shows up there, you may need to expand the window by dragging the top border up to capture more information.  

  1. Close the Web Inspector and you're done! Please attach both the HAR file and the Console log screenshots to your email or case with us so that we can assist further.
