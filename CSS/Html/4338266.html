What’s the article about

Perpetual Software

On-premise software in CSP supports a smooth transition to the cloud by introducing on-premise software in a cloud-focused program. This new offering helps bring the value-added partner to every purchase scenario as they provide a single platform to transact all Microsoft products. CSPs can now sell on-premise software through Partner Center in addition to Open, EA, and other programs currently in use. Straightforward licensing of on-premise software in CSP means cost predictability and a streamlined sales process for you. This business model makes it easy to procure, manage, and price on-premise software for customers, allowing CSP to focus on winning business with an expanded portfolio of IT management value-added solutions.

3.2 Perpetual Software in CSP – Partner Center Purchase Experience

Important Notes:

  • Partners will receive the download link and keys for software purchases via Partner Center.

  • Partners will be responsible for distribution of links and keys to their customers.

  • Cancelled subscriptions within the cancellation policy will trigger blocking of keys for unused activations.

  • Given the high risks with key distribution, Partners are encouraged to use the most secure method to distribute keys to their customers.

There are multiple entry points to the catalog.

This process displays the end to end purchase experience from the customer list.

1. After the Partner logs into Partner Center, they can Add Products for each customer available in the list.

[Asset 4338275]

2. A new window will appear to select the Quantity and then Add to cart.

[Asset 4338276]

3. The partner can do a final review and confirm the purchase by clicking Buy.

Note: Billing frequency of One time means that you pay one time at your next generated invoice.

4. A confirmation window will appear and contain the order details (order ID, currency, and quantity). The Partner can print the confirmation and click Done to close the window. After purchasing the products, the Partner will be redirected to the Order History page.

[Asset 4338277]

5. From the Software management page, the Partner must select the software version, language, and file type. After selecting these parameters for each product, click Submit.

[Asset 4338280]

Note: If the partner wants products of different versions, languages, or file type, they can select a new product version / language / file type from the dropdown menu, and then Submit. The partner can buy different versions of the same product, but cumulatively the number of purchased licenses. For example, if they purchased 4 licenses of Project Professional, they can download 2 installers and activation keys for Project Professional 2016, 1 installer and activation key for Project Standard 2013, and 1 installer and activation key for Project Standard 2010.

[Asset 4338281]

6. After the order has been placed, it will appear in the Order History page, where the partner can also see the previous orders. Note that there is a delay of up to 15 minutes between the order submission time and when it appears in the customer's orders.

Note: The Partner can buy Online services, Azure RI, and perpetual software in a single cart. Though the cart it's single, the order will be split in the background. The partner will see a different recon file for each purchased service.

Once the order has been registered, the Partner can Download, Copy Link, or Get Key for each product.

  • Download:  Download an archive of the software installer.
  • Copy Link: Generates a link to download an archive of the software installer. This can be sent to their customer.
  • Get Key: Generates a 25 alphanumeric product key to use for the software activation.
    Add that there's no API support for software download.

[Asset 4338282]

Note: If the wrong version is delivered to the customer, the partner can go back to the Software management page and choose the appropriate software version.

A recurring billing is a term subscription that is billed on a monthly or an annual basis and auto-renews at the end of the term. For example, Office 365 and Azure Pay-as-you-go subscriptions.

A one-time billing is a term that is billed one-time up front and does not auto-renew at the end of the term. For example, Azure Reservations, Software, and Server Subscriptions (On Prem or via Azure Cloud).

For one-time purchases, Partners will receive a separate invoice and recon file for each different currency, with all purchases for the currency consolidated. For example, for a given month, all one-time purchases in Euro will appear on a separate invoice and recon file for EUR.

On the monthly billing date, partners may receive:

a. Invoice for reoccurring billing

  • Reconciliation file for license-based subscriptions
  • Reconciliation file for usage-based subscriptions

b. Invoice for one-time billing

  • Reconciliation file for one-time orders

Note: EU/EFTA Partners Only - One-time purchases will be denominated in the customer country currency (see customer country/currency matrix), therefore if your purchase includes customers with different country currencies, you will receive a separate invoice and recon file for each unique currency.

If Partners request assistance with a software key, you can use Key Information Tool (KIT) to check the status of their activation key. For deactivated keys because of system issues, a new key would be automatically generated and available in the Partner Center.

KIT is a web-based tool that provides product key information and activation details. You can use KIT to:

  • look up a customer’s product key
  • review their activation summary and history
  • access general product key information
  • find invalid/blocked product keys or keys that have reached their activation limit

For more information, check 3150228 Tools User Guide: Key Information Tool (KIT). 

Advocates are expected to perform basic troubleshooting steps, such as:

  • making sure the partner is trying to activate the key visible on Partner Center on the same media they downloaded from Partner Center
  • the key version and edition correspond to the product media version and edition they are trying to activate (ex.: a key for Windows Server 2019 Enterprise edition will *not* activate a Windows Server 2019 Standard edition media)
  • the key has not reached its activation limit

Escalation points for Product Key issues:

Advocates should perform the basic troubleshooting steps explained above and request screenshots from the partner clearly showing the full key, the media they are trying to activate (with version and edition) and the error received. All product activation issues should be escalated to MPN Global support via child SR to MPN Help Escalations – CSP. This template should be used:

  • Purchase Order ID;
  • Full PK;
  • PK analysis in KIT (including how many activations were done);
  • Confirmation of what product and version the customer is trying to activate (to avoid mismatches between the key and the media);
  • for Win 7 ESU confirmation that the customer is definitely trying to activate ESU (Extended Security Updates) and not Windows 7.

What are Windows Server and SQL Server Subscriptions?

Windows Server and SQL Server Subscriptions sold in CSP include Azure Hybrid Benefit, previously available only through Software Assurance sold in traditional licensing programs. Azure Hybrid Benefit enables significant savings on server software compared to pay-as-you-go rates. By making 1 or 3-year commitments to use both Microsoft server software (via subscriptions) and Azure compute (via Reservations), commercial customers can maximize their savings on cloud compute solutions. This benefit will be especially attractive to cloud-only customers who currently don’t have access to Software Assurance.

Why is this announcement good news for customers?

Commercial customers can save up to 72% versus pay-as-you-go Azure VM pricing simply by purchasing (or “reserving”) virtual machines for a 1 or 3-year period. The 3-year Windows Server subscription combined with Azure Reservations saves customers up to 79% versus pay-as-you-go pricing. Combining pricing with deployment flexibility, customers will get the best overall value when they choose Azure Reserved VM Instances and Server Subscriptions.

Why is this announcement good news for Microsoft partners?

By bringing Azure Reserved VM Instances and Server Subscriptions with Azure Hybrid Benefit to CSP, Microsoft is better enabling partners to grow their cloud solution practice by employing low-cost Azure VMs & server software to target and win more persistent cloud workloads. Without these offerings in the CSP program, Azure partners were able to sell only pay-as-you-go Azure virtual machines and had to use other licensing programs to acquire Azure Hybrid Benefit for Windows Server and SQL Server.

Azure Reserved VM Instances and Server Subscriptions help partners grow revenue and profitability by enabling them to develop and deliver higher-margin, value-add services that can also pull sales of additional software and/or subscriptions as well as ongoing management service fees.

In summary, Server Subscriptions and Azure Reserved VM Instances in CSP enable Microsoft to deliver market-leading customer value on predictable cloud workloads. This empowers partners with the agility, scalability and efficiency they need to profitably target and capture a larger share of the fast-growing cloud compute opportunity.
