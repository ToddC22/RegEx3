1. Overview

Azure Monthly Credit ($100/month) is a benefit that all MAPS and Competency partners are entitled to receive.

Azure Bulk Credit is a benefit offered only to partners that have attained Silver Cloud Platform and Gold Cloud Platform Competency, and to partners that have enrolled in the Azure Expert MSP program.

For Silver Cloud Platform, partners will receive $6,000 credit, and for Gold Cloud Platform, partners will receive $12,000 credit.

For Azure Expert MSP, partners will receive $13,000 credit.

2. Escalation teams

2.1 Renewals:

Microsoft Azure Credit (Monthly/Bulk) renewal scenarios, (activation/assignation process in PC) are handled by MPN team (MPN Global Support team).

Exception: If a partner who has Azure Monthly Credit (0025P) offer is impacted by this error message "You're not eligible for this benefit. This benefit is only available to users with an active Microsoft Partner Network membership” when they are redirected to Azure Portal, they must be guided to raise an Azure support ticket here: https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview

2.2 Benefit transfer:

1. For Microsoft Azure Monthly Credit transfer scenarios, when partners wish to transfer the benefit from one user to another (or wrong association), this can be managed directly in PC, using the “Remove” button to remove the current user assignment, and then assigning the benefit to the desired user. This change can be done by Global admin or Partner MPN admin.

[Asset 4519432]

2. For Microsoft Azure Bulk Credit transfer scenarios, when partners wish to transfer the Azure benefit already assigned in Partner Center to a user, from that user to another one (or wrong association), Frontline should check within CMAT if there is an Active Sponsorship Subscription under the existing user assigned in PC.

[Asset 4519433]

  1. If there is an Active Sponsorship Subscription, partner should be guided to create a new request to Azure Subscription Management Support Team, requesting for the Subscription and the Entitlement to be transferred to the new desired user. The request can be created using this link: https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview
  2. If there is no Active Sponsorship Subscription, it means the partner only assigned the user in PC. In this case, Frontline should escalate to MPN Global Support to check if AIRS was triggered when the user was assigned in Partner Center (renewal scenario).
  • If AIRS is Active/Approved, MPN Global Support will advise that partner should be guided to create a new request to Azure Subscription Management Support Team, requesting for the Entitlement to be transferred to the new desired user.
  • If AIRS is Cancelled/Denied or does not exist, MPN Global Support will engage Partner Center engineering team to remove the user assigned in Partner Center, so that partner can assign it to the desired user and activate it.

Azure Bulk Credit (Offer 0036P) offer can’t be combined with other licensing, prepaid, or credit offers (i.e., Microsoft Enterprise Agreement, Cloud Solution Partner, Azure in Open, Azure Pass, Action Pack, etc.).

​For any additional details regarding this offer, please access the following page: https://azure.microsoft.com/en-us/offers/ms-azr-0036p/.

3. Call to action

Frontline team must escalate renewal scenarios or assign credits issues to MPN Global Support, except the ones impacted by the message “You're not eligible for this benefit. This benefit is only available to users with an active Microsoft Partner Network membership”, which must be addressed per the process described at section 2.1 - Renewals.

MPN Global Support will work with the engineering team to solve any renewal/assignment issues, or any other issue related to the benefit in Partner Center. Based on engineers’ feedback, you will receive guidance provided by MPN Global Support.

MPN Global Support cannot assist with any issues related to Azure Portal.

Very important – this process/ call to action is applicable only for partners who already migrated to Partner Center. For partners who are still in PMC, there are no changes.

4. Assigning the Azure Monthly/Bulk Credit benefit in Partner Center

To assign the benefit in Partner Center to a user, partner’s Global Admin/ MPN Partner Admin should go to the Benefits tab in PC, click on “Azure & Cloud” section, filter the benefits by Subscription type selecting Usage-based, go to the dropdown arrow from the right, under Azure Benefit (Monthly or Bulk) and add the user credentials to which the benefit to be assigned (as in the screenshot below).

[Asset 4519434]

After the credits are assigned in Partner Dashboard, it is very important to activate them using the activation link sent to the user via email or from Partner Dashboard and not login separately in a new tab or a new window to Azure Portal.

Note: To access, view and manage MPN Benefits, Guest users need to have Global Admin/MPN Partner Admin roles. Please find below the experience in Partner Center for a Guest user with no admin privileges on the tenant.

[Asset 4549354]

Very important – Azure Monthly/Bulk Credit in Partner Center can be assigned only to users having an AAD account. The benefit cannot be assigned to users having a free-hosted domain, such as @live.com, @hotmail.com, @gmail.com or any other, only company domains.

A user can only have 1 Azure Monthly Credit $100 assigned to him at the same time!

A user can also have assigned to him the Azure Monthly Credit, Azure Bulk credit, and the Azure Expert MSP bulk credit at the same time, ONLY if the Azure Bulk Credit is assigned before the Azure Monthly Credit.

If a partner is trying to assign the Azure Bulk Credit to a user that is already assigned to Azure Monthly Credit, it will not work because Azure Monthly Credit is mapped in the system as Enterprise Agreement, and Azure Bulk Credit cannot be assigned to users that are associated with an Enterprise Agreement.

If a partner is trying to assign the Azure Monthly Credit to a user that is already assigned to Azure Bulk Credit, it will work.

A user cannot be assigned to both Silver and Gold Azure Bulk Credit in Partner Center, however Azure Subscription Management Support can update the monetary cap in Azure Portal for the same user, upon request, to have a total cap of $18K Azure Bulk Credit ( $6K from Silver Cloud Platform + $12K from Gold Cloud Platform). In Partner Center the user will only be assigned under one of the two Azure Bulk Credit benefits.

5. FAQ                                                                                                    

Q. In Partner Center, Azure credit scenarios are not transferred anymore to AZURE team?
A. They are handled by MPN Global Support, except for the Azure Bulk Credit transfer scenario, where the Frontline team must follow the process described under section 2.2 - Microsoft Azure Bulk Credit transfer scenarios. 

Q. If a partner asks us about a renewal Azure credit issue, should we redirect him to open a ticket with Azure team?
A. No. You should collect all the information they can provide and escalate the case to MPN Global Support.

Q. If a partner reaches us asking how to transfer his credits from a tenant to another one, or assign them to another individual, should we ask him to open a ticket with Azure team?
A. No. For Azure Monthly Credit, you should guide the partner to use the Remove feature from PC to remove the current assignment and assign the benefit to a new desired user. For Azure Bulk Credit, you should follow the process described at section 2.2 - Microsoft Azure Bulk Credit transfer scenarios.
