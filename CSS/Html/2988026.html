1. Documenting the customer interaction

Always document the contact for every customer interaction. This includes creating a new service request (SR), editing an existing SR, or adding case notes, depending on the scenario. Exceptions to this rule are listed in this article.

If the customer is contacting support again about an existing issue, the goal is to continue working from the existing SR, if possible. Reviewing previous SRs should be done during each customer interaction to determine whether the customer has previously contacted us.

2. Reviewing customer information

2.1 Search customer information

Customer information can be located by searching for the customer by their Microsoft account, SR, gamertag, phone number, or serial number. For more details, refer to KB 3065764 Tool User Guide: Membership Support Tool (MST).

Return to top

2.2 Review existing service requests

If you find an existing SR, review the relevancy for the current issue. If the issues are the same, take ownership following the rules outlined in section 4. Review existing cases as necessary:

  • If the customer provides an SR regarding a recent contact that matches their current reason for contacting support.
  • If you notice recently opened SRs on the customer’s account, and you know how to solve the customer’s issue, and sees that further work isn’t being performed by the previous advocate.

If there is no SR with the same issue, proceed to section 3.

3. When to create a new service request

Important notes: If you associate a new SR against a customer’s Microsoft account, ensure the account is authenticated. Refer to KB 4506610 Procedure: Microsoft account verification. You only want to associate an SR to a customer account we know the customer owns.

Keep in mind you only need to authenticate the customer’s account if the supported scenario requires it. If the supported scenario does not require authentication, the SR can be created without customer account association.

4. Case ownership

If the customer contacted Microsoft support within the last 30 days regarding the same issue, take ownership of the SR with the same issue.

If you were the last person that interacted with the customer, you should be the owner of the case.

For a transferred contact: If you get an error saying there is a lock on the case, you will not be able to take ownership until the previous advocate completes the case.  If this does not occur during the customer contact, create a new case.

Exception: Do not take ownership of a case with a scheduled call back OR an escalated case.

  • If the customer calls back before the scheduled time, provide the call back arrangement details to the customer and let them know that they should expect a call back as scheduled.
  • Windows and Office only: If the customer calls back after the scheduled time, perform a live escalation to Tier 2.

On how to take ownership, refer to KB 4050243 Take ownership of a case.

5. Reopening a case

Reopen a case when a customer states they already have an SR, or you have discovered that the customer has an existing closed case for the same issue. This is only applicable if the case was closed within the last 30 days.

To reopen a case, search the case in MST and reopen in ASD.

Refer to the following articles for more details:

  • KB 3201395 Tool User Guide: Membership Support Tool (MST) Search
  • KB 3046180 Tool User Guide: Assisted Support Desktop (ASD)
  • KB 3179995 Tool User Guide: Communication Console

6.1 Preferred email field guidance

  1. After the customer information has been loaded via MST, confirm with the customer their preferred email address. If the customer is having an issue accessing their Microsoft account email, you should ask for an alternative email address.
  2. If a customer provides an email address other than the Microsoft account email that they already have, select Yes in the Does the customer have a Preferred Email that is different than the Customer Account Email above? field and enter the customer's new email address in the Preferred Email field.

Remind them that they can add a contact email address to their Microsoft account.

  • For the customer: Go to https://account.microsoft.com/billing/addresses, select Billing email
  • For the advocate: Via MST -> Update profile -> add to their billing profile

Note: This field should only have one email address, not multiple.

7. Multiple issues during a call/chat

  • Gather information for all the customer's issues.
  • If the issues are related, create only one SR and log the primary reason for contact for the customer.

Examples:

  • The customer has an issue with USB Webcam, USB mouse and USB Keyboard. This issue can be associated to a single case where we describe issues with multiple USB devices.
  • Customer needs to change multiple subscriptions/settings on same Microsoft account.
    • If the issues are unrelated, create a separate SR for each.
    • When you are transferring a customer to a different advocate for one of the multiple issues that you are not able to resolve, create a separate case for the specific issue that you are transferring.

Examples:

  • The customer has an issue installing Office and another issue with viewing webpages.
  • The customer has an office installation technical issue and wants to return the Windows 10 pro upgrade for a refund, which needs to be transferred to different line of business.
    • When dealing with multiple issues, it is important that you negotiate with the customer on priority of the issues. Since you will be working on one issue at a time, we want to start with the issue that matters the most to the customer.
