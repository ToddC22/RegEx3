﻿3. Consult Outcome code options explanation

The Consult Outcome can be selected on the Incident form when you are ready to close the case. Choose the option based on the table below.

Consult Outcome

Description

Assure Support

Support delivered to an Assure subscriber (new purchase or support for an existing subscriber). This also applies to scenario where you are processing a refund of the Assure Service.

Complete Support

Support delivered to a Complete subscriber, may include hardware replacement/repair for covered devices.

Download, Install, Activate – No Charge

Support scenarios in scope for download, installation, and activation of products such as Windows, Office\Microsoft 365, or Windows & Office Updates. Also use this code for scenarios where the customer is experiencing issues immediately after an upgrade or an update.

For example: “My printer stopped working after I upgraded to Windows 10.” We do not have a rule for the time interval between the occurrence of the symptom and the installation date. As long as a customer claims that the issue surfaced after an upgrade or update, support is offered at no charge. This code should also be used for other no-charge scenarios, such as:

  • Microsoft account-related issues
  • Microsoft Family Features
  • Store app issues (including games) and app education
  • Microsoft hardware peripherals (keyboards, mice, wireless display adapters, webcams, headsets, gaming controllers)
  • A customer that lives in a Country where Answer Desk paid service offer is not available (Refer to KB 3049120 Informational: Software Support Services).
  • Issues related to Windows Mixed Reality.

New Product as a Solution

This code should be use under three potential conditions:

  • We convinced the customer to disregard the issue that they have with a third-party product and to use a Microsoft solution instead.
  • No matter what the outcome of the proposal was (for example, if the customer did not buy Windows 10, but we know that in case that is the best solution, it is NPAS). The outcome of the discussion should not depend on the consult outcome.
  • We convinced the customer to disregard the issue that they have with an older Microsoft Product and to use a newer Microsoft solution instead.
  • The only option that we can provide to the customer is a new product because the issue is with a product that is no longer supported.

For example:

  • A customer calls about an issue with Open Office. We convinced the customer to use Microsoft 365 instead and skip the effort to resolve the Open Office issue.
  • A customer calls about an issue where they are trying to share and collaborate on a Word document with Word 2007. We convinced the customer to upgrade to Microsoft 365 as it provides co-authoring features, and other added value such as OneDrive storage.
  • A customer contacts support with a Windows XP or an Office Legacy version (for example, Office 2003) with an issue not related to activation. Since those products are no longer supported, a solution is upgrading to a current version. If the customer upgrades, the code would be new product as a solution.

The customer had a non-genuine product and we convinced him to buy a legitimate PK to NPAS.

Microsoft 365 Post Install Support Excluding DIA

Technical support delivered to an existing Microsoft 365 subscriber only, excluding any download-, install- or activation-related issues.

This code should never be used with the Office Perpetual product category or a Windows category.

One-time Paid Support

Purchase of one-time Answer Desk offer to resolve a single support issue.

  • Premium software support, virus removal and protection and Advanced PC tune-up.

This also includes quick assist cards. This also applies to scenario where we are processing a refund of the One-Time Paid Service fee.

Paid Offer Declined

Customer declined to pay for support and was referred to self-help options.

  • This code does not apply to product sales offer. It only applies to Answer Service offers (Assure, One-time paid).
  • This code should not be used for scenarios where a customer declines the purchase of a product such as Office, Windows, and Hardware.
  • If the customer refuses multiple options (new product, for example: Microsoft 365) and a service offer (Assure or PPI), this is still Paid Offer Declined.

Do not use this code if you are an advocate based in India. A Paid offer decline scenario is impossible for an advocate in India due to the inability to conduct service purchases.

Quick Solution

Quick Fix Scenarios (Quick Solution) are technical support issues that have fast and easy resolutions. It could also be a simple, or ‘how to’, question which can be answered quickly without troubleshooting steps. These should take no more than 5-10 minutes to resolve.

  • If you can resolve the customer's issue, code this as Quick Solution.

If you are not able to resolve the issue, a paid offer is expected, and the consult outcome code should reflect the outcome of the offer (Assure, One-Time Paid, or Paid Offer Declined).

Note: We do not code Microsoft 365 subscription DIA or technical support issues as Quick Solution. Use either: Download, Install, Activate – No Charge or Microsoft 365 Post Install Support Excluding DIA.

Redirect

The issue is not in scope and the customer is redirected. This code should only be determined and used within the first 10-15 minutes of a call. The Redirect consult outcome should only be determined based on initial questioning and diagnostic. This code should only be used with specific scenarios:

  • Office Tech Support to Windows Tech Support and vice versa
  • In-app support for Outlook.com or OneDrive
  • Third-party outside of Microsoft

Do not use this code for the following scenario:

We assist the customer - diagnose and delivered troubleshooting steps. As you troubleshoot the issue, you determined with the customer that they need to contact a third-party for further assistance (for example, OEM and ISP).

4.FAQ about Consult Outcome

Q: How should I code the consult outcome for an issue with Outlook.com and/or OneDrive.

A:

  • These Microsoft online services include support known as In-App support. Within the user interface of these services, there is an option where customer can submit a request for support. The consult outcome should be coded as Redirect. If the customer has an issue with the OneDrive App after it has been installed, direct them to in-app support. Steps can be found in KB 3126549 Procedure: Office: OneDrive - Consumer Support Boundaries
  • If it's an issue where the customer is not able to send/receive outlook.com emails from a desktop mail app (for example, Mozilla Thunderbird, and Outlook 2007), then a paid offering is expected. The outcome of the service offer will help you determine the proper consult outcome code

If it’s an issue specific to a web browser, and troubleshooting can be conducted then a Paid offering is expected. The outcome of the service offer will help you determine the proper consult outcome code

For detailed information on how to code Outlook.com or OneDrive issues that are being redirected to in-app support see the following KB:

  • KB 291054 Procedure: Office: Outlook.com - Consumer Support Boundaries
  • KB 3126549: Procedure: Office: OneDrive - Consumer Support Boundaries

Q: How should I code the consult outcome for an issue with Windows XP, Vista, or Office 2003?

A: These products are no longer supported. The first aspect to determine is if this an activation issue. Even though these products are no longer supported, there are exceptions if it's related to product activation (see KB 2458477 to understand products supported for activation). If it is, then code the consult outcome as DIA. If it's not related to product activation, it should always be New Product as Solution.

Q: If a customer has an Assure subscription, and they have a Windows XP issue that is not related to product activation can I use Assure Support?

A: No. Despite the subscription, Windows XP is no longer supported. The consult outcome should be New Product as Solution.

Q: If a customer has a non-genuine product? How should I code the issue?

A: It's important to capture further details of the scenario. We do not expect or require for you to check if the product is genuine on every call or chat. You may have proceeded with delivering support to the customer with one of the codes listed in the above (Assure, One-Time Paid, Download, Install, Activate – No Charge). However, if early diagnostic and questioning suggest that the customer is using a non-genuine product, then the appropriate code should be New product as solution. 

For example:

  • The symptoms describe an issue that appears to be a malware infection. You complete a sale of Assure and deliver support for the customer. During troubleshooting, you discover that their Windows is non-genuine. You educate the customer that they are a victim of software piracy. The consult outcome is Assure Support.
  • A customer tells you that they just had their friend install Windows on their PC. They are also not able to activate. You ask for licensing details (product key) and discover that it's a non-genuine product. You educate the customer that they are a victim of software piracy. The consult outcome is New Product as Solution.

Q: After some troubleshooting, I discovered and concluded that the customer has a hardware failure (for example, bad hard drive). I redirect the customer. Should I code the consult outcome as Redirect?

A: No. A consult outcome does not mean it's a troubleshooting outcome. Before you proceeded with troubleshooting a proper consult outcome should have already been set (Download, Install, Activate – No Charge, Assure, One-time Paid Support, Microsoft 365 Post Install Support). Even if a redirect is required after troubleshooting was completed, the consult outcome should never be changed to Redirect.

Q: How should I code the consult outcome if the customer has an issue with their Microsoft account?

A: Microsoft account issues are part of the list of issues where no-charge support can be delivered. The consult outcome is Download, Install, Activate – No Charge. The full list of no-charge scenarios:

  • Microsoft account issues
  • Any Store app issues (including games). These are issues with apps that are offered via the Windows Store. This can include download, install, and usage issues.
  • Any Microsoft hardware peripherals (keyboards, mice, Wireless Display Adapter, Webcams, Headsets, Gaming Controllers)

 

Q: How should I code the consult outcome if the question/issue is related only to launching Windows Defender and its functionality?

A: It should be coded as DIA unless it is related to virus infection. If it is a virus infection, it should be a paid scenario.

﻿5. Completing the Incident form (coding)

Important notes:

  • Remember to complete all the fields (marked with a star) when closing. If you don’t complete them, you will receive a prompt to complete the required fields and a red exclamation mark will appear next to the field you missed.

First, you must complete the Incident form before you can close/complete SR. As you start working with the customer, you need to choose options available on the form. The Incident form fields will change based on the product or sub product selected.  

First, the case for the customer should be either created or opened.

Step
Incident form - Screenshot
1. Select the Call Type from the drop down. Note that based on the line of business (LoB) you support, the form will change.

2. In the Ghost/Transfer /Redirect select the drop down and choose options. Note that if you choose an option the form will change based on what you have selected.    

* Refer to Section 1 of this article for details.

3. Complete the greet customer and collect info section. Enter the Customer’s Name, Callback Number and Email address if not already populated.

Enter Subject

Note: Customer account will pre-populate for you if the customer has been verified. Do not enter customer’s info for activation queries.

4. Select the Customer’s Country.

5. You can use Completed Actions Taken and Results: box to document your troubleshooting steps and notes.

Here are some example what you can enter in in the Completed Actions Taken and Results:

[Asset 4493740]

  • Troubleshooting steps taken  
  • Data Collection and Analysis – Anything you need to collect or test?  
  • Research – Anything you need to test or research before contacting the customer.  
  • Next Steps you or the customer are to take 
  • Expectations set with the customer  

Remember that you are not writing notes for just yourself but for the next advocate that may have to look at the history of customer’s issue to help them next time.  

Also see KB 3076798 Procedure: Case Title and Case Notes.
6. Complete the reason for contact section.

7. All fields that have a red exclamation mark are required fields and must be populated in order to Complete the case.

Note: Add Reason for Contact option is used when multiple issues occur on one call instead of creating a new SR. Selecting the Add Reason for Contact will not create a new SR but will amend information to the existing SR.

As you are selecting Product Category, Product, and so on, the form will display fields specific to the problem to complete.

[Asset 4493741]

[Asset 4493747]

8. Complete the Case Documentation. Select Yes in the Add Sales/AB drop down if you made a sale of Windows, Office or another product.

9. In the Promotions field, if you gave a Goodwill token select MTLU 1.

Under Achieve More Conversation? Select Yes or No.  If you selected Yes, fill out the expanded fields.


[Asset 4493743]
 

10. Complete the case disposition section.

11. Select if a customer requires a Scheduled Follow Up (Callback)

12. The Status of the call (Closed: Resolved, Closed: Unresolved, or Open)

13. Select the Consult Outcome code. See Consult Outcome codes explanation in this article.

14.  DTM shared with internal users:  See KB 4135034 Tool User Guide: Data Transfer & Management (DTM), “Using DTM Collaboration feature inside Assisted Support Desktop”

 [Asset 4493749]
 
15. Save the case or Complete (If the Status was set to Open select Complete and the SR will remain open). [Asset 4493750]


For additional information about SR closure, see the following Knowledge base article: KB 3046180 Tool User Guide: Assisted Support Desktop (ASD)
