Assist partners through their Co-Sell Journey

When you have a partner who wants to have a Co-Sell / IP Co-Sell Ready solution, this is what you need to know from your first conversation:

1. Do they have an “In-Market” or built/ready Solution?

  • If yes; make sure it’s published on our MarketPlace, if it’s not published, assist with Marketplace Onboarding (Push Trial or Transact as the ideal listing type because there is built-in technical validation)
     
  • If no; no problem, CED can help build solutions and get them published on our MarketPlace. Partner can still work on the Co-Sell requirements while they get their solution to market

2. Is their Solution:

  1. A Modern Workplace solution e.g. built on Teams, O365, SharePoint Online, etc
    • Is it an IP solution?
  2. A Business Applications/Biz Apps/Dynamics Solution e.g. D365, CRMOL, etc
    • Is it an IP solution?
  3. An Azure Solution e.g. Built on Azure and driving Azure consumption
    • Is it an IP solution?

3. Is the partner an ISV or Services Partner?

If they are an ISV, the Gold competency requirement is automatically waived for them. After you have gathered the basic information from the partner, explain all the requirements they will need to complete (see section Co-Sell Requirements by Solution Type).

The next step is to verify the offer type for their solution and that will determine if you would set up their Account in OCP CRM or go through the Co-Sell option in Partner Center.

[Asset 4577781]

Reference Link: Co-Sell option in Partner Center

If the offer type is Software as a Service (SaaS), Dynamics 365 for Customer Engagement & PowerApps, Dynamics 365 for Operations, Dynamics 365 Business Central or Azure Application, the Co-Sell option is currently available in Partner Center and you can guide the partner on how to complete the Co-Sell requirements using the steps in Co-Sell in Partner Center Guide.

If the offer type is Azure Virtual Machine, Azure Container, Consulting Service, IoT Edge Module or Power BI Service App, the Co-Sell option is not yet available in Partner Center. You will need to send the FY21 Information Needed for OCP CRM.docx and the Bill of Material (BoM) templates they will need to complete. Once the partner has filled out the FY21 Information Needed for OCP CRM, they will need to resend the updated document to the CED Specialist. Once they do so, the next step for the CED Specialist is to set up the account in OCP CRM using the steps below:

Open OCP CRM:
 

Does the partner have an OCP CRM account?

  • If no; create a new account by following the steps outlined in the training video on adding a new account to OCP CRM 
  • If yes; you have three options available:
    • If there is no PDM listed in the Account section in OCP CRM or the PDM’s alias is no longer valid, in the Microsoft Address Book, you need to open a ticket with OCP Support to have the account moved to CED Business Unit and the status set to SW-Only
    • If there is a valid IPDM or PDM-R listed in OCP CRM, then the users have elevated privileges in OCP CRM and can create a new solution under the Engaged, non-CED business unit account, by following the tutorial here
    • If there is a PDM associated with an account that isn’t an IPDM or PDM-R the CED does not work with that partner account

Once you have an OCP CRM Account for the partner, you now need to create the Partner’s solution in OCP CRM, so that it appears in OCP GTM.

Open OCP CRM:

Follow the steps outlined in the training video on adding a solution in OCP CRM and OCP GTM.

  • It can take up-to 24 hours for a new solution, created in OCP CRM, to be live and editable in OCP GTM.
  • Make sure you add the partner contact in OCP GTM (as well as OCP CRM), once you’ve added them, you must click the Invite button in OCP GTM (under the contacts section), this enables the partner to administrate their solution and you can provide them with this guide to navigate OCP GTM.

[Asset 4592390]

  • The partner contact can now do most of the work to get the solution Co-Sell Ready. However, some information cannot be added in OCP GTM, so the CED Specialist will need to add them in OCP CRM for the partner e.g. the Partner description, the solution description, end customer segment, etc. The more you assist, the quicker the partner can start Co-Selling with Microsoft.

  • The Solution Description is shared between the marketplace listing and the Co-Sell Solution finder listing, so this requirement should be updated in Offer listing

Duplicate solutions
 

You may run into a scenario where a solution is created in OCP CRM and published in OCP GTM and the same solution has been created by the partner in Partner Center. If this happens, you will see the same solution twice in OCP GTM and Co-Sell solution finder. In order to prevent this from happening, before publishing the solution for the partner in OCP CRM, check the list here, to ensure that the offer type is not available to publish through Partner Center.

Impact: Sellers will see the same solution twice in Co-Sell solution finder.

Resolution:

If both solutions are still in market, the CED specialist would be responsible for deactivating the solution published in OCP CRM.

[Asset 4592380]

If the solution created in OCP CRM is already Co-Sell Ready and the partner creates a duplicate in Partner Center, the CED Specialist should contact OCP Support to resolve the issue.

Intellectual Property (IP) & Incentive Eligibility

All solution can be IP, but not all IP solutions can be Incentive Eligible.

Both ISV & Services partners can have IP solutions. And both can have Incentive Eligible IP solutions. But no Partner can have a Services solution that is Incentive Eligible.

What is Incentive Eligibility and why is it important to our partners?

IP Incentive Eligible solution provides a revenue percentage to the field seller and helps them hit their scorecard metrics. There is no financial benefit or risk to the partner, but it does make their solution more attractive to Co-Sell with than a similar non-Incentive Eligible solution.

Incentive Eligibility Solution requirements

An Incentive Eligible Modern Workplace solution:

  • Must be a Teams solution
  • Must meet all Co-Sell Requirements
  • Must be an IP
  • Must have Microsoft Teams in the solution description
  • Must have a Teams Sales Play/Sub-type selected

An Incentive Eligible Business Applications Solution:

  • Must be enrolled in the Business Applications ISV Connect Program
  • Must meet all Co-Sell Requirements
  • Must be an IP

An Incentive Eligible Azure Solution:

  • Solution in MarketPlace must be list as Trial or Transact, if not, Solution must be approved as IP by PTC team following the Cloud Enablement Desk: IP (Incentive Eligibility) Technical Validation for Azure Co-Sell Solutions Process
  • Must meet all Co-Sell Requirements
  • Must be an IP
  • $100K Azure Trailing Twelve Months Consumption (TTMC)

CED takes a screenshot from OCP Insights if the consumption is under the partners DPOR

or

Partner takes a screenshot of their own Azure tenant from their Azure Console.

Note: If an ISV were to request an exception for the Azure TTMC, the CED Specialist should fill out the Azure TTMC Exception Request Template.docx and send it to the Co-Sell Desk. The exception is only valid for 6 months and the partner will need to demonstrate the 100K Azure TTMC after the 6 months has ended.

Marketplace Onboarding 

For a partner to become Co-Sell Ready/IP Co-Sell Ready, they must publish their solution to the Azure Marketplace or AppSource Storefront (H1 exception: MW Teams Service).This will be done through the Commercial Marketplace Portal or Office Store Portal in Partner Center. The offer type will determine which portal the partner should utilize.

If the offer type is Azure Application, Azure Container, Virtual Machine, Consulting service, Dynamics 365, IoT Edge module, Managed service, Power BI app or SaaS, the partner should utilize Commercial Marketplace to publish.

If the offer type is Office add-in, SharePoint solution, Power BI visual or Microsoft Teams App, the partner should utilize Office Store to publish.

For guidance on how to publish through Commercial Marketplace, please refer to the links below:

  • Create an Azure Application offer
  • Create an Azure Container offer
  • Create an Azure Virtual Machine offer
  • Create a Consulting Service offer
  • Create a Dynamics 365 Business Central offer
  • Create a Dynamics 365 for Customer Engagement & PowerApps offer
  • Create a Dynamics 365 for Operations offer
  • Create an IOT Edge module offer
  • Create a Managed Service offer
  • Create a Power BI app
  • Create a SaaS offer

For guidance on how to publish through Office Store, please click here.

Once the partner publishes to the Marketplace and is ready to become IP Co-Sell Ready, the publishing option will determine if the solution requires a technical validation.

Marketplace Listing

[Asset 4577795]

Reference Link: https://docs.microsoft.com/en-us/azure/marketplace/determine-your-listing-type

Trial or Transact is a status a Solution can have in either AppSource or Azure Marketplace. If the solution is listed in AppSource or Azure MarketPlace as “Free Trial”, “Test Drive” or “Get it now” then these are Trial or Transact solutions and the technical validation has been done automatically when the solution was listed. This means they can achieve the solution type “Incentive Eligible” in OCP GTM.
 

If an Azure solution is listed as “Contact Me” (e.g. not Trial or Transact) then a PTC technical validation will need to take place and pass the validation to have the solution Type “Incentive Eligible” in OCP GTM.

Steps to complete IP Co-Sell Technical Validation

1. Navigate to the TP&D Field Request Form

2. Enter the Partner’s email address and select Search

3. Create New Contact (if prompted)

4. Search by Partner’s MPN ID/Name and select Create

5. Create New Case

Partner contact: The primary partner POC name

Partner account: Partner company name

Case title: IP Co-Sell Technical Validation

Description: Please provide a detailed breakdown of the support required in the case title

Program: Either CED services or ISV depending on the partner

Service: Advanced

Solution area: Relevant solution area for the partner. See aka.ms/FY21Taxonomy

Key product/technology: Relevant key product/tech area for the partner. See aka.ms/FY21Taxonomy

Priority scenario: Relevant partner priority scenario. See aka.ms/FY21Taxonomy

Language: Partner language

Country: Partner country

6. Select Submit

Note:

  • ISVs do not require MAPS/Competency to complete IP Co-Sell Technical Validation
  • CED Specialist should provide the partner with the IP Co-Sell Validation Architecture Reference Guide in order to prepare for the technical validation
  • It is strongly recommended that the partner completes the technical validation after completing all other Co-Sell requirements
 
Soon all solution will need to be Trial or Transact to be Co-Sell Ready. So you should strongly advise all partners to list their Solutions in our MarketPlace as Trial or Transact.
 

MarketPlace links:

  • Azure MarketPlace: https://azuremarketplace.microsoft.com/en-us/marketplace/
  • AppSource: https://appsource.microsoft.com/en-GB/
