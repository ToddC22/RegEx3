  • If you do not have the option to POBO in a specific country, it likely means that POBO is not offered in that location.
  • If you are assisting a customer with a purchase and cannot POBO, walk them through the online purchase process.

2. Importing cart

Notes:

  • The Import Cart feature allows you to import a cart that is built in the online Microsoft Store. You can now Purchase on Behalf of (POBO) with bundles and other types of store promotions (for example, Build-a-Bundle, Essentials Bundle, Buy M, Get N and Tight Bundles). Note: Dell Preferred Account (DPA) All Access offers are excluded.
  • The customer’s Microsoft account must be verified for the Import Cart feature to work.
  • The Import Cart feature is not limited to store promotions, most carts built in the online store can be imported.

Return to top

2.1 FAQ

Q: What bundles or promotions will work in MST POBO?

A: The Import Cart function works with any cart created on Microsoft Store online, including all the different bundles types and promotions. The exception is Dell Preferred Account (DPA) promotional offers, these will not work.

Q: Do student, military or SMB discounts works with an imported cart?

A: No, discounts can’t be added to an imported cart. Work with the customer to determine the best pricing.

Q: Can multiple bundles or promotions be imported?

A:  Yes, there is no limit to bundles that can be built online and imported. Limitations on promo stacking, cart amounts and promo limits still apply to all orders.

Q: What if the customer changes their mind on one of their bundle or promotion choices and the cart has already been imported to MST?

A: You will close the order in MST, then go back to the online store to rebuild the bundle with the customer’s updated choice(s).

Q: What if the customer wants to add additional items to their order that are not part of the bundle or promotion?

A: If it’s not the customer’s desire to add the items as part of an existing bundle or promotion, select Keep Shopping option in the checkout workflow (see below) and select the additional items. The customer’s original cart still needs to be open in online store, make the edits, then re-import the cart. If it isn’t open, then a new cart must be built and imported.

Q: Can we use this new feature to import Design Lab controllers?

A: No, do not use this new feature for Design Lab controllers, as they must be built by the customer due to the personalization nature of the product. Note: There are no returns, if the color is not what the customer wanted, we are not liable for that choice.

Q: Can we use this new feature to import Buy Online and Pick Up In Store (BOPIS) orders?

A: Yes, but you must change shipping methods to Pick up in Store once the order has been imported into MST.

2.2 How to use Import Cart for POBO

Before using Import Cart for the first time, follow these steps:

  1. Open MST to any Microsoft account.
  2. Navigate to Orders->Import Cart.
  3. On the Import Cart page, select the Microsoft Store link to open the store in a new tab. MST has now been synced with the store session.
  4. Select New Case in ASD, then open the customer’s Microsoft account.
  5. Navigate to Orders->Import Cart.
  6. Select the link to open Microsoft Store in a new tab.
  7. Import Cart should now work as expected, and the old case window can be closed.

2.3 Import Cart isn’t working

Issue:
Error message or cart won’t import:

Symptoms:
Import Cart fails with a red error message. Nothing is imported after selecting Import Cart.

Before using Import Cart for the first time, follow these steps:

  1. Open MST to any Microsoft account.
  2. Navigate to Orders->Import Cart.
  3. On the Import Cart page, select the Microsoft Store link to open the store in a new tab. MST has now been synced with the Store session.
  4. Select New Case in ASD then open the customer’s Microsoft account.
  5. Navigate to Orders->Import Cart.
  6. Select the link to open Microsoft Store in a new tab.
  7. Import Cart should now work as expected, and the old case window can be closed.

If Import Cart still fails, try the following:

  1. Open Internet Explorer standalone.
  2. Go to Settings > Internet Options.
  3. In the General tab, look for the section Browsing History, and Delete.
  4. Select the Cookies, then select Delete.
  5. Close all active Internet Explorer sessions and proceed\close, then relaunch ASD.

Issue:
Order not found error message.

[Asset 4519635]

How to troubleshoot?
When you go to the store website from the Microsoft Store link within the Import Cart workflow, make sure you are not signed in with your @msncs account (profile icon on the top) prior to copying/pasting the order number. If signed in, please sign out, then build the cart again. Signing out should not log you out of ASD session.

Issue:
You cannot switch back to the Import Cart workflow from the Microsoft Store window after building a cart and copying the order number.

[Asset 4519638]

How to troubleshoot?
Double click on the Split Mode Toggler icon and that should resolve the issue.

[Asset 4519640]

If Import Cart still fails, purchase the items individually and provide the discount via CSV.

3. How to resend shipping confirmation/purchase confirmation/shipping label and locate tracking information

Once you’re in MST after account verification, go to the Order section and locate the order and product that you want to resend the Shipping Confirmation or Purchase Confirmation or view tracking information. Select the Action icon and select the appropriate resend option. Look at the Order State to see the tracking number.

Note: You will only have the option to resend the email notification to the customer. You will not have access to the contents of the email that was sent.

Important note: Resending shipping labels is not an available option currently. If customers call about wanting a shipping label that was already sent previously, guide the customer to print out the label in their order history at account.microsoft.com. If that doesn’t work, you will not have a way of resending the shipping label, and instead, need to re-submit a new return line item to send a customer a new shipping label.

4. Signature required for delivery

In certain countries, products with certain properties (like maturity ratings) require an adult to sign for the delivery when being delivered. If a product has one or more of these properties, a message indicating the type of shipping/delivery restriction will be displayed.

Note for German users: The adult signature restriction is in place with messaging indicating that delivery to minors is excluded is displayed, on the Cart, Select PI, and the Review and Place Order pages.

10. How to perform an exchange for the same item

An exchange is performed when the customer wants to replace their item with the same item. The process requires us to receive the item the customer is sending in for exchange before we ship them the same item.

  • Once an exchange has been performed on an item, you cannot cancel the exchange and take another action such as return on the item.
  • If the customer changes their mind on the exchange and instead wants a return, the customer will have to follow through with the exchange before we can initiate a return.
  • The customer will have to ship their item to us, wait for us to send them the new item after their item has been received, then call us again to initiate the return.
Step Screenshot
1. Go to Orders and find the item the customer wants to exchange. Select the Action icon and select Exchange Line Item. [Asset 4021324]
2. Select the address the customer wants to use for the exchange, then the Reason for Exchange, and enter additional details in the comments. Select Next. [Asset 4496425]
3. This is the final summary. Verify the information and select Submit. [Asset 4496414]
4. You will receive confirmation that the exchange request was completed. [Asset 4496417]
5. If there is not enough inventory to perform the exchange, you will receive a message stating the exchange could not be performed, and to escalate the case. [Asset 4496418]
