1. Case creation

  • Whenever you talk to a customer, check if there is a case already in RAVE. Review any open or recent cases to see if the customer is calling back about an existing issue (see section 1.2).
  • If you receive a transfer or the customer mentions they have already worked with us, check ASD to see if any previous Service Requests (SRs) exist for that customer (follow guidance in section 1.3).

1.1 When to create a case in RAVE

Every phone call or chat received or made in RAVE needs to be associated to a case.

Note: Make sure you associate Microsoft accounts whenever possible and are getting accurate contact info when creating cases.

Case creation scenario

Handling

There was no RAVE case created for you.
  1. Search for the customer’s Microsoft account in RAVE.
  2. Authenticate.
  3. Create one manually.

You use Comm Console with RAVE as your case management tool.
  1. Create the case manually in RAVE.
  2. Only create a case in ASD if you need to transfer the case outside of Office.

You are not using OneCloud for your phone and have a stand-alone tool.

or

If OneCloud WWE client is being used as a stand-alone phone client (not integrated).

 
  1. If the customer is already associated to the case, select Customer Name in the upper left.
  2. If the customer is not associated to the case:
    • Create cases manually in RAVE whenever the customer has a Microsoft account you can authenticate.
    • If the customer doesn’t have a Microsoft account you can authenticate, create the case in ASD.

Note:

  • Always check to see if the customer has already contacted us and has a case in RAVE.
  • If you are using OneCloud within RAVE, do not launch OneCloud WWE client stand-alone.

Commercial Presales: Follow your existing guidance for these customers. No RAVE case is expected. 

You are assigned multiple RAVE cases on the same issue.
  1. Close the additional cases as Duplicate.
  2. Provide the case number of the main case.

Important: Do not use this status for duplicate ASD cases, only duplicate RAVE cases.

Escalate the case to Tier 3/Record an Advise Me transaction

Create an ASD case in addition to the RAVE case:

  1. Copy and paste your Internal Notes, Issue Description, and Resolution steps to the ASD case.
  2. Add the RAVE case number to the ASD case in a case note.
  3. Add the ASD SR number to the RAVE case in an internal note.
  4. For tools only accessible via ASD, you only need to create a RAVE case.
  5. Associate the RAVE case to the same customer account you are using with the tool.
    • If the tool or process asks for an ASD case number in a field or note, use the RAVE case number instead.
    • If the tool requires that an ASD case is present, generate an ASD case in addition to the RAVE case.
    • If you are using RAVE and need to use MST/ASD to perform any action on the customer’s account like a refund, cancellation, or new purchase, you must create an MST comment on each account being serviced and cite the RAVE case number:
      • If you cannot find or associate the customer account in RAVE, you will need to generate the ASD case and use the ASD case number in the tool.
      • If you need to transfer the customer's call, see section 4.

1.2 The customer has an existing RAVE case

  1. When you receive a case/support request, the first step to take is to ensure you populate customer’s information (Microsoft account or email/phone number). Use the case you were just assigned to document the whole customer interaction (take case notes, select the appropriate case coding, reference the history as appropriate). See KB 4552001 Procedure: CMF: Case documentation/coding for guidance on case documentation.
  2. Once the customer’s information is populated, you can look for existing case and reference it in the case you just worked on.

Important: Ownership of an existing case associated to the customer account is now required to open and view an existing case, with authenticating to the customer information as a backup method for access.

Ownership scenario

Handling

The case is already assigned to you (you are the owner of the existing case).
  • You can open that case without having to authenticate again.
  • You can access the customers profile without having to authenticate again.
  • You can open cases closed within the last 40 days.
  • You can access other open cases that customer may have.

The case is not already assigned to you (you are not the owner of the existing case).
  • You see an authentication screen: This means the case has contact information associated to it. Round trip a code as normal to access the case.
  • You receive an error message with a folder icon, and no contact information: This means the case does not have contact information associated to it, and you will not be able to access this case. You will need to create a new case.

Note: Make sure you get the customer’s Microsoft account associated to the case if possible, as this allows for the best experience if they need to recontact.

Return to top

1.3 The customer has an existing ASD case

  1. Take ownership of the ASD case (if the case is open).
  2. Inform the customer they will receive an email with a new case number. Let them know that the cases are related to the same issue.
  3. Add a case note in ASD advising the customer support case occurred in RAVE. Provide the RAVE case number.
  4. Add an internal note to the RAVE case: #oldasd <provide SR number>.
  5. Work the customers case using RAVE.
  6. Close the ASD case if it was open.

When finished:

  • Resolved issue: No additional steps.
  • Unresolved issue: Copy all your internal case notes and resolution steps from RAVE and paste them into the ASD case.

Return to top

1.4 The customer has multiple issues as part of the same call or chat

Multiple issues you support: Do not create a second RAVE case, note the second issue in the case notes.

If you need to transfer the case to another Line of Business (LOB) that uses ASD, create the ASD SR for the issue you need to transfer. No new RAVE case is expected for this issue.

2. Customer account association

Always attempt to associate the customer’s Microsoft account to the case. Associating a customer by email only is not possible in RAVE. If you are unable to associate the customer’s account to the case explain why in the notes.

Return to top

3. Taking notes in RAVE

When creating case notes in RAVE:

  • Use the Activity, Blocker, Solution (ABS) standard format. See section 3 of KB 4552001 Procedure: CMF: Case documentation/coding.
     
    • Record your notes in the internal notes field. Copy and paste the ABS elements to the correction sections on the Summary tab.
    • Activity and Blocker go in the Issue Description field within the Summary tab.
    • Solution goes in the Resolution field within the Summary tab.
  • Use the Internal Case Notes field in the Communications tab to log any troubleshooting steps you performed. Notes in this section are intended to help any future advocates understand previously worked issues and steps.

Return to top

4. Transferring the case

4.1 To another LOB by phone:

  1. If the receiving advocate is using RAVE, you do not need to generate an ASD case.
  2. If the receiving advocate is not using RAVE create an ASD case. If the customer has an existing ASD case for this issue, use it for the transfer.
    • If the customer does not have an existing ASD case, create a new case in ASD and add your internal case notes, the RAVE case number, and the issue description.
  3. Add an internal note to the RAVE case with the ASD case number are transferring.
  4. Transfer the customer using the RAVE phone controls and provide the ASD SR number to the receiving advocate.
  5. Close the RAVE case with a Resolution of Transferred.

Return to top

4.2 To another LOB by chat:

There is no way to transfer a RAVE chat to Comm Console. The customer needs to start a new chat in the correct area they need assistance with.

  1. If the customer has an existing ASD case for this issue, use it for the transfer.
    • If the customer does not have an existing ASD case, create a new case in ASD. Add your internal case notes, RAVE case number, and issue description to the ASD case.
  2. Add an internal note to the RAVE case with the ASD case number.
  3. Guide the customer to the correct website to start their chat. Give them guidance on what issue they need assistance with. For example, if they need to chat with Xbox, direct them to support.xbox.com to start a chat.
  1. Provide the customer with the ASD SR number and ask them to provide it to the next advocate.
  2. Close the RAVE case with a Resolution of Transferred.
  3. Close the ASD case.

Standard warm or cold transfer phone guidelines apply. See KB 3163896 Procedure: All-up landing page for Phone and Chat Transfer.

Return to top

4.3 To a previous Tier 3 escalations advocate

Escalation features are not available in RAVE. If Tier 3 has the case, add a case note to the ASD case if you need to send a message to them. Re-escalate the ASD case if it has been with Tier 1 and needs Tier 3’s attention.

  • Add a case note to the ASD case with the RAVE case number.
  • Add an internal note to the RAVE case with the ASD SR number.

Return to top

4.4 Commercial transfers

Do not create an ASD case if transferring to Commercial support.

  • Transfer the customer using the RAVE phone controls.
  • Close the RAVE case with a Resolution of Transferred.
Return to top
 

5. Escalating a case

5.1 Tier 3 escalations

Not available in RAVE right now:

  • If the customer has an existing ASD case, use that case for the escalation.
  • If the customer does not have an existing ASD case, create a new case in ASD. Add your internal case notes, RAVE case number, and issue description to the ASD case. Let the customer know they will receive an email with a new case number, and assure them the cases are related to the same issue.
  • Close the customer’s case using the Close Case action.

Note: Add an internal note to the RAVE case with the ASD case number you escalate. Escalate the case in ASD.

Return to top

5.2 Tier 2 escalations

If your existing process indicates you can transfer to Tier 2 phone support, transfer using Corporate Favorites within RAVE.

  • If the customer has an existing ASD case, use it for the escalation.
  • If the customer does not have an existing ASD case, create a new case in ASD. Add your internal case notes, RAVE case number, and issue description to the ASD case.
  • Inform the customer they will receive an email with a new case number. Let them know that the cases are related to the same issue.
  • Close the customer’s case using the Close Case action.

Note: Add an internal note to the RAVE case with the ASD case number you will escalate.

Return to top

6. Emerging issues

If you receive a case about an emerging issue identified on the Advocate Blog, create an ASD case in addition to the RAVE case.

  1. Copy and paste your Internal Notes, Issue Description, and Resolution steps into the ASD case.
  2. Add the RAVE case number to the ASD case in the note.
  3. Add the ASD SR number to the RAVE case in an internal note.
  4. Indicate the emerging issue color code in the internal case notes.

Return to top

7. Handling customer chats

  • RAVE doesn’t have automatic greetings. When you join a chat, it does not remember to acknowledge the customer right away.
  • Send the rejoin link only when you need the customer to reboot their computer. Do not send one for ghost chats or other cases. The rejoin link expires once you close the case using the case action.
  • If you need to make a call, sign in with OneCloud via the settings menu. Sign out when done.
  • Remember to go to Available after receiving a chat to receive additional ones. If you are unsure how many chat sessions you should be doing at a time, contact your supervisor.
  • If it looks like you have lost connection with the customer while they are in queue for a chat, PossibleChatAbandon – true is added to the Issue Description. There are several reasons why the system might detect this, in some of these cases the customer will still be present, don’t assume they are not there. Give the customer a chance to respond but be ready to start the three strikes process. If you get to the first strike you may want to take another concurrent chat as the customer is likely disconnected.
  • Go to Available manually after completing a chat to receive another one. You will not be automatically made available when finishing a chat.

Return to top

8. Handling customer phone calls

There are four types of phone calls you will receive:

  • IVR Inbound: Customers who called a phone number to reach Microsoft.
  • Web Callback: Customers who requested a callback on the web.
  • Transfers (from non-RAVE advocate): Customers who were transferred to you from an advocate not deployed on RAVE.
  • Transfers (from RAVE advocate): Customers who were transferred to you from an advocate who uses RAVE.

Regardless of the call type you receive make sure you check your presence in the tool:

  • When calling a customer or on a call, you should be set to On a case.
  • After hanging up with a customer, you should be set to After call work.
  • After call work does not end automatically. Make sure you are going to back to Available as soon as you finish wrapping up your case.

Return to top

8.1 IVR Inbound

  • These calls arrive with the customer.
  • A case opens automatically for you.
  • You must associate and authenticate the customer account.

Return to top

8.2 Web callback

When you get a web callback case in RAVE you need to initiate the callback to the customer. A case should open automatically for you, and the customer’s account will be associated and authenticated.

Note: Check the customer’s phone number and make sure to include the correct international dialing prefix. Make sure to collect the international dialing format (customers often enter the phone number as dialed locally).

8.2.1 If you reach the customer

  1. Identify yourself as a representative of Microsoft.
  2. Inform the customer you are contacting them because of the inquiry they requested on our support page.
  3. Confirm you are talking to the person who sent the request. Their name will be on the case in RAVE.

Messaging: “Hello. I am <your name> calling from Microsoft. I see you requested a phone call from our support page. Have I reached <customer name>?”

8.2.2 If you get an answering machine or voicemail

First or second time:

  1. Identify yourself as a representative of Microsoft.
  2. Inform the customer you are contacting them because of the inquiry they requested on our support page.
  3. Let them know you will try again in a minute.

Messaging: “Hello. I am <your name> calling from Microsoft due to a request you submitted on our support page. I’m sorry to have missed you, I’ll try again in a minute.”

Third time:

  1. Identify yourself as a representative of Microsoft.
  2. Inform the customer you are contacting them because of the inquiry they requested on our support page.
  3. Let them know they need to contact us again if they still need assistance.

Messaging: “Hello. I am <your name> calling from Microsoft. I’m sorry I missed the opportunity to assist you today. Please visit our support page again if you still need assistance. Thank you!”

Close the case – Resolution = Unresolved, Contact outcome = No contact

8.2.3 How often do I try to contact the customer?

  • Wait one to two minutes between attempts to reach the customer.
  • Try to contact them three times only.

8.2.4 I am at the end of my shift or need to go on break and cannot reach the customer

If you have not reached the retry limit, talk to your supervisor. They will provide additional guidance or help reassign the case to an available advocate.

8.3 Transfers (non-RAVE source)

These cases come from an advocate who is not using RAVE. These calls will look like IVR inbound volume, but they arrive on a transfer skill.

  • These calls arrive with the customer.
  • A case opens automatically for you.
  • You must associate and authenticate the customer account.

Return to top

8.4 Transfers (RAVE source)

When you receive a transfer from an advocate who also uses RAVE it will look similar to IVR Inbound volume.

  • A case is opened for you.
  • The customer account may be associated, if the original advocate associated the customer account to the case, if they did not, you need to associate and authenticate.
  • Any text boxes that were filled out like case notes, issue description, or resolution steps, will still be filled out in the case. You need to select the product and any other check box or drop downs.

Return to top

8.5 Another call offer arrived while I have one

In some rare cases you may get an offer of a call of the opposite type.

IVR Inbound or transfer with web callback offer:

  • Decline the web call back when it comes in. This routes to another advocate who is available.
  • If you accidently accepted the web callback, handle it once your current case is finished. If the web callback ends and you are unable to handle it or your current case lasts longer than 30 minutes, contact your supervisor and ask them to reassign the web callback to another advocate.

You accepted a web callback then an IVR Inbound or transferred call arrived:

  • Help the customer on the IVR Inbound call. If you are unable to handle the web callback when your IVR Inbound ends, or your current case lasts longer than 30 minutes, contact your supervisor and ask them to reassign the web callback to another advocate.

9. Completing a call or chat

After completing a call or chat, and the customer:

Needs to be transferred or escalated:

  1. Finish documenting the call including transferring your internal notes to the appropriate fields and filling out any product coding.
  2. Take the appropriate action for the customer.
  3. Close the RAVE case using the close case action. Do not use the Get Feedback action.

Needs additional help:

  1. Finish documenting the call including transferring your internal notes to the appropriate fields and filling out any product coding.
  2. Schedule a follow up with the customer if appropriate and close the case tab when complete, see KB 4560056 Procedure: CMF: Callback.
    • You can find your scheduled contacts in My Callbacks in the upper left menu.
    • You can find cases you have left open in My Active Cases in the upper left menu.

Does not need additional help:

  1. Finish documenting the call including transferring your internal notes to the appropriate fields and filling out any product coding.
  2. Use the Get Feedback case action. This ends your current call or chat and sends the customer the survey.
    • Always select the IVR and Web or Chat and Web option.
    • The case will be in a pending status after completing this action.
  3. Close the case using the Close case action.

Note: If you forget to close the case and just close the tab, look for the case in My Pending Cases in the upper left menu.

14. FAQ

Q: I am available, but have not received a call or chat in some time. My coworkers are receiving contacts.

A: Reach out to your supervisor and have them check the skills associated to your account. Improperly configured accounts can result in not receiving customer contacts. If this does not resolve the issue or you receive a Bad State error, try refreshing the browser.

Q: I am on a call with the customer and some of my call controls are grayed out or unresponsive.

A: Save any updates to your case and refresh the browser. This will not impact the phone call with the customer, you will still be able to talk to them. You will not be able to put the case on hold or transfer it until the browser has finished refreshing. If no call controls appear from the beginning of the call, talk to your supervisor and have them check that your OneCloud login and RAVE login match.

Q: Error: “Failed to save closing summary. Please try again.”

A: There are a couple of reasons you may see this error:

  • You are trying to save the case while performing an action via the action menu. For example, you are trying to save your case notes while scheduling a callback.
  • If you enter anything that isn’t a URL into the Bug URL field. Just leave this field blank.

Q: What is Research presence and when should I use it?

A: Only use this if your supervisor indicates you should.

Q: I got an email bounce back, I have not added a customer account to the case.

A: If you have not added a customer account to the case the customer will not receive the email. The tool will send the email to a default email address which will bounce back.
