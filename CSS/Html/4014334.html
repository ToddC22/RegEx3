Procedure: Surface: Using the Microsoft Surface Diagnostic Toolkit

[Asset 4049349]

Who should use this article?

  • Surface

What is this article about?

This article explains how to use the Microsoft Surface Diagnostic Toolkit (SDT) and the Microsoft Surface Diagnostic Toolkit for Business (SDT4B) as an additional requirement to current troubleshooting. This should be followed when a customer is contacting us about their Surface device running Windows 10. If troubleshooting steps prior to that guidance resolves the issue, then SDT or SDT4B is not required, but can be used optionally to add value.

Note: SDT and SDT4B do not work on Surface Duo.

1. SDT use exceptions

SDT or SDT4B is required on every repair order that is setup unless the issue is related to one of the exceptions:

  • Devices running Windows 8.1 or earlier.
  • Customer induced damage scenarios (for example, cracked screen or broken kickstand).
  • No network connectivity as the tool requires an internet connection.
  • Issues where the device will not turn on or boot to Windows.
  • Issues where SDT or Windows Update is not a troubleshooting step (for example, Microsoft Account Forgotten Password or Pen requires a new battery).

2. Overview

The Microsoft Surface Diagnostic Toolkit (SDT) and the Microsoft Surface Diagnostic Toolkit for Business (SDT4B) resolves the most common Surface issues.

Note: For the rest of this article, except when needed, SDT will mean either SDT or SDT4B as appropriate based on customer’s segment.

SDT is downloadable by the customer and can take 10-15 minutes to complete if their device is up to date with all Windows Updates.

The automated tool pinpoints and auto heals issues that can contribute to or cause the symptoms the customer is facing. SDT4B allows the customer to create a support request directly from the tool and includes device information and diagnostic logs, so you can begin troubleshooting right away.

There are four versions of this tool:

  • All devices running Windows 10 (not in S mode) should download and use the executable (.exe) version of the toolkit appropriate for their customer segment. See section 3.1 for appropriate links.
  • Consumer only: Surface Laptop running Windows 10 in S mode should download the app version from the Microsoft Store since executable files won’t install on a device running Windows 10 in S mode.

If SDT is not listed in the troubleshooting steps, but Windows Update is, then SDT should be run in place of Windows Update.

  • SDT is required on every repair order that is setup unless the issue is related to one of the exceptions in section 1. Requirement: The reference ID (19 digits) under the Telemetry section must be recorded in case management notes after the SDT diagnostics have run in the customer’s device.
  • If a customer indicates that they have already run SDT, have the customer start SDT and on the Telemetry tab, verify that the last run was within the last five days. If not within the last five days, the customer will need to carry out a new diagnostic run.
  • If SDT does not resolve the customer’s issue, the next step is to continue standard troubleshooting, or the device reset step.
  • Follow the steps and guidance provided in the section 3, for specific situational handling instructions.

Note: If either of the following conditions are true then include a link to SDT, in any email you may send. SDT may resolve the customer’s issue without need for further support or device servicing. See section 3.1 for appropriate links.

  • The customer is out of warranty (OOW) and declines the paid diagnostics offers.
  • The issue is resolved with standard troubleshooting steps before SDT is used. Providing the information about SDT can be a value add for the customer.

3. Procedures

3.1 SDT download

Note: The online page does not provide screen sequence instructions. If customers ask questions about specific screens, refer to KB 4456776 Tool User Guide: Surface Diagnostic Toolkit (SDT) and access the training content.

Follow existing troubleshooting steps for the customer’s issue but stop at the step where you would install Windows Updates and run SDT instead.

Messaging:

“Please download and run this program on your device. It is a self-healing tool which looks for potential issues and fixes them, as well as ensures your device is completely up to date. The tool takes about 10-15 minutes to run and is a requirement for progressing your repair order. I’ll send you an email with the link now.”

  • SDT
    • Windows 10 Home or Windows 10 Pro: Direct link to the executable http://aka.ms/surfacedt
    • Windows 10 Home S or Windows 10 Pro S: Direct link to app in Microsoft Store http://aka.ms/surfacerepairuwp
  • SDT4B
    • Surface ProX: https://aka.ms/SDTB_tool_ProX
    • All other Surface devices: https://aka.ms/SDT4B_tool

3.2 Customer objects to using SDT

If the customer objects, highlight that running SDT is a requirement as part of our device repairs process and running SDT will avoid potential extensive delays if the device requires repairs.

  • Stop repair order steps if the customer refuses the tool usage and advise the customer that you cannot proceed with the repair request unless SDT diagnostic information (Run ID and date) is provided.
  • See Telemetry screen in KB 4456776 Tool User Guide: Surface Diagnostic Toolkit (SDT) to provide a view of information required.

Return to top

3.3 Customer agrees to using SDT

1. If the customer agrees to proceed, continue with the following and set expectations:

  • The tool takes 10-15 minutes to run unless the tool resolves a Windows Update issue and there are additional Windows Updates that need to be installed.
  • Let the customer know that if the tool does not resolve the issue that there are some additional troubleshooting steps that can be done which may include the need to reset the device.

2. Walk the customer through the steps to download and install the tool. See section 3.1 for appropriate links.

3. Remain on the phone/chat until SDT starts to download Windows Updates.

  • If a Windows Update is pending, inform the customer that you will send an email with the link above so that the customer may finish the SDT steps completely.
  • If the device is already up to date, Windows Update may not initiate – in which case stay on with the customer to finish the steps. If the customer asks to be called back, schedule a call back.
  • If the customer is running Windows 10 in S mode on their Surface Laptop, the executable version of Software Repair Tool will not install or run. In this case:
  1. Have the customer use the app version (http://aka.ms/surfacerepairuwp) available in Microsoft Store instead.
  2. The app needs to run as Administrator. Start menu > find Software Repair Tool > right-click > Select More > Select Run as administrator.
  • If the customer does not want to stay on the call, send the customer an email with this link (Fix common Surface problems using the Surface Diagnostic Toolkit) which provides step by step instructions. Let the customer know there can be a callback to check on the status of their issue and whether the tool helped resolve it. If the customer refuses a callback, offer to send the customer an e-mail and request they reply with the outcome.

4. In case the customer requests a callback and contacts us first, leave notes in the Comments field:

  • What troubleshooting has been completed.
  • Next steps which includes any additional troubleshooting, may include a device restore/recovery/reset.

Note: Consumer: If the customer did not complete running SDT prior to a repair/exchange order being written in MST due to one of the highlighted SDT exceptions, you may receive an error stating that you cannot proceed until running the tool. Review KB 4053533 TUG: MST: Devices section for the exact error message.

This error message indicates that the customer did not run the Surface Diagnostic Toolkit (SDT) within the last five days or did not complete the running of SDT. If the customer did not run the SDT tool yet, have them do so and then create the order again. If the customer is calling back after having been provided these instructions by a different advocate, ensure that the notes are complete, and the customer completed running SDT without issue.

If the customer claims they already ran SDT or is unable to do so, consult with a Tech Lead on next actions. Note that you may see an MST problem type of SDT Bypass (Requires Approval), which is not to be used unless a Tech Lead reviewed and approved use. Do not try to change the problem type in MST to bypass the process.

  1. Code the Did you run Surface Diagnostic Toolkit (SDT) field. Review section 4, for guidance on how to code.
  2. Document specific outcomes in the Comments field in case management tool. Review section 4, for additional guidance.

4. Documentation requirements

Make sure the following data points are recorded in the Comments field in the case management tool:

  • Document that SDT was used as part of troubleshooting and if it resolved the customers issue.
  • Consumer: Use the Did you run Surface Diagnostic Toolkit (SDT) field to accurately describe why you are able to or not able to use SDT.

Value

When to select

SDT is required to be used on every repair order that is setup unless the issue is related to one of the exceptions in this article.

Requirement: The reference ID (19 digits) under the Telemetry section must be recorded in case management notes after the SDT diagnostics have run on the customer’s device.

  • If a customer indicates that they have already run SDT, have the customer start SDT and on the Telemetry tab, verify that the last run was within the last five days. If not within the last five days, the customer will need to carry out a new diagnostic run.
Yes, SDT used and resolved the issue
  • SDT was run with the customer on the call/chat and it resolved the issue. Advocates recorded the Reference ID in the appropriate area(s) of the case management tool.
  • After recommendation, customers ran SDT by themselves after call/chat and it resolved issue. You can find the history within the last five days on the Telemetry tab.
  • Reference ID is required.
  • SDT solved the issue.

Yes, SDT used but did not resolve issue
  • SDT was run with the customer on the call/chat, but it didn’t resolve the issue. Record the Reference ID.
  • After recommendation, customers ran SDT by themselves after call/chat, but it didn’t resolve the issue. You can find the record within the last five days on the Telemetry tab.
  • Reference ID is required.
  • If the issue was not solved by SDT, even if the issue was solved finally by other operations, you still need to use this bucket.

Yes, SDT recommended, but unsure if customer ran
  • SDT was recommended, but the customer was discontinued/unresponsive.
  • The customer said they ran SDT after you recommended it and the issue was resolved but the customer didn't provide the Reference ID.
  • The customer said they ran SDT after it was recommended, and the issue was not resolved, and the customer didn't provide the Reference ID.
  • The customer was out of warranty (OOW) and declined the paid diagnostics offers. Send SDT link for the customer to self-help.

No, issue was resolved, SDT does not apply for this issue
  • Issue did not require SDT because the issue was resolved by troubleshooting steps before running SDT.

Note: If SDT wasn’t applicable to this issue then one of the other options should be selected.

No - Could not install SDT from Store or Download Center

Use in any of the following cases:

  • Customer does not have internet connectivity and was unable to download the tool.
  • Customer is not an administrator on the Surface and is not able to install the tool.
  • An error occurred during the installation of the tool.

No - Customer has already used SDT
  • The customer ran SDT within the last five days before the SR was created. Note: The customer used SDT before contacting Microsoft.
  • Reference ID is required.

No - Device won't turn on or boot to Windows

Use in any of the following cases:

  • Device will not turn on even after troubleshooting.
  • Device will turn on, but unable to boot the device into Windows.

Note: If you can tell that the customer’s issue is indeed a “won’t turn on or boot to Windows” symptom, even if the customer’s call is discontinued/unresponsive or the case can't be continued due to different reasons, choose this taxonomy. If during the troubleshooting, device can turn on and booted into Windows, this taxonomy is not applicable.

No - Physical damage to the device
  • The issue the customer is calling about is related to physical damage of the device such as cracked screen, damage to the buttons or ports, or damage to the case.

Note: If you can tell that the customer’s issue belongs to “Physical damage” scenarios, even if the customer is discontinued/unresponsive or the case can't be continued due to different reasons, choose this taxonomy.

No - SDT installed but would not launch/run

Use in any of the following cases:

  • The customer was able to install the tool, but it would not launch or run due to the application error.
  • The customer was able to install the tool, but it could not run because they do not have administrator rights.
  • The customer is running Windows RT or a version of Windows earlier than Windows 10.
  • The customer is running Windows 10, but they are using a Surface Pro (v1) or Surface Pro 2.

Note: If the SDT issue is resolved, choose the appropriate SDT taxonomy.

Not applicable - Misroute, disconnected, warranty, RRT, or other approved reasons
  • The customers issues are not related to troubleshooting the device, but instead are misroutes, disconnected calls/chats, warranty, or RRT.
  • Choose this taxonomy if the case doesn't belong to any other taxonomy.

Note: Create a Speak Up! item if more taxonomy options are needed.

SDT is not available in my customer's language

The tool is not available in the customer’s language.

No - Customer refused to run SDT
  • Thecustomer said they ran SDT and refused to run it again. Customer could not or was not willing to provide the reference ID or records.
  • SDT was recommended. The customer didn't refuse to run SDT directly; however, they wanted to try other solutions instead of SDT and never ran SDT.
  • The customer refused to run SDT (if SDT is applicable to that case).
