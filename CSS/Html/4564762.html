What is this article about?

This article provides all the details on Assistance Request V2 feature designed for M365 Broad Commercial support cases. AR is not to be used as a collaboration method for Broad Commercial cloud cases between Rave and Service.

Assistance Request 2.0

Examples of requesting assistance versus escalating a case:

  1. A case comes in saying that it is a problem with Exchange. After a few rounds working with the customer, you find that there is an Exchange setting within Outlook that may be the cause. In this instance, you would request assistance from more experienced users (TL (Technical Lead) – L1) within your partner organization. If the TL is still not able to provide a definitive solution to the issue, he can still request for help to a further level of support (SEE -L2, usually a Microsoft escalation engineer). SEE will provide a definitive solution to the Assistance Request. The ambassador will always keep the ownership of the original RAVE case.
  2. A case comes in saying that it’s a problem with Exchange. After several rounds working with the customer, you conclude that there may be a bug in Exchange causing it to work incorrectly. In this instance, you would escalate the case to the product team.

Assistance Request requirements – Users setup

Assistance Request 2.0 feature is only available to ambassadors who are enabled for Tier 1 and/or Tier2 skills and are part of the CSS Concierge RAVE organization.

Please refer to your team manager to complete the following procedure to enable Assistance Request 2.0 experience.

  1. As a Team Manager, select Admin from the main menu and then Queue View.
  2. Locate the user that needs to be enabled for the feature and select the ellipsis in the Name column.
  3. Click Edit.
  4. In the Edit ambassador dialog box please verify that the user belongs to the CSS-Concierge Organization.

    [Asset 4564765]
    The Edit ambassador  dialog box
     
  5. Finally, in the Skill dropdown locate the Tier 1 and Tier 2 Escalation tiers and enable them accordingly, Save.
    • Ambassador – Assign only Tier 1 Skill
    • TL (Technical Lead) L1 – Assign only Tier 1 Skill
    • SEE (Support Escalation Engineer) L2 – Assign Tier 1 and Tier 2 Skills

Request assistance on a case

  1. On the Case detail view, select Request assistance from the Case action dropdown list.

    [Asset 4564767]
    The Request assistance option in Case Action dropdown
     
  2. On the Request Assistance dialog box, select the team from where you need assistance.
    [Asset 4564769]
    The Request assistance dialog box
     
  3. In the Description field, type the exact question(s) you need help with.
  4. Select Request.
  5. A new item will appear in the Assistance area, waiting to be acknowledged by the L1 support.
    [Asset 4564770]
    The Assistance area


To request help from a specific person, update the case from the Assistance requests view.

  1. For your case, select the ellipsis in the Action column and select Update.
    [Asset 4564771]
    The Update assistance action box in the Assistance requests view

    [Asset 4564772]
    The Update assistance dialog box
     
  2. On the Update assistance dialog box, begin typing the person’s name into the Assign to field. Once the first few letters are entered, the person’s name should come up. (The person’s correct team must be selected in the Support area dropdown.)
  3. Select Update.


The same operation can be performed from the Assistance area too, clicking on Update.

Give TL (Technical Lead) – L1 assistance on a case

  1. As a TL - L1 (Tier 1 Skill), go to the Assistance requests view, select Assistance requests from the main menu.
    [Asset 4564773]
    The Assistance request view for TL
    • This screen shows you the case Creation date, Case number, Support area, Customer name, Internal title, Description and Action menu.
    • All columns are sortable, and the list can be filtered.
  1. To accept an assistance request without reading the full details, hover over the case row and select the ellipsis in the Action column and select Accept.
    [Asset 4564771]
    The Accept assistance action box
     
  2. Once you start helping on a case, the case will appear in your My cases view with the “ASSISTANCE” badge next to it.
  3. To add your questions and notes to the case owner, type in the Add internal note tab on the Case detail view.
  4. To email the customer directly – to directly ask an additional question as part of your troubleshooting, for example – type in the Email customer tab on the Case detail view.

TL requesting for further assistance to SEE (Escalate to L2)

As a TL -L1 (Tier 1 Skill), you have the possibility to request assistance to a further level of support (SEE – L2).

  1. On the Case detail view, find the Assistance request in the Assistance area.
  2. Click on Update.  You can perform the same operation from the Assistance requests view.
    [Asset 4564774]
    The Update assistance Button and area
     
  3. The Update assistance dialog box will show. Select Escalate this case to Level 2.
    [Asset 4564775]
    The Update assistance dialog box
     
  4. Select the Support Area and provide a description about the AR L2 escalation, then click on Request.
  5. In the Assistance area you will see a new line created in the Assistance request: “Level 2 Searching”.
    [Asset 4564776]
    The Assistance area with an AR escalated to L2

Close a request for assistance

Either party can close a request for assistance at any time. This closes the assistance request, but the case itself remains open.

  1. On the Case detail view, find the Assistance widget in the righthand sidebar.
  2. Select Close.

The same operation can be performed from the Assistance area too, clicking on Close.

The case will still appear in the My active cases view of the ambassador who owns the case, but it will disappear from the assistance-provider’s My active cases view.

Once a request for assistance is closed, it cannot be re-opened. To request more assistance with the same case, open a new assistance request.

Frequently Asked Questions (FAQs)

  • Ambassador - Add Tier 1 skills​
  • TL (Technical Lead) – Add Tier 1 skills ​
  • SEE (Support Escalation Engineer) – Add Tier 1 & Tier 2 skills​

Tier 1 & Tier 2 skills are only available for the CSS Concierge org​.
Any user who currently utilizes Assistance Requests v1 should not be skilled with either Tier 1 or Tier 2

Q: Can I have multiple AR 2.0 open at the same time for a RAVE Case?

A: YES. There is no limit to the number of the AR you can have for a specific RAVE Case.

Q: Can I reassign an AR 2.0 to a different Tech or Person?

A: YES. You can do it using the UPDATE button. You can reassign to a different Tech or AR owner. The RAVE Case ownership won’t change. The reassignment will happen at the same escalation level. TL->TL, SEE->SEE.

Q: Can I reassign an AR 2.0 to the queue and not to a specific person?

A: No. Reassignment requires the selection of a new AR owner.

Q: If I accept an AR will I be the owner of the RAVE Case and will I receive the survey from the CX?

A: NO. The RAVE Case ownership will remain with the Ambassador and the same will happen with the survey.

Q: How are the AR 2.0 in ordered the RAVE case panel?

They are ordered by creation date.

Q: Will I receive any notification for AR 2.0?

A: Although this has been requested by multiple teams, this has still not planned for now. It’s something under evaluation from the RAVE team. No notifications will be triggered by AR for now.

Q: As a SEE can I send back the AR 2.0 to the TL level (L1)?

A: No. Is not possible to push back AR.

Q: Is it possible to reopen a closed AR 2.0?

A: No. You need to open a new AR.

Q: Can I filter specific AR Notes inside a RAVE Case?

A: Not initially. It is something that will be included with the next release of AR.

Q: Can I use AR 2.0 for GetHelp requests?

A: No. Not all the people in the GetHelp team can access to rave and also because a RAVE case needs to be open to create an AR, this is not always possible. We are suggesting to the GH team to revert back to the mail notification system.

Q: How many levels of escalations there are for an AR 2.0?

A: AR has two escalation tiers. Level 1 = TL and Level 2=SEE.

Q: As a SEE, can I do proactive review on AR 2.0?

A: Yes. Knowing the Case number of a RAVE case, you can still open it using a simple search and leave notes as usual. Your contribution to the case won’t be tracked.

Q: Do we have an AR reporting like we had in EC?

A: We will have soon a basic reporting. We are planning to expand the reporting in the upcoming weeks and months.

Q: Can I bypass TL (L1) like we had in EC?

A: Although we know that this is important for certain teams (like Commerce), this is not currently possible.

Q: Can a SEE check and access to any AR 2.0?

A: Yes, from his dashboard he can see AR across all suppliers and technologies.

Q: Can a TL check and access to any AR 2.0?

A: No, from his dashboard he can see AR created only inside his organization (supplier).

Q: Can I add PII data to an AR 2.0? In EC was prohibited.

A: Yes, you can add PII data to an AR. (only when is really needed). Data related to AR is now stored directly in RAVE.

Q: As an AR user, which Support area should I use?

A: You should use only the support areas available within your organization.

Q: Will AR 2.0 impact existing users? ​

A:

  • No, existing users will continue to see AR 1.0 experience​.
  • AR 2.0 feature is being enabled through Tier 1 skill; only ambassadors who have Tier 1 skill can see the new experience​.
  • If premier ambassadors do not move to Tier 1 or Tier 2 skill, they will continue having the old experience. ​
  • Impact: Volume of ARs will increase in the queue and the existing filters would help mitigate this.
  • Edge case - If a Premier ambassador is also provisioned in consumer BU with Tier 1/Tier2 skills, they would see the new experience.
Q: If my case came with "Enterprise" as Support Area, should I open the AR also using "Enterprise"?

A: No. The "Enterprise" Support Area is generally used for those cases where machine learning cannot identify the exact technology they should be re-routed to. If you are handling an "Enterprise" tagged case that is, for example, a SharePoint case, then we recommend you open the AR using the category that applies to the technology being supported (in this case, "EnterpriseSharePoint"). That will bring more accountability into the TLs/SEEs acknowledging the cases, and it won't harm the normal handling of the case.

This should only be done when cases came as "Enterprise", and we are confident on what's the technology affected in the case. If, on the other hand, the case is a Concierge tagged case, then for those items the category to be used while opening the AR must always be "Concierge”.

Q: Can I send the AR back to the queue when the case has been wrongly acknowledged?

A: No. In these types of scenarios, you need to work with the original AR creator (or its Team Manager), asking that person to close the AR acknowledged by mistake and open a new one (with the same characteristics) that can be accepted by the respective team.

Q: Can a case be closed if there are open ARs?

A: No. All ARs need to be closed before the case can be closed. The closing action should always be performed by the respective owner of the AR (either TLs at L1 or SEEs at L2), and only after that has been completed, the case owner can move forward with the case closure.

If TL/SEE is taking too long to close the AR, engineers can ask TM/POD Lead support on the situation.

A: Since Concierge is a generalist Support Area, there is no field in the AR form for the engineer to specify the technology needed. A: Due to some specific scenarios related to skilling in RAVE and DAO, there may be engineers who get offered with cases that are not from their original Region. Although this is not a common situation, whenever an engineer needs to raise an AR for a case they own and that is outside of their Region, the item should also be acknowledged and worked by the TLs and SEEs from the engineer’s Region.
