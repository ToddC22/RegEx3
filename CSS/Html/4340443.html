The call features available in Rave for Ambassadors or support engineers to use may be different depending on which one was originally set up. There are two types of call systems: one is the headless SIP system (Genesys Softphone), the other one is the Enterprise Voice system (Skype). Always ‘SIGN OUT’ of Rave instead of closing the browser at the end of your shift to ensure OneCloud softphone is fully disconnected. Follow the appropriate procedures in this article based on what call system you have been set up with.

  1.  Search for Genesys on the desktop and open the Genesys softphone. Take note of this 7 digits ID for the place:

  1. From there, go into Rave, right-click on it, and the select My Settings:

  1. In your setting, enter the OneCloud information:
    1. Enter alias (already populated and read only).
    2. Enter password (1).
      Note If this is the first time accessing the OneCloud platform, you will be asked to set the permanent password and confirm it.
    3. Enter place (2), the 7 digit ID number
    4. Select the Use OneCloud SIP Client check box (3).
    5. Select Sign On (4).

Conference Call / warm transfer

Once you are connected with the customer, additional options will open up on OneCloud, including "Conference Call".

  1. While you are connected, you can select the "Conference Call" icon.
  2. A free text box will be enabled, allowing to enter the conference number.
  3. Once you enter the number, select Dial which will place the customer on hold and dial the number.
  4. Then you can use Merge Call to start the 3-way call:

[Asset 4340453]

  1. OneCloud module will then display the current call as well as the "Conferenced Numbers”.

[Asset 4340454]

  1. Then if needed, you can disconnect the conferenced call and remain on the phone with the customer.

[Asset 4340456]

Stop call recordings

RAVE 2.0 now supports the ability to stop call recordings for customers who request to stop the recording during a call. Read the following information to learn how and when to use the stop call recording option.

  1. Continue to inform the customer of the call recording disclosure for all calls.
  2. ONLY IF the customer requests to stop the call recording anytime during the call, select Stop Recording feature.

[Asset 4340457]

  1. If this feature is accidently pushed OR if the customer changes their mind to turn call recording on, you can select the same button to resume the recording.

FAQs-Call recording

When to stop using call recording?

Ambassadors can stop using call recording feature ONLY if a customer verbally requests to stop the recording of the call. The call recording feature will be monitored, and weekly audits will take place to determine if this feature was used properly.

Which customers does this apply to?

This feature can apply to ALL customers (US or Non-US). (Note the feature was designed originally to meet the EU privacy requirement.)

Do we have to continue to say the call disclosure to the customer?

Yes, until the new automated call disclosure is in place for both inbound and outbound calls. A communication will be sent to inform the ambassadors of the change.  When engineers provide the disclosure to the customer, it must state:  "This call may be recorded for quality and training purposes."

What additional changes to customer call recording?

Due to the EU privacy regulations mostly driven by Germany, Non-US customers will have the option to have their support calls with call recordings ON or OFF.

For US customers, they will not receive this option and the default will have the call recordings turned ON.

Should we record calls under NDA?

Recordings are not a compliance requirement but the disclosure is when it does happen. For NDA calls, you don’t need to record them, but if you do, it would be better that you state the disclosure so the customer is aware that the call may be recorded.

 

Do engineers need to state the recording disclosure a second, third, fourth, etc time they call the customer?

This is up to your team based on the frequency of the calls. As long as it's recorded that the disclosure is said once to the customer, this should work. But if you are engaging with a new person in the tenant, you should say the disclosure to the new person.
