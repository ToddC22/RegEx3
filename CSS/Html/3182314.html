1. User roles and permissions ﻿

Everyone who uses Microsoft Advertising is assigned a user role, which gives permissions to perform specific account functions. The person who signs up for the account is given the role of Super Admin, which gives them full permissions, including inviting and deleting other users. If you have more than one person accessing a Microsoft Advertising account, you'll want to know how to invite users and what role to give them.

Microsoft Advertising has four user roles:  

  • Super Admin - This role has full permissions for all accounts. A Super Admin can manage everything related to billing and payments, account details, and other users (including other Super Admins). The Super Admin can specify which accounts other users have access to. When signing up as a new customer, the first user is the Super Admin. 
  • Standard User - This role has permissions to manage campaigns and perform some billing activities on specific accounts. This role cannot add, edit, or delete payment methods or other users, create or delete accounts, or link or unlink from clients.
  • Advertiser Campaign Manager - This role has permissions to view selected accounts and add, edit, or delete campaigns within the selected accounts. The Advertiser Campaign Manager can view payment methods, but cannot manage any billing and payment tasks. 
  • Viewer - This role has read-only permissions. API user. This role can access only those accounts assigned to their user through API web services for programming needs. This is not published in the external user-facing HELP content.

4. Cancel pending user invitation 

If necessary, Super Admins can cancel a user invitation that has not yet been completed by the recipient, and is still showing as pending in the UI.

  1. Select the gear icon, and then select Accounts & Billing.
  2. Select the Users tab.
  3. Select Delete for the pending user invite.
  4. Select OK to confirm.

5. Remove a user

Don’t forget to delete old users from your account. This prevents unauthorized people from accessing or changing your campaigns. Superadmins can delete users from their accounts using the following steps:

  1. Select the gear icon, and then select Accounts & Billing.
  2. Select the Users tab.
  3. Next to the name of the person you want to remove, select Delete.


Note: Before removing a primary user, you must designate a new primary user for the applicable accounts. Every active account requires at least one primary user to ensure delivery for critical alerts. 

Refer advertisers to How do I give someone access to my Microsoft Advertising account? as needed which contains all the above and more.

Removing the primary user from an account

The last user on an account cannot be removed. See Deleting, Inactivating, and Reactivating accounts for guidance on permanently deleting CIDs, which is the only method to remove the last user on a customer shell.

Fraud customers should be directed to the Privacy Response Center to submit a Data Subject Rights request per Procedure: Privacy: Handling data subject rights requests.

Removing a user who no longer works for the company

  1. Send the caller a verification email to the email address they wish to remove from the account.
  2. Once they are able to provide verification, remove the user from the account.

6. User contact information

By keeping contact information current, advertisers make it easier for Microsoft Advertising team members to contact them when it is important to do so. It is the advertiser’s responsibility to maintain this information so that they can continue to receive important notifications in a timely manner.

Microsoft Advertising non-marketing communications (aka Service communications) Modify Microsoft Advertising User Preferences under Settings. Note: Critical alerts cannot be suspended.
Advertising (Ads and non-Ads related) communications Send customer email to adcunsub@microsoft.com (anyone can/should submit here as needed) with the following info:

[Contact Name] has requested to be removed from the direct mail mailing list. Here is the required information:
  • First and last name
  • Customer Name
  • Customer ID
  • Company or organization
  • Mailing address
  • Country
  • Promotional codes (if applicable)
  • Date customer called or contacted you
The advertiser will be removed, but neither you nor the advertiser will receive any type of confirmation. Set expectations with the advertiser that they have been removed from the list, but will not receive a confirmation. Close the ticket.
