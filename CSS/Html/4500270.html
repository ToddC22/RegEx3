1. Windows Store App Automated Troubleshooters

1.1 Windows Store Apps Troubleshooter

If an app fails, you might be prompted to run the troubleshooter.

The troubleshooter can also be started from Start > Settings > Update & Security > Troubleshoot > Windows Store Apps.

2. Reinstall and re-register apps

If running the Windows Store Apps troubleshooter doesn’t resolve the issue, you can try reinstalling and re-registering apps. Note: For apps that were previously uninstalled from the system, these steps will not reinstall the app. 

If you are not able to reinstall the app, then you will need to install the app from the Windows Store instead.

The best results will come from reinstalling and re-registering all apps. If you aren’t willing to do this for all apps, see 2.3 Specific apps.

If the issue is occurring with one user account only, see 2.1 All apps for current account. If it’s happening with all user accounts, see 2.2 All apps for all accounts.

Return to top

2.1 All apps for current account

  1. From the Start button, search for powershell. Select Windows Powershell from the results.
  2. Copy and paste the command below in PowerShell, and then press Enter. See screenshots below.

Code:

Get-AppXPackage | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

  1. When finished, close PowerShell, and try opening the affected app again.

2.2 All apps for all accounts

  1. From the Start button, search for powershell. Right-click Windows Powershell in the results and choose Run as administrator.
  2. Copy and paste the command below in PowerShell, and press Enter.

Code:

Get-AppXPackage -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

  1. When finished, close PowerShell, and try opening the affected app again.

Return to top

2.3 Specific apps

If the customer does not want to re-register all apps, you can try to uninstall and re-install the specific app that is not working.

  1. From the Start button, search for that app.
  2. Right-click the app from the search results and choose Uninstall.
  3. Open the Store app, search for the app, and re-install it.

Some built-in apps might not have the Uninstall option. Those apps can be re-registered using the following steps:

  1. From the Start button, search for cmd. Select Command Prompt from the results.
  2. Copy and paste the command in section 7. Codes for the Windows app you want to reset into the Command Prompt, and press Enter. When finished, you can close the command prompt.

 3. Error: 0x803F8001

There are two main error messages that might include this code.

 3.1 Something unexpected happened

Error Message: Something unexpected happened – Reporting this problem will help us understand it better. You can wait a bit and try again or restart your device. That may help. Code: 0x803F8001

Try again later. This issue can be caused by a Store outage. If this exact error message is encountered with multiple customers, alert CSS Command Center (CSS CC) of a possible outage. Follow KB 3038858 Procedure: How and when to engage CSS Command Center (CSS CC) all regions.

Return to top

3.2 Check your account

Error Message: Check your account - <App Name> is currently not available in your account. Here’s the error code in case you need it. 0x803F8001

  • This issue highlights a problem with the customer’s account and indicates there could be an issue with the customer’s billing profile.
    • The customer’s account may suffer from one of the following scenarios, detailed here:
    • No Billing Profile
    • LiveID mismatched against primary admin (Delegate Admin issue)
    • Profile/region mismatch
    • Incorrect billing account
    • Web only account
    • V1 Gamertag
  • Follow KB 2981105 Procedure: Advanced Capability Advocate engagement to have Advanced Capability Advocates (ACA) investigate the customer’s billing profile.

4. App specific troubleshooting

4.1 Windows Store not able to connect to the internet with error codes

  • Run WSReset: http://aka.ms/diag_Apps10
  • New user
  • Check for TLS activation on Internet settings
  • Remove all proxies and VPN connections if present
  • Reset Internet Settings

Return to top

4.2  Windows Store not launching

  • Run WSReset: http://aka.ms/diag_Apps10
  • New user
  • Remove and reinstall Store using PowerShell on admin mode:
  • Get-AppxPackage *windowsstore* | Remove-AppxPackage
  • Get-AppxPackage -AllUsers| Foreach {Add-AppxPackage -DisableDevelopmentMode -Register “$($_.InstallLocation)\AppXManifest.xml”}
  • In-place upgrade
  • Clean Install

Return to top

4.3 Apps not downloading or can’t download App updates

  • Run WSReset: http://aka.ms/diag_Apps10
  • Check proxy/VPN and remove any susceptible network conflicts
  • Change Network
  • New User
  • Switching Microsoft account on Windows Store
  • Clear Store Cache:
  • Win + r -> %userprofile%\AppData\Local\Packages\winstore_cw5n1h2txyewy\LocalState
  • Rename the “Cache” folder to “Cache_old”
  • Create a new “Cache” folder
  • Re-register Windows Apps using PowerShell:

Get-AppxPackage -AllUsers| Foreach {Add-AppxPackage -DisableDevelopmentMode -Register “$($_.InstallLocation)\AppXManifest.xml”}

  • Reset Windows update Components
  • DISM from source line command:
  • DISM /Online /Cleanup-Image /RestoreHealth /source:WIM:X:\Sources\Install.wim:1 /LimitAccess
  • (X: the drive letter with the Windows image installed)
  • In-place Upgrade
  • Clean Install

Return to top

4.4  Xbox app not working or games launching but not connecting to the profile

  • Create new user
  • From an elevated (admin) PowerShell prompt:

Get-AppxPackage *windowsstore* | Remove-AppxPackage

Get-AppxPackage *xboxapp* | Remove-AppxPackage

Get-AppxPackage -AllUsers| Foreach {Add-AppxPackage -DisableDevelopmentMode -Register “$($_.InstallLocation)\AppXManifest.xml”}

4.5 Unable to access the Microsoft app like Edge/Cortana/Windows Store

  • The common two causes of this issue can be:
  • The Microsoft account has not been verified yet on the computer.
  • Date and time on the computer is set incorrectly.
  • The store app uses https instead of http which is used by other apps, so in order for the SSL handshake to proceed, the clock on the client and server needs to be correct.
  • Follow these steps and check if it helps:
  • Check if the date and time are set correctly:

1) Open the Start screen, type date and time in the search bar and select date and time settings.

2) Check if the date and time is set correctly.

If the issue persists, proceed to next step.

  • Finish the two-step verification process:
  1. Close the Store app completely.
  2. Open Start menu, type settings in the search bar and open Settings.
  3. Select Accounts. It will show the Microsoft account there.
  4. Select the verify link if available.
  5. Once done try starting the store.

If the issue persists, proceed to next step.

  • Check if proxy connection is enabled. If yes, then disable the proxy connection and check the status:
  1. Open Internet Explorer.
  2. Select Tools and then select Internet Options.
  3. Select the Connections tab.
  4. Select the LAN Settings button.
  5. Then uncheck the option Use a proxy server for your LAN.
  6. Select Apply.
  • If the issue still persists, use the Microsoft account Troubleshooter from here:
  • http://download.microsoft.com/download/F/2/4/F24D0C03-4181-4E5B-A23B-5C3A6B5974E3/microsoftaccounts.diagcab

4.8 Edge and Mail not working

  • In Windows 10 Build 1809, Windows Store apps like Edge and Mail won't work if IPv6 in the network adapter is disabled. Desktop apps like Outlook do not experience this issue. If the customer is not able to reach on any websites:
  • Try enabling IPv6 for the adapter to resolve this issue.
  • Right click on the Wi-Fi or ethernet icon on the right side of the taskbar  > open network and internet settings > change network settings > right click on the active connection > properties > there you can activate IPv6
  • If Edge doesn’t work, and when selecting “show more details” you see something like this (INET_E_RESOURCE_NOT_FOUND):
  • Open the following link in a different browser (IE, Chrome, Firefox), https://developer.microsoft.com/microsoft-edge/platform/issues/13779603/   
  • Download and save the Regfix.zip file > Extract the Zip file.
  • After extracting the Zip File, right click over FixTcpipACL and select Run with PowerShell.
  • After the PowerShell window automatically opens up, type A when given the option to do so to allow the execution policy and press Enter.
  • Wait until the command is finished. Open Microsoft Edge.
  • You can also try using the built-in network reset option in the Settings app to reset all the network adapters to their default settings.
  • If the issue only occurs with one email address, it might be isolated to the email account or server settings.
  • Refer to Resolve sync issues in Mail and Calendar apps in Windows 10 – specifically sections:
  • Check the connectivity to your email server
  • Collect the required information from your email provider
  • Check if two-factor authentication is turned on for your account
  • If the Mail app does not launch, try the guidance in the following:
  • Set up email in the Mail app for Windows 10
  • Troubleshooting Mail and Calendar apps for Windows 10
  • Resolve sync issues in Mail and Calendar apps in Windows 10

4.9 Mail sync issues

  • Check app access
  • Go to Settings
  • Open the Privacy tab
  • Open the Contacts sub-menu
  • In the list of apps, check that access for "App Connector" and "Mail and Calendar" are turned on
  • Open the Calendar sub-menu
  • Check the "Let apps access my Calendar" option is turned on
  • In the list of apps, check that access for "App Connector" and "Mail and Calendar" are turned on
  • Set the Credential Manager to Automatic
  • Open Task Manager
  • Go to Services tab
  • At the bottom select Open Services
  • Go to Credential Manager and set to Automatic
  • Restart the service if necessary
  • Delete and create new account
  • Delete the current/upgraded account
  • Create a new Exchange one
  • Have the customer enter their full e-mail and then press next
  • Have the customer enter their password
  • IMPORTANT: If 4 input fields appear, try to login with Username (without the domain name) and the domain name specified in the Domain field.
  • If they use bitdefender internet security program, try turning the bitdefender firewall off.
  • If they use AVG, try turning off the firewall for AVG.
  • Change mailbox sync settings
  • Open the mail app
  • Select the Settings button at the bottom of the left pane, and select Manage Accounts in the Settings flyout
  • Select the account you want to change mailbox sync settings for
  • Select Change mailbox sync settings.
  • Select how often to download new content, and select how far back to download email from on the server, turn on or off Email, Calendar, and/or Contacts under Sync options for what you want, and select Done
  • Select Save to apply
  • Unchecking SSL for incoming and outgoing server
  • Go to account settings > advanced settings and uncheck the require SSL for both incoming and outgoing email.
