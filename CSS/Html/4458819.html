Procedure: Xbox: Web-form Refunds

[Asset 4049349]

Who should use this article?

  • Xbox advocates receiving refund requests via the web-form

What is this article about?

Web-form usage when processing customer refunds to ensure a consistent customer experience around refunds.

Important note: Refunds can be processed for all locales.

2. Locate and review the request in MST/ASD

  1. Sign in to ASD.
  2. Select service request (SR). Note: Subject line will be the title of the order/BULK – with one title being requested.
  3. Load customer’s SR and access it with Microsoft Secured Access and take ownership.
  4. Review Partner Notes in MST Comments section prior to issuing any refunds .
  5. Review the details in the first Metis GetSup to understand the details behind the request date and time.

[Asset 4550806]

[Asset 4550807]

  1. Note in MARS: If the same customer has multiple refund requests across different service requests (SR): This will happen if customers are submitting for a subscription, modern game and legacy game, or forgot to select an order.
  • All refund requests for the same customer (within the transfer to the queue) should be assigned to a single advocate to ensure account limits are accurate. Check the Incident section in MST to confirm multiple submissions and include other possible SRs.
  • Work each SR one by one, oldest to newest. Search for the orders one by one to check if they are refundable or not – searching for multiple orders at once tends to yield inaccurate results.
  1. Copy the link and follow the guidance for processing a refund using MARS.

4.1 Sending decision emails

After processing the refund, do not close the case.

  1. Build and send the e-mail using the Email function in ASD. Do not erase any recipients. In the second metis comment, scroll down to find Email: "name@provider.com".

Note: Check if the customer provided a preferred email other than their Microsoft account. If the email shown is not a Microsoft account email, ask the customer if they have a preferred email for communication. If yes – add the email to the preferred email field in ASD. For MARS – Use the preferred email they submitted the request with.

  1. Wait until the e-mail shows up on My SRs section. Refresh the section if the case doesn't show up.
  2. Go back to the case and close it using the correct case disposition (resolved or unresolved).

Note: If an email template does not exist for a scenario, submit a suggestion via Speak Up!

4.5 Case disposition

Change the Subject to Refund Request or Refund Request Sub (for SRs that contain a subscription), or Refund Request Bulk.  

For SRs that contain a subscription, use the subject Refund Request Sub.

For SRs with multiple orders, Refund Request Bulk.

For all other refund requests, use Refund Request.

5. Case reopens

[Asset 4550817]

  • If a customer responds to an email (auto-generated, more info, or decision), the case is reopened and returns to the queue. Process accordingly by the scenario.
  • The customer responds to a dispute: Follow the process for initial review pulling in any additional information provided and make determination. Consult another advocate or team lead for assistance as needed. Update the solution delivered if needed. Respond to the customer’s email from ASD.
    • After an additional review, decision stands.
    • After further review, refund granted.
  • Customer responds with the additional information requested: Process the request with the additional info (see section 1) and respond to the email thread in ASD with the decision using template text.
  • Customer responds back to thank you: Close the case.
  • If refund fails to process a new case will be opened to retry in MARS.
