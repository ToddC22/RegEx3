ASMS – EA and MCA Credit Request – ADO Ticket Process

Submit a ModTrack request to update this TSG.

Overview

This TSG provides guidance on how to submit Enterprise Agreement (EA) and Modern Commerce Accounts (MCA) Field Led (FL) credit requests through Azure DevOps (ADO).

This process is in place to provide credit compliance. ADO provides user credentials, timestamps, and a log for entered/approved actions that are required for credit compliance audits.

Note: If you are looking for guidance on how to refund a Reserved Instance (RI) customer for Returns/Exchanges, refer to TSG 4054281.

Credit ADO Process

  1. To submit a Credit Request, navigate to the following ADO link: Azure Ops (EA Credit Request Form)
  2. Fill out each of the fields on the form.
    • For more information on each of the fields, see the Field Definitions section.

    • Best Practice: To help identify top ranking customer cases, it is best to title the case with the following information:

      • [Rank (if any)]

      • [Impacted amount (if higher priority is required)]

      • Concise issue statement with billing period

        • Example: PO# change, cancel invoice, credit adjustment query, reason of credit/re-bill etc

      • Enrollment number

      • Customer name

      • SR number 

Example: [Top100] [USD 250K] Credit invoice for July2020 | EA#XXXXXXX | ABC Corp | SR#12007XXXXXXXXX 

  1. Assign the case based on the guidance in the Credit Issue Status Column section below. 
  2. Click the Save button in the upper right to submit the ADO ticket.

Note: ADO, which we use for Credit requests, has attachment file size limits. If the file size limit is exceeded, please visit Azure Ops Billing in order to upload the attachment file for review and paste the file's URL into the Summary of Issue section.

Field Definitions

Review the list of fields below that are required based on the Credit Scenario Type chosen. 

Ticket Status (Section A)

  1. Area – Set to Azure Ops\EA Billing.
  2. Iteration – Set to Azure Ops.

Adjustments Column (Section B)

  1. Adjustment Amount (Local Currency) Requested – Do not populate this field. 
  2. Adjustment Amount (USD) Requested – Do not populate this field.
  3. Currency – Use the three-letter currency abbreviation for your customer’s local currency in this field.
  4. Credit Scenario Type – Select the appropriate reason code from the list.
    • Note: The ticket will automatically highlight the fields that are required with a red exclamation point based on the Credit Scenario Type.
    • More details about credit reason codes can be found in TSG 3045635. 
  5. Adjustment Methodology – Explain how you arrived at your proposed adjustment amount for your customer.
    • Include formulas or justification as appropriate.
      • When creating a ADO for refund requests, the USD credit amount is calculated by using the following formula: Credit in local currency divided by the FX Rate.
      • If you need to calculate credit in local currency based on a given USD amount, please use the following formula: Credit in USD multiplied by the FX Rate.
    • If you need to attach files to the ADO ticket, click the paperclip icon in the upper right corner of the window to do so. Mention which attached files pertain to this section in this field.

Credit Issue Status Column (Section C)

  1. Credit Issue Status – The status you set this to depends on your empowerment level.
    Use the amount in Adjustment Amount (USD) Requested for the following conditions:
    • For Support Engineers: If the amount is $1,000 USD or less in policy, $250 outside of policy, select ASMS – Pre-Approved. Fill out the Approval column.
      • Once ASMS pre-approval is given, be sure to mark the case as "unassigned" and set the status to approved. This status means that it is in MBO's queue and awaiting credit application. 
      • If the amount is above the SE’s empowerment level, select ASMS - Pending Review. Do not fill out the Approval column. Contact your TA/M1 to review the request.
    • For Technical Advisors (TA): If the amount is $2,500 USD or less and in policy, or up to $250 outside of policy, and you have reviewed and approved, select ASMS Pre-Approved. Fill out the Approval column.
    • For Managers (M1): If the amount is $5,000 USD or less and in policy, or up to $500 outside of policy, and you have reviewed and approved, select ASMS Pre-Approved. Fill out the Approval column.
      • If the amount is above the M1’s empowerment level, select AO Billing - Pending Review. Do not fill out the Approval column. If the EA team decides to approve the request, they will fill out the Approval column. In the Discussion section, add a note that you have reviewed the request, so the EA team knows it has been reviewed.
      • Any tickets outside of ASMS empowerment that are in an Approved status and are unassigned are in MBO’s queue for processing and should not be touched/changed.
  2. Source SR – Enter the customer’s Service Request (SR) number from [Asset 4497459]for this field.
  3. The issue is currently resolved – True/False toggle. Toggle to True if your customer’s issue is resolved. Toggle to False if their issue is not resolved.
  4. Date support issue was reported – Enter the date that your customer’s issue was reported.
  5. Products Impacted – Select the products impacted from the provided list. In the instance that more than one product is impacted, list all in the summary of the issue section.
  6. Enrollment # – Enter the customer’s EA Enrollment number.
    • For MCA Field Led (FL): Enter Commerce Root (FL GTM) and Billing Group ID (FL GTM).
  7. For MCA: GTM Application Method – Select the required adjustment method. “Billing Group Adjustment” and “Bulk Billing Group Adjustment” are only available for Field Led & Customer Led (it is not available for Partner Led). 
  8. Summary of Issue – Summarize the customer’s issue, the cause (what led to the issue), as well as the effect especially as it pertains to the customer’s bill (if needed engage additional resources to help craft this section). Use clear, direct language and avoid complex technical language when possible, to prevent any miscommunication with others who may read the description.
    • If the credit is related to backdating of pricing, be sure to include the following details in the Summary of Issue field:
      • Start/End Date
      • Old Pricing/Special Pricing
      • SKU
      • Customer Name
      • EA Number

Approval Column (Section D)

Important: Only fill in these fields if you set Credit Issues Status field to ASAM - Pre-Approved.

  1. Approval Amount (Local Currency) – Enter the amount for the approved credit request in your customer’s local currency.
  2. Approval Amount (USD) – Enter the amount of the approved credit request in USD.
  • Find the conversion rate for the local currency and current month when the credit ADO is created under the FX Rate tab in the upper right corner.
  1. Approved By – Enter your name here.
  2. Approval Date – Enter the date you approved the request.
  3. Decision Comment – Provide justification for your approval decision.
    • Include formulas or rationale as appropriate.
    • If you need to attach files to the ADO ticket, click the paperclip icon in the upper right corner of the window to do so. In the event the file is too large (> 20 MB) to attach use the Azure SharePoint Site to upload the file, AO SharePoint Site.
    • Paste the file's URL into the Summary of Issue section.

Discussion (Section E)

  1. Discussion – Use this section for all communications on the ADO ticket. SEs, TAs, and the EA team will note any requests for information or clarifications to complete their requested actions. Use the @ symbol to mention a person (which will subsequently email that person the comment that they were mentioned in.)
