Technical Guide: Surface: Covers, bases, and keyboards (non-Bluetooth) issues

[Asset 4049349]

Who should use this article?

  • Surface

What is this article about?

  • It will help you in addressing common Surface covers, keyboards, or bases issues that your customer may encounter.
  • It is not for Bluetooth keyboards. For Bluetooth keyboards, see KB 3203673 Technical guide: Surface Bluetooth keyboard issues.

Table of contents

1. Cover - keyboard and touchpad

Confirming the issue

When attempting to use the keyboard and the touchpad, neither work when connected to the Surface. Various scenarios include:

  • The keyboard and touchpad stopped working.
    • Both the keyboard and touchpad do not respond while connected to Surface.
    • The keyboard and touchpad worked without issue beforehand.
    • The keyboard is not out of box retail or out of box replacement.
  • The keyboard and touchpad never worked.
    • Both the keyboard and touchpad do not respond while connected to Surface.
    • The keyboard is out of box retail or out of box replacement.

Confirmation/paraphrase

“Just to make sure I understand your issue: Your Keyboard and touchpad used to work but are not working now.” -OR- “Your Keyboard and touchpad have never worked.”

How to code

After confirming the issue, code the case, based on the customer’s issue:

  • The keyboard and touchpad stopped working
    • Accessories > Covers and Keyboards – Surface > Cover – Keyboard and Touchpad > The keyboard and touchpad stopped working
  • The keyboard and touchpad never worked
    • Accessories > Covers and Keyboards – Surface > Cover – Keyboard and Touchpad > The keyboard and touchpad never worked

3. Disconnect and reconnect the keyboard.

This causes Windows to re-identify the device in Device Manager, which can fix certain issues.

4. Force the device to shut down and restart (varies by new vs old generation).

Studio (all), Laptop (all), Pro (5th Gen), Pro 6, Book 2, and Go: Press and hold the power button for 20 seconds.

Ensure the device is up to date.

5. Verify it works in UEFI
  1. Shut down your Surface and wait about 10 seconds to make sure it's off.
  2. Press and hold the volume-up button on your Surface, and, at the same time, press and release the power button.
  3. The Microsoft or Surface logo appears on your screen. Continue to hold the volume-up button. Release the button once the UEFI screen appears.
  4. Verify if keyboard (arrow key) allows you to progress through the menus, or the touchpad allows control of the mouse arrow.

If they keyboard/touchpad works in UEFI, it shows that the keyboard does appear to function, and there may be a driver/windows drive issue.

6. Run SDT or Windows Updates.

For SDT: Have your customer go to http://aka.ms/surfacedt.

For WU: Go to Settings > Update & Security > Windows Updates > Check for updates

Tip: Restart the Surface after installing updates.

Updates have fixes to common problems, or issues. Making sure the customer has all the latest updates on their Surface will ensure they have the best possible experience.

7. Reset

Good: Fresh Start

Go to Start > Settings > Update & security > Recovery.

Under Fresh Start, select Additional info > Get Started.

Better: Reset from within Windows

Go to Start > Settings > Update & security > Recovery.

Under Reset this PC, select Get Started. Follow on screen instructions.

Best: USB recovery

The customer will need to download and set up a recovery image on a USB drive. Send the USB recovery link to them and help them get started. Information on what size and type of drive is provided at the page.

-OR-

Set up an order to have a USB recovery drive sent to the customer.

 

Fresh Start (good):

  • Fresh Start is the least intrusive form of reset.
  • Customers will not lose any files or personal information.
  • Customers will need to reinstall desktop programs.
  • Fresh Start also ensures the customer has all available updates once complete.
  • If there is an issue with Windows itself, the issue may still occur after a reset.

Reset from within Windows (better):

  • This option uses the devices hard drive partition to reimage the device.
  • This option can delete personal data (the customer can choose to keep their files or remove everything).
  • The customer will need to reinstall desktop apps.
  • The customer will need to run updates upon completion.
  • If there is an issue with the partition, the issue may still occur after the reset.

USB recovery (best):

  • This option reimages the entire device, including the partition on the hard drive.
  • It is the most complete form of reset.
  • Customers will need to back-up their personal data.
  • Customers will need to reinstall desktop apps, and will need to run updates after completion.

If the issue occurs after a reset, then the root cause is very likely hardware related.

Is the issue resolved?

Yes, close the case: Once resolved, have the customer check for updates.

 

For SDT enabled sites, let the customer know about the Surface Diagnostic Toolkit.

For all other sites, check Settings > Update & Security > Windows Update for updates.

No, replace the keyboard: Set up a service order for the customer based on the status of their warranty.

2. Cover – keyboard only

Confirming the issue

When attempting to use the Keyboard and the touchpad, the keyboard does not work, but the touchpad works correctly. Various scenarios include:

  • Delays/slow performance
    • The time between when the customer presses a key to when that key is registered on the screen within the customer’s application is longer than expected.
  • Individual key presses dropped/missed
    • The customer presses a unique, and individual key or keys and those specific keys do not register a response.
    • For example: The “A”, “S”, “D”, and “F” keys work, but the “J”, “K”, “L”, and “;” keys do not
  • Never worked but the touchpad works
    • Out of the box, the keyboard is not working, but the touchpad is functioning while connected to Surface.
  • Stopped working but the touchpad works
    • After functioning previously, the keyboard is no longer working, but the touchpad is while connected to Surface.
  • Adjusting screen brightness – all problems
    • The screen brightness +/- keys are currently only supported on Surface Laptop, Surface Pro Keyboards (1796 and newer), and Surface 3 Keyboard.
    • Pressing the screen brightness keys does not adjust the screen’s brightness.
  • Backlight – all problems
    • The keyboards backlight does not turn on while connected to the Surface.
      • The brightness +/- keys on the Surface Touch or Type Cover do not function on Surface 3, Surface Pro 3, Surface Pro 4, Surface Book, and Surface Laptop models only.
  • Function/FN key – all problems
    • The FN (Function key) does not work when pressed in combination with other keys.
      • Most modern Surface Keyboards will have an LED light on the FN key to indicate whether it is enabled. Older Surface Keyboards may not have this feature.

Confirmation/paraphrase

“Just to make sure I understand your issue: The keys on your keyboard are not working correctly or at all.”

How to code

After confirming the issue, code the case, based on the customer’s issue:

  • Delays/slow performance
    • Accessories > Covers and Keyboards – Surface > Keyboard Only > Delays / Slow Performance
  • Individual key presses dropped/missed
    • Accessories > Covers and Keyboards – Surface > Keyboard Only > Individual Key Presses Dropped / Missed
  • Never worked but the touchpad works
    • Accessories > Covers and Keyboards – Surface > Keyboard Only > Never Worked but the Touchpad Works
  • Stopped working but the touchpad works
    • Accessories > Covers and Keyboards – Surface > Keyboard Only > Stopped Working but the Touchpad Works
  • Adjusting screen brightness – all problems
    • Accessories > Covers and Keyboards – Surface > Keyboard Only > Adjusting Screen Brightness – All Problems
  • Backlight – all problems
    • Accessories > Covers and Keyboards – Surface > Keyboard Only > Backlight – All Problems
  • Function/FN key – all problems
    • Accessories > Covers and Keyboards – Surface > Keyboard Only > Function / FN Key – All Problems

4. Touchpad only–sensitivity

Confirming the issue

While using the touchpad, it does not perform to the customers expectation. Various scenarios include:

  • Sensitivity – accidental taps/jumps
    • The customer complains of their mouse cursor being “jumpy” or sporadic.
    • The customer complains that while typing or during normal use, the touchpad registers taps or clicks without the customer intending to tap or click on the track pad.
  • Sensitivity – delays/slow performance
    • The Surface touchpad has slow or delayed response.
    • The time from when the customer moves their finger on the touchpad to when the mouse is registered on the screen is longer than expected.
    • Customers may use the word “lag” or “delay when describing their mouse or cursor movement

Confirmation/paraphrase

“Just to make sure I understand your issue: Your touchpad is not working correctly.”

How to code

After confirming the issue, code the case, based on the customer’s issue:

  • Sensitivity – accidental taps/jumps
    • Accessories > Covers and Keyboards – Surface > Touchpad Only > Sensitivity – Accidental Taps / Jumps
  • Sensitivity – delays/slow performance
    • Accessories > Covers and Keyboards – Surface > Touchpad Only > Sensitivity – Delays / Slow Performance

5. Fingerprint reader

Confirming the issue

In this scenario, the customer is having issues with their fingerprint reader recognizing their fingerprint. Scenarios include:

  • After setup of Windows Hello, the fingerprint reader does not recognize or does not respond to the customer’s finger print.
  • At the time of sign-in, after Windows Hello setup, when the customer attempts to use the fingerprint reader, an error comes up, and the customer is unable to log in.

This is not a Windows Hello setup scenario. For Windows Hello setup, see Troubleshoot problems with Windows Hello on Surface, and code your ticket to the appropriate “Security, Sign in, and Accounts” issue.

Confirmation/paraphrase

“Just to make sure I understand your issue: When setting up or using the fingerprint reader, your fingerprint is not recognized.”

How to code

After confirming the issue, code the case.

  • Accessories > Covers and Keyboards – Surface > Cover – Fingerprint Reader > Fingerprint Reader Does Not Respond

Good: Fresh Start

Under Reset this PC, select Get Started. Follow on screen instructions.

Best: USB recovery

-OR-

Set up an order to have a USB recovery drive sent to the customer.

Fresh Start (good):

  • Fresh Start is the least intrusive form of reset.
  • Customers will not lose any files or personal information.
  • Customers will need to reinstall desktop programs.
  • Fresh Start also ensures the customer has all available updates once complete.
  • If there is an issue with Windows itself, the issue may still occur after a reset.

Reset from within Windows (better):

  • This option uses the devices Hard Drive Partition to reimage the device.
  • This option can delete personal data (the customer can choose keep their files or remove everything).
  • The customer will need to reinstall desktop apps.
  • The customer will need to run updates upon completion.
  • If there is an issue with the partition, the issue may still occur after the reset.

USB recovery (best):

  • This option reimages the entire device, including the partition on the hard drive.
  • It is the most complete form of reset.
  • Customers will need to back up their personal data.
  • Customers will need to reinstall desktop apps. Customers will need to run updates after completion.

If the issue occurs after a reset, then the root cause is very likely hardware related.
