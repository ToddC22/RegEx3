ASMS – EA Scenarios

Overview

This TSG provides guidance to help determine which Enterprise Agreement (EA) scenarios are handled by ASMS, and which scenarios are handled by the Enterprise Azure Operations (EAO) team.

EA Scope Transition to ASMS

Some new scenarios are now owned by ASMS as part of the EA scope transition. For more information on these scenarios, see the Message Center post.

 

Policy

  • For the scenarios listed under the EAO Scope section, send the entire case to the EAO team (Azure EA Portal Support) as soon as the scenario has been identified. 

  • For some of the scenarios owned by ASMS, additional steps and information can be found in the Help section of the EA Portal. As well as in the respective TSGs. 
    Note: The customer can also access the help section directly by logging into the EA Portal.

Finding an Enrollment Number Without Contacting the Customer

In some cases, customers may not know their Enrollment number or have not provided it in the case. Use the below guidance to find the customer's Enrollment number without having to ask them for it.

  • Use the company name noted in CMAT or in the customer’s signature to search for the enrollment in the EA Portal.

  • If you’re still unable to find the Enrollment number, try searching the domain in CMAT.

  • If the company name contains multiple words, try searching in the EA Portal using individual words. 

  • The EA Enrollment number can also be found through Azure Support Center (ASC) by launching ASC from Service Desk (SD) while the case is open. If the customer has an Enrollment number, it will be displayed on the ASC screen as depicted below:
    [Asset 4467049]

  • If, in ASC or CMAT, the customer's enrollment details show as N/A, do the following to find the customer's Enrollment number:

    1. Navigate to https://dataexplorer.azure.com/clusters/azsc/databases/asccosmos.

      Note: ASC access is required to use the Azure Data Explorer.

    2. On the Query tab, under azsc, select asccosmos.

    3. Run the following command:

      Customer_Subscriptions<Return/Enter key>

      | where SubscriptionId contains "<Subscription GUID>"

      Note: In place of <Subscription GUID>, enter the customer's subscription GUID. In place of <Return/Enter key>, press the return/enter key.

    4. In the results, the customer's Enrollment number will be visible.

  • You can also use the Customer Portal tool to find the Enrollment number using the subscription GUID. Use the following link to access the tool: https://caicustomers.microsoftonline.com/.
  •  If you are unsuccessful in finding the Enrollment Number after attempting the above steps, you may request the information from the customer.
     

Adding the missing Enrollment Number to the case

When working on an EA case, ensure the Enrollment ID is listed in the Enrollment ID field of the case management tool. If it is not listed, you should manually add it before closing the case. 

 

Customer Experience and Provisioning

In addition to the self-serve actions listed in the Help section of the EA Portal, here are some additional actions that can be taken by or on behalf of the customer:

  1. TAM or salesperson wants access to Azure Health Report.

    • Direct them to: https://microsoft.sharepoint.com/teams/ceinsights.

    • If they need support for the tool, they can email the C+E BI Support alias at cebisup@microsoft.com.

  2. EA customer requests to change the Company Name on an EA invoice.

    • Direct the customer to their Account Manager, as this is out of scope for both the ASMS and Enterprise Azure Operations teams.

  3. Azure True-up

    • Refer to the Enterprise Agreement True-up Guide: Click here for more details.

Support Scenarios

The following is a list of scenarios handled by EAO and a list of scenarios handled by ASMS. Leverage this list to determine how to best support your customer.

Note: For guidance supporting customers with Programmatically Created Subscriptions, see TSG 4090378.

EAO Scope

The scenarios in this section are handled by the EAO team. Should you receive a case related to any of the below scenarios, follow the aligned guidance to transfer the case or engage the EAO Team accordingly.

Internal Microsoft Employee Support
Cases submitted by Internal Microsoft Employees that reach the ASMS EAL MCA queue, should be transferred to the MSaaS Azure EA Help - Internal Audiences queue for the EA Portal team to handle the case, irrespective of the scenario/scope. 

ASMS Scope

This section provides guidance for handling EA-related support scenarios that fall under ASMS scope.

Users and Administrators

The TSGs in this section provide guidance on EA Portal roles and permissions.

  • General Information about the EA Portal – Provides general information about the EA Portal.

  • Add or Remove EA Administrator – Provides guidance on how to assist customers in adding and removing EA Administrators.

  • Add A New EA Account – Provides guidance on how to add a new account to an EA via the EA portal.

  • EA Portal Authentication and Access Issues – Provides guidance on how to troubleshoot EA Portal login issues. Also covers adding users to EA accounts, validating and updating user PUIDs.

  • EA Activation Email – Provides guidance on assisting EA customers with email activation issues.

  • EA Price Markup Feature – Provides guidance on assisting customers with the Markup feature, as it relates to Azure pricing.

  • Add Support to EA Enrollment – Provides guidance on adding technical support to EA enrollments.

  • Enrollment not Found in EA Portal – Provides guidance on finding previous enrollments in the Portal.

Enrollment Invoicing

The TSGs in this section provide guidance related to the purchase and billing experience.

  • Add Notification Contact to EA – Provides guidance on adding Notification Contacts to enrollments, as well as stopping or changing the frequency of email notifications.

  • Change EA Invoice Purchase Number – Provides guidance on how customers can change the purchase order (PO) number on their invoice.

  • EA Combined Overage and Marketplace Invoices – Provides guidance on how to read and understand combined Overage and Marketplace invoices.

  • How to apply EA Overage to Azure Pre-payment – Provides guidance on applying EA Overage to Azure Pre-payment.

  • EA Billing Cycles and Channel Modes – Provides guidance on the various billing modes and billing cycles available to EA customers.

  • EA Azure Commitment Discount (ACD) – Provides guidance on EA Azure Commitment Discount (ACD).

  • Understanding EA Billing and Invoice Issues – Provides guidance on troubleshooting EA billing and invoice issues. Applies to Direct Customers and Partners.

  • Azure EA Services and Charges – Provides guidance on understanding Azure services and charges.

  • EA CPS Special Pricing – Provides guidance on Customer/Channel Price Sheet (CPS) Special Pricing.

  • EA Pricing – Provides guidance on EA-related pricing (price protection, regional pricing, general availability (GA) pricing, graduated pricing, conversion from usage details to usage summary, and rounding rules).

  • SIE Credits – When an EA customer requests a Service Impacting Event (SIE) credit, ASMS should work with the EAO team accordingly. Requests for processing SIE credits for indirect customers should come from the Partner. See the following TSGs for additional guidance:

    • TSG 2658533 (EA Credits Escalations)

    • TSG 3119077 (EA SLA and SIE Credit Requests)

    • TSG 4094770 (EA Credit Request – TFS Ticket Process)

  • Change EA Country of Usage on Overage Invoice – Provides guidance on changing the EA Country of Usage on an Overage Invoice.

Usage and API

The TSGs in this section provide guidance on EA reporting.

  • EA Reporting API Troubleshooting – Provides guidance on troubleshooting API reports.
  • EA Reports and Pricelists Requests – Provides guidance on understanding EA reports and pricelists.

Destructive Scenarios

The TSGs in this section provide guidance on terminating and transferring EAs. These scenarios are considered "destructive," and contain actions which must be carried out by managers.

  • Terminating Enterprise Agreements – Provides guidance on how to handle EA termination requests, as well as reactivations of terminated EAs.
  • Transferring EA Enrollments – Provides an overview of EA enrollment transfers, and guidance on how to perform them.
  • Transferring EA Accounts – Provides an overview of EA account transfers, and guidance on how to perform them.
