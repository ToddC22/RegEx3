1. Conversion enabled clicks

Microsoft Advertising uses a system known as conversion enabled clicks to track conversions. This means that a click will not be recorded in our system as a conversion unless it meets the following criteria:

  1. The account has at least one ACTIVE UET goal using the tagid.
  2. The UET tag on the account has at least one event fire in the last seven days (for example, the tag is active on the website).

A conversion will not be registered by Microsoft Advertising unless both 1 and 2 are true.

Conversion reporting time

Microsoft Advertising associates the date of a conversion to the date an ad was clicked (for example, if a customer clicks an ad on day one, then completes a conversion on day three, the conversion will show as occurring on day one when the advertiser checks their reporting).

Advertisers reporting discrepancies in timing between UET tracking and conversions in the UI should check the lookback window for their UET tags, to make sure they’re covering the correct timeframe.

Advertisers reporting discrepancies between Google Ads Analytics and Microsoft Advertising conversion tracking should be made aware that the two systems report differently: Google Ads will report based on the date of the conversion, not the date of the click.

2. Universal Event Tracking (UET)

UET is the conversion tracking tool that replaces Campaign Analytics. This tool allows advertisers to track what customers do on their website after clicking on an ad. By creating one UET tag and placing it across your website, Microsoft Advertising will collect data that allows advertisers to track conversion goals and target audiences with remarketing lists.

UET is available for API users as well.

5.1 Include in “Conversions ”

Include in “Conversions” is a setting within the conversion goal create/edit workflow that allows customers the ability to either include or exclude a specific conversion goal from reporting conversion metrics. Customers can therefore customize reporting grids to see the data that matters most to them so they can better assess the impact of ad campaigns and make better optimization decisions. The goals the customer selects also tells the platform which conversions to consider in automated bidding, helping improve ad matching and performance on search partner sites.

Along with this setting, five reporting columns are available, designed so the advertiser can see all conversion data regardless of the Include in “Conversion” setting selected by the customer. Those columns are: “All conv.,” “All conv. CPA,” “All conv. rate,” “All conv. revenue,” and “All conv. return on ad spend.”

Things to know:

  • New and existing goals can be selected or removed for Include in “Conversions” from the Goal details page within the Conversion goals section of the UI.
  • All existing goals will be defaulted to the Include in “Conversions” setting.
  • Only conversions from goals selected for inclusion will be used to calculate conversion-based metrics, such as conversions, conversion rate, CPA, revenue, and return on ad spend (ROAS).
  • Performance grids will, by default, only show conversion data from goals marked for inclusion. When deselecting a goal from being included, it will only affect reporting data moving forward.
  • If a customer excludes a specific conversion goal from conversions, it will also exclude that goal for automated bidding optimizations with Microsoft Advertising.
  • To see reporting for all your conversion goal types, you can select in the performance grid and Reports page a new set of conversion-based metrics that are preceded by “All,” such as “All conv.” Or “All conv. CPA.”
  • This is a Google parity feature.
  • It’s available for all goal types (such as, UET, Offline, App, and In-Store).

Getting started:

  1. When viewing the details of any existing conversion goal or during the setup of a new goal, the option to Include in “Conversions” is at the bottom of the screen. This box is selected by default.

[Asset 4559029]

Selections can be seen summarized on the Conversion goals summary page.

[Asset 4559030]

  1. If you want to see performance reporting across all conversion goals, not just the ones selected to Include in “Conversions,” use the set of five conversion-based metrics that start with “All”.

[Asset 4559031]

For more information, see Conversion goals: “Conversions” versus “All conversions”.

5.2 View-through conversions

This conversion goal setting is designed to help customers measure the true value of their ads on the Microsoft Audience Network. Unlike the Microsoft Search Network, where users are actively looking for the products or services advertisers offer by typing in a keyword, Microsoft Audience Ads are displayed on the Microsoft Audience Network (MSAN) properties based on the audience an advertiser is targeting and their intent goals. Evaluating the performance of campaigns on MSAN based on click-based conversions may not be optimal and may discount the effectiveness of the ads.

To solve this problem, view-through conversions were introduced. View-through conversions are based on impressions when users see a customer’s ad and later convert on their website. Instead of counting clicks, Microsoft Advertising tracks ad impressions related to conversions on a customer’s site within a time period that they choose. Think of them as impression-based conversions.

The last viewable impressions within the defined view-through window will be credited as view-through conversions for a specific conversion goal. Users who had any ad clicks on either MSAN or the Microsoft Search Network will be automatically excluded from view-through conversion metrics to avoid double counting.

Benefits:

  • Enhance conversion tracking by attributing user actions to impressions.
  • See how users are responding to ads even if they don’t click on them.
  • Get a better understanding of how the Microsoft Audience Network is working.

Things to know:

  • All accounts are enabled for the account-level view-through conversions setting, which includes the view-through conversions in the All conv. column.
  • View-through conversions only support impression traffic on MSAN, which includes audience campaigns as well as search campaigns extended to the MSAN.
  • Historical data for the view-through conversions metric will be shown as null.

Example

Let’s say an online retailer set up a conversion goal to track purchases on the checkout page with a view-through window of four days and a conversion window of five days.

[Asset 4559048]

Set up view-through conversion window

All UET-based conversion goals have a setting called the View-through conversion window on the Goal details page which is set at 1 day by default. The valid range is between 1 minute to 30 days (inclusive).

Only UET-based conversion goals are supported: Destination URL, Duration, Pages viewed per visit, and Event.

Count (All vs. Unique) option determines how to count view-through conversions. A single ad impression can result in either one or multiple conversions within the same view-through conversion window.

[Asset 4559032]

The View-through window can be viewed in the Conversion goals grid. Mobile app install, offline conversions and in-store transaction goals will show a dash “-“.

[Asset 4559033]

Include view-through conversions in “All conv.”

An account-level setting view-through conversions is in the Account level options under Shared Library, enabled by default. When this setting is enabled, view-through conversions will be included in only the All conv. column. There is no impact on the other All conv. metrics (All conv. CPA, or All conv. rate). This setting has no impact on historical data when it’s updated – only new data is affected.

[Asset 4559035]

View-through conversion reporting

  • The metric View-through conv. can be seen in all grids and reports with conversion data. View-through conversions are reported at the impression time.
  • Only Audience ads total (in the grid) and Audience (segment by network) have values for view-through conversion.
  • View-through conversion is always 0 for all other networks (AOL Search, Bing and Yahoo Search, Syndicated Search Partners).
  • View-through conversions are only included in View-through conv. and All conv. (if the associated setting is enabled). It’s never included in the conversion columns (used for click-based conversions).

[Asset 4559036]

For more information, see What are view-through conversions?.

6.1 Validate tag setup

If the advertiser has installed UET Tag Helper more than 24 hours ago, you can validate the tag by looking at the UET tag status.

  1. Turn on UET Tag Helper in Chrome and go to the customer’s website.
  2. If the advertiser has installed UET Tag Helper throughout the website, each page you visit will result in the reset of the count of events seen on the Tag Helper (Green = working, Red = Critical error, Yellow = Minor issue, can be improved).
  3. If you don’t see any numbers (no events) as you browse through pages either:
    • The tag is not implemented or implemented incorrectly
    • The tag is installed only on Conversion page (check with advertiser)
  4. If tag is installed only on conversion pages, check with the client for exact workflow to convert and take those actions while leaving Tag Helper on.

Note: You may not see any events on intermediate pages but should see one on the final conversion page.

Outcomes:

  • No tags found: (The client did not install UET Tag Helper or did not install it correctly), work with the client. Engage support only if client provides supporting data/proof that they have installed it correctly.
  • Tags working fine: Yay – no action required!
  • Tags returns in error: See Results tab for possible solutions.

If the client needs help or wants to see what a correct implementation looks like – they can reference our Microsoft Advertising UET Sample website and review the event fires in Tag Helper for comparison purposes: http://bingadsuet.azurewebsites.net/Index.html.

6.2 Deprecation of redirected Campaign Analytics (CA) to auto-migrated UET tags:

Last year, Microsoft Advertising announced the deprecation of Campaign Analytics (CA) and migrated customers to UET by automatically generating a new UET tag (UETTag_Autogen_for_CA) and conversion goal(s) (UETGoal_Autogen_GoalName). Microsoft Advertising has now stopped redirecting from the old deprecated CA tags.

If advertisers have already established another UET tag and conversion goal, they should be all set. However, if advertisers have noticed a decline in their conversions on the migrated goal(s) “UETGoal_Autogen_GoalName”, it is because we are no longer redirecting Campaign Analytics traffic to UET and they still need to take the final steps to implement their new UET tag and update their goal(s).

To continue tracking conversions through the migrated tag and goal(s), advertisers will need to:

  1. Remove all Campaign Analytics tags from their website and replace them with the UET tag created for migration (UETTag_Autogen_for_CA).
  2. Update their definitions for the goal UETGoal_Autogen_GoalName as appropriate.

7. Sharing UET tags

Once a Customer Hierarchy is built, UET tags can be shared across different accounts and manager accounts within the same hierarchy using the Shared Library.

Sharing can be managed by the UET tag owner or the owner’s manager Super Admin. Sharing permissions will vary based on the scope of the Super Admin’s user access.

For more information about Customer Hierarchy or Shared Library, see the help article https://help.ads.microsoft.com/#apex/3/en/60035/-1. The Shared Library content is at the very bottom of the page.

7.1 Sharing an existing UET tag

There are two places where you can manage your UET tags. While in the manager account:

  1. Navigate to the Accounts tab at the top of the page.
  2. Select UET tags under Shared Library on the left-hand navigation bar.
  3. Here you can see all the UET tags available for the manager account you’ve selected. The other place where you can manage your UET tags is the UET tags page in the Conversion Tracking section on your Campaigns tab.
  • In either place, you can view the UET tag’s owner, sharing scope, and who it’s used by in the UET tag grid. The Scope column lists the accounts that can share within the hierarchy. The Used By column displays the accounts that are currently using a shared tag.
  1. If you’re the owner of the UET tag or the owner’s manager and a Super Admin user, you can share the existing UET tag by selecting Manage Scope.

7.5 FAQ

Q: Is Shared Library available for legacy (non-hierarchy) CID to XID account linking?

A: No, Shared Library is only available for CID-CID linking for accounts within a hierarchy. We don’t support Shared Library for CID to XID linking. Refer to the customer hierarchy resources for more information.

Q: Is Shared Library available to aggregators?

A: No, hierarchy linking and resource sharing is not available to aggregators. You can validate whether an account has been designated as an Aggregator in the UI by checking the Customer Type field under Company Information > Settings and Yahoo accounts.

9. Offline conversion tracking

Let’s say a customer sees your ad, clicks on it, but ends up completing the sale offline – either by calling you or visiting your physical store. How can you track when your search ad leads to a conversion offline and outside of your website? You can import offline conversions, to better measure what happens after your ad was clicked.

Note: Offline conversions are a separate feature from UET. As such, the usage of UET or UET tags is not a prerequisite to setting up offline conversion tracking.

See the Tracking Conversions Offline help page for more information.

Important: Note that once offline conversion data is uploaded it cannot be edited/cancelled, either by the customer or internally by our engineering and support teams. If customers require help with correcting such mistakes, explain that this is not a feature currently supported by offline conversion uploads, and direct them to UserVoice in order to vote on existing requests to prioritise this feature in future tool updates.

9.1 Benefits of offline conversion tracking

Tracking the impact of your campaigns for both online and offline channels will allow for more holistic optimization and better ROI. Importing offline conversions gives you a more comprehensive look at which keywords and targeting criteria (for example, geography or time of day) drive the most cost-effective conversions.

Return to top

9.2 How offline conversion tracking works

Offline conversion tracking works differently depending on whether you’re tracking conversions that start with an ad click vs. a call from your ad.

Return to top

9.2.1 Conversions from clicks

Microsoft Advertising provides you with a unique 32-character ID, called a Microsoft  click ID (MSCLKID), for every click driven to your website from an ad. If a prospective customer clicks your ad and submits their contact info via your website, you’ll have the MSCLKID associated with that click. When you call the customer back later, they “convert” by making a sale offline. To track this offline conversion, you’ll import that MSCLKID back into Microsoft Advertising along with a few details about the type of conversion it was and when it happened. Microsoft Advertising will record this conversion along with your other conversion tracking data, allowing you to see which keywords and ads are contributing not just to new lead submissions, but to actual sales.

Return to top

9.2.2 Conversions from calls

By importing call conversion data into Microsoft Advertising, you can track which of your ads and keywords result in the most sales calls for your business. Instead of using a click ID as with conversions from clicks, you track information about your phone calls that resulted in sales. When you import this data into Microsoft Advertising, these conversions are matched with phone calls from your ads.

Note: This feature is only available to advertisers using a supported call-tracking provider. Advertisers should refer to these providers’ documentation for details on integrating with Microsoft Advertising. For the full list of supported providers, see the help content below.

9.5 Enabling your website to track Microsoft click IDs

When you create your first offline conversion goal, auto-tagging of Microsoft click IDs is automatically turned on. Once auto-tagging is enabled, when an ad is served, the MSCLKID is dynamically appended to your landing page URL. The click ID is unique for each ad click.

After you’ve created one or more conversion goals, you’ll need to enable your website to track conversions. Be sure to consult with your webmaster when completing this process. It involves the following two steps:

  1. Configure your website code to capture and store the MSCLKID in a cookie. This will ensure that the ID follows customers through their session. More details and the sample JavaScript code can be found on the Help pages below.
  2. Configure your website code to store MSCLKID with lead data. When the customer submits a lead gen form, you’ll need to retrieve the MSCLKID from the cookie and send it with the rest of the customer data. This way, all the data is stored in your CRM system and you can track offline conversions for this click. More details and the sample JavaScript code can be found on the help pages below.

9.7 Deleting or editing offline conversion records

Once uploaded, the offline conversion records cannot be altered. None of the following are options:

  1. Deleting the records.
  2. Editing the records (adding a conversion or revenue).
  3. Editing the records (trying to remove the conversion/revenue by adding negative conversion/revenue value).

Inform customers that these features are not currently available for Microsoft Advertising and direct them to the UserVoice forum to provide engineering feedback.

Note: Recreating the goals and reuploading the data will not delete the conversion data already uploaded – so this is not a viable workaround.

9.9 Common offline conversion troubleshooting scenarios

  1. Issue: click ID is lost due to redirection or UET tags are not implemented on all pages.
    • Customer Impact: Example: A customer can see 10 clicks in Microsoft Advertising but is only able to fetch six click IDs when they run their web server logs.
    • Solution: Ensure the customer has tagged their entire website and everything is set up properly. If no issues can be found, escalate to Global Support for investigation.

  1. Issue: Partial attribution for uploaded conversion records. Example: Customer uploaded 100 records but only 80 get attributed. – there is no error report for this.
    • Customer Impact: Example: A customer uploaded 100 records and received a message saying they were all uploaded successfully. When the customer checks back 5 hours later, they see only 80 conversions in their reports. They want to know where the 20 records went.
    • Solution: Escalate to Global Support. They must run logs to identify the cause. A few possible reasons are: The click ID is invalid, the click ID doesn’t belong to the customer’s account, the click ID is older than the lookback window defined for the goal, the conversion time that was entered is too recent, or the conversion goal name that they entered cannot be found on the account.

10.4 Variable revenue

Variable revenue is reporting revenue at the time of conversion when each conversion may have a different value. When creating a goal, advertisers can choose to either:

  • Not associate revenue with the goal.
  • Associate a fixed value for revenue with every conversion for that goal.
  • Report revenue at the time of conversion. Note: Variable revenue is applicable only to destination URLs and event type goals, it is not applicable to duration, pages/visit and mobile app install type goals.

JavaScript:

Non-JavaScript:

<noscript>

10.11 App extension goals & conversions

Products such as App Extensions allow advertisers to show ads to users that click to the app store to promote app downloads. Microsoft Advertising enables mobile app install tracking through partnerships with Kochava, Tune and Appsflyer. Only those advertisers that use one of these partners (and adopt their software development kits (SDKs) into their apps) can track mobile installs as conversions in Microsoft Advertising.

How mobile app installs are tracked as conversions:

  • If the advertiser supplied a Measurement URL from the partner, Microsoft Advertising appends a parameter called bclickid to that URL during ad delivery. When a user clicks on the App Extension, the call goes to the partner, which logs the bclickid and redirects to the app store.
  • If a user installs the app, the first time the app is opened, the partner SDK on the app notifies the partner that the app has been installed and opened. At this point, the partner does conversion attribution.
  • The partner then uploads conversion counts data back to Microsoft Advertising via a server-server interface (a modified version of UET) using the clickID and AppID as the keys.
    Microsoft Advertising then (1) identifies the goals associated with the AppID, (2) checks if clicks are in the conversion window of the goal and (3) associates the conversion with the App Extension.

Note 1: A few callouts for mobile app install goals:

  • Attributes such as Counting option (Unique vs All), Scope (Account vs Customer level goals) and Variable revenue cannot be tracked for mobile app install goals.
  • Assists are not counted.
  • No UET tags are required or associated with mobile app install goals.
  • Tracking status can have only one of two values - Recording conversions and No recent conversions.

Note 2: The current solution for mobile app install tracking has the following issues:

  • Currently there is no clear service level agreement (SLA) established between the partners and Microsoft Advertising for how long the partner takes to report conversions to Microsoft Advertising after users install the apps.
  • Conversion attribution (attributing conversion to a click) is done by the partner and not by Microsoft Advertising. Microsoft Advertising is unaware of the attribution model (such as first click and last click) employed by the partner.
  • App URL and Measurement URL are the same.

11. Troubleshooting

11.1 Conversion tracking not reporting correctly

Use the following troubleshooting guidance if the customer indicates their conversion tracking is inaccurate.

Remember that in Microsoft Advertising the date of conversions is shown as the date when an ad was clicked. If customers are using third-party conversion tracking, these may base conversion date on the date the conversion occurred. If the number of conversions matches, then the date discrepancy would be due to this reason.

Third-party tracking report does not align with Microsoft Advertising conversions

Advise the advertiser that Microsoft Advertising conversion tracking reports are the most effective for monitoring Microsoft Advertising conversions given variance in the methodologies used by different reporting services. Universal event tracking will record a conversion for each time that a page is loaded and meets a goal that is defined in the UI for a user that has clicked on an ad. A single user can trigger multiple conversions.

The main cause of third-party tracking discrepancies is that UET does not have the ability to adjust the dedup value. This means that a conversion will be tracked for every valid code fire. Until functionality exists to allow advertisers to modify the dedup value, discrepancies with third-party reporting are expected and do not indicate an error in the code itself.

Conversion tracking is not reporting accurately in Microsoft Advertising

The advertiser can provide physical and anecdotal evidence that conversions are not being captured by Microsoft Advertising Conversion Tracker.

  • First, confirm with the advertiser that they have an ACTIVE goal in their account using the relevant tagid.
  • Confirm that the UET script is implemented on the website and the conversion event occurred in the last 7 days (the goal will still need to have been active during this time).
  • If the customer tried to test for a conversion and 1&2 were not completed, they will need to test their conversion again.
  • Request the advertiser conduct a test conversion and gather the following information for the Global Support investigation:
    • Exact time conversion occurred
    • Exact steps taken to complete test conversion (specify all navigation and clicks necessary to complete the conversion)

Request Assistance from Global Support.

Complete the issue-specific Category = Global Support Escalations and Issue-specific Type = Conversion Tracking Escalation and submit to Global support for further assistance.

As a best practice, include the following information in your escalation to Global Support:

  • Webpage URL where the Microsoft Advertising analytics script for the specified account is hosted. Note: If the webpage URL may only be viewed by making a purchase, ask for the full HTML source text file.
  • What are the steps/actions involved in completing a Conversion? Example: Click on the Ad, fill the form for sign up of newsletter or add product to shopping cart and complete purchase (include Test Credit Card Instructions).
  • R-link: To obtain the r-link, search for a keyword, right click on the ad, copy the shortcut, and then paste it into a text editor.

Global support will use publicly available test card numbers provided by all major credit carriers to attempt a test conversion and conduct their investigation.

Messaging: "We can have a specialist run a couple of test conversions, but we use publicly available test cards provided by all major carriers."

If Global Support indicates the test cards were unsuccessful to reproduce a conversion: "It seems our technical teams were unable to use test card numbers provided by all major card carriers to attempt a test conversion. Are you able to provide us with a 100% promotion code to conduct a test conversion? Once complete we will inform you of the order generated because of testing so you may cancel it."

11.3 Bing's solution for Apple's Intelligent Tracking Prevention (ITP)

Apple Inc. has introduced a new feature in Safari called Intelligent Tracking Prevention, which may impact the reporting of conversions to Microsoft Advertising from Apple users who have installed the new High Sierra and iOS 11 operating systems. To help ensure that conversions continue to be reported accurately across your Microsoft Advertising account(s), we recommend following the instructions below if you track conversions in Microsoft Advertising using Universal Event Tracking (UET) tags. These steps are consistent with Apple’s guidance on ad attribution.

How to continue tracking conversions if you track conversions in Microsoft Advertising using UET tags?

To continue tracking conversions for Apple users who have installed the High Sierra and iOS 11 operating systems, you will need to enable auto-tagging of Microsoft click ID (MSCKLID). Microsoft Advertising UET will then set a cookie on your website’s domain that captures the ad click ID that brought the user to the website. Conversions on the website will then be attributed back to the search ad click using this ad click ID. This change will go into effect by early 2018.

You can enable auto-tagging on your website by clicking URL options in the Conversion Tracking section in Microsoft Advertising online.

Additional information

  • Conversions that are tracked within 24 hours of the ad being clicked will be completely unaffected by this change to Safari.
  • Conversions will continue to be tracked correctly for users who have recently interacted with Bing services.
  • For users who have not recently interacted with Bing services, this change to Safari will cause conversions to go unreported in Microsoft Advertising until the above workaround is completed.
  • While Safari users are an important segment of the Microsoft Advertising Network, the majority of ad clicks and conversions across the Microsoft Advertising Network come from PC users and reporting for those conversions will be unaffected by changes to Safari.
  • Advertisers should avoid placing the UET tag within iFrames.
  • Advertisers should make sure any redirection trackers they use are also passing the MSCKLID URL parameter before reaching the final landing page.

The impact to reported conversions will be limited to only Apple users who have upgraded their operating system to the most recent version (High Sierra and iOS 11).

12. Terminology quick reference

  • Tag - Universal Event Tracking (UET) tag that is placed on the advertiser website and enables Microsoft Advertising to identify user actions on advertiser site.
  • Tracking code - An html snippet associated and provisioned with the UET tag that advertisers must install on their website for conversion tracking and remarketing.
  • Goal - A way for advertisers to define which user activity to count as a conversion.
  • Goal type - The type of user activity to track as conversions. Microsoft Advertising supports five types of goals: Destination URL, Pages/Visit, Duration on Site, Custom Events and Mobile App Installs.
  • Referrer URL - When a user clicks on a link in a webpage, the user's browser moves to the specified link. When the browser requests the new page, it sends along the URL of the previous page (along with many other details). This previous page URL is called a referrer URL.
  • Custom event – With UET, advertisers can customize the UET tag to report more granular events taken by visitors on the advertiser’s website. Custom events can be used to define either conversion goals or remarketing lists.
  • Revenue - Numeric value that advertisers can associate with their conversions. Reported as the Revenue column in performance and goals report.
  • Count (Unique vs All) - Advertiser specification on whether to count all conversions (of a given conversion goal) generated by a click or just one. All conversions make sense for purchase type goals, unique makes sense for lead-generation type goals.
  • Scope (Account-level vs. Customer-level goals) - When goals are associated to one account (in order to attribute conversions to only clicks from that account), they are referred to as account-level goals. If they are associated with all accounts, they are called customer-level goals.
  • Conversion window - An advertiser’s defined value representing the period of time after a user clicks an advertiser’s ad during which a conversion, such as a purchase, is recorded in Microsoft Advertising.
  • Conversion attribution - The process of assigning credit for a conversion to one or more ad clicks. There are several models of attribution such as last click, first click and time decay. Microsoft Advertising does last ad click attribution.

13. FAQ

Q: An advertiser wants to have a data processing statement/agreement from Microsoft, to comply with GDPR and Privacy Requirements, as they are concerned with Microsoft Advertising collecting customer information as part of UET. How should I respond?

A: UET offers advertisers the ability to place a Microsoft pixel/tag on their sites, which allows Microsoft to track users’ activities on the advertisers’ sites once users click on an ad served by Microsoft Advertising.

Microsoft collects data directly from the user via the UET pixel. Although the advertiser places our pixel on their website, we are not processing the data on the advertiser’s behalf. Therefore, Microsoft is a data controller with respect to data collected via a Microsoft UET tag.

Microsoft honors data subject rights through the standard controls and processes applicable to Microsoft Advertising, including honoring the privacy settings chosen by data subjects through Microsoft’s privacy dashboard.

A Data processing agreement is not required as we are the data controller and users will be able to manage their data settings via Microsoft’s privacy dashboard (www.microsoft.com/privacy).

Q: Is there a best practice for tracking multiple domains using UET tags within the same customer shell?

A: Yes! Advertisers should create separate tags for each domain they wish to track with UET. This helps prevent scenarios where a tag may fail for one domain but remain active for others, leading to a misleading ‘active’ tag status.
