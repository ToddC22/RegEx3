Single user creation

You must be precise in entering the username. If it already exists, you do not receive an error message until you try to save the newly created user.

  1. In the Create User form, you can edit the following fields.
    • User Principal Name (in the format of [alias]@office365support.com or microsoftsupport.com)
    • Password
    • First Name
    • Last Name
    • Usage Location (enter a two-letter country code, in uppercase.)
    • Company Email Address
    • Location (select the supplier and their city office from the drop-down list.)
    • Active Employee (select the check box.)
    • Business Unit
      • Commercial (select the radio button to show selections.)
      • Consumer (select the radio button to show selections.)
    • Line of Business (You can select more than one line of business. For example, Windows Commercial and Premier.)
      [Asset 4577771]
  2. After you've made your entries, select Create. A dialog box pops ups briefly, showing the new user entry being sent to the Azure database. You also receive a confirmation message.
Bulk user creation
  1. In the Bulk Create section, select the Choose File button to select an Excel file to import, and then select Import.
    [Asset 4577772]
  2. If you need the template file, click the blue here link. The template is stored on the MID Management Tool SharePoint site and contains the requirements for bulk creation.
    [Asset 4577773]

VMAS-VDI setup

Setup instructions:
  1. VMAS is automatically provisioned after 24 hours of user creation with PartnersAccess permissions. Use regional portal to access it, see point 3.
  2. Current permission levels are:
    • ExternalConnectivity
      Max Memory per Machine = 16 GB
      Max Memory per Workspace = 32 GB
      Max Number of Machines per User = 12
      Max System Disk Size = 300 GB
      Max Secondary Disk Size = 500 GB
      Max External IPv6 Addresses = 1
      Allowed to Use DNS Records
    • PartnersAccess (Azure)
      Max Memory per Machine = 16 GB
      Max Memory per Workspace = 32 GB
      Max Number of Machines per User = 15
      Max System Disk Size = 300 GB
      Max Secondary Disk Size = 500 GB
      Not Allowed to Use External IPv4 Addresses
      Not Allowed to Use DNS Records
  3. The provisioning is done on a GEO basis with the following permission level:
    1. Americas (NOAM & LATAM)
      https://ncuscommercial.vmasvdi.net
      CSSCommercialVDI_ExternalConnectivity
      CSSCommercialVDI_PartnersAccess
    2. West Europe (EMEA)
      https://wecommercial.vmasvdi.net
      EMEA_CSSCommercialVDI_ExternalConnectivity
      EMEA_CSSCommercialVDI_PartnersAccess
    3. East Asia (APAC)
      https://eacommercial.vmasvdi.net
      APAC_CSSCommercialVDI_ExternalConnectivity
      APAC_CSSCommercialVDI_PartnersAccess

Tools decommissioning procedure

Tools decommissioning procedure

Modern Identity

You can edit a blocked user's properties; however, you cannot unblock a user from within the UMT site. Typically, an admin blocks an account because the user has been terminated, so unblocking a blocked user is a security risk.

  1. In the Returned Users list, go to the Edit/Block User column, and select the Block Account link.
  2. After you block a user, you receive a confirmation.
    [Asset 4577774]

Rave

  1. Browse to Admin - Queue view.
  2. Search for the user by email address.
    [Asset 4498615]
  3. Expand the options and select Delete.
    [Asset 4498616]

OneCloud

  1. Login to configuration manager via OneCloud Genesys and select Persons within the Accounts tile.
    [Asset 4498617]
  2. Browse to OneCloud -> Concierge -> CSS -> select the appropriate supplier folder -> site folder (if applicable) -> either Agents or Non-Agents.
  3. Search the account by email.
    [Asset 4498619]
  4. Confirm that it’s the correct account, select it and click Delete.
    [Asset 4498620]

CMAT

Create a SNOW ticket to request CMAT permission removal.

LMI and VMAS

LMI will be auto disabled in 6 months and VMAS will be disabled as the user is deleted in Modern Identity.
