TUG: ASD: Creating a new case

[Asset 4049349]

Who should use this article?

  • All Consumer

What is this article about?

This article is part of the ASD TUG, and explains how to create a new case within ASD.

To create a new case for an email customer because they don’t have a Microsoft account, follow these steps:

Important:

  • When you create a new SR, use “New;” and when you complete the work, use “Complete.”
  • When the interaction is short, do not save in between creating a new SR and completing it. There are limited case notes.
  • This should apply for any cases under 20 mins.

If substantial case notes have been added to the account during the call, the advocate may save (this should be done sparingly). Otherwise, save should only be used when closing a case, before transferring a case, or when you are done working on a case.

Please remember that when customer and contact data is entered in the appropriate areas, it is saved automatically

Action
1. Search for the customer account. See KB 3201395 Tool User Guide: Membership Support Tool (MST) Search - How to search in MST for guidance.

2. Complete the Incident form:

 Important:

You must complete all fields that have a red exclamation point before you can save the case.

The options in this form change based on what you already selected.

Do not use save button frequently, save should only be used when closing a case, before transferring a case, or when you are done working on a case
3. Select New to create a new case.

4. Continue to complete the sections in the Incident form:

 Important:

Type the case title in the Subject field in the greet customer and collect info section. This field is the same as the subject line in CAP and is displayed when you perform a search on a queue.

5. Select Save or Complete based on the scenario.

 Important:

Save only saves the case. It does not change the status of the case or start any action.

Complete saves the case, closes the case and the chat session, and starts actions.

When the customer does not have a Microsoft account:

If the customer does not have a Microsoft account, but does have an email address, you can create a Service Request with just their email. To do so:

  • Type customer’s email address in the Email field in the Incident form, which is next to the Customer Account field.
  • This will cause the Customer Account field to update with the email entered.
  • You will not be able to edit the email after it is entered.

If you find out you have misentered you will have to start a new Service Request
