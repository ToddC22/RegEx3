1. Device registration

  • The Devices section will display all devices logged into with the currently authenticated Microsoft account.

  • The Register Device section splits the view into “Registered Devices” and “Devices signed into with this Microsoft Account”.
    • Devices listed under “Registered Devices” are Microsoft devices and display a warranty indicator.
    • Devices listed under “Devices signed into with this Microsoft Account” are Microsoft devices as well as any third-party devices that may have signed on using the Microsoft account currently authenticated. Devices in this view will not display any warranty authenticator.
  • To create a repair or accessory order against a device shown under Devices signed into with this Microsoft account, select register device first. Once the device is registered, you can create the repair or accessory order. You can only register Microsoft devices.
    Registered Devices

    [Asset 4339378]

  • An exchanged device will NOT automatically disassociate from the Devices section. Either the customer (through accounts.microsoft.com) or you (through MST) can remove the device from view if necessary.

  • The replacement device does not automatically register to the customer’s Microsoft account. When the customer signs in using their Microsoft account, the device will appear in the Devices section of that Microsoft account.

3. Creating a repair order

  • Before entering the repair or fulfillment workflow in MST, you must either have a ‘saved’ SR (for instance, existing case) loaded without an active order OR select NEW on the incident form to generate a new SR. Otherwise, the tool will display an error message.
  • You must select the customer’s ship to country under Problem Type and Offers. This determines what offers populate based on the customer’s device type and configuration.
  • Surface only: If the device has the Enterprise flag (112), MST will display an alert and return no repair offers. Consult a Tech Lead/Super User to have the flag removed.

Note: For a customer resident in Europe or North America that states they would like to schedule a pickup with the carrier during a repair order creation, escalate the case to a Repair Specialist and capture “courier collection” in case notes. (Asia Pacific/Australia/New Zealand markets already have courier collection in place. India is out of scope - which is walk in center support).

4. Order status and history

MST displays the order status and history of a specific device serial number. MST does not have a view showing all active or past orders for all devices registered on the account.

To see the order status and history, select and open the registered device. This will display a Repair Orders for customer section, if there is an active order or a past order.

While the status will display the serial number of the replacement device, after an exchange, the replacement device will not automatically register to the customer’s Microsoft account. If there is a need, you should register manually.

5. Resending shipping labels

5.1 Important notes

  • Shipping labels can only be resent if the original label was generated and sent by email (SEI for Surface, PEI for Band, ESI for Xbox).
  • You cannot resend shipping labels if the order was created online by the customer. Customers are expected to retrieve their labels online if the order was created online. An alternative is to cancel the order and recreate the order.
  • You can only resend to the email address that was used when the order was created. If a typo or wrong email address is what prompted the email delivery to fail, using the resend function will not fix the issue. If this is the case, cancel the order and recreate it using the correct email address.
  • If the attempt to resend the shipping label displays an error message for a customer order that was previously created by an advocate AND the order is from one of the countries listed above, cancel the order then recreate the order.
  • See the following KB’s to verify if a device is launched in that country:
    • KB 4514977 Procedure: Surface: Services Exchange and Repair Offers Overview
    • KB 967172 Procedure: Xbox: Global Xbox repair and fulfillment information and options
  • All online created orders are excluded.

Note: If a device is not officially released in a country, the resend feature will not be enabled.

6. Cancelling an open order

There are scenarios where a customer would like to cancel an order before it has been processed because they made a mistake but only realized it after they placed the order, such as:

  • They gave the wrong serial number.
  • They used an incorrect shipping address.

There are scenarios where an advocate would like to cancel an order while it is in progress, not limited to the following:

  • The serial number is incorrect as submitted.
  • The service order type was incorrect such as, in warranty instead of out of warranty.
  • Advance exchange and send immediately.

Note: Order types cannot be cancelled once submitted.

You can only cancel an order where there has been no product movement. This means:

  • On a standard exchange/repair, customer has not already shipped out their non-working device.
  • All send immediately/advance exchange activities.

Steps to cancel an open order

  1. Ensure you have the SR associated to the order loaded prior to proceeding (for example, append to existing case). You will receive an error message otherwise and have to redo the steps.

7. Purchasing service plans/warranty dispute

  • Microsoft Complete plans are now sold through Purchase on Behalf of (POBO) process in MST. In Europe, you can only sell the offer to customers who purchased their device through Microsoft Store online.
    • Devices advocate: Always write an order against a registered device. If the customer’s device is not registered, register the device.
    • Non-Devices advocate: If the customer’s device is not already registered and visible in the Devices section of MST, start a New Order under the Orders section of MST and run a serial number look up.  
  • The device must have been purchased within the last 45 days. If the customer is eligible, the offer will be available in the tools.
    • If the customer is within 45 days of device purchase and you receive an error while attempting to follow the Purchase Service Plan process below:
      • Xbox Offline advocate or Surface Super User: Follow KB 2981105 Procedure: Advanced Capability Advocate engagement to escalate to Devices ACA.
      • Surface Repair Specialist – Follow KB 4090113 Procedure: TFS Incident Escalation to escalate to GRAT.

8. Transfer Service Plan

The Transfer Service Plan function under the Devices section of MST allows you to search for past purchase records of Microsoft Complete given an order number or serial number then reattach the plan to the intended device.

Unless the purchase was through Microsoft Store online to where you can use the Orders section to verify the purchase and capture the order number, you will need to collect a valid proof of purchase (Receipt ID not Invoice ID) with Microsoft Complete clearly listed before you can initiate the process.

This function only works for purchases made by customers after September 1st, 2017.

  • Do not use the instructions for this function if the purchase date is prior to this date, instead escalate the case to a Repair Specialist.

What this function allows:

  • Allows you to search for the Microsoft Complete plan purchase record.
  • Finding the purchase record, allows you to reattach to the intended device.

Note: For any scenarios that fall outside of the above, you will escalate a Child SR to a Repair Specialist. (See errors table in section 8.3.) The Repair Specialist should access the Store Warranty Support SharePoint for resolution.

8.3 Error conditions

All error conditions should be escalated to a Repair Specialist

Search (https://ustrl.mp.microsoft.com/v1.0/warranty/fulfillment/orders)

Error Code 400: "Order id, jarvis id, serial number, or purchase date cannot be null."

Double-check that the search query is filled in.

Error Code 404: "No orders found for given parameters."

The search query did not match any results. This is expected for orders older than August 2017.

 

Transfer action (https://ustrl.mp.microsoft.com/v1.0/warranty/fulfillment/transfer)

Error Code 400: "Service contract id cannot be null.", "Device serial number cannot be null.", "Manufacturer cannot be null."

There is either a data issue, or a serial number was not entered.

Error Code 404: "Serial number does not exist or is invalid."

The serial number that was entered may be invalid or was not found in the backend system.

Error Code 403: "Device is already attached to a warranty."

A transfer was attempted to a device that already has a warranty.

Error Code 403: "Invalid service contract id or order was never fulfilled."

There is an error with this order. Escalate to the backend team to resolve.
