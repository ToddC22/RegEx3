ASMS – Azure Pass Sponsorship (243P) Offer

Submit a ModTrack request to update this TSG.

Overview

This article provides information about the Azure Pass Sponsorship offer, and guidance on how to assist customers who may contact support.   

Microsoft created the Azure Pass Sponsorship (243P) offer to encourage potential customers to try the Azure Platform. This program is widely used for sales opportunities, training scenarios, workshops, and much more.

Policy 

  • The maximum Azure Pass Sponsorship value is $1,500 per pass. Azure Pass Sponsorship requires a promo code to activate the offer.
  • Each account is limited to one concurrent Azure Pass.
    • If the customer obtains additional Azure Pass promotional codes, advise them that they will need to redeem that promo code after the current offer has ended or with a new account.
  • Azure Pass Sponsorship does not require a payment instrument (PI) or SMS check.
  • Accounts associated with an EA enrollment are not eligible for an Azure Pass Sponsorship.
  • Customers may have multiple Azure Pass Sponsorship subscriptions on a single account. However, all subscriptions under that Account ID (whether a Microsoft Account or Org ID) will use credits from the same pool.
  • There is no monthly credit cap. There is only an overall usage cap (the maximum consumption value for an Azure Pass Sponsorship), which allows customers to spend Azure credits at their own pace up to the usage cap. 
  • The Azure Pass Sponsorship will end when the customer reaches either the usage cap or the duration of the subscription has come to an end, whichever occurs first.
  • The Azure Pass Sponsorship offer will have the same data center and service level restrictions as the legacy (120P-130P) Azure Pass offers.

Customer Experience and Provisioning

Subscription Signup

If the customer redeemed their promo code but failed to sign up for the subscription, direct the customer to the sign up link for the Azure Pass Sponsorship offer: Sign Up Offer Azure Pass Sponsorships

Activate Promo Code

If customers need assistance activating their promo code, provide them with the following link: Creating a Microsoft Azure Pass Subscription.

In a situation where the customer has redeemed the code under a specific account but does not see the subscription, and the SE has verified the customer's account, instruct the customer to do the following:  

  1. Log into the Accounts Portal. 
  2. Select Add Subscription. 
  3. Select the Upgrade Offer (Azure Pass Sponsorship) and the subscription will activate.

Note: If the customer has a Free Trial active, you will need to engage your TA who will work to get the Free Trial deprovisioned so that the Azure Pass subscription can be activated. 

Monetary Credit

Azure Pass Sponsorship offers flexible credits. Customers can use their credits at any time during the Azure Pass Sponsorship offer (similar to the 36P Sponsorship offer). Promo codes can be redeemed on the Azure Pass site.

The customer can check their Azure Pass Sponsorship credit balance at https://www.microsoftazuresponsorships.com/. The credit balance will not be available in https://portal.azure.com.

Support Scenarios

Promo Code/Azure Pass Sponsorship Entitlement Verification

You can use AIRS to look up the Account ID to verify if the customer has an Azure Pass Sponsorship. If the customer redeems the Azure Pass Sponsorship promo code with an incorrect Microsoft account ID, they will need to contact their internal Microsoft contact who issue them the code to recieve a new promo code.

Use the steps below to verify a customer’s Account ID:  

  1. Go to the AIRS site.
  2. Click on Sign in.
  3. Click on List Registrations.
  4. Uncheck box titled “Pending business desk approval”.
  5. Under Account owner Live id – enter the Email in which the sponsorship was granted and select the Equals radio button underneath the text box.
  6. Click on Filter and then click on the tab under the “#” symbol to run the query.
     

Ask the customer if they already redeemed the pass with their account ID via the Azure Pass website:

  • If yes, the customer will need to attempt to redeem the subscription via the Azure Portal.
  • If not redeemed, instruct the customer to follow the prompts to redeem the promo code.

If you need to contact the Azure Pass team for further assistance,  you can send a task to the Azure Sponsorship Offer queue.

Azure Pass Subscription and Ownership Transfers

SOTs of Azure Pass subscriptions from one account to another are not supported. Performing a subscription transfer will cause the subscriptions end date to reset; allowing the customer to continue using their services without paying. All new monetary credit Azure Pass subscriptions will have a subscription end date and will not allow for renewal.

However, should the customer need to tranfer thier services to a new account, we can suggest a migration instead. If the customer decides to use this workaround, follow the guidance in the migration section below. 

Example: If you have a 12 month pass that expires tomorrow on 7/16/19, and an SOT is performed on 7/15/19, the new subscription end date will be 7/15/20. Resetting the subscription end date and allowing the customer another year of service.  

Converting Existing Subscriptions to Azure Pass Sponsorship

It is not possible to convert an existing subscription to an Azure Pass Sponsorship.

Converting Expired Azure Pass Sponsorships to Another Offer

Expired Azure Pass Sponsorship subscriptions (243P) will not be automatically converted to Pay-As-You-Go (PAYG). 

If there is no PI and the customer wants to re-enable the subscription after it was disabled at the end of their term, the subscription can still be converted to a PAYG. At this time, the customer will need to remove the monetary cap, add a credit card to re-enable the subscription. If the customer doesn’t have the self-serve option, ASMS may need to re-enable the subscription on their behalf. See TSG 2647585 for guidance on how to convert a subscription.

For guidance on adding a credit card, reference Add, Update, or Remove a Credit or Debit Card for Azure.

If the Azure Pass subscription is disabled and a migration to an EA subscription or another offer (Offer Details) that doesn't require a credit card needs to occur, you can do the following: 

  • Engage the Azure Pass team to have them provide a second pass with $50 of usage so that the subscription can be re-enabled during the migration phase and then disabled again, once complete.  

Note: EA customers with an active EA enrollment cannot sign up for Azure Pass Sponsorship subscriptions.

Data Migration

If a customer requests a data migration, confirm if the customer purchased the Azure subscription using the same MSA ID. Once confirmed, you can work with the customer to perform a data migration.

As stated above SOTs are not supported for this offer type. The workaround for this, is to have the customer request a new promo code from the resource that originally provided them the code. Once approved, the following will need to take place:

  1. The customer will need to create a new Azure Pass Subscription using the new code.
  2. The customer will need to activate the code using a new Azure Account. 
    • The customer will not be able to activate another Azure Pass under the same account until the current offer has ended, due to the design of the system.
  3. An ASMS engineer will perform a data migration of the services from the old pass subscription to the new one.

For guidance on performing a simple data migration, reference TSG 2856526 and for performing a forced migration, reference TSG 2892044.

Internal Users

If an internal user (Microsoft employee) is requesting an Azure Pass Sponsorship subscription, direct them to the Azure Pass Request Portal. If there are questions regarding this portal, direct them to email AzurePassSupport@microsoft.com.

Note: This alias is for internal use only and should not be shared with external customers.

Azure Pass Extensions

If a customer requests to have additional credits or time added to their Azure Pass Sponsorship, direct the customer to their Account Rep or Azure Pass Distributor. These types of requests are at the funding discretion of the Microsoft employee who distributed the code. The user will need to redeem a new promo code if the Azure Pass already ended.

Azure Pass for Microsoft Certified Trainer (MCT)

If an MCT requests an Azure Pass, direct them to the following link to activate the offer:
https://vouchers.cloudapp.net/AzurePass/
 

For support with any issues related to an Azure Pass Voucher claim, see the MCT Training Support site here.

Engagement and Escalation Path

For help or guidance with any of the above Support Scenarios follow these steps:

  1. Engage your TA.
  2. If you need additional guidance after you have engaged your TA first, send a task to the Azure Sponsorship Offer queue.
